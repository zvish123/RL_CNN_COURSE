שפת שאילתות מובנית - Structured Query Language

ברוכים הבאים למסע הלמידה שלכם בעולם מסדי הנתונים!

📊 מה זה SQL?

SQL היא שפת תכנות מיוחדת לעבודה עם מסדי נתונים. היא מאפשרת לנו לשמור, לשלוף, לעדכן ולנהל מידע בצורה יעילה.

🎯 למה ללמוד SQL?

SQL נמצאת בשימוש בכל אפליקציה, אתר ומערכת שמשתמשת במידע. זו מיומנות חיונית בעולם ההייטק!

🚀 מה נלמד היום?

נלמד לכתוב שאילתות, ליצור טבלאות, לחבר ביניהן ולשלוף מידע בדיוק כמו מתכנתים מקצועיים!

SQL קלה ללמידה, חזקה בשימוש, ומבוקשת מאוד בשוק העבודה. רוב החברות בעולם ההייטק משתמשות בה!

🗄️ מהו מסד נתונים?

מסד נתונים הוא כמו ארגז כלים מאורגן שבו אנחנו שומרים מידע בצורה מסודרת. במקום קבצים מפוזרים, כל המידע נמצא במקום אחד!

דוגמה מהחיים:

ספרייה: ספרים מאורגנים לפי קטגוריות, מחברים, שנות הוצאה

מסד נתונים: טבלאות מאורגנות עם מידע על תלמידים, ציונים, מורים

📋 מבנה מסד נתונים רלציוני:

  • טבלאות (Tables): כמו גליונות Excel - מידע מאורגן בשורות ועמודות
  • שדות (Fields/Columns): העמודות - כל עמודה מייצגת סוג מידע (שם, גיל, וכו')
  • רשומות (Records/Rows): השורות - כל שורה מייצגת פריט אחד של מידע
  • מפתח ראשי (Primary Key): מזהה ייחודי לכל רשומה

טבלת תלמידים לדוגמה:

תעודת זהות (מפתח ראשי) שם גיל כיתה
123456789 יעל כהן 16 יא1
987654321 דוד לוי 17 יא2
555555555 מיכל אברהם 16 יא1
כל רשומה בטבלה היא ייחודית! המפתח הראשי (כמו תעודת זהות) מבטיח שלא יהיו שתי רשומות זהות לחלוטין.

🏗️ יצירת טבלה - CREATE TABLE

לפני שנוכל לעבוד עם מידע, צריך ליצור טבלה שתכיל אותו. זה כמו לבנות את השלד של הטבלה.

תחביר בסיסי:

CREATE TABLE table_name (
    column1 datatype,
    column2 datatype,
    column3 datatype,
    PRIMARY KEY (column_name)
);

📌 סוגי נתונים נפוצים:

  • INT - מספרים שלמים (1, 42, -7)
  • VARCHAR(n) - טקסט באורך משתנה (עד n תווים)
  • DATE - תאריך (2024-01-15)
  • DECIMAL(p,s) - מספרים עשרוניים (3.14, 99.99)
  • BOOLEAN - ערך אמת/שקר (TRUE/FALSE)

דוגמה מעשית - טבלת תלמידים:

CREATE TABLE students (
    student_id INT NOT NULL,
    first_name VARCHAR(50) NOT NULL,
    last_name VARCHAR(50) NOT NULL,
    age INT,
    grade VARCHAR(10),
    email VARCHAR(100),
    PRIMARY KEY (student_id)
);

הסבר השדות:

  • student_id: מזהה ייחודי לכל תלמיד (מפתח ראשי)
  • NOT NULL: שדה חובה - חייב להכיל ערך
  • VARCHAR(50): טקסט עד 50 תווים
  • PRIMARY KEY: מגדיר את המפתח הראשי - המזהה הייחודי

נסו בעצמכם!

צרו טבלת "books" (ספרים) עם השדות הבאים:

  • book_id (מפתח ראשי, מספר שלם)
  • title (כותרת, טקסט עד 100 תווים)
  • author (מחבר, טקסט עד 50 תווים)
  • year_published (שנת פרסום, מספר שלם)
  • price (מחיר, עשרוני)

פתרון:

CREATE TABLE books (
    book_id INT NOT NULL,
    title VARCHAR(100) NOT NULL,
    author VARCHAR(50) NOT NULL,
    year_published INT,
    price DECIMAL(10, 2),
    PRIMARY KEY (book_id)
);
תמיד בחרו שמות שדות ברורים ומתארים! במקום "n" כתבו "name", במקום "d" כתבו "date". הקוד שלכם צריך להיות קריא.

➕ הוספת נתונים - INSERT INTO

אחרי שיצרנו טבלה, הגיע הזמן למלא אותה במידע! פקודת INSERT מאפשרת לנו להוסיף רשומות חדשות.

תחביר בסיסי:

INSERT INTO table_name (column1, column2, column3)
VALUES (value1, value2, value3);

דוגמה - הוספת תלמיד:

INSERT INTO students (student_id, first_name, last_name, age, grade)
VALUES (1, 'יעל', 'כהן', 16, 'יא1');

לפני ההוספה:

student_id first_name last_name age grade
טבלה ריקה
↓

אחרי ההוספה:

student_id first_name last_name age grade
1 יעל כהן 16 יא1

💡 שיטות שונות להוספה:

1. הוספה עם ציון כל השדות:

INSERT INTO students 
VALUES (2, 'דוד', 'לוי', 17, 'יא2', 'david@example.com');

2. הוספה של חלק מהשדות:

INSERT INTO students (student_id, first_name, last_name)
VALUES (3, 'מיכל', 'אברהם');
-- שאר השדות יהיו NULL

3. הוספת מספר רשומות בבת אחת:

INSERT INTO students (student_id, first_name, last_name, age, grade)
VALUES 
    (4, 'רון', 'מזרחי', 16, 'יא1'),
    (5, 'נועה', 'שמיר', 17, 'יא3'),
    (6, 'אורי', 'גולן', 16, 'יא2');
כשמוסיפים טקסט (VARCHAR), חייבים להשתמש במרכאות גבשות 'כך'. מספרים לא צריכים מרכאות!
❓ מה לא בסדר בפקודה הזו?
INSERT INTO students (student_id, first_name)
VALUES (7, דני);
חסר נקודה-פסיק בסוף
student_id צריך להיות טקסט
חסרים שדות נוספים
השם דני צריך להיות במרכאות
🎉 מצוין! ערכי טקסט חייבים להיות במרכאות. הפקודה הנכונה: 
INSERT INTO students (student_id, first_name)
VALUES (7, 'דני');

🔍 שליפת נתונים - SELECT (חלק 1)

פקודת SELECT היא הפקודה הכי חשובה ב-SQL! היא מאפשרת לנו לשלוף מידע מהטבלאות.

שליפה בסיסית - כל הנתונים:

SELECT * FROM students;
-- הכוכבית (*) אומרת "כל השדות"

תוצאה:

student_id first_name last_name age grade
1יעלכהן16יא1
2דודלוי17יא2
3מיכלאברהם16יא1
4רוןמזרחי16יא1
5נועהשמיר17יא3

שליפת שדות ספציפיים:

SELECT first_name, last_name, grade 
FROM students;

תוצאה - רק השדות שבחרנו:

first_name last_name grade
יעלכהןיא1
דודלוייא2
מיכלאברהםיא1
רוןמזרחייא1
נועהשמיריא3

שליפה עם תנאי - WHERE:

SELECT * FROM students
WHERE age > 16;
-- שלוף רק תלמידים מעל גיל 16
student_id first_name last_name age grade
2דודלוי17יא2
5נועהשמיר17יא3

🎯 אופרטורים לשימוש עם WHERE:

  • = - שווה ל
  • > - גדול מ
  • < - קטן מ
  • >= - גדול או שווה ל
  • <= - קטן או שווה ל
  • <> או != - שונה מ

דוגמאות נוספות:

-- תלמידים מכיתה יא1
SELECT * FROM students
WHERE grade = 'יא1';

-- תלמידים שלא בכיתה יא2
SELECT * FROM students
WHERE grade <> 'יא2';

-- תלמידים בגיל 16 בדיוק
SELECT first_name, last_name 
FROM students
WHERE age = 16;
WHERE הוא כמו פילטר - הוא מאפשר לנו לבחור רק את השורות שעונות על תנאי מסוים!

🔍 שליפת נתונים - SELECT (חלק 2)

שילוב תנאים - AND / OR:

AND - שני התנאים חייבים להתקיים:

SELECT * FROM students
WHERE age = 16 AND grade = 'יא1';
-- תלמידים בגיל 16 וגם מכיתה יא1

OR - אחד מהתנאים צריך להתקיים:

SELECT * FROM students
WHERE grade = 'יא1' OR grade = 'יא2';
-- תלמידים מכיתה יא1 או מכיתה יא2

שילוב AND ו-OR:

SELECT * FROM students
WHERE (grade = 'יא1' OR grade = 'יא2') 
  AND age >= 17;
-- תלמידים מיא1 או יא2 שגילם 17 ומעלה

💭 הסבר:

AND: כמו "וגם" - שני התנאים חייבים להיות נכונים

OR: כמו "או" - מספיק שאחד מהתנאים נכון

סוגריים: קובעים סדר ביצוע, בדיוק כמו במתמטיקה!

חיפוש בטקסט - LIKE:

LIKE מאפשר חיפוש דינמי בשדות טקסט. משתמשים בסימנים מיוחדים:

  • % - מייצג אפס תווים או יותר (כלשהם)
  • _ - מייצג תו אחד בדיוק

דוגמאות עם LIKE:

-- שמות שמתחילים ב"י"
SELECT * FROM students
WHERE first_name LIKE 'י%';

-- שמות שמסתיימים ב"ל"
SELECT * FROM students
WHERE first_name LIKE '%ל';

-- שמות שמכילים "ו" באמצע
SELECT * FROM students
WHERE first_name LIKE '%ו%';

-- שמות בני 4 אותיות שמתחילים ב"נ"
SELECT * FROM students
WHERE first_name LIKE 'נ___';

תחום ערכים - BETWEEN:

-- תלמידים בגילאים 16-17
SELECT * FROM students
WHERE age BETWEEN 16 AND 17;

-- שווה ערך ל:
SELECT * FROM students
WHERE age >= 16 AND age <= 17;

בדיקת השתייכות - IN:

-- תלמידים מכיתות מסוימות
SELECT * FROM students
WHERE grade IN ('יא1', 'יא2', 'יא3');

-- קצר וקריא יותר מאשר:
SELECT * FROM students
WHERE grade = 'יא1' OR grade = 'יא2' OR grade = 'יא3';
❓ איזו שאילתה תשלוף תלמידים ששמם מתחיל ב"ד" או מתחיל ב"מ"?
WHERE first_name LIKE 'ד%' OR first_name LIKE 'מ%'
WHERE first_name = 'ד%' OR first_name = 'מ%'
WHERE first_name IN ('ד', 'מ')
WHERE first_name BETWEEN 'ד' AND 'מ'
🎉 נכון! LIKE משתמש עם % לחיפוש דינמי, ו-OR מחבר בין שני התנאים.

📊 מיון ותיחום תוצאות

מיון תוצאות - ORDER BY:

ORDER BY מאפשר למיין את התוצאות בסדר עולה או יורד.

מיון בסדר עולה (ברירת מחדל):

SELECT * FROM students
ORDER BY age;
-- מיון לפי גיל מהקטן לגדול

מיון בסדר יורד:

SELECT * FROM students
ORDER BY age DESC;
-- מיון לפי גיל מהגדול לקטן

מיון לפי מספר שדות:

SELECT * FROM students
ORDER BY grade ASC, age DESC;
-- מיון לפי כיתה (עולה), ובתוך כל כיתה לפי גיל (יורד)

דוגמה:

לפני המיון (סדר הכנסה):

1יעל16יא1
2דוד17יא2
3מיכל16יא1
5נועה17יא3
↓ ORDER BY age ASC

אחרי המיון (לפי גיל עולה):

student_idfirst_nameagegrade
1יעל16יא1
3מיכל16יא1
2דוד17יא2
5נועה17יא3

הגבלת כמות תוצאות - LIMIT:

-- הצג רק 3 תלמידים ראשונים
SELECT * FROM students
LIMIT 3;

-- הצג 5 התלמידים הצעירים ביותר
SELECT * FROM students
ORDER BY age ASC
LIMIT 5;

-- הצג 3 התלמידים המבוגרים ביותר
SELECT first_name, last_name, age 
FROM students
ORDER BY age DESC
LIMIT 3;

-- הצג רשימת כיתות (כל כיתה פעם אחת)
SELECT DISTINCT grade

תרגיל: כתבו שאילתה שמציגה את 10 התלמידים הצעירים ביותר, רק את השם והגיל, ממוין מהצעיר לגדול.

SELECT first_name, age
FROM students
ORDER BY age ASC
LIMIT 10;

🧮 פונקציות אגרגציה

פונקציות אגרגציה מאפשרות לבצע חישובים על קבוצות נתונים - כמו ממוצע, סכום, ספירה ועוד.

📌 הפונקציות העיקריות:

COUNT() - ספירה

סופר כמה שורות יש

SELECT COUNT(*)
FROM students;

AVG() - ממוצע

מחשב ממוצע של ערכים

SELECT AVG(age)
FROM students;

SUM() - סכום

מחבר את כל הערכים

SELECT SUM(age)
FROM students;

MAX() - מקסימום

מוצא את הערך הגבוה ביותר

SELECT MAX(age)
FROM students;

MIN() - מינימום

מוצא את הערך הנמוך ביותר

SELECT MIN(age)
FROM students;

דוגמאות מעשיות:

-- כמה תלמידים יש בטבלה?
SELECT COUNT(*) AS total_students
FROM students;

-- מה גיל הממוצע?
SELECT AVG(age) AS average_age
FROM students;

-- מי התלמיד הצעיר והמבוגר ביותר?
SELECT MIN(age) AS youngest, 
       MAX(age) AS oldest
FROM students;

-- כמה תלמידים יש בכיתה יא1?
SELECT COUNT(*) AS students_in_11_1
FROM students
WHERE grade = 'יא1';

💡 AS - כינוי לעמודות:

AS מאפשר לתת שם ברור יותר לעמודה בתוצאה. במקום "AVG(age)" נראה "average_age"

קיבוץ נתונים - GROUP BY:

GROUP BY מאפשר לחשב סטטיסטיקות על קבוצות שונות של נתונים.

-- כמה תלמידים יש בכל כיתה?
SELECT grade, COUNT(*) AS num_students
FROM students
GROUP BY grade;

תוצאה:

grade num_students
יא13
יא21
יא31
-- מה הגיל הממוצע בכל כיתה?
SELECT grade, 
       AVG(age) AS avg_age,
       COUNT(*) AS num_students
FROM students
GROUP BY grade
ORDER BY grade;

סינון קבוצות - HAVING:

HAVING כמו WHERE, אבל עבור קבוצות (אחרי GROUP BY).

-- הצג רק כיתות עם יותר מ-2 תלמידים
SELECT grade, COUNT(*) AS num_students
FROM students
GROUP BY grade
HAVING COUNT(*) > 2;
WHERE מסנן שורות לפני הקיבוץ, HAVING מסנן קבוצות אחרי הקיבוץ!
❓ מה תעשה השאילתה הזו?
SELECT grade, AVG(age)
FROM students
GROUP BY grade
HAVING AVG(age) > 16.5;
תציג את כל הכיתות
תציג תלמידים מעל גיל 16.5
תציג כיתות שהגיל הממוצע בהן מעל 16.5
תספור כמה תלמידים בכל כיתה
🎉 נכון! HAVING AVG(age) > 16.5 מסנן קבוצות (כיתות) שהגיל הממוצע שלהן גבוה מ-16.5.

✏️ עדכון נתונים - UPDATE

פקודת UPDATE מאפשרת לשנות ערכים קיימים בטבלה.

⚠️ אזהרה חשובה!

זהירות! UPDATE בלי WHERE מעדכן את כל השורות בטבלה!

תמיד בדקו את ה-WHERE לפני הרצת UPDATE!

תחביר בסיסי:

UPDATE table_name
SET column1 = value1, column2 = value2
WHERE condition;

דוגמה 1 - עדכון שדה אחד:

-- עדכן את הכיתה של התלמיד עם מספר 1
UPDATE students
SET grade = 'יב1'
WHERE student_id = 1;

לפני העדכון:

student_idfirst_namegrade
1יעליא1
2דודיא2
↓

אחרי העדכון:

student_idfirst_namegrade
1יעליב1
2דודיא2

דוגמה 2 - עדכון מספר שדות:

-- עדכן גיל וכיתה של תלמידה
UPDATE students
SET age = 17, grade = 'יב1'
WHERE first_name = 'מיכל';

דוגמה 3 - עדכון מרובה:

-- העלה את כל התלמידים מיא1 ליב1
UPDATE students
SET grade = 'יב1'
WHERE grade = 'יא1';
-- זה יעדכן את כל התלמידים שבכיתה יא1!

דוגמה 4 - עדכון עם חישוב:

-- הוסף שנה לגיל של כל התלמידים
UPDATE students
SET age = age + 1;
-- לא מומלץ! יעדכן את כולם כי אין WHERE

-- גרסה טובה יותר:
UPDATE students
SET age = age + 1
WHERE grade = 'יא1';
טיפ מקצועי: לפני UPDATE, הריצו SELECT עם אותו WHERE כדי לראות אילו שורות יושפעו! 
-- קודם בדקו:
SELECT * FROM students WHERE grade = 'יא1';
-- אחר כך עדכנו:
UPDATE students SET grade = 'יב1' WHERE grade = 'יא1';

תרגיל מעשי:

נניח שהתגלתה טעות - כל התלמידים ששמם מתחיל ב"י" צריכים להיות בכיתה יב2.

כתבו את פקודת ה-UPDATE המתאימה:

-- פתרון:
UPDATE students
SET grade = 'יב2'
WHERE first_name LIKE 'י%';
❓ מה יקרה אם נריץ את הפקודה הזו?
UPDATE students
SET age = 18;
תמחק את כל התלמידים
תשנה את הגיל של כל התלמידים ל-18
תשנה רק את התלמיד הראשון
תגרום לשגיאה
🎉 נכון! בלי WHERE, ה-UPDATE משפיע על כל השורות בטבלה. זו סיבה טובה תמיד לכלול WHERE!

🗑️ מחיקת נתונים - DELETE

פקודת DELETE מסירה רשומות מהטבלה.

⚠️ אזהרה קריטית!

🚨 זהירות מרבית! DELETE בלי WHERE מוחק את כל הנתונים!

DELETE הוא בלתי הפיך - אין "ביטול"! המידע נמחק לצמיתות!

כלל זהב: תמיד בדקו עם SELECT לפני DELETE!

תחביר בסיסי:

DELETE FROM table_name
WHERE condition;

דוגמה 1 - מחיקת רשומה אחת:

-- מחק תלמיד עם מספר זיהוי 3
DELETE FROM students
WHERE student_id = 3;

לפני המחיקה:

student_idfirst_namegrade
1יעליא1
2דודיא2
3מיכליא1
4רוןיא1
↓ DELETE

אחרי המחיקה:

student_idfirst_namegrade
1יעליא1
2דודיא2
4רוןיא1

דוגמה 2 - מחיקת מספר רשומות:

-- מחק את כל התלמידים מכיתה יא3
DELETE FROM students
WHERE grade = 'יא3';

דוגמה 3 - מחיקה עם תנאים מורכבים:

-- מחק תלמידים מתחת לגיל 16 בכיתה יא1
DELETE FROM students
WHERE age < 16 AND grade = 'יא1';

-- מחק תלמידים שאין להם email
DELETE FROM students
WHERE email IS NULL;

❌ הדוגמה המסוכנת ביותר:

DELETE FROM students;
-- ⚠️ מוחק את כל התלמידים! אין WHERE!

💡 שיטת עבודה נכונה:

  1. שלב 1: הריצו SELECT עם אותו WHERE
  2. שלב 2: בדקו שאתם רואים בדיוק מה שצריך למחוק
  3. שלב 3: רק אז החליפו SELECT ב-DELETE
-- שלב 1: בדיקה
SELECT * FROM students WHERE grade = 'יא3';

-- אם הכל בסדר, שלב 2: מחיקה
DELETE FROM students WHERE grade = 'יא3';
רוב מערכות ניהול בסיסי נתונים מאפשרות להגדיר "מצב בטוח" שלא מאפשר DELETE או UPDATE בלי WHERE. זו הגדרה מומלצת!

תרגיל:

נניח שיש לכם טבלה עם מידע על הזמנות ישנות. רוצים למחוק הזמנות מלפני 2020.

כתבו את הפקודות הנכונות:

-- 1. קודם בדקו מה תמחקו:
SELECT * FROM orders 
WHERE order_year < 2020;

-- 2. אם הכל נראה טוב, מחקו:
DELETE FROM orders 
WHERE order_year < 2020;
❓ איזו מהפקודות הבאות היא הבטוחה ביותר?
DELETE FROM students;
DELETE FROM students WHERE age > 0;
DELETE FROM *;
DELETE FROM students WHERE student_id = 5;
🎉 נכון! DELETE עם מפתח ראשי ספציפי הוא הבטוח ביותר - זה ימחק רשומה אחת בלבד.

🔗 קשרים בין טבלאות

במסדי נתונים אמיתיים, המידע מפוזר על פני מספר טבלאות. נצטרך לדעת לחבר ביניהן!

למה לפצל לטבלאות?

  • 📉 למנוע כפילויות מידע
  • 🎯 לארגן טוב יותר את הנתונים
  • ⚡ לשפר ביצועים
  • 🔧 להקל על תחזוקה

סוגי קשרים:

1:1 - יחיד ליחיד

דוגמה: תלמיד ← → דרכון

לכל תלמיד דרכון אחד, לכל דרכון תלמיד אחד

1 ↔ 1

1:N - יחיד לרבים

דוגמה: כיתה ← → תלמידים

לכיתה יש הרבה תלמידים, לתלמיד כיתה אחת

1 ↔ ∞

N:M - רבים לרבים

דוגמה: תלמידים ← → קורסים

תלמיד לומד הרבה קורסים, בקורס הרבה תלמידים

∞ ↔ ∞

מפתחות זרים - Foreign Keys:

מפתח זר הוא שדה בטבלה שמצביע על מפתח ראשי בטבלה אחרת - כך יוצרים את הקשר!

דוגמה - קשר 1:N (כיתה ← תלמידים):

טבלת Classes (כיתות):

class_id (PK) class_name
1יא1
2יא2
3יא3

טבלת Students (תלמידים):

student_id (PK) name class_id (FK)
101יעל1
102דוד2
103מיכל1
104רון1

class_id בטבלת Students מצביע על class_id בטבלת Classes

יצירת מפתח זר:

CREATE TABLE students (
    student_id INT PRIMARY KEY,
    name VARCHAR(50),
    class_id INT,
    FOREIGN KEY (class_id) REFERENCES classes(class_id)
);

💡 יתרונות מפתחות זרים:

  • שומרים על שלמות הנתונים - לא יכול להיות class_id שלא קיים
  • מונעים מחיקת כיתה שיש בה תלמידים
  • יוצרים קשר לוגי בין הטבלאות
כלל אצבע: הצד "רבים" (N) תמיד מחזיק את המפתח הזר!
כיתה אחת ← הרבה תלמידים, אז התלמידים מחזיקים את class_id

🔗 חיבור טבלאות - JOIN (חלק 1)

JOIN מאפשר לשלוף מידע משתי טבלאות או יותר במקביל, על ידי חיבורן לפי שדה משותף.

טבלאות לדוגמה:

INNER JOIN - החיבור הבסיסי:

מחזיר רק שורות שיש להן התאמה בשתי הטבלאות.

SELECT students.name, classes.class_name, classes.teacher
FROM students
INNER JOIN classes 
    ON students.class_id = classes.class_id;

שימוש בכינויים (Aliases):

כדי להקל על הכתיבה, נותנים לטבלאות כינויים קצרים:

SELECT s.name, c.class_name, c.teacher
FROM students AS s
INNER JOIN classes AS c 
    ON s.class_id = c.class_id;
-- s = students, c = classes

הוספת תנאים:

-- הצג תלמידים וכיתות שלהם, רק מכיתה יא1
SELECT s.name, c.class_name, c.teacher
FROM students s
INNER JOIN classes c 
    ON s.class_id = c.class_id
WHERE c.class_name = 'יא1';

💡 הסבר המבנה:

  • SELECT: בוחרים שדות משתי הטבלאות
  • FROM: הטבלה הראשונה
  • INNER JOIN: הטבלה השנייה שרוצים לחבר
  • ON: תנאי החיבור - איזה שדות מתאימים
  • WHERE: תנאים נוספים לסינון

דוגמה מתקדמת - חיבור 3 טבלאות:

-- טבלאות: students, classes, teachers
SELECT s.name AS student_name,
       c.class_name,
       t.teacher_name,
       t.subject
FROM students s
INNER JOIN classes c ON s.class_id = c.class_id
INNER JOIN teachers t ON c.teacher_id = t.teacher_id
ORDER BY c.class_name, s.name;
תמיד ציינו מאיזו טבלה בא כל שדה (table.field) כדי למנוע בלבול כשיש שדות עם אותו שם!

תרגיל: יש לכם טבלת orders (הזמנות) ו-customers (לקוחות).

כתבו שאילתה שמציגה את שם הלקוח ותאריך ההזמנה לכל הזמנה:

SELECT c.customer_name, o.order_date
FROM orders o
INNER JOIN customers c 
    ON o.customer_id = c.customer_id;

טבלאות לדוגמה:

INNER JOIN

מחזיר רק שורות עם התאמה בשתי הטבלאות

SELECT *
FROM students s
INNER JOIN classes c
ON s.class_id = c.class_id;

תוצאה: יעל (יא1), דוד (יא2)

LEFT JOIN

כל השורות מהטבלה השמאלית + התאמות מהימנית

SELECT *
FROM students s
LEFT JOIN classes c
ON s.class_id = c.class_id;

תוצאה: יעל (יא1), דוד (יא2), רינה (NULL)

RIGHT JOIN

כל השורות מהטבלה הימנית + התאמות מהשמאלית

SELECT *
FROM students s
RIGHT JOIN classes c
ON s.class_id = c.class_id;

תוצאה: יעל (יא1), דוד (יא2), NULL (יא3)

FULL JOIN

כל השורות משתי הטבלאות

SELECT *
FROM students s
FULL JOIN classes c
ON s.class_id = c.class_id;

תוצאה: הכל - יעל, דוד, רינה, יא3

דוגמה מפורטת - LEFT JOIN:

SELECT s.name, 
       COALESCE(c.class_name, 'לא משובץ') AS class_name
FROM students s
LEFT JOIN classes c 
    ON s.class_id = c.class_id;

💡 COALESCE:

פונקציה שמחזירה את הערך הראשון שאינו NULL. שימושית ב-LEFT/RIGHT JOIN!

COALESCE(c.class_name, 'לא משובץ') - אם class_name הוא NULL, הצג "לא משובץ"

איך לבחור?
  • INNER JOIN: כשרוצים רק התאמות מלאות
  • LEFT JOIN: כשרוצים לראות הכל מהטבלה הראשית (כמו: כל התלמידים, עם או בלי כיתה)
  • RIGHT JOIN: נדיר - בדרך כלל משתמשים ב-LEFT JOIN במקום
❓ איזה JOIN נשתמש כדי למצוא תלמידים שלא משובצים לכיתה?

📍 טבלת Enrollments מחברת בין Students ל-Courses
(ואפשר להוסיף עוד מידע כמו ציון!)

יצירת הטבלאות:

-- טבלת תלמידים
CREATE TABLE students (
    student_id INT PRIMARY KEY,
    name VARCHAR(50)
);

-- טבלת קורסים
CREATE TABLE courses (
    course_id INT PRIMARY KEY,
    course_name VARCHAR(50)
);

-- טבלת חיבור - עם שני מפתחות זרים!
CREATE TABLE enrollments (
    student_id INT,
    course_id INT,
    grade INT,
    PRIMARY KEY (student_id, course_id),
    FOREIGN KEY (student_id) REFERENCES students(student_id),
    FOREIGN KEY (course_id) REFERENCES courses(course_id)
);

💡 שימו לב:

  • המפתח הראשי של Enrollments הוא משולב משני שדות
  • זה מבטיח שתלמיד לא יכול להירשם לאותו קורס פעמיים
  • אפשר להוסיף שדות נוספים (כמו grade, enrollment_date)

שאילתות מעשיות:

1. הצג את כל הקורסים של יעל:

SELECT c.course_name, e.grade
FROM students s
JOIN enrollments e ON s.student_id = e.student_id
JOIN courses c ON e.course_id = c.course_id
WHERE s.name = 'יעל';

2. הצג את כל התלמידים בקורס מתמטיקה:

SELECT s.name, e.grade
FROM courses c
JOIN enrollments e ON c.course_id = e.course_id
JOIN students s ON e.student_id = s.student_id
WHERE c.course_name = 'מתמטיקה';

3. כמה תלמידים רשומים לכל קורס?

SELECT c.course_name, COUNT(e.student_id) AS num_students
FROM courses c
LEFT JOIN enrollments e ON c.course_id = e.course_id
GROUP BY c.course_name;
טבלת חיבור היא הדרך התקנית ליישם קשר רבים-לרבים. כמעט כל מערכת אמיתית משתמשת בזה!

תרגיל מעשי:

נניח שיש ספרים ומחברים. ספר יכול להיות למספר מחברים, ומחבר יכול לכתוב מספר ספרים.

🎓 שאילתת סיכום מתקדמת:

הנה דוגמה לשאילתה שמשלבת הרבה מה שלמדנו:

-- מצא את התלמידים שהממוצע שלהם מעל 90
-- והם לומדים לפחות 3 קורסים
SELECT 
    s.name AS student_name,
    COUNT(e.course_id) AS num_courses,
    AVG(e.grade) AS avg_grade,
    MAX(e.grade) AS highest_grade
FROM students s
INNER JOIN enrollments e ON s.student_id = e.student_id
INNER JOIN courses c ON e.course_id = c.course_id
GROUP BY s.student_id, s.name
HAVING AVG(e.grade) > 90 AND COUNT(e.course_id) >= 3
ORDER BY avg_grade DESC;
