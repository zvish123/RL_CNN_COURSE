עבודת צוות, Git ו-GitHub — יישום בפרויקט

מקימים repository פעיל, מחלקים תפקידים, ופותחים את לוח המשימות

⏱ זמן משוער: 45-60 דק' 👥 כל הצוות יחד 💻 דורש חשבון GitHub לכל חבר צוות

בדף ההסבר למדתם למה עובדים עם branches, Pull Requests, Issues ו-Kanban בעבודת צוות. כאן תיישמו את זה בפועל על ה-repository של הפרויקט שלכם. בסוף הדף תקבלו ✅ repository פעיל עם README, לוח Kanban, ומשימות מחולקות.

1

הקמת ה-Repository

משימה 1.1

יוצרים את המאגר

אחד מחברי הצוות יוצר repository חדש ב-GitHub (פרטי או ציבורי לפי הנחיית המורה) ומוסיף את כל חברי הצוות כ-collaborators.

  • שם repository נבחר ועקבי (לדוגמה: vault-x-team3)
  • כל חברי הצוות מקבלים הזמנה ומקבלים אותה בפועל
  • נוצר branch ראשי main עם קובץ ראשוני (לדוגמה .gitignore)
⚠ ייבדק ע"י המורה — בדיקת קישור פעיל בפועל
2

חלוקת תפקידים

משימה 2.1

מי עושה מה — ושם ה-branch של כל אחד

לכל תפקיד — כתבו את שם חבר הצוות. שם ה-branch כבר מוצע (אפשר לשנות אם הצוות מעדיף שמות אחרים, אך חשוב להישאר עקביים).

תפקידBranchשם חבר הצוות
אחראי שרתfeature/server
אחראי לקוח / Agentfeature/client-agent
אחראי פרוטוקול ואבטחהfeature/protocol-security
אחראי UIfeature/ui
אחראי בדיקות ותיעודfeature/tests-docs
⚠ ייבדק ע"י המורה
3

כתיבת README ראשוני

משימה 3.1

ממלאים את התוכן, ואז מעתיקים בפועל לקובץ README.md ב-repository

מילוי השדות כאן הוא טיוטה — התוצר הסופי הוא קובץ README.md שקיים בפועל ב-repository שלכם.

# [שם הפרויקט יופיע כאן] ## מה הפרויקט עושה [התיאור יופיע כאן] ## איך מריצים [הפקודות יופיעו כאן] ## חברי הצוות ותחומי אחריות - [יושלם לפי משימה 2] ## מבנה הקבצים ...
⚠ ייבדק ע"י המורה — בקובץ README.md בפועל ב-repository
4

פתיחת לוח Kanban ו-Issues ראשוניים

משימה 4.1

כל תפקיד פותח לפחות Issue אחד לעבודה הראשונה שלו

לכל Issue: כותרת ברורה (לא "באג"), label לפי תחום, ושם האחראי. מלאו כאן, ואז פתחו בפועל ב-GitHub Issues + הוסיפו ללוח Kanban (GitHub Projects).

⚠ ייבדק ע"י המורה — בדיקת Issues ולוח פעילים בפועל

בסיום מילוי הדף — הפיקו קובץ JSON עם כל התשובות, והגישו אותו למורה.
