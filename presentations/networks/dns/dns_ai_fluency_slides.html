03

שלושת השחקנים ב-DNS

Recursive Resolver

השרת שהלקוח פונה אליו ישירות. עושה את כל הבירור מול שאר השרתים ומחזיר תשובה סופית אחת.

Root & TLD Servers

שרתי השורש מפנים לפי סיומת (.com/.il) לשרת ה-TLD, שמפנה לשרת האחראי על הדומיין.

Authoritative NS

השרת ש"הכי יודע" — מחזיק בפועל את הרשומות של הדומיין, ונותן תשובה רשמית.

04

תהליך הפענוח (Resolution)

לקוח שואל: מה ה-IP?→ Recursive Resolver→ Root Server→ TLD Server→ Authoritative NS→ תשובה + שמירה ב-cache לפי TTL

רוב השאילתות היומיומיות נענות ישירות מה-cache — התהליך המלא קורה בעיקר בפעם הראשונה שדומיין נשאל.

06

DNS כמשטח תקיפה

DNS Tunneling

קידוד מידע בתוך תתי-דומיין או רשומות TXT, ושליחתו החוצה כשאילתות "תמימות" שעוברות דרך חומות אש. סימן: נפח TXT חריג, תוויות אקראיות, מקור פנימי בודד.

DGA — Domain Generation Algorithms

תוכנה זדונית יוצרת אלפי דומיינים "אקראיים" כדי לשמור ערוץ עם C2 גם אם חוסמים דומיינים בודדים. סימן: דומיינים רבים ושונים שמפנים לאותו IP.

הגנות

DNSSEC — חתימות קריפטוגרפיות שמאמתות שהתשובה לא זויפה. DoH/DoT — מצפינים את השאילתה עצמה מפני האזנה.

07

הדגמת קוד — שאילתות DNS ב-Python

# pip install dnspython
import dns.resolver

def query_domain(domain, record_types=("A", "TXT", "MX")):
    resolver = dns.resolver.Resolver()
    for rtype in record_types:
        try:
            answer = resolver.resolve(domain, rtype)
            print(f"[{rtype}] {domain}")
            for rdata in answer:
                print(f"  -> {rdata}  (TTL={answer.rrset.ttl}s)")
        except dns.resolver.NoAnswer:
            print(f"[{rtype}] אין רשומה כזו")

הרצה בפועל תדפיס את הרשומות האמיתיות של הדומיין, כולל שדה TTL — הזמן שהתשובה נשמרת ב-cache.
