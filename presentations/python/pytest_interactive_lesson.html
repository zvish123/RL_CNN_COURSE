📚 מה זה pytest?

pytest היא ספרייה חיצונית פופולרית מאוד ב-Python (מותקנת עם pip install pytest) שמאפשרת לנו לכתוב בדיקות יחידה (Unit Tests) בצורה פשוטה, קצרה וקריאה - בלי לכתוב מחלקות וירושה.

🔍 עקרונות בסיסיים:

בדיקת יחידה (Unit Test) = בדיקה של חלק קטן בקוד (פונקציה אחת)
מרכיב תפקיד דוגמה
פונקציית בדיקה רגילה אין צורך במחלקה או בירושה def test_addition():
test_* פונקציות בדיקה test_addition()
assert פקודת הבדיקה היחידה שצריך assert a == b
pytest (פקודת מסוף) הרצת הבדיקות מפעיל את כל הטסטים
כל פונקציית בדיקה חייבת להתחיל במילה test_ אחרת pytest לא יזהה אותה!

✔️ פקודת assert - כלי הבדיקה שלנו

ב-pytest יש לנו רק פקודה אחת לבדיקה: assert. כל מה שצריך זה ביטוי Python רגיל שמחזיר True או False.

מה רוצים לבדוק הביטוי ב-pytest דוגמה
a שווה ל-b assert a == b assert 5 == 2+3
a לא שווה ל-b assert a != b assert 5 != 6
x הוא True assert x assert 5 > 3
x הוא False assert not x assert not (5 < 3)
a נמצא ב-b assert a in b assert 3 in [1,2,3]
x הוא None assert x is None assert x is None
נזרקת חריגה pytest.raises(Error) pytest.raises(ZeroDivisionError)

                # קובץ: statistics.py

def calculate_average(numbers):
    """
    מחשבת ממוצע של רשימת מספרים
    מחזירה None אם הרשימה ריקה
    """
    if not numbers:
        return None
    
    if not all(isinstance(n, (int, float)) for n in numbers):
        raise ValueError("כל האיברים חייבים להיות מספרים")
    
    return sum(numbers) / len(numbers)

🎯 המשימה שלך:

כתוב בדיקות לפונקציה הבאה שמחשבת עצרת (factorial) של מספר: 

                # קובץ: math_functions.py

def factorial(n):
    """
    מחשבת עצרת של מספר n
    עצרת של n = n * (n-1) * (n-2) * ... * 1
    דוגמה: factorial(5) = 5 * 4 * 3 * 2 * 1 = 120
    """
    if not isinstance(n, int):
        raise TypeError("הקלט חייב להיות מספר שלם")
    
    if n < 0:
        raise ValueError("אין עצרת למספר שלילי")
    
    if n == 0 or n == 1:
        return 1
    
    result = 1
    for i in range(2, n + 1):
        result *= i
    return result

  • ✅ קלט לא תקין: מספרים שליליים, מספרים עשרוניים, מחרוזות

⭐ עקרונות חשובים לבדיקות טובות

זכור: מטרת הבדיקות היא למצוא בעיות! כשבדיקה נכשלת - זה טוב, כי מצאנו באג לפני שהמשתמש נתקל בו! 🎯

🎓 סיכום - מה למדנו?

✨ הנקודות החשובות:

  1. pytest - ספרייה חיצונית ופופולרית לבדיקות ב-Python
  2. פונקציות רגילות - בלי מחלקה ובלי ירושה, רק def שמתחיל ב-test_
  3. assert - פקודה אחת ויחידה, עם כל ביטוי Python שתרצי
  4. בודקים מקרים רגילים, מקרי קצה, גבולות וחריגות (עם pytest.raises)
  5. כל בדיקה עצמאית ובודקת דבר אחד
  6. בדיקות = תיעוד - הן מראות איך הקוד אמור לעבוד

🎯 התבנית הבסיסית לזכירה:

                    import pytest

def test_normal_case():
    # בדיקת מקרה רגיל
    result = my_function(valid_input)
    assert result == expected_output

def test_edge_case():
    # בדיקת מקרה קצה
    result = my_function(edge_input)
    assert result == expected_output

def test_invalid_input():
    # בדיקת קלט לא תקין
    with pytest.raises(SomeError):
        my_function(invalid_input)
