להבין מהו פרוטוקול TLS, איך הוא עובד, ואיך להשתמש בו בפרויקטי סיום בתכנות וסייבר

מה זה TLS?

הגדרה

TLS = פרוטוקול להצפנת תקשורת בין שני צדדים (לקוח ↔ שרת)

שלושת העקרונות המרכזיים:

🔐 הצפנה (Encryption)

המידע מוצפן כך שגם אם מישהו "מצותת" לתנועה ברשת - הוא לא יכול לקרוא את התוכן

✅ אימות (Authentication)

הלקוח יודע בוודאות שהוא מדבר עם השרת האמיתי ולא עם מתחזה

🛡️ שלמות (Integrity)

אי אפשר לשנות את המידע בדרך מבלי לגלות את ההפרה

💡 דוגמה מהחיים

כל אתר שמתחיל ב-https:// משתמש ב-TLS!

איך TLS עובד? - 5 שלבים

1

הלקוח מבקש להתחבר

"שלום שרת, רוצה ליצור חיבור מאובטח"

2

השרת שולח תעודה דיגיטלית (Certificate)

בתוכה יש:

  • מפתח ציבורי (Public Key)
  • שם השרת

✅ אם התעודה אמיתית - ממשיכים

4

יצירת מפתח משותף

הלקוח מייצר מפתח סימטרי ושולח אותו לשרת מוצפן עם המפתח הציבורי → רק השרת יכול לפענח!

תקשורת מוצפנת

עכשיו יש לשני הצדדים מפתח סימטרי משותף - כל התקשורת מוצפנת במהירות וביעילות! 🚀

תעודות מקומיות vs. תעודות מאושרות

תכונה תעודה מקומית (Self-Signed) תעודה מאושרת (CA-Signed)
מחיר 🆓 חינם 💰 בתשלום / חינם (Let's Encrypt)
יצירה ⚡ מיידי - שורת פקודה אחת ⏱️ דורש תהליך אימות
אמון דפדפנים ❌ הדפדפן מציג אזהרה ✅ הדפדפן סומך אוטומטית
שימוש 📚 פיתוח, למידה, בדיקות מקומיות 🌐 אתרים ציבוריים, production
רמת הצפנה 🔒 זהה! הצפנה מלאה 🔒 זהה! הצפנה מלאה
מתאים לפרויקט סיום ✅ מצוין! ⭐ מצוין++ (אם אפשרי)

איך מקבלים תעודות?

🔧 תעודה מקומית (Self-Signed) - לפרויקטי לימוד

פקודה אחת ב-Terminal/PowerShell/WSL:
openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
    -keyout key.pem -out cert.pem

זה יוצר שני קבצים:

  • cert.pem - התעודה הציבורית
  • key.pem - המפתח הפרטי

🌐 תעודה מאושרת - לפרויקטים אמיתיים

app = Flask(__name__)

@app.get("/")
def home():
    return {"status": "secure connection", "message": "TLS פעיל!"}

def get_users():
    return {"users": ["אליס", "בוב", "צ'רלי"]}

if __name__ == "__main__":
    app.run(
        host="0.0.0.0", 
        port=5000,

    )

שילוב TLS ב-Sockets - רמה 2

🔌 מתאים לפרויקטי TCP ישירים:

צ'אט, שרת-לקוח מותאם אישית, משחקים מרובי משתתפים

קוד שרת עם TLS:

secure_server.py
import ssl
import socket

# יצירת הקשר SSL
context = ssl.SSLContext(ssl.PROTOCOL_TLS_SERVER)
context.load_cert_chain(certfile="cert.pem", keyfile="key.pem")

# יצירת סוקט רגיל
with socket.socket(socket.AF_INET, socket.SOCK_STREAM) as sock:
    sock.bind(("0.0.0.0", 5001))
    sock.listen(5)
    print("🔒 שרת מאובטח עם TLS רץ על פורט 5001...")
    
    # עטיפת הסוקט ב-SSL
    with context.wrap_socket(sock, server_side=True) as ssock:
        conn, addr = ssock.accept()
        print(f"✅ חיבור מוצפן מ: {addr}")
        conn.send("Hello over TLS! שלום מוצפן!".encode('utf-8')
        
        data = conn.recv(1024)
        print(f"📨 התקבל מוצפן: {data.decode()}")
        conn.close()

קוד לקוח עם TLS:

secure_client.py
import ssl
import socket

# יצירת הקשר SSL ברירת מחדל
context = ssl.create_default_context()
context.check_hostname = False  # לתעודות מקומיות
context.verify_mode = ssl.CERT_NONE  # לתעודות מקומיות

# התחברות לשרת
with socket.create_connection(("localhost", 5001)) as sock:
    with context.wrap_socket(sock, server_hostname="localhost") as ssock:
        print("🔒 מחובר לשרת בצורה מאובטחת!")
        
        # קבלת הודעה מוצפנת
        data = ssock.recv(1024)
        print(f"📨 התקבל: {data.decode()}")
        
        # שליחת הודעה מוצפנת
        ssock.send("Hi server! היי שרת!".encode('utf-8')

⚠️ שימו לב

ה-verify_mode = ssl.CERT_NONE מתאים רק לתעודות מקומיות בפיתוח!

טעויות נפוצות ואיך להימנע מהן ⚠️

❌ טעות #1: שימוש ב-verify_mode = CERT_NONE בפרודקשן

הבעיה: זה משאיר אתכם חשופים למתקפות MITM!

הפתרון: השתמשו בזה רק בפיתוח מקומי. בפרויקט אמיתי - תמיד אמתו תעודות!

❌ טעות #2: שכחתם את הקבצים cert.pem ו-key.pem

הבעיה: השרת לא יעלה!

הפתרון: ודאו שהקבצים נמצאים באותה תיקייה של הקוד, או ציינו נתיב מלא.

❌ טעות #3: ניסיון גישה ב-http במקום https

הבעיה: השרת לא יענה!

הפתרון: זכרו - אחרי הפעלת TLS צריך לגשת ל-https:// ולא ל-http://

❌ טעות #4: שיתוף המפתח הפרטי (key.pem)

הבעיה: זה כמו לשתף סיסמה - כל מי שיש לו את המפתח יכול להתחזות לשרת!

  • Flask + TLS = קל ומהיר ליישום
  • Sockets + TLS = מאובטח כל תקשורת TCP
