שכבת הרשת ושכבת הקו: IP / ICMP / DHCP / ARP

בשלבים האחרונים עברנו על שכבות גבוהות (אפליקציה, תעבורה). הפעם יורדים לשכבות הנמוכות — איפה שמתבררות שאלות בסיסיות כמו "איך המחשב שלי מגלה את כתובת ה-IP שלו", ו"מה קורה כשהשרת לא מגיב כלל".

IP — כתובת, ניתוב, TTL

  • כל מכשיר ברשת מזוהה בכתובת IP — ייחודית (בתוך הרשת המקומית, או גלובלית)
  • פקטת IP עוברת מנתב לנתב (router) בדרך ליעד — לא תמיד באותו מסלול בכל פעם
  • TTL (Time To Live) — מספר שמתחיל בערך מסוים ויורד ב-1 בכל נתב; מגיע ל-0 ➜ הפקטה נזרקת. מונע "פקטות נצחיות" שתועות ברשת

ICMP — ping ואבחון רשת

ICMP הוא פרוטוקול עזר של שכבת הרשת, לא לשליחת מידע אפליקטיבי אלא לדיווח על מצב הרשת.

Echo Request
→
השרת
→
Echo Reply

זו בדיוק פקודת ping שמכירים מהטרמינל.

  • ping מצליח = יש Reply, ואפשר למדוד זמן תגובה (latency)
  • ping נכשל / timeout = או שהשרת לא זמין, או שחומת אש חוסמת ICMP
  • הודעת TTL Exceeded — שימושית לכלי traceroute (לא חלק מהשלב, רק לידיעה)

תזכורת: Ethernet ו-MAC

  • שכבת הקו (Data Link) עובדת עם כתובת MAC — 6 בתים, ייחודית לכל כרטיס רשת
  • Ethernet הוא הפרוטוקול שמעביר frames ברשת המקומית, בעזרת כתובות MAC
  • בניגוד ל-IP (שמשתנה כשעוברים רשת), MAC נשאר קבוע לכרטיס הרשת עצמו

זה אותו שדה שמצאתם ב-Wireshark בשלב 10: Ethernet II, Src: ..., Dst: ...

מה זה Gateway?

ה-Gateway (נתב ברירת המחדל) הוא המכשיר שדרכו עוברת כל תעבורה שיוצאת מהרשת המקומית שלכם לאינטרנט.

  • כל מחשב ב-LAN "יודע" מי ה-Gateway שלו (כתובת IP + MAC)
  • תזכורת משלב 10-11: ARP spoofing מתבסס בדיוק על שיקור ל-MAC של ה-Gateway — אם מישהו "משתלט" על הזהות הזו, כל התעבורה שלכם לאינטרנט עוברת דרכו
  • לכן שינוי לא צפוי ב-MAC של ה-Gateway הוא דגל אדום משמעותי

DHCP — איך מקבלים IP אוטומטית

כשמחשב מתחבר לרשת, הוא לא "יודע" איזו כתובת IP פנויה. DHCP פותר את זה בארבעה שלבים:

DISCOVER
→
OFFER
→
REQUEST
→
ACK

המחשב "צועק" DISCOVER לכל הרשת, שרת ה-DHCP מציע (OFFER) כתובת, המחשב מבקש אותה רשמית (REQUEST), והשרת מאשר (ACK).

בתוך ה-ACK מגיעים גם כתובת ה-Gateway וכתובות שרתי ה-DNS — לא רק ה-IP עצמו!

⏸ תחנת עצירה

מה עושים השבוע?

פותחים את דף העבודה — מודול דיאגנוסטיקה לשכבות נמוכות, וביחד עם הצוות:

  • בוחרים אחת מחמש האפשרויות, בהתאם לסוג הפרויקט שלכם
  • מתכננים את הקלט/פלט של המודול (מה הוא בודק, מה הוא מחזיר)
  • ממשים גרסה ראשונית, ומגישים קובץ JSON עם התשובות

בדיקת הבנה

למה שינוי MAC של ה-Gateway נחשב חשוד, יותר משינוי MAC רגיל?

כי MAC לא אמור להשתנות אף פעם בשום מצב
כי כל התעבורה לאינטרנט עוברת דרך ה-Gateway, ולכן השתלטות עליו = השתלטות על כל התעבורה
כי ה-Gateway היחיד שמשתמש בכתובת IPv6
