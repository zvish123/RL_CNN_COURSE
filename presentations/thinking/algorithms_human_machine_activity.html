🧪 בדיקת האלגוריתם

האלגוריתם שלכם צריך לעבור את כל הבדיקות הללו:

✅ בדיקת אותיות רגילות
A + 3 → D, M + 5 → R
🔄 בדיקת מעבר סוף האלפבית
X + 5 → C, Z + 1 → A
🚫 בדיקת רווחים וסימנים
"HELLO!" → "KHOOR!" (! נשאר כמו שהוא)
🔢 בדיקת הזזות גדולות
A + 27 → B (כי 27 = 26 + 1)
🎯 האלגוריתם המושלם להצפנת קיסר
🤖 אלגוריתם הצפנת קיסר (גרסה מדויקת):
קלט: מחרוזת (המילה), מספר (ההזזה)
פלט: מחרוזת מוצפנת

שלבי האלגוריתם:
1. צור מחרוזת ריקה בשם "תוצאה"
2. עבור על כל תו במחרוזת הקלט:
3.    אם התו הוא אות אנגלית (A-Z או a-z):
4.      אם זו אות גדולה (A-Z):
5.        מיקום = (תו - 'A')
6.        מיקום_חדש = (מיקום + הזזה) % 26
7.        תו_חדש = 'A' + מיקום_חדש
8.      אם זו אות קטנה (a-z):
9.        מיקום = (תו - 'a')
10.       מיקום_חדש = (מיקום + הזזה) % 26
11.       תו_חדש = 'a' + מיקום_חדש
12.   אחרת (רווח, סימן פיסוק, מספר):
13.     תו_חדש = התו המקורי (ללא שינוי)
14.   הוסף תו_חדש לתוצאה
15. החזר תוצאה
🎬 הדגמה: "HELLO" עם הזזה +3
שלב 1: תוצאה = "" (מחרוזת ריקה)
שלב 2: תו = 'H' → מיקום = 7 → מיקום_חדש = (7+3)%26 = 10 → 'K'
שלב 3: תו = 'E' → מיקום = 4 → מיקום_חדש = (4+3)%26 = 7 → 'H'
שלב 4: תו = 'L' → מיקום = 11 → מיקום_חדש = (11+3)%26 = 14 → 'O'
שלב 5: תו = 'L' → מיקום = 11 → מיקום_חדש = (11+3)%26 = 14 → 'O'
שלב 6: תו = 'O' → מיקום = 14 → מיקום_חדש = (14+3)%26 = 17 → 'R'
תוצאה סופית: "KHOOR" ✓
🔄 הדגמה: מעבר סוף אלפבית - "XYZ" עם הזזה +5
X → מיקום = 23 → (23+5)%26 = 28%26 = 2 → 'C'
Y → מיקום = 24 → (24+5)%26 = 29%26 = 3 → 'D'
Z → מיקום = 25 → (25+5)%26 = 30%26 = 4 → 'E'
תוצאה: "CDE" ✓
🎉 מה למדנו על אלגוריתמים?
  • 🎯 דיוק מוחלט: כל הוראה חייבת להיות חד-משמעית - "בערך" לא קיים!
  • 🔍 כיסוי מלא: האלגוריתם חייב לטפל בכל המקרים האפשריים
  • 🧪 בדיקה יסודית: צריך לבדוק מקרי קצה (כמו מעבר סוף אלפבית)
  • 🤖 חשיבה כמו מחשב: המחשב עושה רק מה שאומרים לו - בדיוק!
  • 🔧 שלבי ביניים: פירוק המשימה לפעולות קטנות ופשוטות
  • ♻️ שימוש חוזר: אלגוריתם טוב עובד על כל הקלטים מהסוג הנכון
💡 המסר המרכזי:
אלגוריתם טוב = רצף הוראות מדויק שפותר בעיה עבור כל קלט אפשרי
