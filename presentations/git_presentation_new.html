👤הגדרה ראשונית של Git

פתח Terminal ב-PyCharm (Alt+F12) והקלד:

# הגדרת השם שלך
git config --global user.name "השם שלך"

# הגדרת האימייל שלך (אותו אחד מ-GitHub)
git config --global user.email "email@example.com"

# בדיקה שהכל עבד
git config --list

למה זה חשוב?

  • Git צריך לדעת מי אתה
  • כל שינוי יירשם עם השם שלך
  • המורה יוכל לראות מי עשה מה בפרויקט

🎉Commit הראשון שלך!

מה זה Commit?

  • "צילום מצב" של הקוד שלך ברגע מסוים
  • כמו שמירה עם תיאור מה שעשית

איך עושים Commit ב-PyCharm?

1. פתח חלון Commit

Ctrl+K (או כפתור הקומיט הירוק)

2. בחר קבצים

בחר אילו קבצים לכלול (בדרך כלל הכל)

3. כתוב הודעה

הודעה ברורה: "יצירת פרויקט ראשוני"

4. שמור

לחץ Commit

הודעות Commit טובות הודעות Commit גרועות
✅ "הוספת מחשבון בסיסי" ❌ "שינויים"
✅ "תיקון באג בפונקציית חיבור" ❌ "סתם"
✅ "הוספת בדיקת שגיאות" ❌ "עדכון"

🌐חיבור ל-GitHub (המאגר המרוחק)

למה GitHub?

  • גיבוי הקוד בענן ☁️
  • שיתוף עם המורה והכיתה 🤝
  • עבודה מכמה מחשבים 💻

איך לחבר?

1. פתח שיתוף

VCS → Share Project on GitHub

2. התחבר

התחבר עם פרטי GitHub שלך

3. הגדר מאגר

תן שם למאגר (Repository)

4. בחר גישה

בחר אם פרטי (Private) או ציבורי (Public)

מה קרה?

  • הקוד שלך עכשיו גם ב-GitHub
  • יש לך כתובת URL לפרויקט
  • המורה יכול לגשת (אם נתת הרשאה)

📅העבודה היומיומית עם Git

🌅 תחילת עבודה (כל יום)

  1. פתח PyCharm
  2. Ctrl+T (Pull) - קבל עדכונים אחרונים
  3. התחל לעבוד!

⚡ במהלך העבודה

  • כתוב קוד, בדוק שהוא עובד
  • כל 15-30 דקות: Ctrl+K (Commit)
  • הודעות ברורות למה שעשית

🌙 סיום עבודה (כל יום)

  1. Ctrl+K - Commit אחרון
  2. Ctrl+Shift+K (Push) - שלח ל-GitHub
  3. עכשיו הכל מוגב בענן! ✅

חלון Git ב-PyCharm:

Alt+9 - פתיחת חלון Git
• רואה את כל השינויים
• יכול לבטל שינויים לא רצויים
• צפייה בהיסטוריה

⚔️פתרון קונפליקטים (התנגשויות)

מה זה קונפליקט?

כאשר שניכם משנים את אותה שורה בקוד!

איך נראה קונפליקט?

<<<<<<< HEAD
print("השינוי שלי")
=======
print("השינוי של חבר שלי")
>>>>>>> branch-name

איך לפתור ב-PyCharm?

1. זיהוי הקונפליקט

PyCharm יראה לך את הקונפליקט

2. פתיחת כלי המיזוג

לחץ Resolve Conflicts

3. בחירת פתרון

  • Accept Yours - השינוי שלך
  • Accept Theirs - השינוי של החבר
  • Merge - שלב את השניים

4. שמירה

שמור ו-Commit

איך למנוע קונפליקטים?

  • תאמו מי עובד על מה
  • עבדו על קבצים שונים כשאפשר
  • Pull תכוף!

🚨פתרון בעיות נפוצות

❌ "לא יכול לעשות Push"

  1. תמיד Pull לפני Push
  2. בדוק שיש חיבור אינטרנט
  3. ודא שיש הרשאות ל-Repository

😱 "אבדו לי קבצים!"

  1. בדוק Local History: קליק ימני → Local History
  2. חפש בחלון Git → Log
  3. אפשר לשחזר כל Commit קודם

💥 "הקוד לא עובד אחרי Merge"

  1. בדוק שהיו קונפליקטים
  2. הרץ את הקוד וראה מה השגיאות
  3. תקן שגיאות תחביר

🤦 "שכחתי לעשות .gitignore"

# ב-Terminal של PyCharm:
git rm -r --cached __pycache__
git rm -r --cached .idea

פקודות חירום ב-Terminal:

# ביטול כל השינויים המקומיים
git reset --hard HEAD

# ביטול Commit אחרון (בזהירות!)
git reset HEAD~1

# צפייה במצב הנוכחי
git status

🌳עבודה עם Branches (ענפים)

מה זה Branch?

  • "עותק עבודה" של הקוד
  • מאפשר לעבוד על תכונות חדשות בנפרד
  • אפשר למזג בחזרה כשמוכן

מתי להשתמש?

  • עבודה על תכונה גדולה שלוקחת זמן
  • ניסויים שעלולים לשבור את הקוד
  • כל חבר צוות עובד על Branch נפרד

איך ב-PyCharm?

1. יצירת Branch חדש

Git → New Branch

2. תן שם

feature-calculator או bugfix-login

3. עבוד כרגיל

Commit ו-Push כמו תמיד

4. מיזוג

Git → Merge into main
דוגמה מעשית:
main (הענף הראשי)
├── feature-login (תכונת התחברות)
├── feature-calculator (מחשבון)
└── bugfix-crash (תיקון קריסה)

🧮דוגמה מעשית - פרויקט מחשבון

שלב 1: יצירת הפרויקט

# main.py
def add(a, b):
    return a + b

def subtract(a, b):
    return a - b

if __name__ == "__main__":
    print("מחשבון פשוט")
    print(f"5 + 3 = {add(5, 3)}")
    print(f"5 - 3 = {subtract(5, 3)}")

Commit: "יצירת מחשבון בסיסי עם חיבור וחיסור"

שלב 2: הרחבה

# הוספת פונקציות
def multiply(a, b):
    return a * b

def divide(a, b):
    if b == 0:
        return "שגיאה: חלוקה באפס"
    return a / b

Commit: "הוספת כפל וחלוקה עם טיפול בשגיאות"

שלב 3: ממשק משתמש

def main():
    while True:

        print("4. חלוקה")
        print("5. יציאה")
        
        choice = input("בחר פעולה: ")
        # המשך הקוד...

Commit: "הוספת ממשק משתמש אינטראקטיבי"

👥עבודה בצוות - דוגמה מעשית

חלוקת משימות:

אליס 👩‍💻

מודול התחברות (auth.py)

בוב 👨‍💻

מודול ניהול סטודנטים (students.py)

  1. אליס יוצרת Repository ומוסיפה את בוב וצ'רלי

  2. Commit + Push בסוף כל יום
  3. כשמוכן - Merge ל-main

הימנעות מקונפליקטים:

  • כל אחד בקובץ שונה

⭐שיטות עבודה מומלצות (Best Practices)

הודעות Commit טובות ✅ הודעות Commit גרועות ❌
"הוסף פונקציה לחישוב ממוצע" "עדכונים"
"תקן באג בבדיקת סיסמה" "תיקונים"
"עדכן README עם הוראות התקנה" "סתם"

מתי לעשות Commit:

✅ כדאי

  • אחרי שכל פונקציה חדשה עובדת
  • אחרי תיקון באג
  • לפני שמתחילים משימה חדשה

❌ לא כדאי

  • כשהקוד שבור או לא עובד
  • קבצים זמניים
  • שינויים חצי גמורים

מה לכלול ב-Repository:

כן ✅ לא ❌
קוד מקור (.py) קבצים שנוצרים אוטומטית (__pycache__)
קבצי תצורה קבצי IDE מיוחדים (.idea)
תיעוד (README) סיסמאות וקבצים רגישים

📝הכנה למבחן/בחינה

פקודות בסיסיות שחשוב לדעת:

פקודה מה עושה ב-PyCharm
git status מה המצב חלון Git
git add הוספה לשלב הבא סימון קבצים בCommit
git commit שמירת שינויים Ctrl+K
git push שליחה לשרת Ctrl+Shift+K
git pull קבלה מהשרת Ctrl+T

מושגים חשובים:

Repository (מאגר)

המקום שבו נשמר הקוד וההיסטוריה

Commit (שמירה)

"צילום מצב" של הקוד ברגע מסוים

Branch (ענף)

עותק עבודה נפרד של הקוד

Merge (מיזוג)

איחוד שני ענפים

תרגול מומלץ:

  1. צור פרויקט חדש
  2. עשה כמה Commits
  3. צור Branch חדש
  4. מזג בחזרה
  5. שתף עם חבר ותתרגלו conflict
