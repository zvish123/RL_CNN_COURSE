מצגת 01 · מהו gr.update

למה בכלל צריך את זה?

ב-Gradio, כל קומפוננט (Textbox, Button, Column וכו') נוצר פעם אחת בעת טעינת הדף. אבל לפעמים אנחנו רוצים לשנות תכונות שלו בזמן ריצה — בתגובה לפעולת המשתמש.

  • הסתרת/הצגת קומפוננט לפי תנאי
  • שינוי הטקסט שמוצג בשדה
  • נעילת input אחרי שליחה
  • שינוי כותרת, placeholder, סגנון כפתור
  • כל שילוב של אלה יחד — בבת אחת
הרעיון הבסיסי
gr.update() מחזיר אובייקט שאומר לגרדיו: "עדכן את הקומפוננט הזה, ורק את הפרמטרים שציינתי." כל שאר התכונות נשמרות כמות שהן.
⚠ גרסאות חדשות
בגרדיו המודרני אפשר גם להחזיר את הקומפוננט עצמו: gr.Textbox(visible=False) — שקול לחלוטין ל-gr.update(visible=False).
❌ בלי gr.update
# מחזירים רק value
def login(user, pw):
    if ok:
        return "welcome"
    return "error"
# אי אפשר לשנות visible!
✅ עם gr.update
def login(user, pw):
    if ok:
        return gr.update(
            visible=False)
    return gr.update(
        visible=True,
        value="❌ שגיאה")
מצגת 02 · פרמטרים אוניברסליים

פרמטרים שעובדים על כל קומפוננט

פרמטר טיפוס משמעות דוגמה
visible bool האם הקומפוננט מוצג בכלל. False = נעלם לחלוטין, לא תופס מקום בעמוד visible=False
value any הערך הנוכחי — הטקסט בתיבה, הבחירה ב-Dropdown, המצב של Checkbox וכו' value="שלום"
label str הכותרת המוצגת מעל הקומפוננט. אפשר לשנות דינמית לפי הקשר label="שגיאה"
interactive bool True = המשתמש יכול לערוך. False = read-only (שקרא בלבד). שימושי לנעילת שדות אחרי שליחה interactive=False
show_label bool האם בכלל להציג את שורת ה-label. False = הקומפוננט ללא כותרת show_label=False
scale int גודל יחסי בתוך gr.Row. scale=2 = פי שניים רחב מ-scale=1 scale=2
elem_classes list[str] class-ים של CSS שיתווספו לאלמנט — לעיצוב מותאם אישית elem_classes=["red"]
info str טקסט עזרה קטן מתחת ל-label. תומך Markdown. מעולה להוספת הסברים info="3-8 תווים"
💡 כלל זהב
כל פרמטר שהקומפוננט מקבל ב-constructor — אפשר לעדכן אותו ב-gr.update. אם gr.Textbox(lines=3) קיים, אז gr.update(lines=5) יעבוד.

מצגת 04 · gr.skip וטיפים מתקדמים

gr.skip() — לא לשנות כלום

לפעמים יש לנו פונקציה עם מספר outputs, אבל בתנאי מסוים אנחנו לא רוצים לשנות חלק מהם. במקום להחזיר gr.update() ריק, יש פתרון אלגנטי יותר: 

from gradio import skip

def my_fn(x):
    if x == "same":
        return gr.skip()  # ← שמור הכל
    return gr.update(value="שינוי!")
מתי להשתמש ב-skip
כשיש לך טאפל של returns וחלקם לא צריכים שינוי: return gr.skip(), gr.update(...), gr.skip()

⚡ MODERN SYNTAX

✅ גרדיו 4+ — החזרת קומפוננט ישירות
# שקול לחלוטין ל-gr.update
return gr.Textbox(
    lines=5,
    visible=True,
    value="hello"
)
📌 החזרת dict — לדלג על outputs
# עדכון לפי שם, לא לפי סדר
return {
    err: gr.update(visible=True),
    btn: gr.update(interactive=False)
}
⚠ פח נפוץ
לא לבלבל בין gr.update() ריק (שומר ערכים קיימים) לבין gr.update(value="") שמנקה את הערך. ריק ≠ מחיקה.

תבניות נפוצות ב-gr.update

🔐 מסך Login — החלפת panels 
def login(username, password):
    if username not in USERS:
        return gr.update(visible=True),  # login_panel נשאר
               gr.update(visible=False), # main_panel נסתר
               gr.update(visible=True, value="❌ משתמש לא נמצא")  # err מוצג
    if check_password(username, password):
        return gr.update(visible=False), \
               gr.update(visible=True),  \
               gr.update(visible=False, value="")
🔄 נעילת ממשק בזמן עיבוד 
def process(data):
    # בהתחלה — נועל הכל
    yield gr.update(interactive=False, value="⏳ מעבד...")

    result = heavy_computation(data)

    # בסוף — משחרר
    yield gr.update(interactive=True, value="✅ סיום")

def toggle_mode(current_mode):
    if current_mode == "edit":
        return gr.update(value="💾 שמור", variant="primary")
    return gr.update(value="✏️ ערוך", variant="secondary")
