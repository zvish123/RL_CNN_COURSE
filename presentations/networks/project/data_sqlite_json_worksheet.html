תזכורת תאורטית3 דרכים לשמור נתונים, ומתי משתמשים בכל אחת

שיטהמתי מתאימהחיסרון מרכזי
קובץ טקסט פשוט נתונים מאוד פשוטים, לוג חד-כיווני (כמו server.log משלב 8) אין מבנה, קשה לחפש/לסנן בלי לקרוא הכול
קובץ JSON נתונים מבניים עם קינון (אובייקטים בתוך אובייקטים), כמו האירוע שבדוגמה כדי לסנן/לחפש צריך לטעון את כל הקובץ לזיכרון
SQLite (בסיס נתונים) הרבה רשומות, צריך לשלוף/לסנן/למיין לפי שדות במהירות (לדוגמה: "כל האירועים בחומרה גבוהה מהיום") צריך להגדיר מבנה טבלה מראש (סכמה)
דוגמת האירוע מהפרויקט: 
{
  "time": "2026-06-25 10:43:12",
  "client_id": "agent_03",
  "event_type": "DNS_QUERY",
  "severity": "medium",
  "details": {
    "domain": "example.com",
    "count": 12
  }
}

שימו לב: details הוא אובייקט מקונן — זה בדיוק סוג המידע ש-JSON מתאר בקלות, אבל ש-SQLite (כמו כל בסיס נתונים רלציוני) לא תומך בו ישירות בעמודה רגילה.

שאלת הבנה 0.1 (בדיקה אוטומטית)

איך נשמור את שדה ה-details בטבלת SQLite?

✅ נכון. הפתרון הפרקטי ביותר (ושמתאים לפרויקט שלכם): json.dumps(details) לפני השמירה, ו-json.loads(row['details']) כששולפים. כך עמודת details בטבלה היא טקסט רגיל מנקודת מבט SQLite, אבל היא "מבינה" מבנה מקונן מנקודת מבט הקוד שלכם. (אפשרות 3 גם תקפה אם יש לכם תמיד את אותם שני שדות ב-details, אבל פחות גמישה אם event_type שונים מצריכים details שונה).

משימה 2.1

השלימו את מחלקת SqliteDb

מאפיינים: connection, cursor. מתודות: __init__, create_table, insert, select, update, delete, commit.

▶ קוד זה דורש sqlite3 אמיתי — להריץ ב-PyCharm/IDE מקומי
⚠️ לעולם אל תכניסו ערכים לשאילתה עם f-string ישירות (לדוגמה f"...WHERE name='{user_input}'") — זו פגיעות אבטחה קלאסית בשם SQL Injection. תמיד תעבירו את הערכים בנפרד עם ? placeholders: cursor.execute("INSERT INTO t VALUES (?, ?)", (val1, val2)). שימו לב שזה רלוונטי גם ב-VaultGuard — כל מה שמגיע מהלקוח (כולל בקשת LOGIN) הוא קלט חיצוני שאסור "להדביק" לתוך שאילתה כטקסט גולמי.

משימה 4.2 (בדיקה אוטומטית)

מה הבעיה בשורה where_clause=f"severity='{severity}'" בקוד שלמעלה?

✅ נכון. SQL Injection רלוונטי גם ל-SELECT, לא רק ל-INSERT/UPDATE! אם severity מגיע ממקור שלא בשליטתכם המלאה, צריך לבנות את ה-where_clause עם placeholders ולא f-string, או לפחות לוודא (validate) שהערך הוא אחד מתוך רשימה סגורה (whitelist) של ערכים מותרים לפני שמרכיבים את השאילתה.
