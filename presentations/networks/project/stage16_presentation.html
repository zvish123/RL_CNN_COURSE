ממשק משתמש
לוח בקרה למנהל המערכת

בנינו שרת, אבטחה, ניטור והתראות. השלב האחרון: לבנות מסך שמציג את כל זה למנהל בצורה ברורה ושימושית — בלי לבלבל בין הלוגיקה לבין התצוגה.

למה זה השלב האחרון, ולא הראשון?

בכל שלב קודם בנינו לוגיקה: קוד שעושה דברים (מאמת, מצפין, מנטר, שולח התראות). הממשק הוא השכבה שמציגה את מה שהלוגיקה כבר מייצרת.

  • אם בונים ממשק לפני שיש לוגיקה — מציגים נתונים מדומים שלא קיימים באמת
  • אם בונים ממשק בלי הפרדה מהלוגיקה — כל שינוי בעיצוב מסכן את הקוד שעובד
  • לכן: לוגיקה קודם (שלבים 1–15), ממשק שמציג אותה בסוף (שלב 16)

העיקרון המרכזי: הפרדה בין לוגיקה לממשק

הלוגיקה (Backend)

קוד שבודק מי מחובר, מה קרה ברשת, אם יש התראה. לא יודע איך זה ייראה על המסך.

🖥️

הממשק (Frontend / Template)

לא מחליט מה קרה ברשת — הוא רק מציג את הנתונים שהלוגיקה מסרה לו.

בפלאסק: ה-route מחזיר נתונים, ה-template (HTML) מציג אותם. אם רוצים לשנות צבע כפתור — לא נוגעים בקוד שמזהה התקפות.

בהירות

האם הדבר החשוב ביותר (התראה!) בולט מיד, או טובע בתוך נתונים?

  • זיהוי מיידי — מצב השרת (תקין/בעיה) נראה במבט ראשון, לא דורש חיפוש

כפתורי פעולה — רק מה שמותר

בלוח בקרה למנהל, לא כל פעולה מוצגת לכל אחד. כפתור = הרשאה. אם המנהל לא רשאי לבצע פעולה — הכפתור לא קיים, לא "אפור וחסום" (זה מבלבל).

  • "נתק לקוח" — דורש הרשאת admin, ולא operator
  • "חסום IP" — פעולה קריטית → דורשת חלון אישור (מניעת טעויות)
  • "רענן נתונים" — פעולה בטוחה, ללא צורך באישור

ברמת הקוד: ה-route בודק הרשאה בשרת (לא רק מסתיר כפתור ב-HTML) — אחרת אפשר לעקוף בקלות.

הקוד: route שמפריד נתונים מתצוגה

from flask import Flask, render_template

app = Flask(__name__)

# הלוגיקה (שלבים קודמים) מחזירה רק נתונים - לא HTML
def get_connected_clients():
    return [{"name": "dana_92", "status": "authenticated"}]

def get_recent_alerts():
    return [{"text": "5 login נכשלו", "level": "high"}]

# הממשק (route) רק אוסף ומעביר ל-template
@app.route('/dashboard')
def dashboard():
    return render_template(
        'dashboard.html',
        clients=get_connected_clients(),
        alerts=get_recent_alerts(),
        server_status="online"
    )

שימו לב: ה-route לא בודק סיסמאות ולא מנתח תעבורה — הוא רק קורא מהפונקציות של השלבים הקודמים, ומעביר ל-template.

ה-template: רק תצוגה, בלי לוגיקת אבטחה

<div class="clients-box">
  <h3>לקוחות מחוברים</h3>
  {% for c in clients %}
    <div>{{ c.name }} - {{ c.status }}</div>
  {% endfor %}
</div>

קוד ה-Jinja בתבנית רק עובר על רשימה ומציג אותה. הוא לא בודק מי מורשה, לא מצפין, לא שואל "האם זה IP חשוד" — כל זה כבר הוכרע בפייתון לפני שהגיע לכאן. אם תרצו לשנות צבע רקע — תיגעו רק כאן, לא בקוד האבטחה.

בדיקת הבנה

מנהל לוחץ "חסום IP" בלוח הבקרה. איפה צריכה להתבצע בדיקת ההרשאה של המנהל?

רק בקוד ה-JavaScript שמסתיר/מציג את הכפתור
