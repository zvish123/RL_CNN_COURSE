🚀 מהי fastai?

fastai היא ספריית Python שמאפשרת לכם לבנות מודלים של בינה מלאכותית לזיהוי תמונות בקוד פשוט ומינימלי.

🎓 הסבר המושג: למידה עמוקה (Deep Learning)

למידה עמוקה היא שיטה שבה המחשב לומד לזהות דפוסים בתמונות (או בנתונים אחרים) על ידי "הסתכלות" על המון דוגמאות. בדיוק כמו שילד לומד לזהות חתול אחרי שראה הרבה חתולים - כך גם המחשב.

למה fastai מיוחדת?

  • פשוטה לשימוש - קוד קצר ומובן, מתאים למתחילים
  • חזקה - מבוססת על PyTorch ומכילה best practices מובנים
  • מהירה - ניתן לקבל תוצאות מרשימות תוך דקות
  • גמישה - אפשר להתאים אותה למגוון משימות

💭 אנלוגיה מהחיים:

תארו לעצמכם שאתם לומדים לנהוג. אפשר ללמוד כל פרט טכני על איך המנוע עובד, או פשוט ללמוד את הבסיס ולהתחיל לנהוג. fastai היא כמו מורה טוב לנהיגה - היא מלמדת אתכם את מה שצריך כדי להתחיל, בלי להכביד עליכם בפרטים מיותרים.

🎯 דוגמה למה אפשר לבנות:

  • מסווג שמזהה בעלי חיים בתמונות (חתול, כלב, ציפור...)
  • מזהה רגשות בפנים (שמח, עצוב, כועס...)
  • סורק תמונות רפואיות לזיהוי מחלות
  • מזהה סוגים שונים של פירות או ירקות
  • בודק איכות מוצרים על פס ייצור

⚙️ איך מתחילים לעבוד?

📝 הסבר המושג: Notebook

Jupyter Notebook הוא סביבת עבודה אינטראקטיבית שבה אפשר לכתוב קוד Python, להריץ אותו, ולראות תוצאות מיד. זה כמו מחברת דיגיטלית שמשלבת קוד, טקסט והסברים.

☁️ מהו Google Colab?

Google Colab (או Colaboratory) הוא שירות חינמי של גוגל שמאפשר להריץ Jupyter Notebooks ישירות בדפדפן - ללא שום התקנה במחשב!

יתרונות עצומים למתחילים:

  • חינמי לחלוטין - לא צריך לשלם כלום
  • GPU חינמי! - כרטיס מסך מקצועי לאימון מהיר (נצלול לזה בהמשך)
  • אין התקנות - Python, fastai והכל כבר מותקן
  • עובד בכל מקום - צריך רק דפדפן ואינטרנט
  • שמירה אוטומטית - הקבצים נשמרים ב-Google Drive
  • שיתוף קל - כמו Google Docs, אפשר לשתף עם מורים/חברים

⚡ מהו GPU ולמה זה חשוב?

GPU (Graphics Processing Unit) הוא מעבד גרפי שמיועד במקור לעיבוד תמונות ומשחקים, אבל מסתבר שהוא מושלם גם ללמידה עמוקה!

  • מהירות אדירה: GPU מאמן מודלים פי 10-100 מהר יותר מ-CPU רגיל!
  • בחינם ב-Colab: גוגל נותנים לכם גישה ל-GPU מקצועי בחינם
  • דוגמה: אימון שלוקח 2 שעות על המחשב שלכם - יקח 5 דקות על GPU

⚠️ חשוב: אל תשכחו להפעיל GPU! ללא זה האימון יהיה איטי מאוד.

📁 הכנת נתונים

הכנת הנתונים היא השלב הכי חשוב! מודל טוב מתחיל מנתונים מאורגנים כראוי.

🎓 הסבר המושג: DataLoaders

DataLoaders הוא אובייקט שאחראי על טעינת התמונות, חלוקה ל-batches (קבוצות), ביצוע augmentation (שינויים קלים בתמונות), וכל מה שקשור להזנת הנתונים למודל בזמן האימון.

שלוש דרכים נפוצות לארגן נתונים:

אפשרות 1: לפי תיקיות

המבנה הכי פשוט - כל קטגוריה בתיקייה נפרדת:

# מבנה התיקיות:
# data/
#   ├── cats/
#   │   ├── cat1.jpg
#   │   ├── cat2.jpg
#   └── dogs/
#       ├── dog1.jpg
#       └── dog2.jpg

dls = ImageDataLoaders.from_folder(
    path,                  # נתיב לתיקיית הבסיס
    valid_pct=0.2,        # 20% מהנתונים לבדיקה (validation)
    item_tfms=Resize(224),  # שינוי גודל כל התמונות ל-224x224
    bs=32                   # batch size - כמה תמונות בכל פעם
)

אפשרות 2: לפי שם הקובץ

כאשר שם הקובץ מכיל את הקטגוריה:

# שמות קבצים: Persian_123.jpg, Golden_Retriever_456.jpg

dls = ImageDataLoaders.from_name_re(
    path, 
    get_image_files(path),     # מביא את כל התמונות
    pat=r'(.+)_\d+\.jpg$',     # regex שמחלץ את שם הגזע
    item_tfms=Resize(224)
)

# הסבר ה-regex:
# (.+) - תופס את כל התווים עד ה-underscore
# \d+ - מספרים
# \.jpg$ - סיומת jpg

🎓 הסבר המושג: Validation Set

Validation set (מערך בדיקה) הוא חלק מהנתונים ששומרים בצד ולא משתמשים בו לאימון. הוא משמש לבדיקה האם המודל באמת למד או רק "שינן" את התמונות שראה. זה כמו מבחן הפתעה בבית הספר!

🎛️ פרמטרים חשובים בהכנת נתונים

תיאור ערכים מומלצים פרמטר
כמה תמונות בכל קבוצה (batch) שמועברת למודל בפעם אחת 16-64 (תלוי בזיכרון) bs
(batch_size)
אחוז הנתונים ששומרים לבדיקה (ולא לאימון) 0.2 (20%) valid_pct
שינוי גודל התמונות - חייב להיות אחיד 224, 299, 448 item_tfms
(Resize)
שינויים אקראיים בתמונות (סיבוב, חיתוך, תאורה) aug_transforms() batch_tfms

💡 איך לבחור batch size?

  • batch size גדול (64) - מהיר יותר, צריך הרבה זיכרון RAM/GPU
  • batch size קטן (16) - איטי יותר, מתאים למחשבים חלשים
  • כלל אצבע: התחילו מ-32, אם יש שגיאת זיכרון - הקטינו ל-16

🎓 הסבר המושג: Data Augmentation

Data Augmentation הוא טכניקה שיוצרת גרסאות מעט שונות של אותן תמונות (סיבוב קל, חיתוך, שינוי תאורה). זה עוזר למודל ללמוד טוב יותר ולא להיתקע על התמונות המדויקות שראה.

אנלוגיה: אם אתם לומדים לזהות כלבים, עדיף לראות כלבים מזוויות שונות, בתאורות שונות, ובגדלים שונים - כך תוכלו לזהות כל כלב!

קוד מלא עם augmentation:

dls = ImageDataLoaders.from_folder(
    path,
    valid_pct=0.2,
    item_tfms=Resize(224),
    batch_tfms=aug_transforms(
        mult=1.5,          # עוצמת הטרנספורמציות
        do_flip=True,      # היפוך אופקי
        max_rotate=15,    # סיבוב עד 15 מעלות
        max_lighting=0.3  # שינוי תאורה
    ),
    bs=32
)

# הצגת דוגמאות לתמונות עם augmentation
dls.show_batch(max_n=9)

📸 איסוף והכנת תמונות - השלב הקריטי

איכות הנתונים קובעת 80% מהצלחת המודל! תמונות טובות = מודל טוב.

🎓 הסבר המושג: "Garbage In, Garbage Out"

GIGO - עיקרון בסיסי במדעי המחשב: אם נכניס נתונים גרועים, נקבל תוצאות גרועות. המודל הכי מתוחכם לא יכול לפצות על תמונות רעות!

כמה תמונות צריך?

הערות מומלץ מינימום סוג הפרויקט
מתאים ללימוד ותרגול 100-200 לקטגוריה 50 פרויקט תלמיד
מספיק לתוצאות טובות 500-1000 לקטגוריה 200 פרויקט רציני
רמה מקצועית 5000+ לקטגוריה 1000 ייצור/תעשייה
כשיש קטגוריות רבות 10,000+ 5000 פרויקט מורכב

💡 כללי אצבע חשובים:

  • איכות > כמות - 200 תמונות טובות עדיפות על 1000 גרועות
  • איזון - כל קטגוריה צריכה מספר דומה של תמונות (±20%)
  • גיוון - תמונות מזוויות שונות, תאורות שונות, רקעים שונים
  • validation set - שימרו 20% מהתמונות לבדיקה (לא לאימון!)

🎓 למה לא צריך תמונות ענקיות?

המודל ממילא משנה את כל התמונות לגודל אחיד (למשל 224x224). תמונות בגודל 10MB לא ישפרו את הדיוק, רק יאטו את הטעינה והעיבוד. תמונה בגודל 1-2MB מספיקה לחלוטין!

תנאי תאורה וצילום

☀️ תאורה טובה = תוצאות טובות

  • תאורה טבעית - אור יום עדיף על תאורה מלאכותית (פחות צללים קשים)
  • הימנעו מניגודיות קיצונית - לא צילום נגד השמש
  • תאורה אחידה - הימנעו מצללים חזקים על האובייקט
  • גוון - הימנעו מגוון צהבהב/כחלחל קיצוני (White Balance)
  • גיוון מבוקר - צלמו באור יום, בתאורה מלאכותית, בצל - אבל כל תמונה איכותית

✅ תמונות טובות vs ❌ תמונות גרועות

✅ תמונות טובות:

  • חדות ומפוקסות
  • תאורה טובה ואחידה
  • האובייקט ברור ומרכזי
  • רקע לא מסיח דעת
  • צבעים נאמנים

❌ תמונות גרועות:

  • מטושטשות/לא חדות
  • כהות מדי או בהירות מדי
  • האובייקט קטן או חלקי
  • רקע עמוס ומבלבל
  • גוון צבע מעוות

📱 טיפים מעשיים לצילום:

  • נקו את העדשה - עדשה מלוכלכת = תמונות מטושטשות
  • החזיקו יציב - השתמשו בחצובה או שתי ידיים
  • מרחק נכון - האובייקט צריך לתפוס 50-80% מהתמונה
  • פורמט - JPEG בסדר, PNG טוב יותר (אבל גדול יותר)
  • צלמו יותר - קל למחוק תמונות גרועות מאשר לחזור ולצלם

🎨 גיוון תמונות והתייעצות עם מומחים

חשיבות הגיוון

🌈 צלמו מזוויות ומצבים שונים

  • זוויות צילום - מלמעלה, מהצד, מלפנים, בזווית 45°
  • מרחקים - צילום קרוב (close-up), בינוני, ורחוק
  • רקעים - רקע לבן, צבעוני, טבעי, במקום העבודה
  • אוריינטציה - אנכי (portrait) ואופקי (landscape)
  • הקשר - האובייקט לבד, עם אובייקטים נוספים, ביד אדם
  • מצבים - שלם, חתוך, פתוח, סגור (תלוי במשימה)

דוגמה: צילום תפוח

טוב: תפוח על רקע לבן מזוויות שונות ✅

טוב יותר: + תפוח על שולחן עץ, בסלסלה, ביד ✅✅

מושלם: + תפוח חצוי, נגוס, עם עלה, בצל בבוקר/אחה"צ ✅✅✅

ככל שהתמונות מגוונות יותר, המודל יוכל לזהות תפוחים במצבים שונים!

יודעים איזה מחלות חשובות לזהות, איך הן נראות בשלבים שונים רופא, אחות, טכנאי רנטגן זיהוי מחלות
מכירים פגמים אופייניים, תקנים תעשייתיים, דרישות איכות מהנדס איכות, עובד קו ייצור בקרת איכות
יודעים איך צמחים חולים נראים, מתי להתריע, מה דחוף אגרונום, גנן מקצועי מחלות צמחים
מכירים מיני בעלי חיים נדירים, הבדלים עדינים, התנהגות ביולוג, צפר, וטרינר זיהוי בעלי חיים
יודעים מה מותר/אסור על פי חוק, דרישות תיעוד, פרטיות עורך דין, קצין אבטחת מידע פרויקט עסקי/ציבורי
מבינים את צרכי המשתמשים, תרחישי שימוש, חוויית משתמש המשתמשים הפוטנציאליים כל פרויקט!

💬 שאלות חשובות לשאול מומחים:

  1. אילוצים: "מה בהחלט אסור לפספס?" (למשל: פגם מסוכן במוצר)
  2. קריטריונים: "איך אתם מזהים X? מה התכונות המובחנות?"
  3. קשיים: "מה המקרים הכי מאתגרים? איפה קל לטעות?"
  4. הקשר: "באיזה תנאים זה בדרך כלל מופיע? (תאורה, רקע, זווית)"
  5. דרישות: "מה רמת הדיוק שנדרשת? (95%? 99%? 99.9%?)"
  6. edge cases: "מה המקרים החריגים שצריך לטפל בהם?"

📖 סיפור אמיתי

תלמידה בנתה מזהה מחלות עור. בהתחלה צילמה תמונות מהאינטרנט. אחרי שהתייעצה עם רופאת עור, גילתה ש:

  • צריך תמונות בגדלים שונים של הפצע (לעקוב אחרי התפתחות)
  • צריך תמונות מגווני עור שונים (לא רק עור בהיר)
  • תאורת פלאש משנה צבעים - עלולה לטעות בדימום/אודם
  • חלק מהמחלות דומות מאוד - צריך תכונות נוספות (גיל, מיקום)

התוצאה: השיפרה את המודל מ-70% ל-92% דיוק!

✅ רשימת בדיקה לאיכות נתונים

לפני שמתחילים לאמן - בדקו שהנתונים תקינים!

🔍 בדיקות חובה לפני אימון

  1. כמות:
    • יש לפחות 50 תמונות לכל קטגוריה? (100+ עדיף)
    • כל הקטגוריות מאוזנות? (±20% הבדל מקסימלי)
  2. איכות טכנית:
    • כל התמונות נפתחות ללא שגיאות?
    • אין תמונות מטושטשות/כהות/בהירות מדי?
    • רזולוציה מספקת (לפחות 400x400)?
  3. תיוג נכון:
    • כל תמונה בתיקייה/קטגוריה הנכונה?
    • אין תמונות שגויות או מפוקפקות?
    • בדקתם לפחות 10% מהתמונות ידנית?
  4. גיוון:
    • יש וריאציה בזוויות, תאורות, רקעים?
    • לא כל התמונות מאותו מקור (למשל רק מגוגל)?
    • יש דוגמאות למקרים קשים/גבוליים?

🛠️ כלים לניקוי נתונים

from fastai.vision.all import *

# 1. בדיקה שכל התמונות נפתחות תקין
failed = verify_images(get_image_files(path))
failed.map(Path.unlink)  # מחיקת תמונות פגומות

# 2. בדיקת גודלים
for img_path in get_image_files(path):
    img = PILImage.create(img_path)
    if img.size[0] < 400 or img.size[1] < 400:
        print(f'תמונה קטנה מדי: {img_path}')

# 3. בדיקת איזון קטגוריות
for category in path.ls():
    n = len(list(category.glob('*.jpg')))
    print(f'{category.name}: {n} תמונות')

# 4. הצגת תמונות אקראיות לבדיקה ויזואלית
dls = ImageDataLoaders.from_folder(path)
dls.show_batch()  # בדקו שהתיוג נכון!

🎯 טיפ חשוב: ImageNet Cleaner

fastai כוללת כלי מדהים לניקוי נתונים אחרי אימון ראשוני:

# הממשק מציג:
# - תמונות שהמודל הכי טעה בהן
# - אפשרות למחוק או לשנות קטגוריה
# - תמונות דומות שאולי תויגו לא נכון

זה חוסך המון זמן! המודל עוזר לכם למצוא תמונות בעייתיות.

⚠️ דגלים אדומים - מתי להיזהר

  • דיוק 100% - חשוד! בדקו אם יש דליפת מידע (data leakage)
  • פער גדול - 95% באימון, 60% בבדיקה = overfitting חמור
  • קטגוריה אחת מושלמת - אחרות גרועות = בעיה בנתונים
  • המודל לא משתפר - אחרי 5 epochs בלי שיפור = בעיה

🏗️ בניית המודל

עכשיו נגיע לחלק המרגש - יצירת המודל שיזהה תמונות!

🎓 הסבר המושג: Transfer Learning

Transfer Learning זה כמו ללמוד צרפתית אחרי שכבר יודעים ספרדית - אתם מתחילים עם ידע בסיסי ורק מתאימים אותו למשימה החדשה.

המודל מתחיל ממשקלים שכבר אומנו על מיליוני תמונות (ImageNet), וצריך רק "לכוונן" אותו למשימה הספציפית שלכם. זה חוסך המון זמן!

1יצירת Learner

learn = vision_learner(
    dls,                      # הנתונים שהכנו קודם
    resnet34,                 # ארכיטקטורת המודל
    metrics=error_rate        # מה למדוד (אחוז שגיאות)
)

# הפונקציה vision_learner יוצרת מודל מוכן לעבודה!

🏛️ ארכיטקטורות נפוצות (מומלצות למתחילים):

  • resnet18 - קטן ומהיר, טוב לניסויים
  • resnet34 - איזון טוב בין מהירות לדיוק
  • resnet50 - גדול ומדויק יותר, צריך יותר זיכרון

💡 טיפ: התחילו תמיד מ-resnet34, זה ה-"סוס עבודה" של fastai

2אימון המודל - השיטה הפשוטה

learn.fine_tune(3)

# 3 = מספר epochs (מחזורים)
# בכל epoch, המודל עובר על כל הנתונים פעם אחת
# fine_tune עושה אופטימיזציה אוטומטית

🎓 הסבר המושג: Epoch

Epoch הוא מחזור אחד שבו המודל עובר על כל התמונות באימון. בכל epoch, המודל לומד ומשפר את עצמו. בדרך כלל צריך 3-10 epochs כדי לקבל מודל טוב.

✅ מה קורה ב-fine_tune?

  1. שלב 1: "הקפאת" השכבות הראשונות (שכבר יודעות לזהות קצוות וצבעים)
  2. שלב 2: אימון השכבה האחרונה (ספציפית למשימה שלכם)
  3. שלב 3: "הפשרת" כל השכבות ואימון עדין של כל המודל

זו הגישה המומלצת למתחילים - פשוט קוראים ל-fine_tune וזה עובד!

⚙️ פרמטרים מתקדמים לאימון

אחרי שהבנתם את הבסיס, אפשר לשלוט יותר בתהליך האימון

תיאור ערכים מומלצים פרמטר
כמה פעמים המודל עובר על כל הנתונים 3-10 epochs
גודל הצעד בלמידה - קובע כמה המודל משתנה בכל צעד 1e-3 (0.001) lr
(learning rate)
כמה epochs לאמן רק את הראש (לפני כל המודל) 1-2 freeze_epochs
משקל לאיזון בין קטגוריות לא מאוזנות None (רוב המקרים) class_weight

🎓 הסבר המושג: Learning Rate

Learning Rate (קצב למידה) הוא הפרמטר החשוב ביותר! הוא קובע כמה "גדולים" השינויים שהמודל עושה בכל צעד.

  • LR גבוה מדי: המודל "מדלג" ולא מתכנס לפתרון טוב
  • LR נמוך מדי: המודל לומד לאט מדי ועלול להיתקע
  • LR מתאים: המודל לומד מהר ומגיע לתוצאות טובות

🔍 מציאת Learning Rate אופטימלי

fastai כוללת כלי מדהים למציאת ה-LR הטוב ביותר:

# מציאת learning rate מומלץ
learn.lr_find()

# הפונקציה מציגה גרף שעוזר לבחור LR
# מחפשים נקודה שבה הגרף יורד בחדות
# לרוב זה סביב 1e-3 או 1e-2

הגרף מראה איך ה-loss משתנה עם LR שונים. בחרו נקודה שבה הגרף יורד במהירות, לפני שהוא מתחיל לעלות.

🎯 אימון עם פרמטרים מפורשים

learn.fine_tune(
    epochs=5,              # 5 מחזורי אימון
    base_lr=1e-3,          # learning rate בסיס
    freeze_epochs=1       # קפוא epoch אחד בהתחלה
)

# או שליטה מלאה עם fit_one_cycle:
learn.fit_one_cycle(
    5,                    # epochs
    lr_max=1e-3           # learning rate מקסימלי
)

🎯 שימוש במודל - חיזוי תוצאות

אחרי האימון, זמן להשתמש במודל כדי לזהות תמונות חדשות!

1הצגת תוצאות על נתוני הבדיקה

# מציג דוגמאות של תמונות עם החיזויים
learn.show_results()

# כל תמונה תוצג עם:
# - התווית האמיתית (actual)
# - החיזוי של המודל (predicted)
# - רמת הביטחון (confidence)

2חיזוי לתמונה ספציפית

# טעינת תמונה חדשה
img = PILImage.create('path/to/your/image.jpg')

# קבלת חיזוי
pred_class, pred_idx, probs = learn.predict(img)

# הדפסת התוצאות
print(f'חיזוי: {pred_class}')
print(f'ביטחון: {probs[pred_idx]:.2f}')

# probs מכיל הסתברויות לכל הקטגוריות
print(f'כל ההסתברויות: {probs}')

🎓 הסבר המושג: Confidence (ביטחון)

Confidence הוא מספר בין 0 ל-1 שמראה כמה המודל "בטוח" בחיזוי שלו. למשל:

  • 0.95 (95%) - המודל מאוד בטוח
  • 0.70 (70%) - המודל די בטוח, אבל יש ספק
  • 0.51 (51%) - המודל כמעט לא בטוח (רק קצת יותר מחצי)

דוגמה מלאה - מא' ועד ת'

from fastai.vision.all import *

# 1. הכנת נתונים
path = untar_data(URLs.PETS)
dls = ImageDataLoaders.from_name_re(
    path, get_image_files(path),
    pat=r'(.+)_\d+\.jpg$',
    item_tfms=Resize(224),
    batch_tfms=aug_transforms(),
    bs=32
)

# 2. בניית מודל
learn = vision_learner(dls, resnet34, metrics=error_rate)

# 3. אימון
learn.fine_tune(3)

# 4. שימוש במודל
img = PILImage.create('my_pet.jpg')
pred, idx, probs = learn.predict(img)
print(f'זה {pred}! (ביטחון: {probs[idx]*100:.1f}%)')

Confusion Matrix (מטריצת בלבול) היא טבלה שמראה את כל החיזויים של המודל - נכונים ושגויים. היא עוזרת להבין איפה המודל טועה.

✅ מה נחשב לתוצאות טובות?

  • 85%-95% - טוב מאוד, בסדר לרוב המשימות
  • 70%-85% - סביר, צריך שיפור (יותר נתונים או epochs)
  • מתחת ל-70% - חלש, צריך לבדוק מה הבעיה

🧠 שאלה 1: הבנת מושגים בסיסיים

מהו Transfer Learning?
טכניקה להעברת תמונות בין תיקיות שונות במחשב לצורך ארגון הנתונים בצורה יעילה יותר
שיטה להעברת נתונים מהמחשב האישי לשרת בענן בזמן אימון המודל כדי לחסוך זמן עיבוד
שימוש במודל שכבר אומן על מיליוני תמונות והתאמה שלו למשימה הספציפית שלנו, במקום להתחיל מאפס
העברת הידע שלמדנו בשפת תכנות אחת לשפת תכנות אחרת על מנת לשפר את יכולות הקוד

💡 עצה לפתרון:

חישבו על האנלוגיה שנתנו: זה כמו ללמוד צרפתית אחרי שכבר יודעים ספרדית - אתם לא מתחילים מאפס, אלא משתמשים בידע קיים ומתאימים אותו.

🧠 שאלה 2: פרמטרים ואופטימיזציה

מה הפרמטר החשוב ביותר שקובע את מהירות וטיב הלמידה של המודל?
batch_size - הפרמטר שקובע כמה תמונות עוברות במודל בו-זמנית ומשפיע על מהירות האימון
learning_rate - הפרמטר שקובע את גודל הצעדים בלמידה, ערך נכון קריטי להצלחת האימון
validation_split - האחוז של הנתונים ששומרים לבדיקה כדי למדוד את ביצועי המודל באופן אובייקטיבי
image_size - הגודל לו משנים את כל התמונות, משפיע על דיוק המודל וכמות הזיכרון הנדרשת

💡 עצה לפתרון:

זכרו שזה הפרמטר שאפילו יש לו כלי מיוחד למציאתו (lr_find()), והסברנו שערך לא נכון שלו יכול "לדלג" על הפתרון או לגרום ללמידה איטית מדי.

🧠 שאלה 3: הכנת נתונים

מדוע חשוב להשתמש ב-Data Augmentation?
כדי להגדיל את גודל הקבצים ולשפר את איכות התמונות המקוריות לפני האימון
כדי להאיץ את תהליך האימון על ידי שינוי אוטומטי של גודל התמונות לפורמט מהיר יותר
כדי לתקן תמונות פגומות או מטושטשות ולהפוך אותן לשימושיות יותר עבור המודל
כדי ליצור וריאציות של התמונות (סיבובים, חיתוכים, שינויי תאורה) ולמנוע overfitting

💡 עצה לפתרון:

חישבו על המטרה - אנחנו רוצים שהמודל יראה את אותם אובייקטים במצבים שונים, לא רק במצב אחד קבוע. זה כמו ללמוד לזהות חבר גם כשהוא עומד בזווית אחרת.

🧠 שאלה 4: הבנת קוד

מה עושה השורה הבאה?
learn = vision_learner(dls, resnet34, metrics=error_rate)
רק מייבאת את המודל resnet34 מהספרייה ושומרת אותו במשתנה מבלי לחבר אותו לנתונים
יוצרת מודל מלא לזיהוי תמונות עם ארכיטקטורת resnet34, מחברת לנתונים, ומגדירה מדד הערכה
מאמנת את המודל באופן אוטומטי עד שהוא מגיע לרמת דיוק מינימלית שהוגדרה מראש
מריצה בדיקה על הנתונים כדי לוודא שהם מתאימים לארכיטקטורה של resnet34 לפני האימון

💡 עצה לפתרון:

vision_learner היא פונקציה שיוצרת מודל מוכן לעבודה. היא מקבלת 3 פרמטרים: את הנתונים, את סוג המודל, ואיך למדוד ביצועים. זו פעולת הקמה, לא אימון.

🧠 שאלה 5: יישום מעשי

יש לכם dataset של 1000 תמונות. המחשב שלכם נתקע עם שגיאת זיכרון בזמן האימון. מה כדאי לעשות ראשון?
להוריד את מספר ה-epochs מ-10 ל-3 כדי שהאימון ייקח פחות זמן ויצרוך פחות משאבי זיכרון
להחליף את המודל מ-resnet34 ל-resnet18 כי הוא קטן יותר ולכן צורך הרבה פחות זיכרון בזמן אימון
להקטין את ה-batch_size מ-64 ל-16 כי זה ישנה כמה תמונות נטענות לזיכרון בו-זמנית
להקטין את ה-learning_rate כי ערך נמוך יותר דורש פחות חישובים מורכבים בכל צעד

💡 עצה לפתרון:

שגיאת זיכרון קורית בדרך כלל כי המודל מנסה לעבד יותר מדי תמונות בו-זמנית. איזה פרמטר קובע כמה תמונות נטענות לזיכרון בפעם אחת?

🧠 שאלה 6: הכנת תמונות

אתם בונים מזהה מחלות צמחים. עם מי הכי חשוב להתייעץ לפני שמתחילים לאסוף תמונות?
עם מהנדס תוכנה שיסביר איך לבנות את המודל בצורה הכי יעילה ומהירה מבחינה טכנית
עם אגרונום או גנן מקצועי שמכירים את המחלות, איך הן נראות, ומתי חשוב לזהות אותן
עם צלם מקצועי שיסביר איך לצלם תמונות באיכות הגבוהה ביותר עם ציוד מתאים
עם מעצב גרפי שיעזור לארגן את התמונות בצורה ויזואלית ומסודרת במחשב

💡 עצה לפתרון:

חישבו - מי מבין הכי טוב את הבעיה שאתם מנסים לפתור? מי יכול להגיד לכם אילו מחלות חשובות, איך הן נראות, ומה ההבדלים המכריעים בין מחלה אחת לאחרת?

💻 דוגמה מעשית מלאה

בואו נבנה יחד מזהה פירות - מתחילה ועד סוף!

המשימה: זיהוי 5 סוגי פירות

תפוח 🍎, בננה 🍌, תפוז 🍊, תות 🍓, אבטיח 🍉

1איסוף תמונות (השלב החשוב ביותר!)

תכנון:

  • מטרה: 150 תמונות לכל פרי (750 סה"כ)
  • גיוון: שלם, חתוך, על צלחת, בסלסלה, ביד
  • תאורה: אור טבעי, תאורת מטבח, חוץ
  • רקעים: לבן, שולחן עץ, דשא

ביצוע:

  • קנו/צלמו פירות מסוגים שונים
  • צלמו מזוויות שונות (מלמעלה, מהצד, בזווית)
  • בדקו שהתמונות חדות ובתאורה טובה

2ארגון התמונות

# מבנה התיקיות:
# fruits/
#   ├── apple/
#   │   ├── img1.jpg
#   │   └── img2.jpg
#   ├── banana/
#   ├── orange/
#   ├── strawberry/
#   └── watermelon/

3הקוד המלא

from fastai.vision.all import *

# הכנת נתונים
path = Path('fruits')
dls = ImageDataLoaders.from_folder(
    path,
    valid_pct=0.2,              # 20% לבדיקה
    item_tfms=Resize(224),
    batch_tfms=aug_transforms(  # שינויים אקראיים
        do_flip=True,
        max_rotate=20,
        max_lighting=0.3
    ),
    bs=32
)

# בדיקה מהירה
dls.show_batch(max_n=9)

# בניית מודל
learn = vision_learner(
    dls, 
    resnet34, 
    metrics=error_rate
)

# מציאת learning rate טוב
learn.lr_find()

# אימון
learn.fine_tune(5, base_lr=1e-3)

# הצגת תוצאות
learn.show_results()

# ניתוח שגיאות
interp = ClassificationInterpretation.from_learner(learn)
interp.plot_confusion_matrix()
interp.plot_top_losses(9)

# שימוש במודל על תמונה חדשה
img = PILImage.create('my_fruit.jpg')
pred, idx, probs = learn.predict(img)
print(f'זה {pred}! ביטחון: {probs[idx]*100:.1f}%')

🎉 תוצאה צפויה: דיוק של 90-95% אחרי 5 epochs! השקיעו זמן באיסוף תמונות טובות - זה ישתלם!

🔧 בעיות נפוצות ופתרונות

❌ שגיאה: RuntimeError - CUDA out of memory

הבעיה: הזיכרון של כרטיס המסך אזל

פתרון:

  • הקטנת batch_size (מ-64 ל-32 או ל-16)
  • שימוש במודל קטן יותר (resnet18 במקום resnet34)
  • הקטנת גודל התמונות (Resize(128) במקום 224)

❌ המודל לא משתפר (accuracy נמוך)

סיבות אפשריות:

  • מעט מדי נתונים - צריך לפחות 100 תמונות לקטגוריה
  • נתונים לא מאוזנים - קטגוריה אחת עם 1000 תמונות ואחרת עם 50
  • Learning rate לא נכון - השתמשו ב-lr_find()
  • מעט epochs - נסו להגדיל מ-3 ל-10

❌ Overfitting - דיוק טוב באימון, גרוע בבדיקה

הבעיה: המודל "שינן" את התמונות במקום ללמוד

פתרון:

  • הוספת data augmentation חזק יותר
  • הגדלת validation set (valid_pct=0.3)
  • הקטנת מספר epochs
  • שימוש במודל קטן יותר

✅ טיפים לשיפור ביצועים:

  • אספו יותר תמונות - זו התרופה הטובה ביותר!
