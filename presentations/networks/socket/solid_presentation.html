BaseCommandHandler — מחלקת בסיס
COMMAND PATTERN05 / 11
🧱 מחלקת בסיס אבסטרקטית
from abc import ABC, abstractmethod class BaseCommandHandler(ABC): @abstractmethod def can_handle(self, action: str) -> bool: """האם המחלקה מסוגלת לטפל בפעולה?""" pass @abstractmethod def handle(self, msg: ProtocolMessage, client_socket, protocol: ChatProtocol, context: CommandContext ) -> ProtocolMessage: """בצע את הפעולה ומחזיר תשובה""" pass # ─── שימוש בשרת ─── def get_handler(self, action: str): for handler in self.handlers: if handler.can_handle(action): return handler return UnknownCommandHandler()
💡 למה ABC ו-@abstractmethod?
  • 🔒
    ABC — מונע יצירת instance ישירות של BaseCommandHandler
  • 📋
    @abstractmethod — מחייב כל תת-מחלקה לממש את המתודות. שכחת? Python זורק שגיאה בטעינה
  • 🔌
    Polymorphism — השרת קורא handler.handle() בלי לדעת איזה handler זה
✅ דוגמה — RegisterCommandHandler
class RegisterCommandHandler(BaseCommandHandler): def can_handle(self, action) -> bool: return action == "register" def handle(self, msg, socket, protocol, ctx): if not msg.username or not msg.password: return ProtocolMessage( action="register", status="error", error="Missing username or password") success = ctx.user_repo.register_user( msg.username, msg.password) if not success: return ProtocolMessage( action="register", status="error", error="User already exists") return ProtocolMessage( action="register", status="ok", message="Registered successfully")
💡 תשומת לב
handle() לא שולחת לסוקט — היא מחזירה ProtocolMessage. השרת אחראי לשלוח. הפרדת האחריות!

class ChatServer: def __init__(self, host, port): self.host = host self.port = port # ─── Dependency Injection ─── self.user_repo = InMemoryUserRepository() self.client_manager = ClientManager() self.context = CommandContext( self.user_repo, self.client_manager ) # ─── רשימת handlers ─── self.handlers = [ RegisterCommandHandler(), LoginCommandHandler(), MessageCommandHandler(), ListUsersCommandHandler(), WhoAmICommandHandler(), PrivateMessageCommandHandler(), HelpCommandHandler(), LogoutCommandHandler(), UnknownCommandHandler(), # ← תמיד אחרון! ] def handle_client(self, sock, addr): protocol = ChatProtocol() # instance פרטי while True: raw = sock.recv(1024) msgs = protocol.feed_data(raw) for msg in msgs: handler = self.get_handler(msg.action) response = handler.handle( msg, sock, protocol, self.context) sock.sendall(protocol.encode(response))
🔑 CommandContext — מזריק תלויות
class CommandContext: def __init__(self, user_repo, client_manager): self.user_repo = user_repo self.client_manager = client_manager
💡 Dependency Injection
Handler לא יוצר UserRepository בעצמו — הוא מקבל אותו מה-Context. כך ניתן להחליף DB ללא שינוי ב-handlers.
🔄 זרימת בקשה ב-ChatServer
1
recv() ← bytes מהסוקט
2
protocol.feed_data() → ProtocolMessage
3
get_handler(action) → מוצא את ה-handler המתאים
4
handler.handle(msg, context) → ProtocolMessage
5
protocol.encode(response) → sendall()
שים לב
get_handler מבצע Polymorphism: הוא לא יודע איזה handler מחזיר — רק שיש לו can_handle ו-handle.

CLIENT MANAGER07 / 11
👥 client_manager.py
class ClientManager: def __init__(self): self._clients: List[ConnectedClient] = [] self._lock = threading.Lock() def add_client(self, client: ConnectedClient) -> None: with self._lock: # מונע הוספה כפולה for c in self._clients: if c.socket_obj == client.socket_obj: return self._clients.append(client) def get_client_by_socket(self, sock): with self._lock: for c in self._clients: if c.socket_obj == sock: return c return None def broadcast(self, msg, exclude_sock=None): with self._lock: clients = self._clients[:] # copy! for client in clients: if client.socket_obj == exclude_sock: continue client.socket_obj.sendall( client.protocol.encode(msg))
📦 models.py — ConnectedClient
@dataclass class ConnectedClient: socket_obj: socket.socket username: str protocol: ChatProtocol # כל לקוח שומר את הפרוטוקול שלו — # כך broadcast יכול להשתמש ב-encode # של כל לקוח בנפרד
🔐 Thread Safety
  • 🔒
    threading.Lock() — מגן על הרשימה משינויים מקבילים
  • 📋
    copy before iterate — self._clients[:] ב-broadcast מונע crash אם לקוח מתנתק תוך כדי שליחה
  • 🧩
    SRP — ClientManager אחראי רק לחיבורים. לא יודע על DB, פרוטוקול JSON, או לוגיקת פקודות
  • 🔄
    protocol פרטי לכל לקוח — כל ConnectedClient נושא ChatProtocol משלו עם buffer פרטי

# ─── ממשק (interface) ─── class IUserRepository(ABC): @abstractmethod def register_user(self, username, password) -> bool: pass @abstractmethod def validate_user(self, username, password) -> bool: pass @abstractmethod def user_exists(self, username) -> bool: pass # ─── מימוש: זיכרון ─── class InMemoryUserRepository(IUserRepository): def __init__(self): self._users = {} # {username: password} def register_user(self, username, password): if username in self._users: return False self._users[username] = password return True def validate_user(self, username, password): return self._users.get(username) == password

💡 DIP — Dependency Inversion Principle
ChatServer תלוי ב-IUserRepository (הפשטה) — לא ב-InMemoryUserRepository (מימוש). שינוי ה-DB לא משפיע על שום קוד אחר.
📚 Repository Pattern
דפוס עיצוב שמסתיר את פרטי ה-storage מאחורי ממשק אחיד. הקוד שמשתמש ב-repository לא יודע אם הנתונים בזיכרון, SQLite, MongoDB, או Redis.

class PrivateMessageCommandHandler(BaseCommandHandler): def can_handle(self, action): return action == "private_message" def handle(self, msg, sock, protocol, ctx): sender = ctx.client_manager.get_client_by_socket(sock) if sender is None: return ProtocolMessage( action="private_message", status="error", error="You must login first") target = ctx.client_manager.get_client_by_username( msg.target) if target is None: return ProtocolMessage( action="private_message", status="error", error="Target user is not connected") # שולח לtarget ישירות private = ProtocolMessage( action="private_message", from_user=sender.username, message=msg.message) target.socket_obj.sendall( target.protocol.encode(private)) return ProtocolMessage( action="private_message", status="ok", message="Private message sent")

✅ OCP — Open/Closed Principle
שתי הפקודות החדשות נוספו ללא שינוי שורה אחת ב-server.py, ב-ChatProtocol, ב-ClientManager. רק הוספנו — לא שינינו.

🎯 מה השתפר ב-maintainability?
כל קובץ עכשיו קטן וממוקד. אם יש באג ב-login — פותחים רק את LoginCommandHandler. אם רוצים לשנות את ה-DB — משנים רק את ה-Repository.
🚀 מה השתפר ב-extensibility?
הוספת group_chat, user_status, file_transfer — כל אחד מהם הוא מחלקה נפרדת. לא נוגעים בקוד קיים. פחות סיכון לשבור דברים.
🧪 מה השתפר ב-testability?
ניתן כעת לכתוב: handler.handle(msg, mock_socket, protocol, mock_context) — mock קל ל-context. בעבר זה דרש socket אמיתי ו-DB.
