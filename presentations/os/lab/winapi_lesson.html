WinAPI — הגשר לOS

כל תוכנית Windows — Chrome, Notepad, malware — מדברת עם OS דרך Windows API. זהו ממשק של אלפי פונקציות ב-DLL שמאפשרות גישה למשאבי מערכת.

▶
// שכבות המערכת
User Mode → Kernel Mode
תוכניות רגילות רצות ב-User Mode — גישה מוגבלת. WinAPI מתרגם קריאות ל-Kernel Mode דרך syscalls. אם תוכנית קורסת ב-User Mode — לא קורסת Windows.
▶
// ה-DLLs הראשיות
kernel32 · user32 · ntdll · advapi32
kernel32.dll = קבצים, processes, זיכרון. user32.dll = חלונות, אירועים, קלט. ntdll.dll = ה-DLL הנמוך ביותר, ישיר ל-kernel. advapi32.dll = Registry, Security, Services.
▶
// naming convention
FunctionNameA vs FunctionNameW
כל פונקציה עם מחרוזות קיימת ב-2 גרסאות: A = ANSI (ASCII ישן), W = Wide (Unicode, UTF-16). תמיד עדיף W בקוד מודרני. למשל: CreateFileA / CreateFileW.

🔐 קשר לסייבר

Malware עוקף EDR על ידי קריאה ישירה ל-ntdll.dll — דרך syscalls ישירים שמדלגים על הmonitoring של kernel32. טכניקה: Direct Syscall / Hell's Gate.

HANDLE — מפתח לכל אובייקט

Handle = מספר שלם שמייצג "כרטיסייה" לאובייקט Windows. כמו מספר תור — Windows מחזיק את הפרטים, אתה רק מחזיק את המספר.

// C — Handle לקובץ
HANDLE hFile = CreateFileW(
    L"C:\\test.txt",       // path
    GENERIC_READ,          // גישה
    FILE_SHARE_READ,       // שיתוף
    NULL,                  // security
    OPEN_EXISTING,         // פתח קיים
    FILE_ATTRIBUTE_NORMAL,
    NULL
);

if (hFile == INVALID_HANDLE_VALUE) {
    // שגיאה — בדוק GetLastError()
    DWORD err = GetLastError();
}

// תמיד לסגור handle!
CloseHandle(hFile);

⚠️ Handle Leak

לא קראת CloseHandle? Handle נשאר פתוח. עם אלפי leaks — המערכת מתקשה ליצור handles חדשים. malware מנצל זאת לDoS.

🗂️ סוגי Handles

HANDLE = קובץ, pipe, thread
HWND = חלון
HKEY = Registry key
HMODULE = DLL/EXE טעון
SC_HANDLE = Service

💡 GetLastError()

כל WinAPI function שנכשלת — מחזירה NULL/FALSE. GetLastError() נותן קוד שגיאה מפורט.

Handles — מה נכון?

קוד C פותח 1000 קבצים בלולאה ולא קורא ל-CloseHandle. מה יקרה?

א כלום — Windows סוגר handles אוטומטית בזמן ריצה
ב Handle leak — המערכת צורכת משאבים ועשויה להגביל יצירת handles חדשים
ג הקובץ יינעל ולא ניתן לפתיחה — אבל אין בעיה אחרת
ד שגיאה מיידית בפתיחת הקובץ הראשון

💡 Windows סוגר handles כשהתהליך מסתיים

כן — אבל בזמן ריצה, leak של אלפי handles יכול לגרום לכשלים. Process Explorer מציג את כל ה-handles הפתוחים של כל process.

📋 Clipboard מ-Python

import ctypes

def read_clipboard():
    u32 = ctypes.windll.user32
    k32 = ctypes.windll.kernel32

    u32.OpenClipboard(0)
    # CF_UNICODETEXT = 13
    h = u32.GetClipboardData(13)
    if h:
        ptr = k32.GlobalLock(h)
        text = ctypes.wstring_at(ptr)
        k32.GlobalUnlock(h)
        u32.CloseClipboard()
        return text
    u32.CloseClipboard()
    return ""

print(read_clipboard())

// API Reference — לחץ לפרטים

🃏 פונקציות WinAPI — מהנפוצות ביותר

CreateFile
פתיחת/יצירת קובץ, pipe, device. מחזיר HANDLE. שמות עם A/W suffix.
ReadFile
קריאת נתונים מ-HANDLE (קובץ/pipe). מחזיר bytes שנקראו ב-byref.
VirtualAlloc
הקצאת זיכרון וירטואלי. בסיס ל-shellcode injection. MEM_COMMIT + PAGE_EXECUTE.
RegOpenKey
פתיחת Registry key. HKEY handle. malware: HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Run
SetWindowsHookEx
התקנת Hook ל-events. WH_KEYBOARD_LL = keylogger. WH_MOUSE = mouse tracking.
CreateRemoteThread
יצירת thread בprocess אחר. שלב קריטי ב-process injection. דורש PROCESS_ALL_ACCESS.
GetProcAddress
כתובת פונקציה ב-DLL בזמן ריצה. Malware: לוטן API addresses בזמן ריצה כדי לחמוק מImport scan.
🔐 Cyber
כל API שמוצג כאן — אנליסטים מחפשים אותו ב-Import Table של malware!

🔧 WinSpy — System Monitor ב-Python

כלי ניטור בסיסי שמשתמש ב-WinAPI לאיסוף מידע מהמערכת:

#!/usr/bin/env python3
"""WinSpy — System Monitor | מגמת סייבר"""
import ctypes, ctypes.wintypes as wt, os, sys

k32  = ctypes.windll.kernel32
u32  = ctypes.windll.user32
ntdll = ctypes.windll.ntdll

def banner():
    print("""
╔══════════════════════════════╗
║  WinSpy v1.0 | WinAPI Demo  ║
╚══════════════════════════════╝""")

def get_system_info():
    print("\n[+] SYSTEM INFORMATION")
    buf = ctypes.create_unicode_buffer(256)
    sz  = wt.DWORD(256)

    k32.GetComputerNameW(buf, ctypes.byref(sz))
    print(f"  Computer : {buf.value}")

    k32.GetUserNameW(buf, ctypes.byref(sz))
    print(f"  User     : {buf.value}")

    print(f"  PID      : {os.getpid()}")
    print(f"  Admin    : {ctypes.windll.shell32.IsUserAnAdmin()}")

def list_processes():
    print("\n[+] RUNNING PROCESSES")
    psapi = ctypes.windll.psapi
    pids  = (wt.DWORD * 1024)()
    needed = wt.DWORD()
    psapi.EnumProcesses(pids, ctypes.sizeof(pids),
                         ctypes.byref(needed))
    count = needed.value // ctypes.sizeof(wt.DWORD)
    print(f"  Total: {count} processes")
    for i in range(min(count, 10)):
        print(f"  PID: {pids[i]}")

def read_clipboard():
    print("\n[+] CLIPBOARD CONTENT")
    u32.OpenClipboard(0)
    h = u32.GetClipboardData(13)  # CF_UNICODETEXT
    if h:
        ptr  = k32.GlobalLock(h)
        text = ctypes.wstring_at(ptr)
        k32.GlobalUnlock(h)
        print(f"  {text[:80]}..." if len(text) > 80 else f"  {text}")
    else:
        print("  Clipboard ריק")
    u32.CloseClipboard()

if __name__ == "__main__":
    banner()
    get_system_info()
    list_processes()
    read_clipboard()

🔐 Keylogger Hook — הבנה

קרא את הקוד. השלם את הפרמטרים החסרים:

import ctypes
from ctypes import wintypes

# Hook callback לכל הקשה במקלדת
HOOKPROC = ctypes.CFUNCTYPE(wintypes.LPARAM,
    wintypes.INT, wintypes.WPARAM, wintypes.LPARAM)

def hook_callback(nCode, wParam, lParam):
    if nCode >= 0 and wParam == 0x0100:  # WM_KEYDOWN
        vk_code = ctypes.cast(lParam, ctypes.POINTER(
                    ctypes.c_int))[0]
        print(f"Key: {vk_code}")
    return ctypes.windll.user32.CallNextHookEx(
        None, nCode, wParam, lParam)

hook = ctypes.windll.user32.SetWindowsHookExW(
    ,   # WH_KEYBOARD_LL = 13
    HOOKPROC(hook_callback),
    ,          # NULL for global
    0
)
