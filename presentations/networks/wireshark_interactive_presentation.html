🎯 מה זה Wireshark?

Wireshark היא תוכנת הסנפה וניתוח פרוטוקולי רשת המובילה בעולם - חינמית, בקוד פתוח ועוצמתית!

למה נשתמש ב-Wireshark?

  • 🔍 איתור בעיות ברשת - בעיות קישוריות, השהיות, אובדן חבילות
  • 🛡️ ניתוח אבטחה - זיהוי חדירות ובדיקות אבטחה
  • 📊 ניתוח ביצועים - זיהוי צווארי בקבוק ברשת
  • 📚 למידה על פרוטוקולי תקשורת - הבנת HTTP, TCP/IP, DNS ועוד
⚠️ חשוב לדעת: כלי זה עוצמתי ויכול לשמש גם לשימושים לגיטימיים וגם לפעילות זדונית. חשוב להשתמש בו באופן אתי ומשפטי בלבד!

🔗 שכבת הקישור - Link Layer

פרוטוקול Ethernet

שכבת הקישור אחראית על העברת מסגרות (Frames) בין מכשירים ברשת המקומית.

מידע בכותרת Ethernet:

  • Source MAC Address - כתובת MAC של השולח (6 בתים)
  • Destination MAC Address - כתובת MAC של המקבל
  • EtherType - איזה פרוטוקול בשכבה העליונה (0x0800=IP, 0x0806=ARP)

פרוטוקולים נוספים בשכבת הקישור:

ARP

Address Resolution Protocol - ממיר כתובות IP לכתובות MAC

LLDP

Link Layer Discovery Protocol - מגלה מכשירים מחוברים ברשת

דוגמת סינון לשכבת הקישור:

# סינון לפי כתובת MAC מקור
eth.src == 00:11:22:33:44:55

# סינון לפי כתובת MAC יעד
eth.dst == ff:ff:ff:ff:ff:ff

# סינון חבילות ARP בלבד
eth.type == 0x0806

❓ שאלות הבנה

שאלה 1: מדוע צריך כתובת MAC בנוסף לכתובת IP?
תשובה: כתובת IP משמשת לניתוב בין רשתות שונות, בעוד כתובת MAC משמשת לזיהוי מכשיר ספציפי ברשת המקומית. הנתב צריך לדעת את כתובת ה-MAC כדי לשלוח את החבילה למכשיר הנכון ברשת הפנימית.
שאלה 2: מתי נשתמש בפרוטוקול ARP?
תשובה: משתמשים ב-ARP כאשר מכשיר מכיר את כתובת ה-IP של מכשיר אחר ברשת המקומית, אבל לא יודע את כתובת ה-MAC שלו. המכשיר שולח שידור ARP Request ומקבל בחזרה ARP Reply עם כתובת ה-MAC.

🌐 שכבת הרשת - Network Layer

פרוטוקול IP (Internet Protocol)

שכבת הרשת אחראית על ניתוב חבילות בין רשתות שונות באמצעות כתובות לוגיות (IP).

שדות חשובים בכותרת IP:

שדה תיאור דוגמה
Version גרסת IP (4 או 6) 4
Source IP כתובת IP של השולח 192.168.1.100
Destination IP כתובת IP של המקבל 8.8.8.8
TTL Time To Live - מספר הקפיצות המקסימלי 64, 128
Protocol פרוטוקול בשכבה העליונה 6=TCP, 17=UDP
Checksum בדיקת תקינות הכותרת 0x5b2a

פרוטוקול ICMP

ICMP משמש להודעות שליטה ואבחון ברשת:

  • Ping - Echo Request (Type 8) ו-Echo Reply (Type 0)
  • Destination Unreachable - כאשר לא ניתן להגיע ליעד
  • Time Exceeded - כאשר ה-TTL הגיע ל-0

דוגמאות סינון לשכבת הרשת:

# סינון לפי כתובת IP (מקור או יעד)
ip.addr == 192.168.1.1

# סינון לפי כתובת IP מקור
ip.src == 192.168.1.1

# סינון לפי כתובת IP יעד
ip.dst == 8.8.8.8

# סינון חבילות עם TTL מסוים
ip.ttl == 128

# סינון חבילות ICMP
icmp

🚚 שכבת התעבורה - TCP

TCP (Transmission Control Protocol)

TCP הוא פרוטוקול חיבורי המבטיח העברה אמינה של נתונים.

מאפיינים עיקריים:

  • 🔐 Connection-Oriented - יוצר חיבור לפני העברת נתונים
  • ✅ Reliable - מבטיח שהנתונים יגיעו במלואם ובסדר הנכון
  • 🔄 Flow Control - מנגנוני בקרת זרימה
  • 📦 Ordered Delivery - שומר על סדר החבילות

שדות חשובים בכותרת TCP:

שדה תיאור
Source Port פורט המקור (16 ביט)
Destination Port פורט היעד (80=HTTP, 443=HTTPS, 21=FTP)
Sequence Number מספר סידורי לעקיבה אחר סדר החבילות
Acknowledgment Number מספר המאשר קבלת חבילות
Flags דגלים: SYN, ACK, FIN, RST, PSH, URG
Window Size גודל החלון לבקרת זרימה

TCP 3-Way Handshake

תהליך יצירת חיבור TCP:

  1. SYN → הלקוח שולח בקשת חיבור
  2. ← SYN-ACK השרת מאשר ומבקש חיבור חזרה
  3. ACK → הלקוח מאשר - החיבור נוצר!

דגלי TCP חשובים:

  • SYN - Synchronize: פתיחת חיבור
  • ACK - Acknowledge: אישור קבלה
  • FIN - Finish: סגירת חיבור
  • RST - Reset: איפוס חיבור (בעיה)
  • PSH - Push: העבר מיד לאפליקציה
# סינון חבילות TCP
tcp

# סינון לפי פורט
tcp.port == 80

# סינון חבילות SYN בלבד
tcp.flags.syn == 1 and tcp.flags.ack == 0

# סינון חבילות שמכילות נתונים
tcp.len > 0

📡 שכבת התעבורה - UDP

UDP (User Datagram Protocol)

TCP ✅

  • חיבורי
  • אמין
  • איטי יותר
  • בקרת שגיאות
  • בקרת זרימה

UDP ⚡

  • חסר חיבור
  • לא אמין
  • מהיר מאוד
  • ללא בקרת שגיאות
  • ללא בקרת זרימה

מתי נשתמש ב-UDP?

  • 🎮 משחקים מקוונים - מהירות חשובה מדיוק
  • 📹 סטרימינג וידאו/אודיו - VoIP, YouTube Live
  • 🌐 DNS - שאילתות קצרות ומהירות
  • 📡 DHCP - הקצאת כתובות IP
  • 📊 SNMP - ניטור רשת

כותרת UDP - פשוטה וקצרה!

שדה גודל תיאור
Source Port 16 ביט פורט המקור
Destination Port 16 ביט פורט היעד
Length 16 ביט אורך הודעת UDP
Checksum 16 ביט בדיקת תקינות (אופציונלי) 
# סינון חבילות UDP
udp

# סינון DNS (פורט 53)
udp.port == 53

# סינון DHCP (פורטים 67-68)
udp.port == 67 or udp.port == 68

# סינון לפי אורך
udp.length > 100

🌍 שכבת האפליקציה - HTTP/HTTPS

HTTP (HyperText Transfer Protocol)

HTTP הוא הפרוטוקול המשמש לגלישה באינטרנט - העברת דפי HTML, תמונות, סרטונים ועוד.

שיטות HTTP נפוצות:

  • GET - בקשה לקבלת משאב (דף, תמונה)
  • POST - שליחת נתונים לשרת (טופס)
  • PUT - עדכון משאב קיים
  • DELETE - מחיקת משאב
  • HEAD - בקשת כותרות בלבד

קודי תשובה (Status Codes):

קוד משמעות דוגמה
1xx מידע 100 Continue
2xx הצלחה ✅ 200 OK
3xx הפניה 301 Moved Permanently
4xx שגיאת לקוח ❌ 404 Not Found
5xx שגיאת שרת 💥 500 Internal Server Error

HTTP vs HTTPS

HTTP 🔓

  • פורט 80
  • ללא הצפנה
  • גלוי לכולם!
  • מסוכן לנתונים רגישים

HTTPS 🔐

  • פורט 443
  • מוצפן (TLS/SSL)
  • מוגן מהאזנה
  • בטוח לנתונים רגישים
⚠️ חשוב להבין: ב-HTTPS, Wireshark יראה את החיבור והכותרות (handshake), אבל לא יוכל לראות את התוכן המוצפן ללא מפתחות הפענוח! 
# סינון HTTP בלבד
http

# סינון בקשות GET
http.request.method == "GET"

# סינון תשובות עם קוד 404
http.response.code == 404

# סינון לפי כתובת URL
http.request.uri contains "example.com"

# סינון כותרות User-Agent
http.user_agent contains "Chrome"

🎣 Capture Filters - מסנני לכידה

מה זה Capture Filter?

שים לב: מסנן לכידה פועל ברמת ה-Driver ומונע מחבילות להיות מוקלטות בכלל! אם תטעה - לא תוכל לשחזר את החבילות.

מתי להשתמש?

  • 📊 כאשר יש תעבורה רבה מאוד ברשת
  • 💾 כדי לחסוך מקום בדיסק
  • ⚡ כדי לשפר ביצועים
  • 🎯 כאשר יודעים בדיוק מה מעניין אותנו

תחביר בסיסי:

# סינון לפי כתובת IP
host 192.168.1.1

# סינון לפי כתובת מקור
src host 192.168.1.1

# סינון לפי כתובת יעד
dst host 8.8.8.8

# סינון לפי פורט
port 80

# סינון לפי פורט מקור
src port 443

# סינון לפי פרוטוקול
tcp
udp
icmp

# שילוב תנאים
src host 192.168.1.1 and tcp and port 80

אופרטורים לוגיים:

אופרטור משמעות דוגמה
and / && וגם host 1.1.1.1 and port 80
or / || או port 80 or port 443
not / ! לא not port 22

❓ תרגול Capture Filters

שאלה 1: איך נכתוב מסנן שילכוד רק תעבורת HTTP?
תשובה: port 80 או tcp port 80
שאלה 2: איך נכתוב מסנן שילכוד תעבורה מ-192.168.1.1 או ל-8.8.8.8?
תשובה: host 192.168.1.1 or host 8.8.8.8
שאלה 3: איך נכתוב מסנן שילכוד הכל חוץ מ-DNS?
תשובה: not port 53

🔍 Display Filters - מסנני תצוגה

מה זה Display Filter?

יתרון גדול: פועל על החבילות שכבר נקלטו! אפשר לנסות סינונים שונים בלי לאבד מידע.

Display Filters הם הרבה יותר חזקים!

  • 🎯 יכולים לגשת לכל שדה בפרוטוקול
  • 🔤 תומכים ב-Regular Expressions
  • 📊 מאפשרים השוואות מורכבות
  • 🎨 צביעת חבילות לפי כללים

אופרטורי השוואה:

אופרטור משמעות דוגמה
== / eq שווה ip.addr == 1.1.1.1
!= / ne לא שווה tcp.port != 80
> / gt גדול מ frame.len > 1000
< / lt קטן מ ip.ttl < 10
contains מכיל http contains "password"
matches תואם regex http.request.uri matches "\.jpg$"

דוגמאות שימושיות:

# כל התעבורה מכתובת או אליה
ip.addr == 192.168.1.100

# חבילות TCP עם דגל SYN
tcp.flags.syn == 1

# חבילות גדולות במיוחד
frame.len > 1400

# שגיאות HTTP
http.response.code >= 400

# בקשות HTTP לדומיין ספציפי
http.host == "example.com"

# חבילות שמכילות טקסט מסוים
frame contains "password"

# DNS queries לדומיין מסוים
dns.qry.name contains "google"

# TCP Retransmissions (שליחה חוזרת)
tcp.analysis.retransmission

# חבילות מוצפנות TLS
tls

🔄 Follow TCP Stream - מעקב אחר שיחה

מה זה TCP Stream?

כל תקשורת TCP מורכבת ממספר חבילות. "Follow TCP Stream" מאחד את כל החבילות של שיחה אחת ומציג את התוכן המלא!

איך להשתמש?

  1. מצא חבילת TCP שמעניינת אותך
  2. לחץ עליה עם כפתור ימני
  3. בחר: Follow → TCP Stream
  4. יפתח חלון עם כל השיחה!

💾 שמירה וטעינה של הסנפות

שמירת קובץ הסנפה

צעדים לשמירה:

  1. עצור את ההסנפה (Ctrl+E)
  2. לחץ על שמור: File → Save (Ctrl+S)
  3. בחר מיקום ושם לקובץ
  4. בחר פורמט: .pcap או .pcapng

PCAP

  • פורמט קלאסי
  • נתמך בכל הכלים
  • מוגבל בפיצ'רים

PCAPNG ⭐

  • פורמט מודרני
  • תומך במטא-דאטה
  • מומלץ לשימוש!

אפשרויות שמירה מתקדמות:

Packet Range:

  • All packets - כל החבילות
  • Displayed - רק חבילות שעברו את הסינון
  • Selected packet - חבילה בודדת
  • Range - טווח מספרים

טעינת קובץ הסנפה:

  • 🖱️ לחיצה כפולה על קובץ .pcap/.pcapng
  • 📂 File → Open (Ctrl+O)
  • 🔄 File → Merge - למיזוג מספר קבצים

💡 טיפים חשובים:

  • שמור קבצים בשם משמעותי: "http_traffic_2024_01_15.pcap"
  • הוסף הערות לקבצים עם אפשרות "Capture comments"
  • שמור סינונים שימושיים: Analyze → Display Filter Macros
  • גבה קבצים חשובים - הם יכולים להכיל ראיות!

🧪 תרגול וניתוח מקרים

מקרה 1: אתר לא נטען

תסריט: משתמש מתלונן שאתר מסוים לא נטען אצלו.

שאלה: איזה מסננים תשתמש כדי לנתח את הבעיה?
תשובה:
  1. בדוק DNS: dns - האם השם הומר לכתובת?
  2. בדוק TCP: tcp.flags.syn == 1 - האם החיבור החל?
  3. בדוק HTTP: http - מה קוד התשובה?
  4. בדוק Retransmissions: tcp.analysis.retransmission
שאלה: מצאת שיש הרבה TCP Retransmissions. מה זה אומר?
תשובה: TCP Retransmissions מצביעים על אובדן חבילות ברשת. זה יכול להיות בגלל:
  • רשת עמוסה או איטית
  • בעיות בחומרת הרשת (כבל, מתג)
  • תנועה גבוהה מדי
  • הפרעות אלחוטיות (ברשת WiFi)

מקרה 2: חשד לפריצה

תסריט: מערכת האבטחה מדווחת על פעילות חשודה מכתובת IP מסוימת.

שאלה: איך תאתר את כל הפעילות מכתובת זו?
תשובה: 
# כל התעבורה מהכתובת או אליה
ip.addr == 192.168.1.50

# סריקת פורטים? בדוק SYN ללא ACK
ip.src == 192.168.1.50 and tcp.flags.syn == 1 and tcp.flags.ack == 0

# ניסיונות חיבור כושלים?
ip.src == 192.168.1.50 and tcp.flags.reset == 1
שאלה: מצאת הרבה חבילות SYN לפורטים שונים. מה זה אומר?
תשובה: זו כנראה סריקת פורטים (Port Scan)! התוקף מנסה לזהות אילו שירותים פתוחים במערכת. זו פעילות חשודה מאוד שיכולה להיות הכנה למתקפה.

מקרה 3: סיסמה נחשפה?

תסריט: אתר שלך משתמש ב-HTTP (ולא HTTPS). האם סיסמאות יכולות להיחשף?

שאלה: איך נוכל לראות אם נשלחו סיסמאות בטקסט פשוט?
תשובה: 
# חיפוש המילה "password" בתעבורה
http contains "password"

# או חיפוש POST requests (שבהם שולחים טפסים)
http.request.method == "POST"

# אפשר גם:
frame contains "username" or frame contains "passwd"

מסקנה: כן! ב-HTTP הכל גלוי. זו הסיבה ש-HTTPS חיוני!

🔓 תרגיל מעשי: הסנפת סיסמה גלויה

⚠️ אזהרה חשובה!

תרגיל זה מדגים מדוע חשוב להשתמש ב-HTTPS ולא ב-HTTP. סיסמאות שנשלחות ב-HTTP עוברות בטקסט גלוי וניתן ללכוד אותן!

שלבי התרגיל:

  1. התחל הסנפה ב-Wireshark
  2. היכנס לאתר: 
    http://cyber.org.il/networks/links/plain-password.html
  3. הזן סיסמה כלשהי (לדוגמה: "MySuperPassword123")
  4. לחץ על Register
  5. עצור את ההסנפה ב-Wireshark
  6. הכנס את הפילטר: 
    frame contains "txtPa"
  7. מצא את הסיסמה בחבילה שנלכדה!

מה ראינו?

הסיסמה שהזנת עברה בטקסט גלוי ברשת! כל מי שמבצע הסנפה ברשת יכול לראות אותה.

  • 🔴 HTTP - לא מוצפן, מסוכן לסיסמאות
  • 🟢 HTTPS - מוצפן, בטוח לסיסמאות

💡 לקח חשוב:

אף פעם אל תזין סיסמאות, פרטי כרטיס אשראי, או מידע רגיש באתרים שמשתמשים ב-HTTP!

תמיד בדוק שיש נעילה 🔒 בשורת הכתובת של הדפדפן!

שאלה: איך אפשר למנוע מתוקפים ללכוד את הסיסמה שלי?
תשובות:
  • ✅ השתמש רק באתרים עם HTTPS
  • ✅ התקן תוסף דפדפן "HTTPS Everywhere"
  • ✅ השתמש ב-VPN ברשתות WiFi ציבוריות
  • ✅ אל תזין מידע רגיש ברשתות לא מוכרות

🌊 Follow TCP Stream

מה זה Follow TCP Stream?

תכונה חזקה ב-Wireshark שמאפשרת לראות את כל השיחה בין לקוח לשרת כטקסט רצוף אחד, במקום חבילות בודדות!

איך זה עובד?

ללא Follow Stream

רואים מאות חבילות נפרדות, קשה להבין את השיחה המלאה

עם Follow Stream ⭐

רואים את כל השיחה בחלון אחד, ברור וקריא!

איך משתמשים?

  1. מצא חבילת TCP שמעניינת אותך
  2. לחץ עליה בכפתור ימני
  3. בחר: Follow → TCP Stream
  4. יפתח חלון עם השיחה המלאה!

🎯 תרגיל מעשי: Follow TCP Stream

תרחיש: נריץ לקוח-שרת פשוט ונראה את כל השיחה

  1. התחל הסנפה ב-Wireshark
  2. הרץ תוכנית לקוח-שרת (למשל מתרגיל 2.6 מהספר)
  3. שלח כמה פקודות מהלקוח לשרת:
    • TIME
    • NAME
    • RAND
    • EXIT
  4. עצור את ההסנפה
  5. סנן: tcp
  6. בחר חבילה → Follow → TCP Stream
  7. תראה את כל הפקודות והתשובות בחלון אחד!

💡 שימושים מעשיים:

  • ניתוח פרוטוקולי טקסט (HTTP, SMTP, FTP, POP3)
  • דיבאג של תקשורת לקוח-שרת
  • זיהוי בעיות בפרוטוקול
  • למידה איך עובדים פרוטוקולים שונים
⚠️ שים לב: Follow Stream עובד רק על פרוטוקולים חיבוריים (TCP, TLS). ב-UDP אין "שיחה" רציפה!
שאלה: מה ההבדל בין צפיה בחבילות בודדות לבין Follow Stream?
תשובה: כשצופים בחבילות בודדות, רואים כל חבילה בנפרד עם כל הכותרות והפרטים הטכניים. Follow Stream לוקח את התוכן בלבד מכל החבילות של אותה שיחה ומציג אותו כטקסט רצוף, מה שמקל מאוד על הבנת מה עבר בתקשורת.

🎯 הבנת Encapsulation:

זכור: כל שכבה מוסיפה את ה-header שלה למידע שמועבר!

  • 📱 Application → מידע המשתמש (HTTP, DNS...)
  • 📦 Transport → מוסיפה TCP/UDP header (פורטים)
  • 🌐 Network → מוסיפה IP header (כתובות IP)
  • 🔗 Data Link → מוסיפה Ethernet header (כתובות MAC)

המידע של שכבת האפליקציה עובר דרך כל השכבות!
