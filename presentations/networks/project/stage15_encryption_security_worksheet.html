מחלקת SecurityManager

אוסף מתודות אבטחה שמגנות על התקשורת בפרויקט שלכם: הצפנה, hash לסיסמאות, אימות לקוח, ומניעת replay. התוצר השני - פרק כתוב שמסביר אילו חולשות זיהיתם ואיך כל הגנה פותרת אותן.

משימה 1.1

מה ה"payload הרגיש" הספציפי בפרויקט שלכם?

לדוגמה: תוכן קובץ בכספת, פרטי התחברות, פקודת ניהול. מה קורה אם תוקף שמסניף את הרשת רואה אותו בטקסט פתוח?

משימה 1.2

אילו מבין חמשת האיומים בטבלה רלוונטיים אצלכם, ואילו לא?

לא כל פרויקט חשוף לכל האיומים - הסבירו למה כל אחד רלוונטי או לא.

2

מימוש SecurityManagerמתודה לכל הגנה רלוונטית

משימה 2.1

encrypt_payload / decrypt_payload

הצפנה סימטרית ל-payload הרגיש שהגדרתם. אפשר להשתמש בספריית cryptography (Fernet) או דומה.

משימה 2.2

hash_password / verify_password

משימה 2.3

authenticate_client

בחרו שיטה (השוואת hash סיסמה, JWT, או חתימה) והסבירו את הבחירה לפני הקוד.

משימה 2.4

validate_message — הגנת anti-replay

הגדירו חלון זמן מקסימלי להודעה, ומימוש שבודק timestamp + nonce (כמו בדף ההסבר).

משימה 2.5 — הגשה

הריצו בדיקה לכל מתודה

  • encrypt_payload + decrypt_payload מחזיר את הטקסט המקורי בדיוק
  • verify_password מצליח עם סיסמה נכונה, נכשל עם סיסמה שגויה
  • authenticate_client דוחה לקוח לא מאומת
  • validate_message דוחה הודעה ישנה/חזרתית, ומקבל הודעה תקינה
  • שם קובץ הקוד: security_manager_שם_הצוות.py
