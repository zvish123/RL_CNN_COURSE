יסוד ראשון

משתנים

המקום שבו שומרים את ראיות התיק

message = "KHOOR"
key = 3

message שומר את ההודעה החשודה; key שומר את המפתח שבעזרתו ננסה לפענח אותה.

משתנה הוא מקום בזיכרון בעל שם, המאפשר לשמור, לעדכן ולקרוא מידע במהלך התוכנית — בדיוק כמו תיקיית ראיות עם תווית.

איסוף מודיעין

קלט (input)

החוקר מזין מידע למערכת

x = input([prompt])

message = input("Enter message: ")
key = input("Enter key: ")

תהליך הקלט

המשתמש מקליד במקלדת → input() קולט את התווים → התוכנית מעבדת → print() מציג תוצאה למסך.

שימו לב: input() מחזיר תמיד טקסט (str) — גם אם החוקר הקליד מספר!

input עוצר את התוכנית ומחכה למידע מהמשתמש. עד שהוא לא מקבל קלט (ולוחץ Enter), שום שורת קוד אחרי לא תתבצע — ממש כמו לעמוד בנקודת בידוק ולהמתין לתשובת החוקר.

עיבוד ראיה

המרת טיפוסים

מטקסט למספר שאפשר לחשב איתו

key = int(input("Enter key: "))
balance = float(input("Enter score: "))

אם ננסה לחשב key + 1 בלי להמיר את הקלט ל־int, פייתון תזרוק שגיאה — כי "3" (טקסט) ו־1 (מספר) הם טיפוסים שלא ניתן לחבר ישירות.

מכיוון ש־input מחזיר מחרוזת, ולעבודה עם מפתח הצפנה אנחנו צריכים מספר שלם, נדרשת המרה מפורשת באמצעות int(). דפוס דומה קיים גם להמרה למספר עשרוני (float()) או חזרה לטקסט (str()).

דיווח ממצאים

פלט (print)

מציגים את תוצאות החקירה

print(*objects, sep=' ', end='\n')

print("Hello", "world", "!")
# Hello world !

print("Hello", "world", sep="-", end="")
print("!")
# Hello-world!

name = "avi"; age = 18
print(f"{name} age is {age}")
# avi age is 18

print מציג מידע למסך, ומשמש גם כ"דוח שטח" לדיבוג — לבדוק בזמן אמת מה הערך של משתנה.

תרגיל בית הספר

משימת החוקר

חיברו את כל הכלים לתוכנית אחת

message = input("Message: ")
key = int(input("Key: "))

print(message)
print(key)

כתבו תוכנית שמקבלת הודעה ומפתח מהחוקר, ומדפיסה אותם בחזרה כדי לוודא שהקלט נקלט נכון לפני שמתחילים בפענוח בפועל.

תא חקירה #1

מרחק בין שתי נקודות ציתות

מיקום על מפת הרשת

שני אנטנות ציתות ממוקמות ברשת קואורדינטות: (x1, y1) ו־(x2, y2). כדי לדעת אם הן בטווח תקשורת, יש לחשב את המרחק בין הנקודות.

x1, y1, x2, y2 = 1, 2, 4, 6
distance = ((x2 - x1)**2 + (y2 - y1)**2) ** 0.5
print(distance)
# 5.0

זהו ביישום ישיר של נוסחת המרחק האוקלידי. שימו לב לסדר פעולות החשבון: קודם הסוגריים, אחר כך החזקה (**), ולבסוף החיבור והשורש — בדיוק כמו תרגיל ה־PRACTICE המקורי, רק שכאן "המרחק" קובע אם שתי אנטנות ציתות יכולות "לדבר" זו עם זו.

תא חקירה #2

ממוצע קודי התווים

מדד לזיהוי דפוס הצפנה

חוקרים מסוימים בודקים את הממוצע של קודי ה־ASCII באות הראשונה והאחרונה של ההודעה, כדי לזהות חשד ראשוני לסוג הצפנה.

code1 = int(input("Enter ASCII code 1: "))
code2 = int(input("Enter ASCII code 2: "))
avg = (code1 + code2) / 2
print(avg)

המבנה זהה לתרגיל ה־PRACTICE המקורי (avg = (num1+num2)/2) — רק ההקשר השתנה. זו נקודה מרכזית בתכנות: אותו דפוס לוגי (קליטת שני ערכים, חישוב ממוצע, הדפסה) משמש בעולמות תוכן שונים לחלוטין.

תא חקירה #3

תקציב מפתחות הסטה

ניצול תקציב ניחושים

לחוקר יש תקציב חישובי של 101 "נקודות ניסיון". כל ניסיון הסטה בודדת (Caesar shift) עולה 7.5 נקודות, וכל ניסיון התאמת תדירויות עולה 8.2 נקודות. אם ביצע 5 ניסיונות הסטה ו־6 ניסיונות התאמת תדירויות — כמה נקודות נשארו לו?

shift_tries = 5; shift_cost = 7.5
freq_tries = 6; freq_cost = 8.2

remaining = 101 - (shift_tries * shift_cost + freq_tries * freq_cost)
print(remaining)

חישוב ישיר: 5×7.5 + 6×8.2 = 37.5 + 49.2 = 86.7, ולכן 101 - 86.7 = 14.3 נקודות נשארות. בתרגיל ה־PRACTICE המקורי נוצל כל התקציב בדיוק (5 אגסים + 8 תפוחים = 0 עודף) — נסו למצוא שילוב ניסיונות שמנצל את כל 101 הנקודות בדיוק!
