🐳 מה זה Docker?

📦
קונטיינר (Container)
חבילה שמכילה את הקוד + כל מה שצריך להרצתו. פועל מבודד מהמחשב המארח.
📸
Image (אימג')
תמונת מצב קפואה של האפליקציה. כמו "תבנית" שממנה יוצרים קונטיינרים.
📋
Dockerfile
קובץ הוראות לבניית האימג'. שורה אחר שורה — Docker בונה לפי זה.
🏬
Docker Hub
מאגר מרכזי (כמו GitHub לקוד) שבו שומרים ומשתפים Image-ים.
Dockerfile = מתכון הפיצה (רשימת מרכיבים + הוראות אפייה)
Image = הפיצה הקפואה, ארוזה, מוכנה לאפייה
Container = הפיצה האפויה שאוכלים עכשיו
Docker Hub = חנות הפיצות הקפואות (אפשר להוריד של אחרים!)

🎮 משחק ניחוש מספרים — Flask

נבנה שרת Flask פשוט — משחק ניחוש מספרים בין 1–100 עם 4 ניסיונות. נארוז אותו ב-Docker ונריץ אותו בכל מקום.
app.py — שרת המשחק Python 🐍 
from flask import Flask, request, jsonify
import random

app = Flask(__name__)
target = random.randint(1, 100)
attempts = 0

@app.route('/guess', methods=['POST'])
def guess():
    global attempts
    data = request.json
    num = data.get('number')
    attempts += 1
    
    if attempts > 4:
        return jsonify({"msg": "הפסדת! נגמרו הנסיונות", "target": target})
    if num == target:
        return jsonify({"msg": f"זכית! ניחשת ב-{attempts} ניסיונות"})
    return jsonify({"msg": "גבוה! 📈" if num > target else "נמוך! 📉"})

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=5000)
localhost / 127.0.0.1 = "רק אני עצמי יכול לגשת אליי"
0.0.0.0 = "כולם שיכולים להגיע לנתב הרשת שלי, יכולים לגשת אליי"

בתוך Docker, הקונטיינר הוא עולם נפרד. אם נאמר localhost, הדפדפן שלנו לא יצליח לגשת לשרת — כי "localhost" בתוך הקונטיינר זה לא אנחנו. 0.0.0.0 מאפשר לתקשורת לחצות את גבול הקונטיינר.

📋 כתיבת ה-Dockerfile

Dockerfile ללא סיומת! 

FROM python:3.11-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install -r requirements.txt

COPY . .

EXPOSE 5000

CMD ["python", "app.py"]
FROM python:3.11-slim נקודת התחלה — Base Image מ-Docker Hub. python:3.11-slim = גרסה קטנה ויעילה
WORKDIR /app יוצרים תיקיית עבודה בתוך האימג'. כל הפקודות הבאות ירוצו מתוכה
COPY . . מעתיקים את הקבצים מהמחשב שלנו לתוך האימג'
RUN pip install מתקינים ספריות בתוך האימג' — לא על המחשב האישי!
CMD ["python","app.py"] הפקודה שתרוץ אוטומטית כשמפעילים קונטיינר

🔨 בניית האימג' — docker build

קודם ניצור קובץ requirements.txt:

requirements.txt
flask==3.0.0

עכשיו נבנה את האימג':

Terminalbash
# מהתיקייה שמכילה את Dockerfile
docker build -t guess-game .

# -t = שם תג (tag) לאימג'
# .  = הנקודה = "כאן נמצא ה-Dockerfile"

🔄 מה קורה בזמן ה-Build?

  • ⬇️מוריד Base Image (python:3.11-slim) מ-Docker Hub
  • 📁יוצר תיקיית /app בתוך האימג'
  • 📋מעתיק את הקבצים
  • 📦מריץ pip install — מתקין Flask
  • ✅מייצר Image ID ייחודי (hash)
לצפייה בכל האימג'ים
docker images
Docker שומר כל שורת Dockerfile כשכבה (layer) בנפרד. אם שכבה לא השתנתה — Docker לא בונה אותה מחדש, אלא לוקח מה-cache. זו הסיבה שמעתיקים requirements.txt ומריצים pip לפני COPY . . — כך שינוי בקוד לא ייגרום להתקנה מחדש של כל הספריות.

🚀 הרצת הקונטיינר — docker run

Terminalbash
docker run -d -p 5001:5000 --name my-game guess-game

# -d        = detached — רץ ברקע (לא נתקע בטרמינל)
# -p 5001:5000 = PORT MAPPING
# --name    = שם ידידותי לקונטיינר

🔌 Port Mapping — גשר בין הקונטיינר למחשב

מחשב שלנו :5001 → :5000 קונטיינר

http://localhost:5001/guess → Flask שרץ בקונטיינר על port 5000
הקונטיינר הוא רשת מבודדת — כמו מחשב בתוך מחשב. יש לו כתובת IP משלו ופורטים משלו. כברירת מחדל, אי אפשר "להגיע" אליו מהדפדפן שלנו. Port mapping יוצר "מנהרה" שמחברת פורט במחשב שלנו לפורט בתוך הקונטיינר.

דוגמה: אפשר להריץ 3 עותקי משחק מאותו Image:
-p 5001:5000 → משחק 1
-p 5002:5000 → משחק 2
-p 5003:5000 → משחק 3

🏬 Docker Hub — GitHub לאימג'ים

Docker Hub הוא מאגר ציבורי (ופרטי) לאימג'ים. אפשר להוריד אימג'ים מוכנים של כמעט כל דבר — MySQL, Redis, Nginx — ולדחוף אימג'ים שיצרנו.
Push & Pull — שיתוף האימג' שלנו
# 1. התחברות ל-Docker Hub
docker login

# 2. תיוג האימג' עם שם משתמש
docker tag guess-game myusername/guess-game:v1

# 3. דחיפה ל-Hub
docker push myusername/guess-game:v1

# --- על מחשב אחר ---
# 4. הורדה
docker pull myusername/guess-game:v1

# 5. הרצה ישירה (אם לא קיים מקומי — יוריד אוטומטית)
docker run -p 5001:5000 myusername/guess-game:v1

docker exec -it my-postgres \
