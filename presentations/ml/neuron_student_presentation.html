🎯 המטרה

הנוירון הוא מכונת למידה — לאחר אימון, הוא יכול לחזות תוצאות עבור נתונים שלא ראה מעולם.

ŷ = g(w₁·x₁ + w₂·x₂ + w₃·x₃ + b)

אלגוריתם

חלחול קדימה — Forward Propagation

תהליך החישוב מקלט לפלט

1
קבלת קלטים

וקטור הפיצ'רים x₁, x₂, ..., xₙ נכנס לנוירון

2
כפל במשקלים

כל קלט מוכפל במשקל המתאים: w₁·x₁, w₂·x₂, ...

3
חיבור + bias

סכום כל המכפלות + ה-bias: Σ = Σwᵢxᵢ + b

4
פונקציית אקטיבציה

העברת Σ דרך g(): ŷ = g(Σ)

Python חלחול קדימה ידני 
def predict_grade(hours, avg, missing,
                     weights, bias):
    # שלב 1+2: כפל קלטים במשקלים
    z = (weights[0] * hours +
         weights[1] * avg +
         weights[2] * missing)

    # שלב 3: הוספת bias
    z = z + bias

    # שלב 4: פונקציית אקטיבציה (לינארית)
    prediction = z  # g(z) = z

    return prediction

# דוגמה:
weights = [0.5, 0.5, 0.5]
bias = 40

result = predict_grade(5, 75, 3,
                        weights, bias)
print(f"חיזוי ציון: {result}")
# חיזוי ציון: 81.5
✏️ תרגיל — חשב בעצמך

נתון: hours=8, avg=85, missing=2, w=[0.5, 0.5, 0.5], b=40

חשב ידנית את ŷ לפי הנוסחה: ŷ = 0.5×8 + 0.5×85 + 0.5×2 + 40

פונקציית שגיאה — MSE

איך מודדים כמה המודל טועה?

❌ הבעיה

המשקלים הראשוניים שרירותיים — החיזויים לא מדויקים. צריך דרך לכמת את גודל הטעות.

📐 הנוסחה

MSE = (1/n) · Σ(yᵢ - ŷᵢ)²
סמלמשמעות
nמספר הדוגמאות (שורות)
yᵢהערך האמיתי של דוגמה i
ŷᵢהחיזוי של הנוירון לדוגמה i
(y-ŷ)²ריבוע הפרש = עונש על טעות

Activation Functions

כיצד הנוירון מעבד את הסכום הליניארי?

פונקציהטווחשימוש עיקרי
Linear(-∞,∞)רגרסיה / שכבת פלט
ReLU[0,∞)שכבות נסתרות — הנפוצה ביותר
Sigmoid(0,1)סיווג בינארי
Tanh(-1,1)כשצריך ערכים שליליים
Softmax[0,1]סיווג רב-קטגוריאלי

🔴 ReLU — המלכה

ReLU(z) = max(0, z)

מחזירה z אם חיובי, אחרת אפס. מהירה מאוד לחישוב ועוזרת לרשת ללמוד קשרים לא-ליניאריים.

🟢 Sigmoid

σ(z) = 1 / (1 + e⁻ᶻ)

מוציאה ערכים בין 0 ל-1. אידיאלי לסיווג — האם הדוגמה שייכת לקטגוריה?

🟣 Tanh

tanh(z) = (eᶻ - e⁻ᶻ) / (eᶻ + e⁻ᶻ)

דומה ל-Sigmoid אבל סימטרית סביב 0 — מתאימה כשצריך ערכים חיוביים ושליליים.

בניית נוירון עם Keras

3 שלבים: הגדרה → קימפול → אימון

Python — שלב 1: הגדרת מבנה
from tensorflow.keras.models import Sequential
from tensorflow.keras.layers import Dense

# נוירון יחיד עם 3 קלטים
model = Sequential([
    Dense(1,                    # נוירון אחד
          activation='linear',  # פונקציית אקטיבציה
          input_shape=(3,))     # 3 פיצ'רים
])
Python — שלב 2: קימפול
from tensorflow.keras.optimizers import Adam

model.compile(
    optimizer=Adam(learning_rate=0.1),
    loss='mean_squared_error',
    metrics=['mae']
)
Python — שלב 3: אימון
history = model.fit(
    X_train, y_train,
    validation_data=(X_test, y_test),
    epochs=100,
    verbose=1
)
# הדפסת המשקלים שנמצאו
weights = model.get_weights()
print("w, b:", weights)

Overfitting — מכה בלמידת מכונה

מתי מודל "לומד יותר מדי"?

🔴 מה זה Overfitting?

המודל לומד את הנתונים בצורה מושלמת מדי — כולל "רעש" ואנומליות. התוצאה: ביצועים מצוינים על נתוני אימון, ביצועים גרועים על נתונים חדשים.

🟢 איך מזהים?

Train Loss ↓ ירד מאוד
Validation Loss ↗ עלה — סימן אדום!

🛡️ איך מונעים Overfitting?

1
Validation Set

חלק מהדאטה לבדיקה תוך כדי אימון — מזהה overfitting

2
Early Stopping

עצירת האימון כשה-validation loss מפסיק לרדת

3
Dropout / Regularization

הוספת "עונשים" לפרמטרים גדולים מדי

4
הגדלת הדאטה

יותר נתונים = קשה יותר "לזכור" הכל

סיכום

מה למדנו?

🧠 הנוירון

יחידה מתמטית שמקבלת קלטים (features), מכפיל במשקלים, מוסיף bias, ומעביר דרך פונקציית אקטיבציה

ŷ = g(Wx + b)

📊 MSE

מודד את הטעות הממוצעת בריבוע. המטרה: למזער אותו על ידי מציאת w ו-b אופטימליים

MSE = (1/n)Σ(y-ŷ)²

🏋️ אימון

תהליך מסודר שמעדכן w ו-b כדי להקטין את השגיאה — בדרך כלל Gradient Descent

w ← w - lr·∇MSE

⚡ Activation

פונקציה שמוסיפה אי-ליניאריות: ReLU, Sigmoid, Tanh, Softmax — כל אחת לשימוש שונה

⚠️ Overfitting

מודל שלמד "יותר מדי" — מזהים עם validation set ומונעים עם Early Stopping / Dropout

🔗 הצעד הבא

מנוירון אחד לרשת נוירונים — שכבות מרובות של נוירונים שיכולים לפתור בעיות הרבה יותר מורכבות
