🎰 מה זו שיטת מונטה קרלו?

הגדרה:

שיטת מונטה קרלו היא טכניקה סטטיסטית המשתמשת בדגימות אקראיות לפתרון בעיות מורכבות. בהקשר של למידה מחיזוקים במשחקים, השיטה לומדת מניסיון דרך משחקים רבים.

📍 מקור השם

השם "מונטה קרלו" נבחר בהשראת העיר מונטה קרלו במונקו, הידועה בבתי הקזינו שלה.

היסטוריה: השיטה פותחה במעבדה הלאומית לוס אלמוס בארצות הברית במהלך מלחמת העולם השנייה על ידי המתמטיקאים סטניסלב אולם וג'ון פון נוימן לפיתוח נשק גרעיני.

📊 אלגוריתם הניקוד

חישוב ניקוד אחורה

הניקודים של הלוחות ברשימה נקבעים במעבר אחורה:

grade(state) = grade(next_state) × gamma

gamma (גאמא): פרמטר למידה, המשקף את המרחק של הלוח מהלוח הסופי

ערך טיפוסי: בין 0.8 ל-0.9 (לעיתים גבוה יותר)

משמעות: ככל שהלוח רחוק יותר מהניצחון, הציון שלו נמוך יותר

דוגמה מספרית (gamma = 0.9)

מיקום במשחק לוח אחרון לוח 6 לוח 5 לוח 4 לוח 3 לוח 2 לוח 1
ניקוד 1.0 0.9 0.81 0.729 0.6561 0.59049 0.531441

💡 הבנה: המהלך האחרון מקבל את הציון המלא, ומהלכים קודמים מקבלים ציונים יורדים

📚 מילון הלמידה (Q-Table)

מבנה המילון

מפתח (Key): מחרוזת (string) של מצב הלוח

ערך (Value): זוג סדור (ניקוד ממוצע, מספר הופעות)

דוגמה למילון

Q_Table = {
    'x--------': (0.531441, 1),
    'x---o----': (0.59049, 1),
    'x---o---x': (0.6561, 1),
    'x-o-o---x': (0.729, 1),
    'x-o-o-x-x': (0.81, 1),
    'x-ooo-x-x': (0.9, 1),
    'x-ooo-xxx': (1.0, 1)
}

עדכון המילון

אם הלוח חדש: נוסיף אותו למילון עם מספר מופעים 1 והניקוד שקבל

אם הלוח קיים: נגדיל את מספר המופעים ונעדכן את הניקוד הממוצע

חישוב ממוצע מצטבר

ממוצע חדש = (ממוצע קיים × מספר נתונים + נתון חדש) / (מספר נתונים + 1)

💻 קוד דוגמה - עדכון המילון

def update_q_table(Q_table, board_state, grade):
    # המרת הלוח למחרוזת (מפתח)
    state_key = ''.join(board_state)
    
    if state_key in Q_table:
        # הלוח קיים - עדכון ממוצע
        current_avg, count = Q_table[state_key]
        
        # חישוב ממוצע מצטבר
        new_avg = (current_avg * count + grade) / (count + 1)
        new_count = count + 1
        
        Q_table[state_key] = (new_avg, new_count)
    else:
        # לוח חדש - הוספה למילון
        Q_table[state_key] = (grade, 1)
    
    return Q_table


# דוגמה לשימוש
Q_table = {}
board = ['x', '-', '-', '-', '-', '-', '-', '-', '-']
grade = 0.531441

Q_table = update_q_table(Q_table, board, grade)
print(Q_table)
# פלט: {'x--------': (0.531441, 1)}

🔍 חקירה מול ניצול (Exploration vs Exploitation)

הדילמה:

האם לבחור במהלך הכי טוב שאנחנו מכירים, או לנסות מהלכים חדשים?

🎯 ניצול (Exploitation)

בחירת המהלך הטוב ביותר לפי הידע הנוכחי

יתרון: מקסום הסיכוי לניצחון

חיסרון: עלול לפספס מהלכים טובים יותר

🔬 חקירה (Exploration)

בחירת מהלכים אקראיים כדי לגלות אסטרטגיות חדשות

יתרון: למידת דרכים חדשות

חיסרון: עלול להפסיד משחקים

⚖️ האיזון - אסטרטגיה ε-greedy

ε (epsilon): פרמטר השולט על שיעור החקירה

  • בהסתברות ε (למשל 10%) - בחר מהלך אקראי (חקירה)
  • בהסתברות 1-ε (למשל 90%) - בחר את המהלך הטוב ביותר (ניצול)

דוגמה קודית

import random

def choose_move(board, Q_table, epsilon=0.1):
    if random.random() < epsilon:
        # חקירה - מהלך אקראי
        return random_move(board)
    else:
        # ניצול - מהלך טוב ביותר
        return best_move(board, Q_table)

💡 טיפ: בתחילת הלמידה השתמש ב-ε גבוה (0.3), ובהדרגה הפחת אותו (0.05)

💻 קוד מלא - מונטה קרלו ל-XO

import random

class MonteCarloXO:
    def __init__(self, gamma=0.9):
        self.Q_table = {}
        self.gamma = gamma
        self.ai_player = 'X'
        self.human_player = 'O'
    
    def play_random_game(self):
        # משחק רנדומלי בין שני שחקנים
        board = ['-'] * 9
        states = []  # שמירת כל המצבים
        
        while not self.game_over(board):
            move = self.random_move(board)
            board[move] = self.ai_player
            states.append(board.copy())
            
            if self.game_over(board):
                break
                
            move = self.random_move(board)
            board[move] = self.human_player
        
        return states, self.get_result(board)
    
    def grade_states(self, states, result):
        # מתן ניקוד לכל המצבים
        graded = []
        current_grade = result
        
        for state in reversed(states):
            graded.append(''.join(state), current_grade))
            current_grade *= self.gamma
        
        return graded
    
    def train(self, num_games=100000):
        # אימון המערכת
        for i in range(num_games):
            states, result = self.play_random_game()
            graded_states = self.grade_states(states, result)
            
            for state, grade in graded_states:
                self.update_q_table(state, grade)
            
            if (i + 1) % 10000 == 0:
                print(f'סיימנו {i+1} משחקים')

בחירת מהלך מיטבי

def best_move(self, board):
    # מציאת המהלך הטוב ביותר
    available_moves = [i for i, cell in enumerate(board) 
                       if cell == '-']
    
    best_score = -999
    best_move = None
    
    for move in available_moves:
        # נסה את המהלך
        test_board = board.copy()
        test_board[move] = self.ai_player
        
        # בדוק את הציון
        state_key = ''.join(test_board)
        
        if state_key in self.Q_table:
            score, _ = self.Q_table[state_key]
            
            if score > best_score:
                best_score = score
                best_move = move
        else:
            # מצב שלא ראינו - העדפה למרכז/פינות
            if move == 4:  # מרכז
                best_move = move
                best_score = 0.5
    
    return best_move if best_move is not None else available_moves[0]

📖 פעולות על מילון (Dictionary) בפייתון

מה זה מילון?

מילון הוא אוסף של זוגות מפתח-ערך. בשיטת מונטה קרלו, המפתח הוא מצב הלוח והערך הוא הניקוד.

פעולות בסיסיות

# יצירת מילון
Q_table = {}

# הוספת ערך
Q_table['x--------'] = (0.5, 1)

# קריאת ערך
score, count = Q_table['x--------']

# בדיקה אם מפתח קיים
if 'x--------' in Q_table:
    print("המצב קיים!")

# שימוש ב-get (בטוח יותר)
result = Q_table.get('x--------', (0, 0))  # ערך ברירת מחדל

# מעבר על כל המפתחות
for state in Q_table.keys():
    print(state)

# מעבר על כל הערכים
for score, count in Q_table.values():
    print(f"ציון: {score}, מופעים: {count}")

# מעבר על זוגות מפתח-ערך
for state, (score, count) in Q_table.items():
    print(f"{state}: {score}")

⚠️ חשוב: מפתחות במילון חייבים להיות hashable (בלתי ניתנים לשינוי) - מחרוזות, מספרים, tuples

🚀 נקודות העשרה מתקדמות

1️⃣ First-Visit vs Every-Visit Monte Carlo

First-Visit MC

מעדכן ערך רק בפעם הראשונה שמצב מופיע באפיזודה

יתרון: נכון מבחינה תיאורטית

Every-Visit MC

מעדכן ערך בכל פעם שמצב מופיע

יתרון: מתכנס מהר יותר

2️⃣ On-Policy vs Off-Policy

On-Policy: לומד מהמדיניות שהוא משתמש בה (למשל ε-greedy)

Off-Policy: לומד ממדיניות אחת אבל משתמש באחרת (למשל Importance Sampling)

3️⃣ Function Approximation

במשחקים מורכבים (כמו שחמט), אי אפשר לשמור כל מצב במילון. פתרון:

  • שימוש ברשתות נוירונים
  • למידה של פונקציה שמעריכה מצבים
  • דוגמה: AlphaGo

4️⃣ חישוב π בשיטת מונטה קרלו

ניתן להשתמש במונטה קרלו לחישוב π:

  • זורקים נקודות אקראיות בריבוע
  • סופרים כמה נופלות במעגל החוסם
  • π ≈ 4 × (נקודות במעגל / סך הנקודות)

⚠️ אתגרים ומגבלות

1️⃣ יעילות חישובית

דורשת מספר רב של סימולציות, מה שעלול להיות בעייתי במשחקים מורכבים

  • XO: צריך אלפי משחקים
  • שחמט: צריך מיליוני משחקים
  • Go: צריך מיליארדי משחקים

2️⃣ איזון בין חקירה לניצול

קושי במציאת האיזון הנכון

  • יותר מדי חקירה → הפסדים מיותרים
  • יותר מדי ניצול → תקיעות במקסימום מקומי

3️⃣ תלות בזמן ריצה

איכות התוצאות תלויה במספר הסימולציות

✅ פתרונות

  • שימוש ב-Function Approximation (רשתות נוירונים)
  • אופטימיזציות כמו Importance Sampling
  • שילוב עם שיטות אחרות (MCTS - Monte Carlo Tree Search)
  • שימוש במחשוב מקבילי

✍️ תרגיל מעשי

🎯 המשימה שלך:

הטמע את שיטת מונטה קרלו במשחק XO!

📝 שלבי ביצוע:

שלב 1: בניית התשתית

  • צור מחלקה MonteCarloAgent
  • הוסף משתנה Q_table (מילון)
  • הוסף פרמטרים: gamma, epsilon

שלב 2: יישום פונקציות עזר

  • random_move() - מהלך אקראי
  • game_over() - בדיקת סוף משחק
  • get_result() - קביעת תוצאה

שלב 3: אימון

  • משחקים רנדומליים (100,000+)
  • שמירת מצבים
  • חישוב ניקודים
  • עדכון Q_table

שלב 4: משחק

  • בחירת מהלך מיטבי
  • טיפול במצבים לא ידועים
  • ממשק משתמש

🚀 אתגר נוסף: הוסף שמירה וטעינה של המודל, התאם את epsilon לפי מספר המשחקים

📚 סיכום - מה למדנו?

✅ עקרונות יסוד:

  • מונטה קרלו לומדת מאפיזודות שלמות
  • השיטה משתמשת בדגימה אקראית לפתרון בעיות
  • המערכת משפרת את עצמה דרך ניסיון וטעייה

🔧 מרכיבים טכניים:

  • Q-Table - מילון המאחסן ערכי מצבים
  • Gamma (γ) - פקטור היוון למהלכים מוקדמים
  • Epsilon (ε) - איזון בין חקירה לניצול
  • ממוצע מצטבר - עדכון יעיל של ערכים

💡 יישומים:

  • משחקי לוח (XO, שחמט, Go)
  • רובוטיקה ותכנון מסלולים
  • ניהול משאבים ואופטימיזציה
  • סימולציות וחישובים מדעיים
