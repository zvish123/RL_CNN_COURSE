מה זה Hash?

פונקציית גיבוב היא כלי קריפטוגרפי שממיר כל קלט לפלט קבוע — ואי אפשר לחזור ממנו.

SHA-256("hello") →
2cf24dba5fb0a30e26e83b2ac5b9e29e1b161e5c1fa7425e73043362938b9824
🔁

דטרמיניסטי

אותו קלט תמיד מייצר אותו פלט. "hello" → תמיד אותו hash.

⚡

חישוב מהיר

ניתן לחשב hash של כל גודל קלט בזמן קצר מאד.

🚫

חד-כיווני

לא ניתן לחשב את הקלט מה-hash. זה מה שהופך אותו לכלי אבטחה.

📏

פלט קבוע

בין "a" לרומן שלם — ה-hash תמיד אותו אורך (256 ביט ב-SHA-256).

נסה בעצמך — אפקט המפולת + bcrypt

HASH OUTPUT: ---
"hello"
---
"hellO" — אות שונה אחת בלבד!
---
ביטים שונים בין שני ה-hash: ~128 — זהו אפקט המפולת (Avalanche Effect)

🧂 bcrypt בפעולה — שתי סיסמאות זהות, שני hash שונים!

סיסמה א׳: "1234"
לוחץ "יצור"...
סיסמה ב׳: "1234" (גיבוב חדש)
לוחץ "יצור"...
מדוע הם שונים? bcrypt מוסיף salt אקראי לכל סיסמה לפני הגיבוב. כך גם אם שתי משתמשות בחרו "1234" — בבסיס הנתונים יופיעו ערכים שונים לחלוטין. Rainbow Tables חסרות ערך.
מסקנה: SHA-256 של "1234" תמיד זהה — bcrypt של "1234" תמיד שונה. זו ההבדל הקריטי לאבטחת סיסמאות.
CONCEPT

Hash לעומת הצפנה

מאפיין Hash (גיבוב) Encryption (הצפנה)
כיוון חד-כיווני בלבד דו-כיווני (אפשר לפענח)
מפתח ללא מפתח דרוש מפתח
שימוש עיקרי שמירת סיסמאות, בדיקת שלמות העברת מידע סודי
דוגמאות SHA-256, bcrypt, MD5 AES, RSA, ChaCha20
ניתן לשחזר? לא כן (עם המפתח)
המסקנה: לשמירת סיסמאות — תמיד גיבוב, לא הצפנה. אם יש מפתח הצפנה שנגנב — הכל נחשף. Hash בלתי-הפיך מעיקרו.
PROPERTIES

מאפייני פונקציית Hash

🔄

דטרמיניסטי

אותו קלט = אותו פלט תמיד, בכל מחשב, בכל זמן.

🌊

אפקט המפולת (Avalanche Effect)

שינוי ביט אחד בקלט משנה ~50% מהביטים בפלט — לכן לא ניתן "להתקרב" לתשובה.

💥

עמיד בפני התנגשויות (Collision Resistance)

קשה מאד מבחינה חישובית למצוא שני קלטים שונים עם אותו hash.

Salt אוטומטי

bcrypt מחשב את הפונקציה אלפי פעמים — כך brute-force לוקח שנים במקום שניות.

תרשים: שמירה והשוואה

🔒 הרשמה — שמירה

סיסמה מהמשתמש "1234"
↓
bcrypt.gensalt() יוצר salt אקראי חדש
↓
bcrypt.hashpw() סיסמה + salt → hash
↓
שמירה ב-DB $2b$12$salt+hash (לא הסיסמה!)

✅ כניסה — השוואה

סיסמה מוזנת "1234" (ניחוש?)
↓
שליפת hash מ-DB $2b$12$y6z8r...
↓
bcrypt.checkpw() מחלץ salt מה-hash, מגבב שוב, משווה
↓
True / False בלי לפענח — רק להשוות!
bcrypt_demo.py
import bcrypt # הרשמה def register(password): hashed = bcrypt.hashpw( password.encode(), bcrypt.gensalt() ) return hashed # שמור ב-DB # כניסה def login(password, stored_hash): return bcrypt.checkpw( password.encode(), stored_hash )
ATTACKS

התקפות על Hash

Rainbow Tables

⚠ סכנה עם MD5/SHA-1

מילון מחושב מראש של hash→סיסמה. מפצח MD5 של "password123" ב-0.01 שניות.

סיסמהMD5 Hash
password5f4d...cc4
123456e10a...b25
hello5d41...22a

Brute Force

⚠ קשה אך אפשרי

ניסוי של כל הסיסמאות האפשריות. MD5: מיליארד לשנייה. bcrypt cost=12: ~250 לשנייה.

מהירות פיצוח (GPU A100):
MD5: 60,000,000,000/שנייה
SHA-256: 3,000,000,000/שנייה
bcrypt: ~4,000/שנייה

כיצד Salt מגן?

✓ הגנה יעילה

כל סיסמה מקבלת salt אקראי. Rainbow tables חסרות ערך — הן לא מכילות את ה-salt.

hash("1234") → זהה תמיד!
hash("1234"+salt1) → a9f3...
hash("1234"+salt2) → x7b1...

סיכום הגנות

✓ bcrypt = הפתרון

bcrypt פותר את כל הבעיות: Salt אוטומטי, איטיות מכוונת, cost factor ניתן לשינוי.

✅ Salt מובנה — מנטרל Rainbow Tables
✅ איטי — מקשה על Brute Force
✅ Cost Factor — מסתגל לחומרה חדשה

import gradio as gr import bcrypt # מסד נתונים בזיכרון USERS = { "alice": bcrypt.hashpw( b"1234", bcrypt.gensalt()), "bob": bcrypt.hashpw( b"qwerty", bcrypt.gensalt()), } def login(username, password): if username not in USERS: return ("", gr.update(visible=True), gr.update(visible=False), "❌ משתמש לא נמצא") ok = bcrypt.checkpw( password.encode(), USERS[username]) if ok: return (username, gr.update(visible=False), gr.update(visible=True), "") return ("", gr.update(visible=True), gr.update(visible=False), "❌ סיסמה שגויה") def logout(): return ("", gr.update(visible=True), gr.update(visible=False), "")

import gradio as gr import bcrypt USERS = { "alice": bcrypt.hashpw(b"1234", bcrypt.gensalt()), "bob": bcrypt.hashpw(b"qwerty", bcrypt.gensalt()), } def login(username, password): if username not in USERS: return "", gr.update(visible=True), gr.update(visible=False), "❌ משתמש לא נמצא" ok = bcrypt.checkpw(password.encode(), USERS[username]) if ok: return username, gr.update(visible=False), gr.update(visible=True), "" return "", gr.update(visible=True), gr.update(visible=False), "❌ סיסמה שגויה" def logout(): return "", gr.update(visible=True), gr.update(visible=False), "" def tab1_action(user, text): # בדיקת הרשאה גם בצד השרת! if not user: return "⛔ אין הרשאה — יש להתחבר תחילה" return f"📋 [{user}] עיבד: " + text.upper() def tab2_action(user, number): if not user: return "⛔ אין הרשאה — יש להתחבר תחילה" return f"🔢 [{user}] תוצאה: " + str(number ** 2) with gr.Blocks(title="Login + Tabs") as app: user_state = gr.State("") # ─── לוח Login ─── with gr.Column(visible=True) as login_panel: gr.Markdown("## 🔐 כניסה למערכת") username = gr.Textbox(label="שם משתמש") password = gr.Textbox(label="סיסמה", type="password") login_btn = gr.Button("כניסה", variant="primary") err_msg = gr.Textbox(visible=False, label="שגיאה") # ─── אזור מוגן עם Tabs ─── with gr.Column(visible=False) as main_panel: with gr.Row(): welcome_md = gr.Markdown("## ✅ ברוך הבא!") logout_btn = gr.Button("🚪 Logout", scale=0) with gr.Tabs(): with gr.Tab("📋 Tab 1 — עיבוד טקסט"): t1_input = gr.Textbox(label="הכנס טקסט") t1_btn = gr.Button("עבד") t1_output = gr.Textbox(label="תוצאה", interactive=False) with gr.Tab("🔢 Tab 2 — חישוב מספרים"): t2_input = gr.Number(label="הכנס מספר") t2_btn = gr.Button("חשב בריבוע") t2_output = gr.Textbox(label="תוצאה", interactive=False) # ─── חיבור אירועים ─── login_btn.click( fn=login, inputs=[username, password], outputs=[user_state, login_panel, main_panel, err_msg]) logout_btn.click( fn=logout, outputs=[user_state, login_panel, main_panel, err_msg]) t1_btn.click( fn=tab1_action, inputs=[user_state, t1_input], outputs=t1_output) t2_btn.click( fn=tab2_action, inputs=[user_state, t2_input], outputs=t2_output) app.launch()
הטריק הכפול: הטאבים נסתרים עם visible=False בממשק, וגם כל פונקציה בודקת if not user — הגנה כפולה.
למה הגנה כפולה? visible=False רק מסתיר בממשק. משתמש מתקדם יכול לקרוא לAPI ישירות. בדיקת user_state בפונקציה עצמה היא ההגנה האמיתית.

השוואה בין שיטות Hash

סנן לפי שימוש:
אלגוריתם שנת יצירה גודל פלט מהירות Salt מובנה בטיחות כיום שימוש מומלץ Python
MD5 1991 128 bit מהיר מאד לא 💀 שבור Checksum בלבד, לא לסיסמאות hashlib.md5()
SHA-1 1995 160 bit מהיר לא ⚠️ מיושן הימנע — נשבר ב-2017 hashlib.sha1()
SHA-256 2001 256 bit מהיר לא ✅ בטוח קבצים, TLS, בלוקצ'יין — לא לסיסמאות! hashlib.sha256()
SHA-512 2001 512 bit מהיר לא ✅ בטוח חתימות דיגיטליות, מסמכים hashlib.sha512()
SHA-3 2015 224–512 bit בינוני לא 🏆 מצוין ארכיטקטורה שונה מ-SHA-2, עמיד יותר hashlib.sha3_256()
bcrypt 1999 184 bit איטי בכוונה ✓ כן 🏆 מצוין סיסמאות — הבחירה הסטנדרטית bcrypt.hashpw()
scrypt 2009 גמיש איטי מאד ✓ כן 🏆 מצוין סיסמאות — עמיד גם מול GPU/ASIC hashlib.scrypt()
Argon2 2015 גמיש איטי מאד ✓ כן 🏆 מצוין סיסמאות — זוכה תחרות Password Hashing 2015 argon2-cffi
PBKDF2 2000 גמיש איטי ✓ כן ✅ בטוח סיסמאות, גזירת מפתחות (TLS) hashlib.pbkdf2_hmac()
BLAKE2 2012 256/512 bit מהיר מאד לא 🏆 מצוין checksum מהיר, חלופה ל-SHA-2 hashlib.blake2b()
מקרא
💀 שבור ⚠️ מיושן ✅ בטוח 🏆 מצוין
MD5/SHA-1 בהדרגה מוחלף מומלץ מומלץ מאוד

לסיסמאות → bcrypt / Argon2 / scrypt
