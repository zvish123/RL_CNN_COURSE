הגדרה: צינור דו-כיווני של מידע בין שני מחשבים

תכונות Socket:

  • זורם בו זרם של bytes (מידע)
  • דו-כיווני - אפשר לשלוח ולקבל
  • יש לו נקודת התחלה וסיום

🎯 איך מזהים Socket?

כל Socket מזוהה על ידי שני פרטים:

1️⃣ כתובת IP (מזהה המחשב)

192.168.1.100
או
www.google.com → 142.250.185.46

2️⃣ מספר Port (מזהה התוכנה)

טווח: 0-65535

דוגמאות נפוצות:
- HTTP: 80 (אתרי אינטרנט)
- HTTPS: 443 (אתרים מאובטחים)
- FTP: 21 (העברת קבצים)
- SSH: 22 (התחברות מרחוק)
כתובת מלאה: 192.168.1.100:80

🏢 אנלוגיה: בניין משרדים

🏢 בניין (IP: 1.2.3.4)

קומה 80: HTTP - אתר אינטרנט 🌐
קומה 443: HTTPS - אתר מאובטח 🔒
קומה 21: FTP - העברת קבצים 📁
קומה 25: SMTP - שליחת אימייל 📧

כתובת IP = כתובת הבניין
Port = קומה בבניין
כתובת מלאה = בניין + קומה = 1.2.3.4:443

🤔 שאלת הבנה #1

על שרת בכתובת 1.2.3.4 רצים:

  • Port 80: אתר אינטרנט
  • Port 443: אתר מאובטח (HTTPS)
  • Port 25: שרת אימייל

שאלה: לאיזה כתובת צריך להתחבר כדי לגלוש באופן מאובטח?

א. 1.2.3.4:80
ב. 1.2.3.4:443
ג. 1.2.3.4:25
✅ נכון! Port 443 הוא HTTPS - גלישה מאובטחת

מה זה?

  • "כתובת הבית" של המחשב
  • localhost - "המחשב שלי"
  • משמש לבדיקות: שרת ולקוח על אותו מחשב

דוגמה מעשית:

אם אתה מפתח אתר על המחשב שלך ורוצה לראות אותו:

http://127.0.0.1:8080
או
http://localhost:8080

שתי הכתובות מצביעות על אותו מקום - המחשב שלך!

💻 כתיבת לקוח ב-Python

import socket

# יצירת Socket
my_socket = socket.socket(socket.AF_INET, socket.SOCK_STREAM)

# התחברות לשרת
my_socket.connect(('127.0.0.1', 1729))

# שליחת הודעה
my_socket.send("Hello Server!")

# קבלת תשובה
data = my_socket.recv(1024)  # קבל עד 1024 bytes
print(data)

# סגירת החיבור
my_socket.close()

הסבר הפרמטרים:

  • AF_INET = IPv4 (פרוטוקול האינטרנט)
  • SOCK_STREAM = TCP (חיבור אמין)
  • 1024 = מקסימום bytes לקרוא

🖥️ כתיבת שרת ב-Python

import socket

# יצירת Socket
server_socket = socket.socket(socket.AF_INET, socket.SOCK_STREAM)

# קשירה לכתובת ופורט
server_socket.bind(('0.0.0.0', 1729))

# האזנה לחיבורים
server_socket.listen(1)

print("Server is listening...")

# קבלת חיבור מלקוח
client_socket, address = server_socket.accept()
print(f"Connected to {address}")

# קבלת מידע
data = client_socket.recv(1024)

# שליחת תשובה
client_socket.send("Hello from Server!")

# סגירה
client_socket.close()
server_socket.close()

🔄 הבדלים בין שרת ללקוח

שרת לקוח
bind() - קושר לפורט connect() - מתחבר לשרת
listen() - מאזין אין צורך
accept() - ממתין ללקוח מתחבר מיד
0.0.0.0 - כל הכתובות IP מדויק של השרת
💡 זכור: השרת ממתין, הלקוח יוזם!

📋 מה זה פרוטוקול תקשורת?

פרוטוקול = חוקים להעברת מידע בין שרת ללקוח

דוגמה: פרוטוקול פשוט

לקוח → שרת: "TIME"
שרת → לקוח: "14:30:25"
לקוח → שרת: "NAME"
שרת → לקוח: "MyServer"
לקוח → שרת: "EXIT"
שרת: סוגר חיבור

🎯 עקרונות תכנון פרוטוקול

1. ברור וחד-משמעי

כל צד יודע מה לצפות ומה לשלוח

2. קצר ויעיל

לא לבזבז bandwidth - פקודות קצרות

3. עמיד בשגיאות

מה קורה אם יש בעיה? צריך לטפל!

תרגיל חשיבה:

עצב פרוטוקול לשרת שמחזיר מזג אויר. איזה פקודות תגדיר?

⚠️ בעיה נפוצה: אורכי הודעות

בעיה: איך הלקוח יודע כמה bytes לקרוא?

פתרון 1: אורך קבוע

# השרת תמיד שולח 100 bytes (עם padding)

# השרת שולח קודם את האורך

🧠 שאלת הבנה #2

מה יקרה אם השרת לא רץ והלקוח מנסה להתחבר?

א. הלקוח יחכה לנצח
ב. Connection Refused - הלקוח יקבל שגיאה
ג. החיבור יצליח בכל מקרה
✅ נכון! אם אין שרת שמאזין בפורט, הלקוח יקבל שגיאת Connection Refused

🛡️ טיפול בשגיאות - Try-Except

חובה בעבודה עם רשתות! 

import socket

try:
    my_socket = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
    my_socket.connect(('127.0.0.1', 1729))
    my_socket.send("Hello")
    data = my_socket.recv(1024)

except socket.timeout:
    print("Connection timed out!")

except ConnectionRefusedError:
    print("Server is not running!")

except Exception as e:
    print(f"Error: {e}")

finally:
    my_socket.close()  # תמיד לסגור!

למה חשוב?

  • מונע קריסת התוכנית
  • מאפשר לטפל בבעיות רשת
  • משחרר משאבים (close) גם במקרה של שגיאה
