מהי רשת נוירונים חוזרת?

🔄 רשת חוזרת (RNN)

רשת נוירונים המסוגלת לעבד רצפי נתונים באורך משתנה ולשמור מידע ממצבים קודמים

x₁
→
RNN
→
h₁
→
y₁

מאפיינים עיקריים:

  • קלט ופלט משתנים בזמן
  • זיכרון פנימי (hidden state)
  • שיתוף משקלים בין צעדי זמן
  • יכולת לעבד רצפים באורך משתנה
4

ארכיטקטורת RNN

מבנה הרשת החוזרת

x(t)
⬇
RNN Cell
⬇
h(t)
⬇
y(t)
h(t) = tanh(W_h * h(t-1) + W_x * x(t) + b)
y(t) = W_y * h(t) + b_y

רכיבי המודל:

  • x(t) - קלט בזמן t
  • h(t) - מצב נסתר בזמן t
  • y(t) - פלט בזמן t
  • W_h, W_x, W_y - מטריצות משקלים

למידה ב-RNN: חלחול לאחור בזמן

🔄 Backpropagation Through Time (BPTT)

אלגוריתם למידה המתאים לרשתות חוזרות - מחשב גרדיאנטים לאורך הרצף הזמני

שלב 1: Forward Pass

העברת הנתונים קדימה דרך הרשת לאורך כל צעדי הזמן

שלב 2: חישוב Loss

חישוב הפסד בכל צעד זמן והפסד כולל

שלב 3: Backward Pass

חישוב גרדיאנטים מהסוף לתחילה (unfolding בזמן)

שלב 4: עדכון משקלים

עדכון פרמטרי הרשת על בסיס הגרדיאנטים

⚠️ אתגר: Vanishing Gradient Problem

ברצפים ארוכים, הגרדיאנטים נוטים להיעלם, מה שמקשה על למידת תלות ארוכת טווח

מודל שפה (Language Model)

📝 מודל שפה

מודל הלומד להעריך הסתברות למילה הבאה ברצף, ומסוגל ליצור טקסט חדש

P(w₁, w₂, ..., wₙ) = ∏ᵢ₌₁ⁿ P(wᵢ | w₁, w₂, ..., wᵢ₋₁)

דוגמה אינטראקטיבית

קלט: "מחר השמש"

⬇

מודל השפה חוזה: "תזרח" (0.8), "תשקע" (0.15), "תיעלם" (0.05)

סוגי מודלים:

  • Character-level: חיזוי תו הבא
  • Word-level: חיזוי מילה הבאה
  • Subword-level: חיזוי חלקיק מילה הבא

דגימה שלילית (Negative Sampling)

⚡ Negative Sampling

טכניקה לייעול אימון מודלי שפה ע"י דגימה של דוגמאות שליליות במקום חישוב על כל המילים

איך זה עובד?

במקום: חישוב softmax על כל 50,000 מילים

עושים: 1 דוגמה חיובית + 5-20 דוגמאות שליליות

אספקט Softmax רגיל Negative Sampling
מורכבות זמן O(V) - לכל מילה בוקבולרי O(k) - k דוגמאות שליליות
זיכרון רב מעט
דיוק מושלם קירוב טוב
מהירות אימון איטי מהיר
# דוגמה קונספטואלית # במקום לחשב הסתברות לכל 50,000 מילים: # P(target|context) = softmax(u_target · v_context) # דוגמה שלילית - רק 5-20 מילים: positive_sample = "מלך" # המילה הנכונה negative_samples = ["כדור", "ריקוד", "מחשב", "עץ", "שמש"] # מחשבים הסתברות רק עבור אלה!

מנגנון קשב (Attention)

👁️ Attention Mechanism

מנגנון המאפשר למודל להתמקד בחלקים רלוונטיים של הקלט בכל צעד

בעיית וקטור ההקשר הבודד

❌ במשפטים ארוכים, וקטור אחד לא מספיק

✅ Attention פותר זאת ע"י התמקדות דינמית

Attention(Q,K,V) = softmax(QK^T/√d_k)V

דוגמה: תרגום עם Attention

אנגלית: "The quick brown fox jumps"

עברית: "השועל החום הזריז קופץ"

Attention: כשמתרגמים "החום" המודל מתמקד ב-"brown"

יתרונות Attention:

  • טיפול ברצפים ארוכים
  • פתרון בעיית Bottleneck
  • פרשנות טובה יותר
  • ביצועים משופרים

יישום מעשי - קוד Python

import torch import torch.nn as nn class SimpleRNN(nn.Module): def __init__(self, input_size, hidden_size, output_size): super(SimpleRNN, self).__init__() self.hidden_size = hidden_size self.rnn = nn.RNN(input_size, hidden_size, batch_first=True) self.fc = nn.Linear(hidden_size, output_size) def forward(self, x): # x shape: (batch_size, seq_length, input_size) h0 = torch.zeros(1, x.size(0), self.hidden_size) out, _ = self.rnn(x, h0) # out shape: (batch_size, seq_length, hidden_size) out = self.fc(out[:, -1, :]) # אחרון בלבד return out # יצירת המודל model = SimpleRNN(input_size=10, hidden_size=20, output_size=5)

  • הגדרת ארכיטקטורת הרשת

  • הגדרת Loss function ו-Optimizer

  • הערכה ובדיקה

מדדי הערכה למודלי שפה

מדד תיאור טווח ערכים עדיפות Perplexity מדד לביטחון המודל 1 - ∞ נמוך יותר BLEU Score השוואה לתרגום אמת 0 - 1 גבוה יותר Accuracy אחוז ניחושים נכונים 0% - 100% גבוה יותר F1-Score ממוצע הרמוני Precision & Recall 0 - 1 גבוה יותר
Perplexity = 2^(-1/N * Σlog₂P(wᵢ))

דוגמה: Perplexity

אם המודל בטוח ב-100% בכל מילה → Perplexity = 1

אם המודל מנחש אקראית מ-1000 מילים → Perplexity = 1000

אתגרים ופתרונות ב-RNN

אתגר תיאור הבעיה פתרונות
Vanishing Gradient גרדיאנטים נעלמים ברצפים ארוכים LSTM, GRU, Gradient Clipping
Exploding Gradient גרדיאנטים גדלים מאוד Gradient Clipping, Learning Rate
חישוב איטי עיבוד סדרתי לא מקבילי Transformer, CNN, Parallelization
זיכרון מוגבל קושי בזכירת מידע רחוק Attention, External Memory

טכניקות אופטימיזציה:

  • Gradient Clipping: הגבלת גודל הגרדיאנט
  • Dropout: מניעת Overfitting
  • Batch Normalization: ייצוב האימון
  • Learning Rate Scheduling: התאמת קצב הלמידה
