מגמת סייבר - כיתות י"א-י"ב

🎯 מטרות התרגיל

תרגיל זה מיועד לפתח את יכולות החקירה והניתוח שלכם במערכות הפעלה ואפליקציות. תלמדו כיצד תוכנות שומרות מידע, איפה הן מאחסנות הגדרות, וכיצד מערכת ההפעלה Windows מנהלת פונקציונליות מיוחדת.

📚 מה תלמדו?

  • עבודה עם Windows Registry
  • זיהוי קבצי תצורה של אפליקציות
  • ניתוח מסדי נתונים (SQLite)
  • הבנת COM Objects במערכת Windows
  • שימוש בכלי ניתוח תוכנה מקצועיים

📋 הוראות כלליות

המבנה: התרגיל בנוי מ-4 משימות עיקריות, כאשר כל משימה מתמקדת בהיבט אחר של ניתוח תוכנה.

משך ביצוע משוער: 3-4 שעות (ניתן לפצל למספר מפגשים)

עבודה: ניתן לעבוד בזוגות, אך כל תלמיד חייב להגיש דוח אישי

⚠️ חשוב! חלק מהמשימות דורשות גישה ל-Registry ולקבצי מערכת. ודאו שאתם עובדים על מחשב מאושר ולא על מערכת ייצור קריטית. תמיד צרו גיבוי לפני שינויים ב-Registry.

🛠️ כלים מומלצים

  • Process Monitor (Procmon) - מעקב אחר פעולות מערכת בזמן אמת
  • Registry Editor (regedit) - עריכת ה-Registry
  • DB Browser for SQLite - צפייה ועריכת קבצי SQLite
  • Everything Search - חיפוש קבצים מהיר
  • Notepad++ - עורך טקסט מתקדם
  • PowerShell / CMD - שורת פקודה

זיכרון המיקום ב-Registry Editor

💡 רקע תיאורטי

Windows Registry הוא מסד נתונים היררכי שמאחסן הגדרות ואפשרויות עבור מערכת ההפעלה והאפליקציות. כל אפליקציה יכולה לשמור העדפות משתמש, מיקומים אחרונים, והגדרות שונות ב-Registry.

למה זה חשוב? הבנת Registry חיונית לחקירות פורנזיות, ניתוח malware, והבנת התנהגות תוכנות.

🎯 המשימה

שמתם לב שכאשר אתם פותחים את regedit.exe, הוא תמיד נפתח למיקום האחרון שצפיתם בו? מצאו איך ואיפה Registry Editor שומר את המידע הזה.

💡 רמזים ושאלות מנחות:
  • באיזה ענף (Hive) של ה-Registry סביר שתוכנה תשמור העדפות משתמש?
  • איך תוכלו לעקוב אחר הפעולות שמבצע regedit בזמן הסגירה?
  • נסו להשתמש ב-Process Monitor - מה regedit קורא וכותב?
  • חפשו מפתחות הקשורים ל-"LastKey" או "Regedit"

📝 מה לכלול בדוח

  1. את הנתיב המדויק ב-Registry שבו נשמר המידע
  2. את שם ה-Value וסוג הנתונים שבו (String, DWORD, וכו')
  3. הסבר איך זיהיתם את המיקום (איזה כלים השתמשתם?)
  4. צילום מסך של ה-Registry Key הרלוונטי
  5. ניסוי: שנו את ה-Value ידנית ובדקו שזה באמת משפיע על regedit
  6. מדוע לדעתכם Microsoft בחרו לשמור את המידע ב-Registry ולא בקובץ תצורה רגיל?

🔬 שאלת העמקה

חקרו אילו תוכנות נוספות של Windows שומרות את המיקום האחרון שבו היו? (למשל: Event Viewer, Services, Performance Monitor). האם יש דפוס משותף לאופן השמירה?

Python Startup Script - ברכת בוקר טוב!

💡 רקע תיאורטי

בדומה לקובץ .bashrc בסביבות Linux, Python תומך בהרצת קובץ אתחול אוטומטי בכל פעם שנפתח ה-interpreter האינטראקטיבי. זה מאפשר להגדיר פונקציות נפוצות, לייבא מודולים, או להציג הודעות.

שימושים פרקטיים:

  • טעינת ספריות נפוצות אוטומטית
  • הגדרת פונקציות עזר
  • קביעת נתיב לפרויקטים
  • שיפור history וצבעים

🎯 המשימה

מצאו כיצד לגרום ל-Python interpreter להדפיס הודעה מותאמת אישית (למשל "בוקר טוב!" או "Ready to code!") בכל פעם שנפתח מצב אינטראקטיבי חדש.

💡 רמזים ושאלות מנחות:
  • חפשו את משתנה הסביבה PYTHONSTARTUP
  • באיזה ספריה נמצאים קבצי תצורה של Python במערכת שלכם?
  • האם יש קובץ ברירת מחדל שנטען תמיד?
  • איך תוכלו לבדוק אילו קבצים Python טוען בהפעלה?

📝 מה לכלול בדוח

  1. הסבר מפורט על שיטת ה-startup של Python
  2. את הנתיב המלא לקובץ שיצרתם
  3. את תוכן הקובץ (כולל הערות מסבירות)
  4. צילום מסך של Python interpreter עם ההודעה המותאמת
  5. הדגמה של 3 דברים שימושיים נוספים שניתן לעשות בקובץ startup
  6. איך תוכלו להשבית את ה-startup script באופן זמני?
# דוגמה לקובץ startup מתקדם:
import sys
import os
from datetime import datetime

# צבעים ב-terminal
class Colors:
    HEADER = '\033[95m'

# הודעת ברכה

חקרו כיצד עובד הנושא ב-IDE כמו PyCharm או VSCode. האם הם משתמשים באותו מנגנון? האם יש להם מנגנון משלהם?

Chrome Passwords Database - SQLite Forensics

💡 רקע תיאורטי

SQLite הוא מנוע מסד נתונים קל משקל, עצמאי וללא צורך בשרת. הוא נמצא בשימוש נרחב באפליקציות, דפדפנים, מכשירי סלולר ועוד.

Chrome ו-Chromium משתמשים ב-SQLite לאחסון:

  • סיסמאות שמורות
  • היסטוריית גלישה
  • Cookies
  • Bookmarks
  • Extensions data

חשיבות אבטחתית: הבנת אחסון סיסמאות חיונית להבנת איומים ולהגנה על מידע.

🎯 המשימה

מצאו את מסד הנתונים שבו Chrome שומר את פרטי ההתחברות (usernames) של המשתמש. נתחו את המבנה שלו והבינו כיצד המידע מאוחסן.

⚠️ אזהרת אבטחה! סיסמאות ב-Chrome מוצפנות. במשימה זו תתמקדו בשמות המשתמש ובמבנה הנתונים בלבד. אין לנסות לפענח סיסמאות ללא אישור מפורש. זהו תרגיל לימודי על SQLite ולא על פריצה.
💡 רמזים ושאלות מנחות:
  • היכן Chrome שומר את נתוני המשתמש? (רמז: AppData)
  • חפשו קובץ בשם "Login Data" או דומה
  • השתמשו ב-DB Browser for SQLite לפתוח את הקובץ
  • Chrome חייב להיות סגור כדי לגשת לקובץ (למה?)
  • איזה טבלאות קיימות במסד הנתונים?

📝 מה לכלול בדוח

  1. הנתיב המדויק לקובץ מסד הנתונים
  2. רשימת הטבלאות במסד הנתונים והתפקיד של כל אחת
  3. מבנה הטבלה הראשית (שמות עמודות וסוגי נתונים)
  4. צילום מסך של המידע ב-DB Browser (מוסתר/מטושטש כראוי!)
  5. הסבר איך Chrome מגן על הסיסמאות (חיפוש עצמאי)
  6. SQL Query שמחזיר רק את כתובות האתרים ושמות המשתמש
  7. למה לדעתכם Chrome בחר ב-SQLite ולא במסד נתונים אחר?
-- דוגמת SQL Query לניתוח:
SELECT 
    origin_url as Website,
    username_value as Username,
    date_created as FirstSaved
FROM logins
WHERE username_value != ''
ORDER BY date_created DESC
LIMIT 10;

🔬 שאלות העמקה

  1. חקרו את ההבדלים בין אחסון הסיסמאות ב-Chrome, Firefox ו-Edge
  2. מהם ה-DPAPI של Windows וכיצד הם משפיעים על הצפנת הסיסמאות?
  3. האם ניתן לייצא את הנתונים בפורמט JSON/CSV? איך?
  4. מה קורה כאשר משתמש מחובר ל-Google Account? איפה נשמר הסנכרון?

Recycle Bin - COM Objects ו-Shell Extensions

💡 רקע תיאורטי

COM (Component Object Model) היא טכנולוגיה של Microsoft שמאפשרת לתוכנות לתקשר זו עם זו. ה-Shell של Windows משתמש ב-COM כדי להרחיב את הפונקציונליות שלו.

Shell Extensions מאפשרות לתוסיפים:

  • להוסיף פריטים לתפריט ההקשר (Context Menu)
  • לשנות סמלים (Icons)
  • להגדיר עמודות נוספות בתצוגת פרטים
  • לספק Property Pages

Recycle Bin הוא דוגמה מצוינת - הוא נראה כמו תיקייה רגילה, אך מתנהג אחרת לחלוטין!

🎯 המשימה

Explorer מתייחס לסל המיחזור (Recycle Bin) בצורה מיוחדת - יש לו תפריט הקשר שונה, סמל מיוחד, ופונקציות ייחודיות. מצאו מי אחראי על הפונקציונליות המיוחדת הזו ואיפה היא מוגדרת במערכת.

💡 רמזים ושאלות מנחות:
  • Recycle Bin מזוהה על ידי CLSID (Class ID) ייחודי
  • ה-CLSID הוא: {645FF040-5081-101B-9F08-00AA002F954E}
  • חפשו את ה-CLSID הזה ב-Registry
  • היכן מוגדרות ה-Shell Extensions?
  • איזה קובץ DLL מספק את הפונקציונליות?
  • השתמשו ב-ShellExView (אופציונלי) לראות Extensions

📝 מה לכלול בדוח

  1. את ה-CLSID של Recycle Bin ומשמעותו
  2. את המיקום המדויק ב-Registry שבו מוגדרים פרטי ה-COM Object
  3. את שם קובץ ה-DLL שמספק את הפונקציונליות
  4. רשימה של Shell Extensions שמוגדרות ל-Recycle Bin
  5. צילומי מסך של ה-Registry Keys הרלוונטיים
  6. השוואה: מצאו תיקייה מיוחדת נוספת (למשל Control Panel, Network) ובדקו את ה-COM שלה
  7. נסו למצוא איפה שמור הסמל (Icon) של Recycle Bin (מלא/ריק)
# PowerShell Script לחיפוש Shell Extensions: Get-ChildItem -Path "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\Explorer\ShellIconOverlayIdentifiers" | ForEach-Object { $name = $_.PSChildName $clsid = (Get-ItemProperty $_.PSPath).'(default)' [PSCustomObject]@{ Name = $name CLSID = $clsid } } | Format-Table -AutoSize

🔬 שאלות העמקה

  1. חקרו איך Malware משתמש ב-Shell Extensions להסתרה או להתמדה (Persistence)
  2. מצאו 3 דוגמאות של תוכנות צד שלישי שמוסיפות Shell Extensions
  3. איך תוכלו להשבית Shell Extension מסוימת?
  4. מה ההבדל בין ה-Recycle Bin של כל משתמש לבין התיקייה המערכתית $Recycle.Bin?
  5. ניסו ליצור COM Object פשוט משלכם (בונוס!)

🌟 משימות נוספות

5️⃣ Browser Extensions Analysis

מצאו היכן Chrome שומר את ההרחבות (Extensions) המותקנות. נתחו את מבנה התיקיות, מצאו את קובץ ה-manifest.json, והבינו את המבנה.

6️⃣ Environment Variables

מצאו היכן Windows שומר את משתני הסביבה (Environment Variables) ב-Registry. בדקו את ההבדל בין משתני מערכת למשתני משתמש.
