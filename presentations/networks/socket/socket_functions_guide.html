פונקציות שימושיות נוספות ב־socket

דף לימוד לתלמידי תיכון: הסבר פשוט וברור על פונקציות חשובות מעבר ל־bind, listen, accept, connect, send ו־recv.

מטרה: להבין מתי כדאי להשתמש בכל פונקציה, מה היא עושה בפועל, ואיך היא נראית בקוד אמיתי.

למה בכלל צריך פונקציות נוספות?

כשבונים שרת או לקוח בסיסי, אפשר להסתפק בכמה פקודות עיקריות. אבל ברגע שרוצים קוד אמיתי, נוח יותר, בטוח יותר ויציב יותר — צריך להכיר עוד כלים מתוך המחלקה socket.

  • שליטה בהתנהגות החיבור — למשל timeout או blocking.
  • שליטה בהגדרות socket — למשל שימוש חוזר בפורט.
  • קבלת מידע על הכתובות — מי התחבר, לאיזה פורט התחברנו.
  • תמיכה ב־UDP — עם sendto ו־recvfrom.

3 פונקציות שחייבים להכיר

  1. setsockopt() — כדי להימנע מבעיית Address already in use.
  2. settimeout() — כדי שהתוכנית לא תחכה לנצח.
  3. getpeername() — כדי לדעת מי הצד השני.
אלה פונקציות שממש משדרגות שרתים של תלמידים — גם מבחינת מקצועיות וגם מבחינת נוחות בדיבוג.

1. setsockopt()

מגדירה אפשרויות מיוחדות של ה־socket.

import socket

server_socket = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
server_socket.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)

למה זה חשוב? אם סוגרים שרת ומריצים אותו שוב מהר, לפעמים הפורט עדיין תפוס. הפקודה הזו מאפשרת שימוש חוזר מהיר.

2. getsockname()

מחזירה את הכתובת והפורט המקומיים של ה־socket.

print(client_socket.getsockname())
# למשל: ('127.0.0.1', 5000)

שימושי במיוחד כשנותנים למערכת לבחור פורט אוטומטית בעזרת פורט 0.

3. getpeername()

מחזירה את הכתובת והפורט של הצד השני בחיבור.

print(client_socket.getpeername())
# למשל: ('192.168.1.10', 52341)

טוב מאוד ללוגים, דיבוג ותיעוד מי התחבר לשרת.

4. settimeout()

קובעת מקסימום זמן המתנה לפעולות כמו קבלה או חיבור.

client_socket.settimeout(5)

try:
    data = client_socket.recv(1024)
except socket.timeout:
    print("Timeout")

במקום שהתוכנית תיתקע לנצח, היא תעצור אחרי 5 שניות.

5. setblocking()

קובעת אם ה־socket יעבוד במצב blocking או non-blocking.

sock.setblocking(False)

ב־blocking התוכנית מחכה. ב־non-blocking היא לא מחכה ואם אין מידע — מתקבלת שגיאה מיד.

6. shutdown()

סגירה חלקית של החיבור.

sock.shutdown(socket.SHUT_WR)

אומר: אני לא שולח יותר, אבל עדיין מוכן לקבל.

7. sendto()

שליחה ב־UDP.

sock.sendto(b"hello", ("127.0.0.1", 5000))

ב־UDP אין צורך ב־connect רגיל, אלא שולחים ישירות לכתובת.

8. recvfrom()

קבלה ב־UDP.

data, addr = sock.recvfrom(1024)

מחזירה גם את ההודעה וגם את כתובת השולח.

9. connect_ex()

בדומה ל־connect(), אבל מחזירה קוד שגיאה במקום לזרוק חריגה.

result = sock.connect_ex(("127.0.0.1", 80))
if result == 0:
    print("Port open")

שימושי מאוד לסריקת פורטים ולבדיקות.

דוגמת קוד מלאה: שרת עם SO_REUSEADDR ו־timeout

import socket

HOST = "127.0.0.1"
PORT = 5000

server_socket = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
server_socket.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)
server_socket.bind((HOST, PORT))
server_socket.listen(1)
server_socket.settimeout(10)

print("השרת ממתין לחיבור...")

try:
    client_socket, addr = server_socket.accept()
    print("Client connected:", addr)
    print("Peer:", client_socket.getpeername())
    print("Local:", client_socket.getsockname())

    data = client_socket.recv(1024)
    print("Received:", data.decode())
    client_socket.sendall("שלום מהשרת".encode())
    client_socket.close()

except socket.timeout:
    print("לא התחבר לקוח בזמן")

finally:
    server_socket.close()
הסבר על הדוגמה
  • setsockopt מאפשר הפעלה מחדש של השרת בלי להיתקע על אותו פורט.
  • settimeout(10) מונע המתנה אינסופית ללקוח.
  • accept() מחזיר socket חדש לחיבור עם הלקוח.
  • getpeername() מציג מי התחבר.
  • getsockname() מציג על איזה כתובת/פורט השרת מדבר בפועל.

טבלת סיכום

פונקציה מה היא עושה מתי נשתמש
setsockopt() מגדירה אפשרויות ל־socket כששרת נפתח שוב על אותו פורט
getsockname() מחזירה כתובת ופורט מקומיים לבדיקה, לוגים ודיבוג
getpeername() מחזירה את כתובת הצד השני כשרוצים לדעת מי התחבר
settimeout() קובעת זמן המתנה מקסימלי כדי לא להיתקע לנצח
setblocking() מעבירה בין blocking ל־non-blocking בשרתים מתקדמים
shutdown() סגירה חלקית של החיבור כשמסיימים לשלוח אבל עדיין רוצים לקבל
sendto() שליחה ב־UDP בתקשורת ללא חיבור
recvfrom() קבלה ב־UDP כשצריך גם את כתובת השולח
connect_ex() חיבור עם קוד תוצאה במקום חריגה סריקות ובדיקות פורטים

בדיקה עצמית

שאלה 1: איזו פונקציה עוזרת כאשר מקבלים Address already in use?

התשובה היא setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1).

שאלה 2: איזו פונקציה תחזיר את כתובת הצד השני בחיבור?

התשובה היא getpeername().

שאלה 3: באיזה זוג פונקציות משתמשים בדרך כלל ב־UDP?

בדרך כלל משתמשים ב־sendto() ו־recvfrom().

מסר מקצועי לסיום

תלמיד שיודע רק send ו־recv מבין את הבסיס. תלמיד שמכיר גם setsockopt, settimeout ו־getpeername כבר כותב קוד רשת ברמה הרבה יותר מקצועית.

ההמלצה שלי: בכל שרת TCP לימודי, הוסיפו לפחות SO_REUSEADDR וחשבו האם צריך גם timeout.
