🎯 תנאי עצירה (Base Case):

  • ✅ מישהו ניצח
  • ✅ הלוח מלא (תיקו)

💡 למה זה עובד? כל קריאה רקורסיבית פותרת בעיה קטנה יותר, עד שמגיעים למצב פשוט שאפשר להעריך ישירות!

💻 מבנה הקוד - חלק 1

הגדרת הפונקציה והפרמטרים:

def minimax(self, board, depth, is_maximizing):
    # board - מצב הלוח הנוכחי
    # depth - עומק הרקורסיה (כמה צעדים עמוק אנחנו)
    # is_maximizing - האם זה תור המחשב (True) או השחקן (False)
    
    # שלב 1: בדיקת תנאי עצירה
    winner = self.winner(board)
    
    if winner == self.ai_player:
        return 1  # המחשב ניצח
    elif winner == self.human_player:
        return -1  # השחקן ניצח
    elif self.is_board_full(board):
        return 0  # תיקו

📝 הסבר על הפרמטרים:

  • board: רשימה בגודל 9 המייצגת את הלוח (0-8 = מיקומים)
  • depth: מונה שעוקב אחרי כמה צעדים בעתיד אנחנו
  • is_maximizing: דגל בוליאני - True = תור MAX, False = תור MIN

🎯 תנאי העצירה החשובים:

לפני שממשיכים ברקורסיה, בודקים אם המשחק נגמר:

  • האם יש מנצח? → החזר +1 או -1
  • האם הלוח מלא? → החזר 0
  • אחרת → המשך בחיפוש

💻 מבנה הקוד - חלק 2: שחקן MAX

כאשר זה תור המחשב (מקסימיזציה):

    if is_maximizing:  # תור המחשב
        best_score = -math.inf  # מתחילים מהציון הכי נמוך
        
        # עוברים על כל מיקום בלוח
        for i in range(9):
            if board[i] == ' ':  # אם המיקום פנוי
                
                # מנסים את המהלך
                board[i] = self.ai_player
                
                # קריאה רקורסיבית - בודקים מה יקרה
                # שימו לב: is_maximizing הופך ל-False!
                score = self.minimax(board, depth + 1, False)
                
                # מבטלים את המהלך (backtrack)
                board[i] = ' '
                
                # שומרים את הציון הטוב ביותר
                best_score = max(score, best_score)
        
        return best_score

🔍 מה קורה כאן?

  1. מאתחלים: best_score = -∞ (הכי נמוך שיש)
  2. עוברים: על כל מיקום פנוי בלוח
  3. מנסים: שמים את הסימן שלנו
  4. בודקים: קוראים רקורסיבית לבדוק מה יקרה
  5. מחזירים: את הלוח למצב הקודם
  6. שומרים: את הציון הגבוה ביותר

💡 הטריק: שימו לב שב-is_maximizing משנים ל-FALSE בקריאה הבאה, כי עכשיו תור היריב!

🎯 העיקרון: כל צד מנסה לממש את המטרה שלו - MAX רוצה ציון גבוה, MIN רוצה ציון נמוך!

💻 מציאת המהלך הטוב ביותר

הפונקציה שבוחרת את המהלך:

def find_best_move(self, board):
    best_score = -math.inf
    best_move = None
    
    # עוברים על כל מיקום פנוי
    for i in range(9):
        if board[i] == ' ':
            # מנסים את המהלך
            board[i] = self.ai_player
            
            # מקבלים את הציון למהלך הזה
            score = self.minimax(board, 0, False)
            
            # מבטלים את המהלך
            board[i] = ' '
            
            # אם זה המהלך הטוב ביותר עד כה - שומרים אותו
            if score > best_score:
                best_score = score
                best_move = i
    
    return best_move

🎯 שלבי הפעולה:

  1. עוברים על כל מיקום פנוי בלוח
  2. לכל מיקום - מריצים את minimax
  3. minimax מחזיר ציון למהלך הזה
  4. שומרים את המיקום עם הציון הגבוה ביותר
  5. מחזירים את המיקום הטוב ביותר

💡 שימו לב: בקריאה הראשונה ל-minimax אנחנו מעבירים False ל-is_maximizing, כי אנחנו רוצים לדמיין מה השחקן יעשה אחרי המהלך שלנו!

🚀 נקודות העשרה מתקדמות

1️⃣ Alpha-Beta Pruning (קיצוץ אלפא-בטא)

הבעיה: מינימקס בודק את כל המהלכים האפשריים - זה יכול להיות איטי!

הפתרון: Alpha-Beta Pruning "מקצץ" ענפים בעץ שאנחנו יודעים שלא יובילו למהלך הטוב ביותר.

def minimax_alpha_beta(board, depth, alpha, beta, is_maximizing):
    # alpha = הציון הטוב ביותר ל-MAX עד כה
    # beta = הציון הטוב ביותר ל-MIN עד כה
    
    if is_maximizing:
        for move in possible_moves:
            score = minimax_alpha_beta(..., alpha, beta, False)
            alpha = max(alpha, score)
            if beta <= alpha:
                break  # קיצוץ!
        return alpha

💡 יתרון: יכול להפחית את מספר הצמתים שנבדקים ב-50% ויותר!

2️⃣ הגבלת עומק (Depth Limit)

במשחקים מורכבים (כמו שח-מט), אי אפשר לבדוק עד הסוף. פתרון:

  • מגבילים את עומק החיפוש (למשל, 5 מהלכים קדימה)
  • משתמשים בפונקציית הערכה במקום ניצחון/הפסד
  • הפונקציה נותנת "ציון" למצב הלוח

3️⃣ פונקציות הערכה (Evaluation Functions)

def evaluate_board(board):
    score = 0
    # בשחמט למשל:
    score += count_pieces(board) * 10  # כמות כלים

🎯 למה זה עובד כל כך טוב?

מינימקס מניח משחק אופטימלי משני הצדדים, מה שמתאים למצבים תחרותיים רבים בעולם האמיתי!

⚡ ניתוח מורכבות ואופטימיזציות

📊 מורכבות זמן

מורכבות: O(b^d)

  • b = branching factor (כמה מהלכים אפשריים בכל שלב)
  • d = depth (עומק העץ)

דוגמה: XO בתחילת המשחק: b≈9, d≈9 → כ-387,420,489 מצבים!

🚀 שיפורים ואופטימיזציות:

1. Alpha-Beta Pruning

שיפור: O(b^(d/2)) במקרה הטוב

מפחית את מספר הצמתים בכמעט פי 2!

2. Move Ordering (סדר מהלכים)

בודקים מהלכים "טובים" לפני "רעים":

  • מהלכים שניצחו בעבר
  • שליטה במרכז
  • תקיפת כלים חשובים

3. Transposition Tables (טבלאות טרנספוזיציה)

שומרים מצבים שכבר חישבנו:

memo = {}  # מילון לשמירת תוצאות

def minimax_memo(board, depth, is_max):
    board_key = tuple(board)  # המרה למפתח
    if board_key in memo:
        return memo[board_key]  # החזרת תוצאה שמורה
    
    result = minimax(board, depth, is_max)
    memo[board_key] = result  # שמירה
    return result

📈 השוואת ביצועים (XO):

  • מינימקס רגיל: ~300,000 צמתים

📚 סיכום - מה למדנו?

✅ עקרונות יסוד:

  • מינימקס הוא אלגוריתם לקבלת החלטות במשחקים תחרותיים
  • השחקנים מתחלפים: MAX רוצה ציון גבוה, MIN רוצה ציון נמוך
  • האלגוריתם בונה עץ של כל המהלכים האפשריים

🔄 רקורסיה:

  • הפונקציה קוראת לעצמה כדי לבדוק מהלכים עתידיים
  • יש תנאי עצירה ברורים (ניצחון/הפסד/תיקו)
  • מבטלים מהלכים (backtracking) כדי לנסות אפשרויות אחרות

💻 הקוד:

  • פונקציה אחת מטפלת בשני המקרים (MAX ו-MIN)
  • משתמשים ב-math.inf למינימום/מקסימום התחלתי
  • הפרמטר is_maximizing קובע את התנהגות השחקן

🚀 שיפורים מתקדמים:

  • Alpha-Beta Pruning: מפחית צמתים מיותרים
  • Depth Limiting: מגביל עומק למשחקים מורכבים
  • Evaluation Functions: מעריכים מצבים ביניים
  • Memoization: שומר תוצאות לחיפושים חוזרים
