מה זה ממשק משתמש (UI) ולמה אנחנו צריכים אותו?

עד עכשיו כל ההרצות שלכם היו דרך שורת פקודה. מי שרוצה להריץ ניסוי — צריך לדעת Python, להבין את הקוד, ולדעת לשנות פרמטרים בצורה ידנית. ממשק משתמש הוא שכבה גרפית שמאפשרת לכל אחד להשתמש בכלי שבניתם — גם מי שלא יודע לתכנת.

❌ בלי UI
  • צריך להבין את הקוד
  • שינוי פרמטרים = עריכת קוד
  • תוצאות — רק טקסט בטרמינל
  • בלי גרפים, בלי השוואה
✅ עם UI (Gradio)
  • ממשק גרפי נגיש לכל אחד
  • Sliders לשינוי פרמטרים
  • גרפים ותרשימים אוטומטיים
  • ייצוא תוצאות בקליק

Interface vs. Blocks — שתי פילוסופיות

🚀 gr.Interface — מהיר ופשוט

לפונקציה אחת, קלטים פשוטים, ללא ורסטיליות. תוצאה מיידית.

PYTHON 
import gradio as gr

def greet(name):
    return f"שלום, {name}!"

gr.Interface(
    fn=greet,
    inputs="text",
    outputs="text"
).launch()
🏗️ gr.Blocks — גמיש ומלא

פריסה מלאה, מספר פונקציות, events מורכבים. כמו HTML+JS.

PYTHON 
with gr.Blocks() as demo:
    title = gr.Markdown("# כותרת")
    with gr.Row():
        inp = gr.Textbox(label="הכנס שם")
        out = gr.Textbox(label="תוצאה")
    btn = gr.Button("שלח")
    btn.click(fn=greet,
              inputs=inp,
              outputs=out)
demo.launch()
🔍 פירוק שורה-שורה לחץ לפתיחה ▼

title = gr.Markdown("# כותרת")
→
מוסיפים כותרת טקסטואלית לממשק. ה-# הוא תחביר Markdown — יוצר כותרת גדולה (<h1>). המשתנה title מאחסן את הרכיב — אפשר לעדכן אותו מאוחר יותר.
with gr.Row():
→
פותחים שורה אופקית. כל רכיב שנוסיף בתוכה יופיע זה-לצד-זה (ולא אחד מתחת לשני).
inp = gr.Textbox(label="הכנס שם")
→
רכיב קלט — תיבת טקסט שהמשתמש מקליד בה. label= קובע את הכיתוב שמופיע מעל השדה בממשק. inp הוא שם המשתנה — נשתמש בו כ-inputs=inp בהמשך.
out = gr.Textbox(label="תוצאה")
→
רכיב פלט — תיבת טקסט שהממשק כותב אליה את התוצאה. label= מסמן למשתמש שזה פלט ולא קלט. out ישמש כ-outputs=out בקישור לכפתור.
btn = gr.Button("שלח")
→
יוצרים כפתור. הטקסט "שלח" הוא מה שרואים על הכפתור. לבד הכפתור לא עושה כלום — צריך לחבר אותו לפונקציה בשורה הבאה.
btn.click(fn=greet,
          inputs=inp,
          outputs=out)
→
חיבור הכפתור לפונקציה — הלב של Gradio.

inputs=inp מה לשלוח לפונקציה — הערך שבתיבת inp.
outputs=out לאן לכתוב את ה-return של הפונקציה — ישירות לתוך out.
demo.launch()
→

הסבר עמוק

מבנה Blocks — המושגים הבסיסיים

Blocks עובד כמו HTML — יש מיכלים (containers) שמחזיקים רכיבים (components). הסדר בקוד = הסדר על המסך. לחצו על כל מושג להרחבה ודוגמת קוד.

gr.Blocks() המיכל הראשי — עוטף את כל הממשק
לחץ להרחבה ▼

מה זה? gr.Blocks() הוא מיכל השורש של כל ממשק Gradio. כל רכיב שתוסיף חייב להיות בתוך ה-Blocks. משתמשים בו עם with כדי שהקוד בפנים יתרשם כחלק מהממשק.

פרמטרים שימושיים:

  • theme=gr.themes.Soft() — ערכת עיצוב
  • title="כותרת הדף" — כותרת שמופיעה בטאב הדפדפן
  • css="..." — CSS מותאם אישית
PYTHON — gr.Blocks() 
import gradio as gr

# הגדרת הממשק — כל מה שבתוך ה-with שייך לממשק
with gr.Blocks(
    theme=gr.themes.Soft(),
    title="ניסויי XO",
    css=".my-btn { background: blue; }"
) as demo:

    # כאן מגדירים את כל הרכיבים...
    gr.Markdown("# שלום עולם!")

# מחוץ ל-with — מפעילים את השרת
demo.launch()
📺 מה רואים בדפדפן: כותרת דף "ניסויי XO" בטאב, ממשק עם עיצוב Soft, וכיתוב "שלום עולם!" בגדול.
gr.Markdown() טקסט עשיר — כותרות, הדגשות, רשימות, קישורים
לחץ להרחבה ▼

מה זה? gr.Markdown() מאפשר להציג טקסט מעוצב בתוך הממשק באמצעות תחביר Markdown — בדיוק כמו ב-README ב-GitHub. זה לא רכיב קלט/פלט, אלא תצוגה סטטית בלבד.

תחביר Markdown עיקרי:

  • # כותרת ראשית → <h1>
  • ## כותרת משנה → <h2>, ### → <h3>
  • **מודגש** → מודגש
  • *נטוי* → נטוי
  • `קוד inline` → קוד
  • - פריט → רשימת תבליטים
  • [טקסט](URL) → קישור
PYTHON — gr.Markdown() דוגמאות 
with gr.Blocks() as demo:

    # כותרת ראשית של הממשק
    gr.Markdown("# 🧪 מערכת ניסויי XO")

    # תת-כותרת עם תיאור
    gr.Markdown("## הגדרת פרמטרים")
    gr.Markdown("הגדר את **gamma** ו-**exploration rate** לניסוי.")

    # Markdown מורכב — כמה שורות
    gr.Markdown("""
    ### 📖 הוראות שימוש
    1. הגדר את שם הניסוי
    2. כוון את הפרמטרים בעזרת ה-sliders
    3. לחץ על **הרץ ניסוי**

    > ⚠️ ריצה עם יותר מ-50,000 משחקים עלולה לקחת זמן רב.
    """)

    # Markdown דינמי — מתעדכן מתוך פונקציה!
    results_md = gr.Markdown("_ממתין לתוצאות..._")

    # ניתן להשתמש ב-results_md כ-output של פונקציה:
    # btn.click(fn=run, outputs=results_md)

demo.launch()
💡 טיפ: gr.Markdown() יכול לשמש גם כ-output — הפונקציה מחזירה מחרוזת Markdown ורכיב המD מתעדכן בזמן אמת. שימושי להצגת תוצאות בפורמט עשיר.
gr.Row() סידור אופקי — רכיבים זה-לצד-זה
לחץ להרחבה ▼

מה זה? כל רכיב שמוסיפים בתוך gr.Row() יופיע באותה שורה אופקית. ברירת המחדל של Gradio היא אנכי (כל רכיב בשורה חדשה) — Row משנה זאת.

מתי משתמשים? כשרוצים קלט ופלט זה-לצד-זה, או מספר כפתורים באותה שורה.

PYTHON — gr.Row() 
with gr.Blocks() as demo:
    gr.Markdown("## קלט ופלט זה-לצד-זה")

    # בלי Row — שני Textbox יופיעו אחד מתחת לשני
    # עם Row — יופיעו זה-לצד-זה :
    with gr.Row():
        inp = gr.Textbox(label="קלט")
        out = gr.Textbox(label="פלט")

    # Row עם שלושה כפתורים:
    with gr.Row():
        btn_run    = gr.Button("▶ הרץ",   variant="primary")
        btn_clear  = gr.Button("🗑 נקה",   variant="secondary")
        btn_cancel = gr.Button("⛔ בטל",   variant="stop")

demo.launch()
📺 מה רואים: שני תיבות טקסט זה-לצד-זה, ומתחתיהן שלושה כפתורים צבעוניים בשורה אחת.
gr.Column() סידור אנכי + שליטה ברוחב עם scale=
לחץ להרחבה ▼

מה זה? gr.Column() משמש בתוך Row כדי לחלק את הרוחב לעמודות. הפרמטר scale= קובע את היחס בין העמודות — scale=2 אומר "פי שניים רחוב מה-scale=1 לצדו".

scale בפועל: אם יש Column עם scale=2 וColumn עם scale=1, הראשון יתפוס ⅔ מהרוחב והשני ⅓.

PYTHON — gr.Column() + scale 
with gr.Blocks() as demo:
    gr.Markdown("## חלוקה לעמודות")

    with gr.Row():
        # עמודה רחבה (⅔) — פרמטרי קלט
        with gr.Column(scale=2):
            gr.Markdown("### 🎛️ פרמטרים")
            name   = gr.Textbox(label="שם ניסוי")
            gamma  = gr.Slider(0, 1, label="Gamma")
            btn    = gr.Button("הרץ", variant="primary")

        # עמודה צרה (⅓) — תוצאות
        with gr.Column(scale=1):
            gr.Markdown("### 📊 תוצאות")
            out = gr.Textbox(label="פלט", lines=5)

demo.launch()
📺 מה רואים: עמודה שמאלית רחבה עם כותרת "פרמטרים" + שדות קלט, ועמודה ימנית צרה עם "תוצאות".
gr.Tab() / gr.Tabs() חלוקה לטאבים — כל טאב עולם נפרד
לחץ להרחבה ▼

מה זה? gr.Tabs() הוא המיכל של כל הטאבים, ובתוכו כל gr.Tab("שם") הוא עולם רכיבים נפרד. רק הטאב הנבחר מוצג בפועל.

מתי משתמשים? כשיש כמה "מצבים" שונים לממשק — למשל ניסוי יחיד / ניסוי אצווה / השוואה.

PYTHON — gr.Tabs() ו-gr.Tab() 
with gr.Blocks() as demo:
    gr.Markdown("# 🧪 מערכת ניסויים")

    with gr.Tabs():

        with gr.Tab("ניסוי יחיד"):
            gr.Markdown("### הגדרת ניסוי אחד")
            name = gr.Textbox(label="שם הניסוי")
            btn1 = gr.Button("הרץ")

        with gr.Tab("אצווה מ-CSV"):
            gr.Markdown("### טעינת קובץ ניסויים")
            file_inp = gr.File(file_types=[".csv"])
            btn2 = gr.Button("הרץ הכל")

        with gr.Tab("📊 השוואה"):
            gr.Markdown("### גרף השוואה")
            plot = gr.Plot(label="השוואת ניסויים")

demo.launch()
📺 מה רואים: שלושה טאבים בראש הממשק. לחיצה על כל אחד מציגה תוכן שונה לגמרי.
gr.Group() קיבוץ ויזואלי — מסגרת סביב רכיבים קשורים
לחץ להרחבה ▼

מה זה? gr.Group() מוסיף מסגרת ויזואלית סביב קבוצת רכיבים. זה לא משנה את הפריסה (לא כמו Row/Column), רק מסמן ויזואלית שהרכיבים שייכים יחד.

שימוש נפוץ: הפרדה בין "הגדרות כלליות" לבין "פרמטרי RL" לבין "תגמולים" — כל אחד ב-Group נפרד עם כותרת Markdown.

PYTHON — gr.Group() עם כותרת Markdown 
with gr.Blocks() as demo:

    with gr.Row():

        with gr.Column():
            # Group 1 — הגדרות כלליות
            with gr.Group():
                gr.Markdown("### 📋 הגדרות כלליות")
                name     = gr.Textbox(label="שם ניסוי")
                num_games = gr.Slider(100, 50000, label="משחקים")

            # Group 2 — פרמטרי RL
            with gr.Group():
                gr.Markdown("### 🧠 פרמטרי RL")
                gamma   = gr.Slider(0, 1, label="Gamma")
                explore = gr.Slider(0, 1, label="Exploration")

demo.launch()
📺 מה רואים: שני "בלוקים" ויזואליים עם מסגרת, כל אחד עם כותרת ורכיבים בתוכו.
gr.Accordion() רכיב מתקפל — מסתיר/מראה תוכן בלחיצה
לחץ להרחבה ▼

מה זה? gr.Accordion() הוא מיכל שהמשתמש יכול לפתוח ולסגור בלחיצה. שימושי להסתרת פרטים מתקדמים שרוב המשתמשים לא צריכים.

פרמטרים:

  • open=True — פתוח כברירת מחדל
  • open=False — סגור כברירת מחדל (נפוץ יותר)
PYTHON — gr.Accordion() 
with gr.Blocks() as demo:
    gr.Markdown("# ממשק ניסויים")

    # פרמטרים רגילים — תמיד מוצגים
    name  = gr.Textbox(label="שם ניסוי")
    gamma = gr.Slider(0, 1, label="Gamma")

    # פרמטרים מתקדמים — מוסתרים כברירת מחדל
    with gr.Accordion("⚙️ הגדרות מתקדמות", open=False):
        gr.Markdown("_הגדרות אלה מיועדות למשתמשים מנוסים_")
        block_r  = gr.Number(value=0.95, label="פרס חסימה")
        rival    = gr.Radio(["random", "heuristic"], label="יריב")

    # לוג ריצה — נפתח רק כשצריך
    with gr.Accordion("📋 לוג ריצה", open=False):
        log = gr.Textbox(lines=8, interactive=False)

    btn = gr.Button("הרץ", variant="primary")

demo.launch()
📺 מה רואים: ממשק נקי עם שדות בסיסיים, ושני כותרות מתקפלות. לחיצה עליהן מגלה פרמטרים מתקדמים ולוג ריצה.
💡 חשוב להבין: with + indentation = הרכיבים שבפנים נמצאים בתוך המיכל. בדיוק כמו <div> ב-HTML. ניתן לקנן מיכלים בתוך מיכלים — Tabs → Tab → Row → Column → Group → Accordion.

PYTHON — פונקציית הרצה + Progress 
from experiment_config import ExperimentConfig
from experiment_runner import ExperimentRunner

def run_experiment(
    name, num_games, gamma, explore,
    win_r, loss_p, tie_r, block_r, rival,
    progress=gr.Progress(track_tqdm=True)
):
    # שלב 1: בנה ExperimentConfig מהפרמטרים של ה-UI
    config = ExperimentConfig(
        name=name,
        description=f"ניסוי מ-UI: {name}",
        num_games=int(num_games),
        gamma=gamma,
        exploration_rate=explore,
        win_reward=win_r,
        loss_penalty=loss_p,
        tie_reward=tie_r,
        blocking_reward=block_r,
        rival_plan=rival
    )

    progress(0, desc="מאתחל טורניר...")

    # שלב 2: הרץ את הניסוי
    runner = ExperimentRunner()
    runner.run_all([config])

    progress(0.9, desc="מחשב סטטיסטיקות...")

    # שלב 3: חלץ תוצאות
    result = runner.results[0]
    summary = (
        f"✅ ניסוי הסתיים!\n\n"
        f"🏆 X ניצח: {result['x_win_rate']*100:.1f}%\n"
        f"💀 O ניצח: {result['o_win_rate']*100:.1f}%\n"
        f"🤝 תיקו:   {result['tie_rate']*100:.1f}%\n"
        f"🧠 מצבים נלמדו: {result['learning_dict_size']}"
    )
    return summary

PYTHON — Grouped Bar Chart להשוואה 
def build_comparison_chart(history):
    if not history:
        return None

    names   = [r['experiment_name'] for r in history]
    x_rates = [r['x_win_rate'] * 100 for r in history]
    tie_rates = [r['tie_rate'] * 100 for r in history]

    x = range(len(names))
    fig, ax = plt.subplots(figsize=(10, 5))

    bars1 = ax.bar([i - 0.2 for i in x], x_rates,
                    0.4, label='X ניצח %', color='#00d4ff')
    bars2 = ax.bar([i + 0.2 for i in x], tie_rates,
                    0.4, label='תיקו %',   color='#10b981')

    ax.set_xticks(x)
    ax.set_xticklabels(names, rotation=15)
    ax.set_ylabel('אחוז (%)')
    ax.set_title('השוואת ניסויים')
    ax.legend()
    ax.set_ylim(0, 100)

    plt.tight_layout()
    return fig
