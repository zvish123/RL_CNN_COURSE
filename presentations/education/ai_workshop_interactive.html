התרגיל:

יש לכם פונקציה לבדיקת פלינדרום שעובדת, אבל לא יעילה:

def is_palindrome(s):
    clean = ""
    for char in s:
        if char.isalnum():
            clean += char.lower()
    
    for i in range(len(clean) // 2):
        if clean[i] != clean[len(clean) - 1 - i]:
            return False
    return True

📝 משימה 3: פרומפטים מומלצים

פרומפט לניתוח הבעיה:

"אני תלמיד כיתה ___, למדנו לאחרונה על Big O Notation. יש לי פונקציה is_palindrome שעובדת נכון: [הקוד שלמעלה] המורה אמר שהיא לא יעילה בגלל בניית המחרוזת clean. אני חושב שזה בגלל ש-Python יוצר עותק חדש בכל += שאלות: 1. מה ה-complexity (Big O) של הקוד הנוכחי? 2. איפה בדיוק הבעיה? 3. אילו אפשרויות יש לשיפור? (רק רשימה, לא קוד) אל תכתוב לי קוד משופר - רק הסבר את העקרונות."

פרומפט לשיפור מבוקר:

"הבנתי שאפשר לשפר עם list comprehension או join(). תן לי רמזים איך להתחיל: 1. איך בונים list של תווים נקיים עם comprehension? 2. איך משווים את התו הראשון עם האחרון בלי אינדקס מורכב? 3. האם יש דרך לעשות את זה in-place בלי clean בכלל? תן לי את השורה הראשונה של הפתרון עם list comprehension, אבל אל תשלים את שאר הפונקציה - אני רוצה לחשוב."
