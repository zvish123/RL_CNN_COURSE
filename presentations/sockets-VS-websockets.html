🔌 מה זה Socket? (בעברית: "שקע")

  • הגדרה פשוטה: נקודת חיבור בין שני מחשבים
  • כמו שקע חשמל שמחבר מכשיר לחשמל - רק שכאן מחברים מחשבים
  • עובד עם פרוטוקולים בסיסיים: TCP (אמין) או UDP (מהיר)
🎮 דוגמה מהחיים: משחק רשת מקומי (LAN Party) בין מחשבים באותו בית

🌐 מהו WebSocket?

  • גרסה משודרגת שעובדת דרך האינטרנט (HTTP)
  • מתאים במיוחד לדפדפנים (כרום, פיירפוקס וכו')
  • תקשורת בזמן אמת - כמו שיחת וידאו או משחק אונליין
📱 דוגמאות מוכרות:
  • דיסקורד - צ'אט בזמן אמת
  • משחקים אונליין - Fortnite, Roblox
  • מסמכי Google - רואים שינויים של אחרים בזמן אמת

💻 דוגמה 1: צ'אט עם Socket רגיל - השרת

# server_socket_chat.py
import socket, threading

clients = []  # רשימה של כל המשתמשים המחוברים

def handle_client(conn, addr):
    print(f"😊 {addr} הצטרף לצ'אט!")
    while True:
        msg = conn.recv(1024)  # מקבל הודעה
        if not msg:
            break
        # שולח את ההודעה לכל השאר
        for c in clients:
            if c != conn:
                c.sendall(msg)
    conn.close()

# יצירת השרת
server = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
server.bind(('localhost', 5000))
server.listen()
print("✅ שרת הצ'אט מוכן ומאזין בפורט 5000...")

# מאזין למשתמשים חדשים
while True:
    conn, addr = server.accept()
    clients.append(conn)
    threading.Thread(target=handle_client, args=(conn, addr)).start()
מה קורה כאן? השרת מחכה למשתמשים, וכשמישהו שולח הודעה - השרת מעביר אותה לכל השאר

💬 דוגמה 1: צ'אט עם Socket רגיל - הלקוח

# client_socket_chat.py
import socket, threading

def receive(sock):
    """פונקציה שמקבלת הודעות כל הזמן"""
    while True:
        try:
            data = sock.recv(1024).decode()
            if data:
                print("\n📩 הודעה חדשה:", data)
        except:
            break

# התחברות לשרת
client = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
client.connect(('localhost', 5000))
print("✅ התחברת לצ'אט!")

# תהליך נפרד לקבלת הודעות
threading.Thread(target=receive, args=(client,), daemon=True).start()

# לולאה לשליחת הודעות
while True:
    msg = input("👤 כתוב הודעה: ")
    client.sendall(msg.encode())
איך זה עובד? התוכנית מתחברת לשרת, שולחת הודעות ומקבלת הודעות מאחרים בו-זמנית

🌐 דוגמה 2: צ'אט עם WebSocket - השרת

# server_ws_chat.py
import asyncio, websockets

clients = set()  # קבוצה של כל המשתמשים

async def chat(websocket):
    """פונקציה שמטפלת בכל משתמש"""
    clients.add(websocket)
    print(f"👤 משתמש חדש! סה״כ: {len(clients)}")
    
    try:
        # מאזין להודעות מהמשתמש
        async for msg in websocket:
            print(f"📨 הודעה התקבלה: {msg}")
            # שולח לכל השאר
            for c in clients:
                if c != websocket:
                    await c.send(msg)
    finally:
        clients.remove(websocket)
        print(f"👋 משתמש התנתק. נשארו: {len(clients)}")

async def main():
    async with websockets.serve(chat, "localhost", 6789):
        print("✅ שרת WebSocket רץ על ws://localhost:6789")
        await asyncio.Future()  # רץ לנצח

asyncio.run(main())
שימו לב: הקוד פה יותר פשוט וקריא! async/await עוזרים לנו לנהל הרבה משתמשים בקלות

  <script>
    const ws = new WebSocket("ws://localhost:6789");
    
    ws.onmessage = e => {
      document.getElementById("chat").innerHTML += 
        "<div>👤 " + e.data + "</div>";
    };
    
    function sendMsg() {
      const txt = document.getElementById("msg").value;
      if (txt) {
        ws.send(txt);
        document.getElementById("msg").value = "";
      }
    }
  </script>
</body>
</html>

❓ בואו נבדוק שהבנתם (לחצו על כל שאלה)

שאלה 1: למה WebSocket יותר מתאים לאתרי אינטרנט?
✅ כי WebSocket עובד ישירות בדפדפן!
Socket רגיל דורש תוכנת לקוח מיוחדת. WebSocket משתמש ב-JavaScript שכל דפדפן מכיר.
בנוסף, WebSocket מבוסס על HTTP - הפרוטוקול שהאינטרנט כולו משתמש בו.
שאלה 2: תנו דוגמה לאפליקציה שבה עדיף להשתמש ב-Socket רגיל
✅ משחק רשת מקומי (LAN) או תקשורת בין שרתים
כשאין צורך בדפדפן ורוצים ביצועים מקסימליים, Socket רגיל הוא הבחירה הנכונה.
דוגמאות: העברת קבצים גדולים, משחקים תחרותיים, מערכות ניטור ובקרה.
שאלה 3: מה עושה השרת כשמשתמש שולח הודעה?
✅ השרת מעביר את ההודעה לכל המשתמשים האחרים
השרת שומר רשימה של כל המשתמשים המחוברים. כשמגיעה הודעה, הוא עובר על הרשימה ושולח לכולם (חוץ מהשולח המקורי).
שאלה 4: באיזה פורמט שולחים הודעות ב-WebSocket?
✅ טקסט רגיל או JSON
JSON זה פורמט נפוץ שנראה כך: {"user": "שם", "message": "הודעה"}
זה מאוד נוח לעבודה עם JavaScript ופייתון.
שאלה 5: מה היתרון של async/await בקוד WebSocket?
✅ מאפשר לשרת לטפל בהרבה משתמשים בו-זמנית!
במקום ליצור thread נפרד לכל משתמש (כמו ב-Socket רגיל), async/await מאפשר קוד יותר פשוט ויעיל.
זה כמו מלצר שמשרת כמה שולחנות במקביל במקום מלצר אחד לכל שולחן.
