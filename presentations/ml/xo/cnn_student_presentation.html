יש לכם מיליון לוחות XO — ועכשיו מה?

📦 מה יש לנו: מילון נתונים

אספנו לוחות XO עם ניקוד ממוצע שמייצג עד כמה המצב "טוב" עבור שחקן מסוים. למשל:

# board → average_score
{
  "XOX|OXO|XOX": 0.85,
  "X..|.O.|...": 0.52,
  "XXO|OOX|XO.": 0.31
}

❓ הבעיה: כיסוי חסר

לוח XO 3×3 יכול לייצר ~250,000 מצבים שונים. גם עם מיליון לוחות — לא כיסינו הכל. מה יקרה כשהמחשב יגיע למצב שמעולם לא ראה?

✨ הפתרון: CNN

רשת קונבולוציה לומדת דפוסים, לא רק שינון. היא מבינה:

  • שלושה X ברצף = ניצחון, בכל מיקום בלוח
  • שני X בשורה = מצב מסוכן / מבטיח
  • מרכז הלוח חשוב מהפינות
  • יחסים בין תאים סמוכים משמעותיים

💡 כמו שחקן מנוסה — CNN לא זוכרת כל משחק, היא מבינה את הלוגיקה.

ידיות הכוונה של CNN

⚙️ פרמטרים שאנחנו קובעים

filters 8 מספר הפילטרים — כמה דפוסים שונים ללמוד
kernel_size 3 גודל הפילטר (3×3) — חלון הסריקה
stride 1 גודל הצעד — כמה תאים לזוז בכל פעם
padding 0 ריפוד בקצוות — שמירת גודל הפלט
epochs 50 מספר מחזורי אימון על כל הנתונים

📊 גודל פלט הקונבולוציה

הנוסחה לחישוב ממדי הפלט:

output_size = 
  (input - kernel + 2×padding) / stride + 1

# דוגמה: לוח 3×3
= (3 - 3 + 2×0) / 1 + 1 = 1

# עם padding=1:
= (3 - 3 + 2×1) / 1 + 1 = 3

עם הגדרות נוכחיות: פלט = 1×1

למה CNN מנצחת FC לנתוני לוח?

📍 מיקום משנה

ב-Fully Connected, X בפינה שמאלית שקול ל-X בפינה ימנית — הרשת לא יודעת שהם שכנים. CNN מכבדת מבנה מרחבי.

🔁 שיתוף פרמטרים

אותו פילטר "שורה" פועל על כל השורות. פחות פרמטרים = פחות overfitting ואימון מהיר יותר.

🎯 עמידות להזזה

שלושה X ברצף — הפילטר יזהה זאת גם אם הם בשורה 1 או שורה 3. אינוואריאנטי למיקום.

🧠 היררכיה

שכבה 1 לומדת תבניות פשוטות (זוגות), שכבה 2 לומדת שלשות, שכבה 3 לומדת אסטרטגיות שלמות.

⚡ יעילות

FC (3×3=9 neurons): ~1,000 פרמטרים
CNN (8 filters 3×3): ~80 פרמטרים

🌍 הכללה

CNN שלמדה על XO 3×3 יכולה — עם שינוי קל — לפעול על 4 בשורה. ידע ניתן להעברה.

אתחול Kernel — נקודת ההתחלה חשובה

🎲 Random Initialization

כל פילטר מאותחל בערכים אקראיים קטנים (ממוצע 0). למה?

  • מונע symmetry breaking — אם כולם זהים, כולם ילמדו אותו דבר
  • ממוצע 0 מונע ערכים שמתפוצצים
  • הרשת "מחפשת" תבניות שונות מנקודות שונות

✨ Smart Initialization

אפשר לאתחל עם פילטרים שיודעים לזהות שורות/עמודות ידנית!

import tensorflow as tf

# פילטר "שורה ראשונה"
row_filter = [[1,1,1],[0,0,0],[0,0,0]]

def custom_init(shape, dtype=None):
  filters = np.zeros(shape)
  filters[:,:,0,0] = row_filter  
  return tf.constant(filters, dtype=dtype)

Conv2D(1, (3,3), 
       kernel_initializer=custom_init)

🏁 He / Glorot Initialization

שיטות אתחול מתקדמות שמתחשבות בגודל הרשת:

  • Glorot (Xavier): לפונקציות סיגמואיד/tanh
  • He initialization: לפונקציית ReLU — הכי נפוצה
Conv2D(8, (3,3),
