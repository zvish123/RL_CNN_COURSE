מהו Process Monitor?

📖 הגדרה

Process Monitor (או בקיצור Procmon) הוא כלי חינמי ומתקדם של Microsoft Sysinternals המאפשר לעקוב ולנתח בזמן אמת אחרי כל הפעילויות שמתרחשות במערכת ההפעלה Windows.

מה הכלי מאפשר לעקוב אחריו?

יצירה, קריאה, כתיבה, מחיקה ושינוי של קבצים ותיקיות

קריאה, כתיבה ומחיקה של מפתחות וערכים ב-Registry

🌐 פעולות רשת

חיבורים, שליחה וקבלה של נתונים דרך הרשת

מדידת ביצועים וזמני תגובה

✨ למה Procmon מיוחד?

  • ניטור בזמן אמת - רואים את כל הפעולות בזמן שהן קורות
  • פילטור מתקדם - יכולת לסנן את המידע בדיוק לפי מה שצריך
  • פירוט מלא - מציג כל פרט קטן על כל אירוע
  • חיפוש מהיר - מציאת מידע ספציפי בקלות
  • שמירת נתונים - אפשרות לשמור ולנתח מאוחר יותר

למה משתמשים ב-Process Monitor?

🎯 תרחישי שימוש נפוצים

1 ניתוח בעיות בתוכנה

כאשר תוכנה לא עובדת כמו שצריך, Procmon עוזר לגלות מה בדיוק התוכנה מנסה לעשות ואיפה היא נתקעת. למשל, אם תוכנה לא מצליחה לפתוח קובץ, נוכל לראות איזה קובץ היא מחפשת ולמה היא לא מוצאת אותו.

2 מעקב אחרי שינויים במערכת

כשמתקינים תוכנה חדשה, אפשר לעקוב אחרי כל השינויים שהיא עושה במחשב - אילו קבצים היא יצרה, אילו ערכים היא שינתה ב-Registry, ולאן היא שמרה קבצים.

3 גילוי malware וחקירת אבטחה

אפשר לגלות תוכנות זדוניות על ידי מעקב אחרי התנהגויות חשודות - למשל תוכנה שמנסה לגשת לקבצים רגישים או להתחבר לאינטרנט בצורה חשודה.

4 למידה על מערכת ההפעלה

הבנה כיצד תוכנות שונות של Windows עובדות "מאחורי הקלעים" - איך הן מתקשרות עם הקבצים, ה-Registry והרשת.

5 אופטימיזציה וביצועים

זיהוי צווארי בקבוק - תוכנות שקוראות/כותבות יותר מדי פעמים לדיסק, או פעולות שלוקחות הרבה זמן.

⚠️ דברים חשובים לזכור

  • Procmon מציג המון מידע - מדובר באלפי אירועים בשנייה!
  • חובה להשתמש בפילטרים כדי למצוא את המידע הרלוונטי
  • הכלי דורש הרשאות מנהל (Administrator) כדי לפעול
  • שימוש בכלי עלול להאט את המחשב בגלל כמות הניטור

ממשק המשתמש של Procmon

🖥️ החלון הראשי

כאשר פותחים את Procmon, רואים חלון המחולק למספר אזורים:

עמודה תיאור
Time הזמן המדויק שבו האירוע התרחש
Process Name שם התהליך שביצע את הפעולה
PID מזהה ייחודי של התהליך (Process ID)
Operation סוג הפעולה שבוצעה (למשל: ReadFile, WriteFile, RegQueryValue)
Path הנתיב המלא של הקובץ או מפתח ה-Registry
Result תוצאת הפעולה (SUCCESS, ACCESS DENIED, NAME NOT FOUND וכו')
Detail פרטים נוספים על הפעולה

🛠️ סרגל הכלים העיקרי

⏸️
Capture / Pause (Ctrl+E)
התחלה והפסקה של לכידת אירועים. כשהכפתור לחוץ - Procmon רושם אירועים, כשלא - הוא עוצר.
🗑️
Clear Display (Ctrl+X)
מנקה את כל האירועים שמוצגים כרגע על המסך. שימושי כשרוצים להתחיל מחדש.
🔍
Find (Ctrl+F)
חיפוש מילת מפתח או ערך ספציפי בתוך כל האירועים.
🎯
Filter (Ctrl+L)
פתיחת חלון הפילטר - הכלי החשוב ביותר! מאפשר לסנן את האירועים לפי קריטריונים שונים.

סינון וחיפוש מידע

🎯 למה הפילטר כל כך חשוב?

Procmon רושם אלפי אירועים בשנייה! בלי פילטר, אי אפשר למצוא כלום. הפילטר מאפשר לנו לראות רק את מה שחשוב לנו.

פתיחת חלון הפילטר

יש שתי דרכים לפתוח את חלון הפילטר:

  • לחיצה על כפתור ה-Filter בסרגל הכלים
  • קיצור דרך: Ctrl+L

🔧 מבנה הפילטר

כל פילטר בנוי מארבעה חלקים:

1 Column (עמודה) - איזה שדה רוצים לסנן (Process Name, Path, Result וכו')
2 Relation (יחס) - איך לבדוק את הערך:
  • is - שווה בדיוק
  • is not - לא שווה
  • contains - מכיל
  • begins with - מתחיל ב...
  • ends with - נגמר ב...
3 Value (ערך) - הערך שאנחנו מחפשים
4 Action (פעולה) - Include (כלול) או Exclude (אל תכלול)

📝 דוגמאות לפילטרים נפוצים

דוגמה 1: הצגת רק אירועים של תהליך בשם notepad.exe
Process Name | is | notepad.exe | Include
דוגמה 2: הסתרת אירועים של תהליכי מערכת
Process Name | is | System | Exclude Process Name | is | svchost.exe | Exclude
דוגמה 3: הצגת רק כשלונות (אירועים שנכשלו)
Result | is not | SUCCESS | Include
דוגמה 4: מציאת כל פעולה הקשורה לתיקייה מסוימת
Path | begins with | C:\Program Files\MyApp\ | Include

⚡ פילטור מהיר מתפריט ההקשר

אפשר גם לעשות פילטור מהיר על ידי לחיצה ימנית על אירוע:

  • Include - כלול רק אירועים עם ערך זהה בשדה זה
  • Exclude - הסתר את כל האירועים עם ערך זהה בשדה זה
  • Exclude Events Before - הסתר את כל האירועים שקרו לפני אירוע זה
  • Exclude Events After - הסתר את כל האירועים שקרו אחרי אירוע זה
טיפ חשוב: כדי לאפס את כל הפילטרים ולחזור להתחלה, לוחצים על כפתור Reset בחלון הפילטר.

דוגמאות מעשיות

דוגמה 1: איך Notepad שומר קובץ?

1 הפעילו את Procmon ולחצו על Ctrl+E כדי להפסיק את הלכידה
2 לחצו על Ctrl+X כדי לנקות את המסך
3 פתחו את חלון הפילטר (Ctrl+L) והוסיפו:
Process Name | is | notepad.exe | Include
4 לחצו על Ctrl+E כדי להתחיל לכידה
5 פתחו את Notepad, כתבו טקסט ושמרו את הקובץ
6 חזרו ל-Procmon - תראו את כל הפעולות ש-Notepad ביצע:
  • CreateFile - יצירת הקובץ
  • WriteFile - כתיבת התוכן
  • SetEndOfFile - סימון סוף הקובץ
  • CloseFile - סגירת הקובץ

דוגמה 2: מה קורה כשתוכנה מחפשת קובץ DLL?

כשתוכנה צריכה קובץ DLL (ספריה דינמית), Windows מחפש אותו במספר מיקומים לפי סדר מסוים. Procmon מאפשר לנו לראות את התהליך הזה!

1 הפעילו את Procmon עם פילטר על תוכנה מסוימת
2 הוסיפו פילטר נוסף:
Path | ends with | .dll | Include
3 הפעילו את התוכנה
4 תראו שורות רבות של NAME NOT FOUND - זה Windows מחפש בכל המיקומים האפשריים!
5 בסוף תמצאו שורה עם SUCCESS - זה המיקום שבו ה-DLL נמצא

דוגמה 3: Boot Logging - מה קורה בהפעלת המחשב?

Procmon יכול לעבוד גם בזמן אתחול המחשב ולתעד את כל הפעילויות!

1 פתחו את Procmon
2 לכו ל: Options → Enable Boot Logging
3 הפעילו מחדש את המחשב
4 לאחר האתחול, פתחו את Procmon והוא יציג את הלוג של ההפעלה
5 תוכלו לראות:
  • אילו דרייברים נטענו
  • אילו שירותים התחילו לפעול
  • איזה תוכנות רצות בהפעלה

תרגילים מעשיים

תרגיל 1: זיכרון ה-Regedit 🔍
השאלה:
שמתם לב שכאשר פותחים את regedit.exe, הוא תמיד זוכר את המיקום האחרון שצפיתם בו ב-Registry ונפתח עם המיקום הזה. איך הוא עושה את זה? איפה הוא שומר את המידע הזה?

מטרה: למצוא איפה Regedit שומר את המיקום האחרון שנפתח.
פתרון תרגיל 1

שלב א': הכנה

1 הפעילו את Procmon
2 עצרו את הלכידה (Ctrl+E) ונקו את המסך (Ctrl+X)
3 הגדירו פילטר:
Process Name | is | regedit.exe | Include

שלב ב': ביצוע הבדיקה

4 התחילו לכידה (Ctrl+E)
5 פתחו את Regedit ונווטו למפתח כלשהו (למשל HKEY_CURRENT_USER\Software)
6 סגרו את Regedit
7 עצרו את הלכידה ב-Procmon

שלב ג': ניתוח התוצאות

8 חפשו פעולות RegSetValue (כתיבה ל-Registry)
9 תמצאו מפתח בשם:
HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Applets\Regedit
10 בתוך המפתח הזה תמצאו ערך בשם LastKey שמכיל את הנתיב האחרון שנפתח!

🎓 מה למדנו?

  • תוכנות Windows שומרות הגדרות משתמש ב-Registry
  • המיקום הנפוץ להגדרות הוא HKEY_CURRENT_USER\Software
  • Procmon מאפשר לנו לראות בדיוק מה נכתב ואיפה
  • פעולות RegSetValue משמשות לשמירת ערכים ב-Registry
טיפ: נסו לפתוח את Regedit שוב ותראו שהוא נפתח במיקום שמופיע ב-LastKey!
תרגיל 2: Python Interpreter Startup 🐍
השאלה:
מי שמכיר Linux מכיר את קובץ .bashrc שרץ בכל פעם שפותחים terminal חדש. האם יש משהו דומה ב-Python? איך אפשר לגרום ל-Python להדפיס "Good Morning!" בכל פעם שפותחים interpreter חדש?

מטרה: למצוא את קובץ ההפעלה האוטומטי של Python.
פתרון תרגיל 2

שלב א': חקירה ראשונית

1 הפעילו את Procmon עם פילטר:
Process Name | is | python.exe | Include Path | ends with | .py | Include
2 התחילו לכידה
3 פתחו Python interpreter (הקלידו python ב-CMD)
4 עצרו את הלכידה

שלב ב': ניתוח

5 חפשו פעולות CreateFile או QueryOpen
6 תמצאו שPython מחפש קובץ בשם:
%USERPROFILE%\pythonrc.py
או
C:\Users\[username]\.pythonrc

שלב ג': הפעלה

7 Python לא מחפש את הקובץ באופן אוטומטי! צריך להגדיר משתנה סביבה:
8 בחלון CMD הקלידו:
set PYTHONSTARTUP=C:\Users\[username]\pythonrc.py
9 צרו את הקובץ pythonrc.py עם התוכן:
print("Good Morning! ☀️") print("Welcome to Python!")
10 עכשיו בכל פעם שתפתחו Python, תראו את ההודעה!

🎓 מה למדנו?

  • Python קורא קובץ הפעלה דרך משתנה הסביבה PYTHONSTARTUP
  • אפשר לראות אילו קבצים תוכנה מנסה לפתוח עם Procmon
  • גם כשלונות (NAME NOT FOUND) נותנים מידע חשוב!
  • אפשר להגדיר התנהגות אוטומטית לכלים שונים
טיפ: כדי שזה יעבוד תמיד, הוסיפו את משתנה הסביבה PYTHONSTARTUP למשתני הסביבה של המערכת דרך Control Panel → System → Advanced System Settings → Environment Variables.
תרגיל 3: Chrome Password Storage 🔐
השאלה:
Google Chrome זוכר את שמות המשתמש והסיסמאות שלנו לאתרים שונים. איפה הוא שומר את המידע הזה? האם זה בקובץ? ב-Registry? במסד נתונים?

מטרה: למצוא את מיקום האחסון של פרטי ההתחברות ב-Chrome.
פתרון תרגיל 3

שלב א': הגדרת הניטור

1 הפעילו את Procmon עם פילטר:
Process Name | is | chrome.exe | Include Path | contains | Login | Include
2 ניקוי והכנה (Ctrl+X ו-Ctrl+E)

שלב ב': ביצוע הפעולה

3 פתחו את Chrome והיכנסו לאתר כלשהו (או הקלידו שם משתמש וסיסמה)
4 אשרו ש-Chrome ישמור את פרטי ההתחברות
5 עצרו את הלכידה

שלב ג': מציאת המיקום

6 חפשו בתוצאות פעולות על קבצים (FileSystem)
7 תמצאו קובץ בשם Login Data במיקום:
C:\Users\[username]\AppData\Local\Google\Chrome\User Data\Default\Login Data

שלב ד': בדיקת סוג הקובץ

8 הקובץ הזה הוא מסד נתונים מסוג SQLite
9 אפשר לפתוח אותו עם כלים כמו:
  • DB Browser for SQLite (תוכנה חינמית)
  • Python עם הספרייה sqlite3

⚠️ אבטחה וצפנה

חשוב לדעת: הסיסמאות שמורות מוצפנות בקובץ! Chrome משתמש ב-Windows DPAPI (Data Protection API) כדי להצפין אותן. זה אומר שרק המשתמש שיצר אותן יכול לפענח אותן.

🎓 מה למדנו?

  • Chrome משתמש במסד נתונים SQLite לשמירת מידע
  • המיקום הוא בתיקיית AppData\Local של המשתמש
  • נתונים רגישים מאוחסנים מוצפנים לצורך אבטחה
  • Procmon עוזר לגלות איפה תוכנות שומרות מידע
מעניין לדעת: Chrome שומר גם את ההיסטוריה, העוגיות, וההרחבות במסדי נתונים SQLite נפרדים באותה תיקייה!
תרגיל 4: Recycle Bin Magic 🗑️
השאלה:
Windows Explorer מתייחס לסל המחזור (Recycle Bin) בצורה מיוחדת - יש לו אייקון שונה, תפריט הקשר שונה, ופונקציות מיוחדות. מי אחראי על ההתנהגות המיוחדת הזאת? איפה זה מוגדר?

מטרה: למצוא את המרכיב שאחראי לפונקציונליות המיוחדת של Recycle Bin.
פתרון תרגיל 4

שלב א': הגדרת Procmon

1 הפעילו את Procmon עם פילטר:
Process Name | is | explorer.exe | Include Path | contains | Recycle | Include
2 ניקוי והתחלת לכידה

שלב ב': ביצוע פעולה

3 פתחו את Windows Explorer
4 לחצו על Recycle Bin (או נווטו אליו)
5 עצרו את הלכידה

שלב ג': חיפוש DLL רלוונטי

6 הוסיפו פילטר נוסף:
Path | ends with | .dll | Include Operation | is | LoadImage | Include
7 תמצאו DLL בשם shell32.dll - זה הקובץ המרכזי!

שלב ד': חיפוש ב-Registry

8 פתחו את Regedit ונווטו אל:
HKEY_CLASSES_ROOT\CLSID\{645FF040-5081-101B-9F08-00AA002F954E}
9 תמצאו את ההגדרות של Recycle Bin:
  • שם התצוגה
  • האייקון
  • ה-DLL שמטפל בו (shell32.dll)

שלב ה': Shell Extension

10 Recycle Bin הוא דוגמה ל-Shell Extension
11 Shell Extension זה רכיב שמרחיב את יכולות Explorer
12 המפתח InProcServer32 מצביע על shell32.dll שמכיל את הקוד

🎓 מה למדנו?

  • Windows משתמש ב-CLSID (Class ID) לזיהוי רכיבים מיוחדים
  • shell32.dll מכיל את רוב הפונקציונליות של Explorer
  • Shell Extensions מאפשרות להרחיב את Explorer
  • Procmon עוזר לגלות אילו DLLs טוענות ומתי
עובדה מעניינת: ה-CLSID של Recycle Bin הוא תמיד אותו הדבר בכל מערכות Windows! אפשר להשתמש בו כדי לגשת ל-Recycle Bin מקוד.

תרגיל 5 (מתקדם): ניתוח JavaScript חשוד 🕵️
התרחיש:
קיבלתם קובץ JavaScript בשם mysterious.js. כשמנסים לקרוא את הקוד - הוא לא ברור. צריך להבין מה הוא עושה ללא להריץ אותו ישירות.

מטרה: להשתמש ב-Procmon כדי לנתח מה הקובץ עושה בצורה בטוחה.
פתרון תרגיל 5

⚠️ אזהרת אבטחה

תרגיל זה מדגים איך לנתח קוד חשוד בצורה בטוחה. אל תריצו קוד לא ידוע על המחשב שלכם ללא אמצעי הגנה מתאימים!

שלב א': הכנת סביבה מבודדת

1 באופן אידיאלי - השתמשו במכונה וירטואלית (VM)
2 נתקו את המכונה מהאינטרנט
3 צרו תיקייה מיוחדת לניתוח: C:\Analysis

שלב ב': הגדרת Procmon למעקב מקיף

4 הפעילו את Procmon עם הפילטרים הבאים:
Process Name | is | wscript.exe | Include Process Name | is | cscript.exe | Include
5 אל תשכחו לסנן את תהליכי המערכת:
Process Name | is | System | Exclude Process Name | contains | svchost | Exclude

שלב ג': הרצה מבוקרת

6 התחילו לכידה ב-Procmon
7 הריצו את הקובץ:
wscript.exe C:\Analysis\mysterious.js
8 המתינו כמה שניות ועצרו את הלכידה

שלב ד': ניתוח התוצאות - מה לחפש?

9 פעולות קבצים חשודות:
  • CreateFile - האם נוצרים קבצים חדשים?
  • WriteFile - האם משהו נכתב?
  • חפשו קבצים ב-Startup או Temp
10 פעולות Registry חשודות:
  • RegSetValue במפתחות Run/RunOnce
  • שינויים בהגדרות אבטחה
  • הוספת שירותים חדשים
11 פעולות רשת:
  • TCP Connect - ניסיונות חיבור לאינטרנט
  • UDP Send - שליחת נתונים
  • הקפידו לזהות את כתובות ה-IP
12 תהליכים חדשים:
  • Process Create - האם הקוד יוצר תהליכים נוספים?
  • האם הוא מפעיל cmd.exe או powershell.exe?

שלב ה': דוגמה לניתוח

נניח שמצאנו את הפעולות הבאות:

CreateFile: C:\Users\[user]\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup\evil.vbs WriteFile: [כתיבת תוכן לקובץ] RegSetValue: HKCU\Software\Microsoft\Windows\CurrentVersion\Run\MyApp = "C:\temp\backdoor.exe" TCP Connect: 192.168.1.100:4444
מה זה אומר?
  • הקוד יצר קובץ VBS שירוץ בכל הפעלה (Startup)
  • הוא הוסיף מפתח Registry שירוץ תוכנה בהפעלה
  • הוא ניסה להתחבר לשרת חיצוני
זו התנהגות של malware! 🚨

🎓 מה למדנו?

  • Procmon הוא כלי חיוני לניתוח אבטחה
  • אפשר להבין התנהגות של קוד ללא לקרוא אותו
  • דפוסים מסוימים מעידים על קוד זדוני
  • חשוב לעבוד בסביבה מבודדת בניתוח קבצים חשודים
טכניקות ניתוח מתקדמות:
  • השתמשו ב-Process Tree כדי לראות תהליכי בת
  • שימו לב לזמנים - האם הכל קורה מהר מדי?
  • חפשו הצפנה או obfuscation של שמות קבצים
  • בדקו אם יש מחיקה של עקבות (ניקוי logs)

נושאים מתקדמים

📊 ייצוא ושמירת נתונים

שמירת לוג לניתוח מאוחר

אפשר לשמור את הפלט של Procmon בפורמטים שונים:

  • PML - הפורמט המקורי של Procmon (כולל הכי הרבה פרטים)
  • CSV - ניתן לפתיחה ב-Excel לניתוח סטטיסטי
  • XML - לעיבוד אוטומטי על ידי סקריפטים

לשמירה: File → Save → [בחרו פורמט]

🤖 שימוש ב-Procmon דרך Command Line

# הפעלה שקטה עם backing file procmon.exe /Quiet /BackingFile C:\logs\output.pml # הרצה עם פילטר מוכן procmon.exe /LoadConfig myfilter.pmc # שמירה ויציאה procmon.exe /SaveAs C:\logs\final.pml /Terminate
שימוש מעשי: אפשר לכתוב סקריפט שמפעיל תוכנה כלשהי, מתעד את כל הפעילות עם Procmon, ואז מנתח את התוצאות אוטומטית!

🔬 ניתוח ביצועים

זיהוי בעיות I/O

מיון לפי עמודת Duration מגלה פעולות שלוקחות הרבה זמן

ספירת פעולות

Tools → Count Occurrences מראה כמה פעמים כל פעולה בוצעה

Process Tree

Tools → Process Tree מציג עץ תהליכים והקשרים ביניהם

🎨 התאמה אישית של Procmon

1 שינוי צבעים: Options → Highlight Colors
2 בחירת עמודות: Options → Select Columns
3 שמירת הגדרות: Options → Save Configuration

💡 טיפים ותרגילים לשיפור

תרגול עצמאי מומלץ:

  1. נתחו איך תוכנת ההתקנה של תוכנה כלשהי עובדת
  2. גלו איפה משחק שומר את קבצי השמירה שלו
  3. מצאו מה קורה כשלוחצים על "Empty Recycle Bin"
  4. בדקו אילו DLLs Microsoft Word טוען בהפעלה
  5. חקרו כיצד Windows Update עובד מאחורי הקלעים
