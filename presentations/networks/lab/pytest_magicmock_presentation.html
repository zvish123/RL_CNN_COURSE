📚 תיאוריה בסיסית
מה זה בדיקות יחידה (Unit Testing)?

לפני שנצלול לקוד — נבין למה בכלל כותבים בדיקות אוטומטיות ומה הבעיה המיוחדת בבדיקת שרתי רשת.

🧪

מה זה?

בדיקת יחידה = בדיקה של פונקציה אחת בבידוד, ללא תלות במערכות חיצוניות.

⚡

למה?

מאפשר לגלות באגים מוקדם, בלי להריץ את כל המערכת. כל שינוי קוד — הרץ את הבדיקות!

🤖

📚 תיאוריה
pytest — מבנה ומוסכמות

pytest היא ספרייה לבדיקות אוטומטיות בפייתון. היא מגלה ומריצה בדיקות לפי מוסכמות שמות פשוטות.

test_example.py
# pytest מוצא אוטומטית קבצים שמתחילים ב test_ # ופונקציות שמתחילות ב test_ import pytest # fixture = פונקציה שמייצרת משאב לבדיקות @pytest.fixture def sample_data(): return {"key": "value"} # כל פונקציה שמתחילה ב test_ = בדיקה אוטומטית def test_something(sample_data): result = do_something(sample_data) # assert = בודק שהתוצאה נכונה; נכשל = הבדיקה נכשלת assert result == "expected" def test_another_thing(): assert 2 + 2 == 4

📁 מוסכמות שמות

  • 📄 קובץ: test_*.py
  • 🔧 פונקציה: def test_*()
  • 🏛️ כיתה: class Test*:

🪄 MagicMock
MagicMock — הסוקט המדומה

MagicMock מ-unittest.mock יוצר עצם שמחקה כל אובייקט אחר — בלי שצריך את האובייקט האמיתי.

test_server.py — mock_socket fixture
from unittest.mock import MagicMock @pytest.fixture def mock_socket(): sock = MagicMock() # יוצר "סוקט מדומה" return sock # --- מה MagicMock יכול לעשות? --- # 1. לקבוע מה recv() יחזיר (כל קריאה תחזיר ערך אחר) sock.recv.side_effect = [ b"register||zvi||password123", # קריאה ראשונה b"logout" # קריאה שנייה ] # 2. לבדוק מה נשלח (שימוש ב-assert_any_call) sock.sendall.assert_any_call(b"OK||Registered successfully") # 3. לבדוק שנסגר sock.close.assert_called()
📥

recv().side_effect

מגדיר רשימה של ערכים שיוחזרו ברצף בכל פעם ש-recv() נקראת. כך מדמים שרצף הודעות מגיע מלקוח.

📤

sendall.assert_any_call()

בודק ש-sendall() נקרא עם ארגומנט מסוים לפחות פעם אחת. כך אוכפים שהשרת שלח את התשובה הנכונה.

נעבור על כל בדיקה בקובץ, ונבין מה היא בודקת, איך היא עובדת, ומה תוצאה צפויה להיות.

🔬 בדיקה 1 — test_register_success ▶
test_register_success
def test_register_success(mock_socket): mock_socket.recv.side_effect = [ b"register||zvi||password123", b"logout" ] handle_client(mock_socket, ('127.0.0.1', 1111)) mock_socket.sendall.assert_any_call(b"OK||Registered successfully")
  • 1
    מגדיר שתי "הודעות" שיגיעו מהלקוח: רישום ואחר כך logout
  • 2
    מריץ את handle_client() עם הסוקט המדומה — השרת "חושב" שקיבל חיבור אמיתי
  • 3
    בודק שהשרת שלח OK||Registered successfully
✅
תוצאה צפויה: PASSED כי שם משתמש "zvi" לא קיים → יירשם בהצלחה → שרת שולח OK
🔬 בדיקה 2 — test_register_invalid_format ▶
test_register_invalid_format
def test_register_invalid_format(mock_socket): mock_socket.recv.side_effect = [ b"register||only_username", # חסר פרמטר שלישי! b"logout" ] handle_client(mock_socket, ('127.0.0.1', 1111)) mock_socket.sendall.assert_any_call(b"ERROR||Invalid register format")
  • 1
    הפורמט הנכון הוא register||user||pass — כאן חסר הסיסמה
  • 2
    השרת מחלק ב-|| ומקבל רק 2 חלקים במקום 3
  • 3
    מצפים שהשרת יזהה זאת ויחזיר שגיאת פורמט
✅
תוצאה צפויה: PASSED — השרת בודק if len(parts) != 3 ומחזיר שגיאה
🔬 בדיקה 3 — test_login_wrong_password ▶
test_login_wrong_password
def test_login_wrong_password(mock_socket): # הכנה: יצירת קובץ משתמשים עם סיסמה ידועה with open(USERS_FILE, "w") as f: json.dump({"zvi": {"username": "zvi", "password": "correct"}}, f) mock_socket.recv.side_effect = [ b"login||zvi||wrong_pass", # ניסיון עם סיסמה שגויה b"logout" ] handle_client(mock_socket, ('127.0.0.1', 1111)) mock_socket.sendall.assert_any_call(b"ERROR||Wrong password")
  • 1
    שלב ה"הכנה" (Arrange): יוצרים קובץ JSON עם משתמש "zvi" וסיסמה "correct"
  • 2
    שלב ה"פעולה" (Act): שולחים ניסיון התחברות עם "wrong_pass"
  • 3
    שלב ה"בדיקה" (Assert): השרת צריך לזהות שהסיסמה שגויה
⚠️
שים לב: Arrange-Act-Assert זהו דפוס (pattern) נפוץ בבדיקות — הכנה, פעולה, בדיקת תוצאה. שם לב שנוצר קובץ JSON בהתחלה!
🔬 בדיקות 4 ו-5 — unknown_action ו-disconnect ▶
test_unknown_action + test_client_disconnect_suddenly
def test_unknown_action(mock_socket): mock_socket.recv.side_effect = [ b"attack||127.0.0.1", # פקודה לא קיימת בפרוטוקול b"logout" ] handle_client(mock_socket, ('127.0.0.1', 1111)) mock_socket.sendall.assert_any_call(b"ERROR||Unknown action") def test_client_disconnect_suddenly(mock_socket): # recv מחזיר bytes ריק = ניתוק mock_socket.recv.return_value = b"" handle_client(mock_socket, ('127.0.0.1', 1111)) # בודק שהשרת סגר את הסוקט mock_socket.close.assert_called()
  • 1
    unknown_action: מוודא שהשרת מוגן מפני פקודות שרירותיות — חשוב לאבטחה!
  • 2
    return_value שונה מ-side_effect — כאן תמיד מחזיר אותו ערך
  • 3
    bytes ריק (b"") הוא איך TCP מסמן שהחיבור נסגר מהצד השני
