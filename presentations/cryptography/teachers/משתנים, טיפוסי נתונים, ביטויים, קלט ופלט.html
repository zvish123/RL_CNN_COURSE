שלב 1: הבעיה

קופת הכיתה הסודית

יש לוועדת הכיתה תקציב של 101 ש״ח.

קניתם 5 תפוחים במחיר 7.5 ש״ח לתפוח ו-6 אגסים במחיר 8.2 ש״ח לאגס.

כמה כסף נשאר?

מה הנתונים?
מה מחשבים?
האם כל המספרים מאותו טיפוס?
מה יקרה אם מחיר ייקלט כטקסט?

🧺

לפני קוד: מפרקים את הבעיה

קלט

תקציב, כמות תפוחים, מחיר תפוח, כמות אגסים, מחיר אגס.

עיבוד

עלות תפוחים + עלות אגסים, ואז תקציב פחות עלות כוללת.

פלט

העלות הכוללת והעודף שנשאר.

1. קבל נתונים ← 2. חשב עלות תפוחים ← 3. חשב עלות אגסים ← 4. חשב עלות כוללת ← 5. חשב עודף ← 6. הדפס תוצאה.

שלב 2: מהו משתנה?

משתנה הוא שם שאנחנו נותנים לערך, כדי שנוכל להשתמש בו שוב בחישוב.

דימוי לתלמידים

משתנה הוא כמו מדבקה על קופסה: השם הוא המדבקה, הערך נמצא בתוך הקופסה.

budget = 101 apples = 5 apple_price = 7.5
שאלה: למה השם apple_price טוב יותר מהשם x?
תשובה: כי הוא מסביר מה הערך מייצג.

טיפוסי נתונים: למה זה חשוב?

int

מספר שלם
5

float

מספר עשרוני
7.5

str

טקסט
"5"

bool

אמת/שקר
True

מה יודפס עבור "5" + "7"?

ביטויים ואופרטורים

ביטוי

חישוב שהמחשב מעריך ומחזיר ממנו ערך.

total_cost = apples * apple_price + pears * pear_price change = budget - total_cost

שאלה מהירה

מה התוצאה?

a = 5 b = 10 result = a + b * 2

25 — קודם כפל, אחר כך חיבור.

קלט ופלט: input ו-print

input

מקבל מידע מהמשתמש, אבל מחזיר אותו כטקסט.

name = input("Enter your name: ")

print

מציג למשתמש תוצאה.

print("The change is:", change)
כי אם נקלוט מחיר בעזרת input בלבד, נקבל טקסט. כדי לחשב מחיר צריך מספר: float(input(...)).

מעבר מהבנה לקוד

לפני שממלאים קוד, כל זוג תלמידים חייב למלא: קלט, פלט, שלבי אלגוריתם ובדיקת דוגמה ידנית.

לא מדלגים

אם לא יודעים להסביר את החישוב במילים, עדיין לא כותבים קוד.

סימן אזהרה

קוד שנראה עובד על דוגמה אחת בלבד אינו הוכחה שהוא נכון.

שאלת מורה: “איזה קלט ישבור לכם את התוכנית?”
