הבעיה — מאיפה המילון בא ומה חסר בו

לפני שמבינים את CNN, צריך להבין מדוע בכלל צריך אותה. הכל מתחיל בשאלה: כיצד מחשב לומד לשחק XO?

שלב א׳ — Monte Carlo + DQN יוצרים את המילון

🎲 כיצד מונטה קרלו צובר ניסיון

המודל משחק מאות אלפי משחקים נגד עצמו. בכל משחק הוא מגיע לרצף של מצבי לוח. בסוף כל משחק הוא מקבל ציון (ניצחון = +1, הפסד = -0.3, תיקו = 0). הציון הזה מתפשט אחורה לכל הלוחות שדרכם עבר — כך כל לוח צובר accumulated_score ו-count.

משחק 1 → ניצחון
→
כל לוח בדרך: score += 1.0
→
משחק 2 → הפסד: score += -0.3
→
ממוצע = accumulated_score

# פסאודו-קוד: כך המילון נבנה במהלך האימון
learning_dict = {}   # המילון שלנו

for game in range(1_000_000):            # מיליון משחקים
    trajectory = []                        # רצף הלוחות במשחק

    while not game_over:
        board_str = board_to_string(board)
        trajectory.append(board_str)
        action = dqn_choose_action(board)   # DQN בוחר מהלך
        board = apply_action(board, action)

    final_score = get_game_result()         # +1 / -0.3 / 0

    # עדכון כל הלוחות שנראו במשחק זה
    for board_str in trajectory:
        if board_str not in learning_dict:
            learning_dict[board_str] = {'accumulated_score': 0, 'count': 0}

        learning_dict[board_str]['accumulated_score'] += final_score
        learning_dict[board_str]['count']             += 1

# בסוף: ממצעים את הציון
for key in learning_dict:
    entry = learning_dict[key]
    entry['accumulated_score'] /= entry['count']

🔀 מרחב מצבים אקספוננציאלי

בכל תא ב-9 תאים יש 3 אפשרויות (X, O, ריק). 3⁹ = 19,683 קומבינציות. אחרי סינון למצבים חוקיים — עדיין מאות אלפים. גם מיליוני משחקים לא יכסו את כולם — חלק מהמצבים נדירים מאוד.

🎯 DQN מנחה לאסטרטגיות ספציפיות

מודל DQN לומד לנגן טוב — כלומר הוא מגיע פעמים רבות לאותם מצבים "טובים". מצבים של שחקן חלש — לוחות עם מהלכים גרועים — נצפים נדירות, כי ה-DQN נמנע מהם.

הפתרון — CNN כ"מרחיב" המילון

כאן נכנסת ה-CNN. במקום לחפש לוח במילון, אנחנו מאמנים רשת נוירונים על הלוחות שכן יש לנו — ומבקשים ממנה ללמוד את הכלל הכללי שמחבר בין מבנה הלוח לניקוד. אחר כך היא יכולה לנבא גם ללוחות שלא ראתה.

💡 מדוע CNN מתאימה בדיוק לבעיה הזו?
  • לוח XO הוא "תמונה" מרחבית — CNN תוכננה לזהות תבניות מרחביות. שלוש X בשורה היא תבנית — בדיוק כמו קצה בתמונה.
  • תבנית ניצחון לא תלויה במיקום — שורה עליונה ושורה תחתונה שתיהן שוות. CNN עם אותו פילטר "רואה" שתיהן — זה נקרא equivariance.
  • הרשת לומדת כלל, לא שינון — "שלוש X בשורה = ניצחון" תמיד נכון, גם ללוח שמעולם לא הופיע בנתוני האימון.
  • המילון החלקי מספיק לאימון — 4,636 דוגמאות עם 578,763 ביקורים מכילות מספיק מידע כדי שהרשת תלמד את הכלל.

python
import json
import numpy as np

# מיפוי תו → מספר
CHAR_MAP = {'X': 1, 'O': -1, ' ': 0}

def board_str_to_matrix(board_str):
    """
    "O X X X O" → numpy array צורה (3,3,1)
    """
    values = [CHAR_MAP[c] for c in board_str]  # רשימה של 9 מספרים
    matrix = np.array(values, dtype='float32')
    matrix = matrix.reshape(3, 3)                # מטריצה 3×3
    matrix = matrix[..., np.newaxis]             # מוסיף ממד ערוץ → (3,3,1)
    return matrix

# דוגמה:
board = board_str_to_matrix("O X X X O")
print(board.shape)   # (3, 3, 1)
print(board[:,:,0])  # [[-1  0  1]  [ 1  1  0]  [-1  0  0]]

צעד ב׳ — accumulated_score לתווית one-hot

מדוע one-hot ולא הציון הגולמי? המודל שלנו מבצע סיווג — הוא מנסה לסווג כל לוח לאחת משלוש קטגוריות. אם נרצה לנבא את הציון המדויק, נשתמש ב-Regression עם Dense(1) ו-loss='mse'.

def score_to_label(score, win_thresh=0.7, loss_thresh=0.3):
    """
    ממיר ציון רציף לתווית קטגורית (one-hot).
    win_thresh / loss_thresh הם היפר-פרמטרים שניתן לשנות.
    """
    if score >= win_thresh:
        return [1, 0, 0]   # ניצחון
    elif score < loss_thresh:
        return [0, 0, 1]   # הפסד
    else:
        return [0, 1, 0]   # ביניים

python — load_data.py
import json
import numpy as np

def load_tournament_data(json_path, win_thresh=0.7, loss_thresh=0.3):
    """
    טוען את tournament_learning.json ומחזיר X, y, weights מוכנים לאימון.

    מחזיר:
        X       : numpy (N, 3, 3, 1)  — לוחות כ-tensor
        y       : numpy (N, 3)         — תוויות one-hot
        weights : numpy (N,)           — count של כל לוח
    """
    with open(json_path, 'r', encoding='utf-8') as f:
        data = json.load(f)

    X_list, y_list, w_list = [], [], []

    for board_str, entry in data.items():
        score = entry['accumulated_score']
        count = entry['count']

        # X: מחרוזת לוח → tensor (3,3,1)
        values = [CHAR_MAP[c] for c in board_str]
        board_arr = np.array(values, dtype='float32').reshape(3, 3, 1)
        X_list.append(board_arr)

        # y: ציון → one-hot
        label = score_to_label(score, win_thresh, loss_thresh)
        y_list.append(label)

        # weight: count (כמה פעמים ראינו את הלוח)
        w_list.append(float(count))

    X       = np.array(X_list)          # (4636, 3, 3, 1)
    y       = np.array(y_list, dtype='float32')  # (4636, 3)
    weights = np.array(w_list, dtype='float32')  # (4636,)

    # נרמול המשקלים — אופציונלי אך מומלץ
    weights = weights / weights.max()           # ל-[0, 1]

    print(f"טעינה הושלמה: {len(X)} מצבי לוח")
    print(f"X.shape = {X.shape}, y.shape = {y.shape}")
    print(f"התפלגות תוויות: ניצחון={y[:,0].sum():.0f}, ביניים={y[:,1].sum():.0f}, הפסד={y[:,2].sum():.0f}")

    return X, y, weights


# שימוש:
X, y, weights = load_tournament_data('tournament_learning.json')

sample_weight — למה זה חשוב?

בלי sample_weight, המודל מתייחס לכל לוח באופן שווה. אבל לוח שנראה 86,575 פעמים אמין הרבה יותר מלוח שנראה פעם אחת!

❌ ללא sample_weight

שגיאה על לוח עם count=1 זוכה לאותו "עונש" כמו שגיאה על לוח עם count=86,575. המודל עשוי להיות מבולבל מלוחות נדירים עם ציון לא מייצג.

✅ עם sample_weight

שגיאה על לוח נפוץ "שוקלת" הרבה יותר בחישוב ה-loss. המודל לומד בעיקר מהלוחות שנצפו הרבה — הנתונים האמינים.

אנלוגיה: דמיין שאתה לומד ממבחנים. 86,575 מבחנים אומרים "X במרכז = טוב" — לעומת מבחן אחד שאומר משהו שונה. את מי תאמין?

נרמול המשקלים

ערכי count נעים בין 1 ל-86,575 — פער עצום. Keras יתקשה אם משקל אחד גדול פי 86,000 מאחר. נרמול ל-[0,1] שומר על היחסים אך מוריד את הקפיצות הקיצוניות.

from tensorflow.keras.models import Sequential
from tensorflow.keras.layers import Conv2D, Flatten, Dense, Dropout
from tensorflow.keras.optimizers import Adam
from tensorflow.keras.initializers import Constant
import numpy as np


# ── custom_initializer: 8 פילטרים ידועים כנקודת התחלה ──────
# הפילטרים מייצגים תבניות ניצחון: שורות, עמודות, אלכסונות
initial_kernels = np.zeros((3, 3, 1, 8), dtype='float32')

# פילטר 0 — שורה עליונה [1,1,1 / 0,0,0 / 0,0,0]
initial_kernels[0, :, 0, 0] = [1, 1, 1]
# פילטר 1 — שורה אמצעית
initial_kernels[1, :, 0, 1] = [1, 1, 1]
# פילטר 2 — שורה תחתונה
initial_kernels[2, :, 0, 2] = [1, 1, 1]
# פילטר 3 — עמודה שמאל
initial_kernels[:, 0, 0, 3] = [1, 1, 1]
# פילטר 4 — עמודה אמצע
initial_kernels[:, 1, 0, 4] = [1, 1, 1]
# פילטר 5 — עמודה ימין
initial_kernels[:, 2, 0, 5] = [1, 1, 1]
# פילטר 6 — אלכסון ראשי (שמאל-עליון לימין-תחתון)
for i in range(3): initial_kernels[i, i, 0, 6] = 1
# פילטר 7 — אלכסון משני (ימין-עליון לשמאל-תחתון)
for i in range(3): initial_kernels[i, 2-i, 0, 7] = 1

custom_initializer = Constant(initial_kernels)


# ── בניית המודל ──────────────────────────────────────────────
model = Sequential([
    # שכבת קונבולוציה: 8 פילטרים 3×3 מאותחלים מידנית
    Conv2D(8, kernel_size=(3, 3),
           activation='relu',
           input_shape=(3, 3, 1),
           kernel_initializer=custom_initializer,
           padding='same'),   # שומר על ממדי הלוח

    # שיטוח 3D → 1D
    Flatten(),

    # שכבה נסתרת: 32 נוירונים
    Dense(32, activation='relu'),
    Dropout(0.3),   # מניעת Overfitting

    # שכבת פלט: 3 מחלקות + softmax
    Dense(units=3, activation='softmax')
])

model.compile(
    optimizer=Adam(learning_rate=0.001),
    loss='categorical_crossentropy',
    metrics=['accuracy']
)

model.summary()

def predict_board(model, board_str):
    """
    מקבל מחרוזת לוח ומחזיר ניבוי.
    עובד גם אם הלוח לא קיים במילון!
    """
    # 1. המרה לטנסור
    board_tensor = board_str_to_matrix(board_str)
    board_tensor = board_tensor[np.newaxis, ...]   # (1, 3, 3, 1) — batch של 1

    # 2. ניבוי
    probs = model.predict(board_tensor, verbose=0)[0]

    # 3. פירוש
    class_names = ['ניצחון', 'ביניים', 'הפסד']
    predicted = class_names[probs.argmax()]
    confidence = probs.max()

    print(f"לוח: {board_str}")
    print(f"ניבוי: {predicted} (בטחון {confidence:.1%})")
    print(f"הסתברויות: ניצחון={probs[0]:.3f} | ביניים={probs[1]:.3f} | הפסד={probs[2]:.3f}")
    return probs

def score_one_board(board_str, board_tensor, model, data, trust=100):
    """
    מחזיר ניקוד ללוח אחד.
    אם קיים במילון עם count >= trust → ניקוד המילון.
    אחרת → CNN חוזה.
    """
    if board_str in data and data[board_str]['count'] >= trust:
        return data[board_str]['accumulated_score']    # מהמילון הישר

    # לא במילון (או count נמוך) — CNN
    t     = board_tensor[np.newaxis, ...]               # (1,3,3,1)
    probs = model.predict(t, verbose=0)[0]
    return float(probs[0])                             # הסתברות לניצחון


def choose_best_move(current_board, model, data, trust=100):
    """
    מקבל לוח נוכחי, בונה את 5 המהלכים האפשריים,
    מנקד כל אחד, ומחזיר את הלוח הטוב ביותר.
    """
    empty_cells = [i for i, c in enumerate(current_board) if c == ' ']
    scores = {}

    for cell in empty_cells:
        # מהלך: שים X בתא הפנוי
        new_board_str = (
            current_board[:cell] + 'X' + current_board[cell+1:]
        )
        tensor = board_to_tensor(new_board_str)
        scores[new_board_str] = score_one_board(
            new_board_str, tensor, model, data, trust
        )

    # החזר את הלוח עם הניקוד הגבוה ביותר
    best = max(scores, key=scores.get)
    print(f"בחרנו: '{best}' עם ניקוד {scores[best]:.3f}")
    return best

"""
xo_cnn_full.py
==============
טוען tournament_learning.json, מכין נתונים, מאמן CNN,
ומדגים חיזוי ללוחות חדשים.
"""

import json
import numpy as np
from sklearn.model_selection import train_test_split
from tensorflow.keras.models import Sequential
from tensorflow.keras.layers import Conv2D, Flatten, Dense, Dropout
from tensorflow.keras.optimizers import Adam
from tensorflow.keras.callbacks import EarlyStopping
from tensorflow.keras.initializers import Constant
from sklearn.metrics import classification_report

# ═══════════════════════════════════════════════════
# 1. קבועים
# ═══════════════════════════════════════════════════
CHAR_MAP    = {'X': 1.0, 'O': -1.0, ' ': 0.0}
WIN_THRESH  = 0.7   # מעל → ניצחון
LOSS_THRESH = 0.3   # מתחת → הפסד
CLASS_NAMES = ['ניצחון', 'ביניים', 'הפסד']

# ═══════════════════════════════════════════════════
# 2. המרת נתונים
# ═══════════════════════════════════════════════════
def board_to_tensor(s):
    """מחרוזת 9 תווים → numpy (3,3,1)"""
    arr = np.array([CHAR_MAP[c] for c in s], dtype='float32')
    return arr.reshape(3, 3, 1)

def score_to_onehot(score):
    """ציון רציף → one-hot [ניצחון, ביניים, הפסד]"""
    if score >= WIN_THRESH:  return [1, 0, 0]
    if score < LOSS_THRESH:  return [0, 0, 1]
    return                            [0, 1, 0]

def load_data(path):
    """טוען JSON ומחזיר X, y, weights"""
    with open(path, encoding='utf-8') as f:
        data = json.load(f)

    X, y, w = [], [], []
    for board_str, entry in data.items():
        X.append(board_to_tensor(board_str))
        y.append(score_to_onehot(entry['accumulated_score']))
        w.append(float(entry['count']))

    X = np.array(X)                     # (N, 3, 3, 1)
    y = np.array(y, dtype='float32')   # (N, 3)
    w = np.array(w, dtype='float32')
    w = np.sqrt(w) / np.sqrt(w).max()   # נרמול שורש

    print(f"נטענו {len(X)} לוחות | X:{X.shape} y:{y.shape}")
    return data, X, y, w

# ═══════════════════════════════════════════════════
# 3. פילטרים מאותחלים
# ═══════════════════════════════════════════════════
def make_initial_kernels():
    """יוצר 8 פילטרים ידועים: שורות, עמודות, אלכסונות"""
    k = np.zeros((3, 3, 1, 8), dtype='float32')
    for row in range(3): k[row, :, 0, row]   = 1  # שורות 0-2
    for col in range(3): k[:, col, 0, col+3] = 1  # עמודות 3-5
    for i in range(3):   k[i, i,   0, 6]     = 1  # אלכסון ראשי
    for i in range(3):   k[i, 2-i, 0, 7]     = 1  # אלכסון משני
    return Constant(k)

# ═══════════════════════════════════════════════════
# 4. בניית מודל
# ═══════════════════════════════════════════════════
def build_model():
    model = Sequential([
        Conv2D(8, (3,3), activation='relu', padding='same',
               input_shape=(3,3,1), kernel_initializer=make_initial_kernels()),
        Flatten(),
        Dense(32, activation='relu'),
        Dropout(0.3),
        Dense(3, activation='softmax')
    ])
    model.compile(
        optimizer=Adam(0.001),
        loss='categorical_crossentropy',
        metrics=['accuracy']
    )
    return model

# ═══════════════════════════════════════════════════
# 5. אימון
# ═══════════════════════════════════════════════════
def train_model(model, X, y, w):
    X_tr, X_val, y_tr, y_val, w_tr, _ = train_test_split(
        X, y, w, test_size=0.2, random_state=42
    )
    es = EarlyStopping(monitor='val_loss', patience=12,
                        restore_best_weights=True)

    history = model.fit(
        X_tr, y_tr,
        sample_weight=w_tr,        # ← המשקלים לפי count
        validation_data=(X_val, y_val),
        epochs=150,
        batch_size=32,
        callbacks=[es],
        verbose=1
    )
    return history

# ═══════════════════════════════════════════════════
# 6. חיזוי
# ═══════════════════════════════════════════════════
def predict_board(model, board_str):
    """ניבוי לכל לוח — כולל לוחות שאינם במילון"""
    t     = board_to_tensor(board_str)[np.newaxis, ...]
    probs = model.predict(t, verbose=0)[0]
    idx   = probs.argmax()
    print(f"לוח '{board_str}' → {CLASS_NAMES[idx]} ({probs[idx]:.1%})")
    print(f"  ניצחון:{probs[0]:.3f}  ביניים:{probs[1]:.3f}  הפסד:{probs[2]:.3f}")
    return probs

# ═══════════════════════════════════════════════════
# 7. הרצה ראשית
# ═══════════════════════════════════════════════════
if __name__ == '__main__':
    # טעינה והכנה
    data, X, y, w = load_data('tournament_learning.json')

    # בניה + אימון
    model = build_model()
    train_model(model, X, y, w)

    # הערכה
    _, X_test, _, y_test, _, _ = train_test_split(
        X, y, w, test_size=0.2, random_state=42)
    y_pred = model.predict(X_test).argmax(axis=1)
    y_true = y_test.argmax(axis=1)
    print(classification_report(y_true, y_pred, target_names=CLASS_NAMES))

    # חיזוי לוחות — כולל חדשים שאינם במילון!
    predict_board(model, "XXX      ")  # ניצחון ברור
    predict_board(model, "OOOXX    ")  # הפסד ברור
    predict_board(model, "XO X  O  ")  # לוח חדש
    predict_board(model, "X   O    ")  # שלב מוקדם

סיכום — מה CNN עושה שמילון לא יכול

📚 מילון בלבד — מגבלות
  • רק 4,636 לוחות ידועים — אבל יש ~255,168 לוחות XO חוקיים אפשריים
  • לוח חדש שלא נצפה = אין תשובה
  • לא לומד כלל כללי, רק שינון
🧠 CNN — יתרונות
  • לומדת מהנתונים: "שלוש X בשורה = ניצחון" — ומכלילה
  • עונה לכל לוח, כולל לוחות שלא ראתה
  • ניתן לשפר על ידי הוספת נתונים
🎯 הגישה המומלצת: השתמש ב-sample_weight=counts כי לוחות שנצפו פעמים רבות ב-RL מכילים ידע אמין יותר. תן למודל ללמוד מהניסיון הצבור — לא רק מהקיום.
