איך יודעים שההודעה לא שונתה — וגם שמי ששלח אותה מכיר את הסוד המשותף?

🔐

הבעיה: Hash רגיל לא מספיק

אם שולחים הודעה עם SHA-256 בלבד, כל אחד יכול לשנות את ההודעה ולחשב Hash חדש.

message = "Transfer 100$"
hash = SHA256(message)

התוקף משנה ל־Transfer 900$ ומחשב Hash חדש. המקבל לא יודע שהייתה התקפה.

🧨

הוא לא מוכיח מי יצר את ההודעה.

שלמות

אם ההודעה השתנתה — ה־HMAC ישתנה.

🗝️

אימות מקור

רק מי שמכיר את המפתח הסודי יכול לייצר HMAC תקין.

הנוסחה האמיתית: מה הם ipad ו־opad?

HMAC(K,M)=H((K ⊕ opad) || H((K ⊕ ipad) || M))

ipad = Inner Pad

רצף קבוע של בתים בערך 0x36. משתמשים בו כדי ליצור מהמפתח הסודי מפתח פנימי:

K ⊕ ipad

opad = Outer Pad

רצף קבוע של בתים בערך 0x5C. משתמשים בו כדי ליצור מהמפתח הסודי מפתח חיצוני:

K ⊕ opad

איך החישוב מתבצע?

שלב פנימי: מחשבים Hash על המפתח הפנימי יחד עם ההודעה.

Inner = H((K ⊕ ipad) || M)

שלב חיצוני: מחשבים Hash נוסף על המפתח החיצוני יחד עם התוצאה הפנימית.

HMAC = H((K ⊕ opad) || Inner)

דוגמה לתלמידים

נניח שהמפתח הוא pizza וההודעה היא "נפגשים ב־17:00". HMAC לא עושה רק Hash(message), אלא יוצר שתי גרסאות שונות של המפתח: אחת פנימית עם ipad ואחת חיצונית עם opad. אם ההודעה תשתנה ל־20:00, או אם משתמשים במפתח אחר, התג הסופי יהיה שונה לחלוטין.

הסבר אינטואיטיבי: ipad ו־opad הם שתי “שכבות ערבול” שונות של אותו מפתח סודי. כך HMAC בטוח יותר מ־Hash פשוט של מפתח + הודעה.

הדגמה חיה: SHA-256 לעומת HMAC

בהדגמה זו מחשבים שני ערכים לאותה הודעה: SHA-256 רגיל, שכל אחד יכול לחשב, ו־HMAC-SHA256 שמחייב ידיעת מפתח סודי. לחיצה על “התוקף משנה שעה” ממחישה ששינוי קטן משנה את הפלט.

SHA-256 של ההודעה

HMAC-SHA256

אימות הודעה

כעת המקבל מקבל חבילה: הודעה + תג HMAC. הוא מחשב HMAC חדש מההודעה ומהמפתח שברשותו ומשווה אותו לתג שהגיע. התאמה פירושה שההודעה לא שונתה ושנוצרה על ידי מי שמכיר את המפתח.

תוצאה

לחצו על אימות

במערכת אמיתית משווים את התג שחושב מחדש לתג שהתקבל.

קוד Python מינימלי

import hmac
import hashlib

SEPARATOR = "||HMAC||"

# צד שולח: יצירת הודעה + תג אימות
message = "נפגשים במעבדה בשעה 17:00"
key = b"cyber-class-secret"

message_bytes = message.encode("utf-8")
signature = hmac.new(key, message_bytes, hashlib.sha256).hexdigest()

# זו החבילה שנשלחת ברשת: ההודעה גלויה, ולידה תג האימות
packet = message + SEPARATOR + signature
print("sent packet:", packet)

# צד מקבל: פירוק החבילה ובדיקת התג
received_message, received_signature = packet.rsplit(SEPARATOR, 1)

new_signature = hmac.new(
    key,
    received_message.encode("utf-8"),
    hashlib.sha256
).hexdigest()

if hmac.compare_digest(received_signature, new_signature):
    print("OK - ההודעה תקינה")
else:
    print("ERROR - ההודעה שונתה או המפתח שגוי")

חשוב: משתמשים ב־compare_digest ולא ב־== כדי להקטין סיכון להתקפות תזמון.

מה HMAC עושה — ומה לא?

✅

עושה

  • מזהה שינוי בהודעה
  • מאמת שמי ששלח מכיר את הסוד
  • מהיר ומתאים ל־API ושרתים
❌

לא עושה

  • לא מצפין את תוכן ההודעה
  • לא נותן אי־הכחשה
  • לא מתאים אם אין סוד משותף

בדיקת הבנה

1. האם HMAC מצפין את ההודעה?
לא. הוא מוסיף תג אימות, אבל ההודעה עצמה יכולה להיות גלויה.
2. למה Hash רגיל לא מספיק?
כי כל אחד יכול לחשב Hash חדש להודעה מזויפת.
3. מה קורה אם המפתח שגוי?
ה־HMAC שיחושב מחדש לא יתאים לתג שהתקבל.
4. האם HMAC נותן אי־הכחשה?
לא. שני הצדדים מכירים את אותו מפתח, לכן אי אפשר להוכיח מי מהם יצר את התג.

משימת תרגול לתלמידים

בנו מערכת קטנה לאימות הודעה

  1. קלט: הודעה ומפתח סודי.
  2. פלט: HMAC-SHA256.
  3. מסך אימות: הודעה + מפתח + תג שהתקבל.
  4. הציגו האם ההודעה תקינה או זויפה.

אתגר: הוסיפו כפתור “תוקף” שמשנה תו אחד ומוכיח שהאימות נכשל.
