בעיה #1: השוואת אובייקטים (__eq__)

🎯 המשימה שלך:

התלמיד רצה ליצור מתודה שמשווה בין שני שיעורים. הקוד למטה לא עובד כמו שצריך. מצא את הבעיה! 

def __eq__(self, other):
    if isinstance(other, Lesson):
        return (self.subject == other.subject and
                self.teacher == other.teacher and
                self.classroom == other.classroom)
    return self.subject == other.subject

🔍 מה הבעיה?

אם other אינו מסוג Lesson, הקוד מנסה לגשת ל-other.subject שעלול לא להתקיים!

דוגמה:

lesson = Lesson("Math", "Avi", 201)
print(lesson == "Math")  # 💥 AttributeError: 'str' has no attribute 'subject'

התוצאה: AttributeError וקריסת התוכנית 

def __eq__(self, other):

            self.teacher == other.teacher and
            self.classroom == other.classroom)

בעיה #2: מתודה או תכונה? 🤔

🎯 המשימה שלך:

התלמיד רצה לבדוק אילו שעות בלו"ז הן שעות חופשיות. הקוד למטה רץ בלי שגיאות, אבל לא מזהה אף שעה חופשית! למה? 

def get_free_hours(self):

    for i in range(len(self.lessons)):

🎯 המשימה שלך:

הפונקציה הזו אמורה לבדוק אם שיעור הוא חופשי. יש לה שתי בעיות חמורות - תוכל למצוא אותן?

בעיה #5: פונקציה לא לפי דרישות 📝

🎯 המשימה שלך:

הפונקציה הזו אמורה להוסיף שיעור ליום. לפי הדרישות, היא צריכה לתמוך בהוספה לשעה ספציפית או למקום הפנוי הראשון. מצא את כל הבעיות! 

def add_lesson(self, subject, teacher, classroom):
    if len(self.lessons) >= 8:
        print("add more than 8 lesson per day")
    lesson = Lesson(subject, teacher, classroom)
    self.lessons.append(lesson)

🔍 שלוש בעיות במקביל:

  1. בעיה 1: לא תומך בפרמטר hour (נדרש בתרגיל!)
  2. בעיה 2: מוסיף שיעור גם כאשר כבר יש 8 שיעורים
  3. בעיה 3: אין return value - לא יודעים אם הצליח

דוגמה לבעיה:

def add_lesson(self, subject, teacher, room, hour=None):
    new_lesson = Lesson(subject, teacher, room)
    
    if hour is None:
        # הוסף למקום הראשון הפנוי
        for i, lesson in enumerate(self.lessons):
            if lesson.is_free():
                self.lessons[i] = new_lesson
                return True  # ✅ הצלחה!
        return False  # ✅ אין מקום
    else:
        # הוסף לשעה ספציפית
        if 1 <= hour <= 8:
            self.lessons[hour-1] = new_lesson
            return True  # ✅ הצלחה!
        return False  # ✅ שעה לא תקינה

העשרה: שימוש מתקדם ב-lambda ו-max 🚀

🎯 אתגר:

איך מוצאים את היום העמוס ביותר בשבוע? יום עמוס = יום עם הכי הרבה שיעורים (הכי פחות חלונות).

נסה להבין איך הקוד הזה עובד לפני שתלחץ על הכפתור! 

def find_busiest_day(self):
    busiest = max(self.days, key=lambda day: 8 - len(day.get_free_hours()))
    return busiest.day_name

🔍 פירוק שלב אחר שלב:

  1. max(self.days, ...) - מחפש את האלמנט המקסימלי ברשימת הימים
  2. key=lambda day: ... - מגדיר איך למדוד "מקסימום"
  3. day.get_free_hours() - מחזיר רשימת שעות חופשיות, למשל [1, 3, 5]
  4. len(...) - כמה שעות חופשיות יש? (בדוגמה: 3)
  5. 8 - len(...) - כמה שיעורים יש? (8 - 3 = 5 שיעורים)
  6. max בוחר את היום עם הערך הגבוה ביותר

דוגמה קונקרטית:

# יום א': 2 חלונות → 8 - 2 = 6 שיעורים
# יום ב': 4 חלונות → 8 - 4 = 4 שיעורים
# יום ג': 1 חלון  → 8 - 1 = 7 שיעורים ← המקסימום!

# max בוחר את יום ג' כי 7 הוא הערך הגבוה ביותר

💡 מה זה lambda?

פונקציה קטנה וחד-פעמית בשורה אחת:

דרך רגילה:

def get_length(text):
    return len(text)

longest = max(words, key=get_length)

עם lambda:

longest = max(words, 
              key=lambda text: len(text))

🎓 דוגמאות נוספות:

# → [2, 4, 6]

העשרה: List Comprehension 📊

🎯 אתגר:

איך יוצרים רשימה של 8 שיעורים חופשיים?
הקוד הראשון הוא הדרך הרגילה, הקוד השני הוא הדרך המתקדמת.
תוכל להבין מה קורה בדרך המתקדמת? 

# דרך רגילה - 4 שורות
lessons = []
for i in range(8):
    lesson = Lesson("חופשי", None, None)
    lessons.append(lesson)

# דרך מתקדמת - שורה אחת!
lessons = [Lesson("חופשי", None, None) for _ in range(8)]

🔍 איך List Comprehension עובד?

התבנית הכללית:

[ביטוי for משתנה in רשימה if תנאי]

דוגמה פשוטה:

# יצירת רשימת ריבועים
squares = [x**2 for x in range(5)]
# → [0, 1, 4, 9, 16]

# מה קורה מאחורי הקלעים:
squares = []
for x in range(5):
    squares.append(x**2)

עם תנאי:

# רק מספרים זוגיים
evens = [x for x in range(10) if x % 2 == 0]
# → [0, 2, 4, 6, 8]

# מה קורה מאחורי הקלעים:
evens = []
for x in range(10):
    if x % 2 == 0:
        evens.append(x)

💡 מתי להשתמש?

✅ טוב לולאה פשוטה שיוצרת רשימה
✅ טוב סינון או טרנספורמציה פשוטה
❌ לא טוב לוגיקה מורכבת או קריאות
❌ לא טוב לולאות מקוננות מורכבות

🎓 דוגמאות מתקדמות:

# → [1, 3, 5, 7]  (שעות 1,3,5,7 חופשיות)

names = ["דני", "שרה", "יוסי"]
upper_names = [name.upper() for name in names]

# יצירת זוגות
pairs = [(x, y) for x in [1, 2] for y in [3, 4]]
# → [(1,3), (1,4), (2,3), (2,4)]

# עם תנאי מורכב
adults = [person["name"] for person in people 
          if person["age"] >= 18 and person["active"]]

❓ מה יהיה התוצאה?

העשרה: מבני נתונים מתקדמים 🗄️

🎯 אתגר:

רוצים לספור כמה פעמים כל מקצוע מופיע, באילו ימים, ובאילו כיתות.
הקוד למטה משתמש ב-setdefault ו-set - למה? 

info = {}
for day in schedule.days:
    for lesson in day.lessons:
        if not lesson.is_free():
            subject = lesson.subject
            info.setdefault(subject, {
                "count": 0, 
                "days": set(), 
                "rooms": set()
            })
            info[subject]["count"] += 1
            info[subject]["days"].add(day.day_name)
            info[subject]["rooms"].add(lesson.classroom)

🔍 מה זה setdefault?

הבעיה: רוצים להוסיף ערך למילון, אבל המפתח אולי לא קיים.

דרך רגילה (ארוכה):

if "Math" not in info:
    info["Math"] = {"count": 0, "days": set(), "rooms": set()}
info["Math"]["count"] += 1

עם setdefault (קצרה):

info.setdefault("Math", {"count": 0, "days": set(), "rooms": set()})
info["Math"]["count"] += 1

setdefault בודק: אם המפתח לא קיים - יוצר אותו עם הערך שנתנו. אם קיים - לא עושה כלום.

💡 למה set ולא list?

תכונה list set ✅
כפילויות מאפשר כפילויות ערכים ייחודיים בלבד
מהירות חיפוש O(n) - איטי O(1) - מהיר!
סדר שומר סדר לא מובטח סדר

במקרה שלנו:

  • רוצים ימים ייחודיים (לא לספור יום ראשון 5 פעמים)
  • רוצים כיתות ייחודיות
  • לא אכפת לנו מהסדר
  • → set מושלם!

🎓 דוגמה קונקרטית:

# נניח יש לנו מתמטיקה 3 פעמים:
# יום א' בכיתה 201
# יום א' בכיתה 201  (שוב!)
# יום ג' בכיתה 202

info["Math"] = {
    "count": 3,                    # 3 שיעורים
    "days": {"Sunday", "Tuesday"}, # 2 ימים ייחודיים
    "rooms": {201, 202}            # 2 כיתות ייחודיות
}

# אם היינו משתמשים ב-list:
# "days": ["Sunday", "Sunday", "Tuesday"]  # כפילות! ❌
# "rooms": [201, 201, 202]                 # כפילות! ❌

❓ מה יודפס?

days = set()
days.add("Sunday")
days.add("Monday")
days.add("Sunday")
print(len(days))

א. 3
ב. 2
ג. 1
ד. Error

סיכום ונקודות מפתח 🎯

✅ שבע הזהב של קוד איכותי:

  1. תמיד החזר False במתודות בוליאניות (לא None!)
  2. השתמש ב-raise לשגיאות קריטיות, לא ב-print
  3. סוגריים למתודות - is_free() לא is_free
  4. בדוק טיפוסים לפני גישה לתכונות (isinstance)
  5. אתחל ערכי ברירת מחדל ב-constructor
  6. שים לב לאופרטורים - += לא =+
  7. החזר ערכים מפונקציות (לא רק לבצע פעולות)

🚀 טכניקות מתקדמות שלמדנו:

  • Lambda functions - פונקציות קצרות וחד-פעמיות
  • List comprehension - יצירת רשימות בשורה אחת
  • max() with key - מציאת מקסימום לפי קריטריון
  • setdefault() - יצירת ערכי ברירת מחדל במילון
  • set() - רשימות ללא כפילויות
  • Exception handling - טיפול נכון בשגיאות
