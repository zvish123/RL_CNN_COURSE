Exception Handling

מטרות השיעור:

✓ הבנת מושג החריגות (Exceptions)

✓ שימוש ב-try/except

✓ טיפול בשגיאות נפוצות

✓ כתיבת קוד חסין תקלות

🤔 מה זה חריגה (Exception)?

חריגה היא אירוע שמפריע לזרימה הרגילה של התוכנית

דוגמה לשגיאה:

x = int(input("הכנס מספר: "))
print(x * 2)

# אם המשתמש יכניס "abc" במקום מספר:
# ValueError: invalid literal for int() with base 10: 'abc'

פתרון עם try/except:

try:
    x = int(input("הכנס מספר: "))
    print(x * 2)
except ValueError:
    print("זה לא מספר תקין! נסה שוב.")
החריגה מאפשרת לנו להמשיך להריץ את התוכנית גם כשמתרחשת שגיאה!

📋 מבנה try/except

try:
    # קוד שעלול לגרום לשגיאה
    # אם השגיאה קורית - קופצים ישירות ל-except
    
except סוג_השגיאה:
    # קוד שירוץ אם השגיאה התרחשה
    # כאן אנחנו מטפלים בבעיה

⚙️ איך זה עובד?

  • Python מנסה להריץ את הקוד בבלוק ה-try
  • אם הכל תקין - הקוד ב-except מדולג
  • אם יש שגיאה - הקוד ב-try נעצר ועוברים ל-except
  • אחרי except התוכנית ממשיכה רגיל
שאלת הבנה: מה יקרה אם נכניס את המספר 10? 
try:
    num = int("10")
    print("המספר הוא:", num)
except ValueError:
    print("שגיאה!")
print("המשך התוכנית")
תשובה: הפלט יהיה: 
המספר הוא: 10
המשך התוכנית
הקוד ב-except לא ירוץ כי לא הייתה שגיאה!

🔍 סוגי חריגות נפוצות בפייתון

ValueError

ערך לא תקין לפעולה

int("abc")

ZeroDivisionError

חלוקה באפס

10 / 0

IndexError

אינדקס מחוץ לטווח

lst[10]

KeyError

מפתח לא קיים במילון

dict["x"]

TypeError

פעולה על טיפוס לא נכון

"5" + 5

FileNotFoundError

קובץ לא נמצא

open("x.txt")
כל חריגה בפייתון היא אובייקט מסוג Exception. אפשר לתפוס חריגות ספציפיות או כלליות!

💻 דוגמאות מעשיות

דוגמה 1: חלוקה בטוחה

def safe_divide(a, b):
    try:
        result = a / b
        return result
    except ZeroDivisionError:
        print("שגיאה: לא ניתן לחלק באפס!")
        return None

print(safe_divide(10, 2))   # 5.0
print(safe_divide(10, 0))   # הודעת שגיאה + None

דוגמה 2: קריאת קלט מספרי

def get_age():
    while True:
        try:
            age = int(input("מה הגיל שלך? "))
            if age < 0 or age > 120:
                print("גיל לא הגיוני!")
                continue
            return age
        except ValueError:
            print("נא להכניס מספר תקין!")
שאלה: מדוע משתמשים ב-while True בדוגמה השנייה?
תשובה: כדי לאפשר למשתמש לנסות שוב ושוב עד שהוא מכניס ערך תקין. הלולאה נמשכת עד ש-return מתבצע בהצלחה.

🔀 טיפול במספר סוגי שגיאות

ניתן לטפל במספר סוגי חריגות שונים:

try:
    num = int(input("הכנס מספר: "))
    result = 100 / num
    my_list = [1, 2, 3]
    print(my_list[num])
    
except ValueError:
    print("הקלט חייב להיות מספר!")
    
except ZeroDivisionError:
    print("לא ניתן לחלק באפס!")
    
except IndexError:
    print("המספר גדול מדי לרשימה!")
    
except Exception as e:
    print(f"שגיאה כללית: {e}")
Exception הוא סוג כללי שתופס כל שגיאה. כדאי להשתמש בו בסוף, אחרי except ספציפיים!

סדר חשוב! 🎯

  • Python בודק את ה-except לפי הסדר מלמעלה למטה
  • הבלוק הראשון שתואם את החריגה - מתבצע
  • שאר הבלוקים מדולגים
  • except כללי תמיד בסוף!

🏁 הבלוק finally - תמיד מתבצע!

הבלוק finally מבוצע תמיד - עם או בלי שגיאה!

try:
    file = open("data.txt", "r")
    content = file.read()
    print(content)
    
except FileNotFoundError:
    print("הקובץ לא נמצא!")
    
finally:
    # זה ירוץ תמיד - גם אם הייתה שגיאה!
    try:
        file.close()
        print("הקובץ נסגר")
    except:
        pass

מתי משתמשים ב-finally? 🤔

  • ✓ סגירת קבצים
  • ✓ שחרור משאבים (חיבורי רשת, מסדי נתונים)
  • ✓ ניקוי זיכרון
  • ✓ כל פעולה שחייבת להתבצע בסוף
דוגמה נוספת: 
def foo(num):
    try:
        print(8 / num)
        return "success"
    except:
        print("wrong")
        return "error"
    finally:
        print("byebye")  # ירוץ תמיד!

📝 שאלות הבנה

שאלה 1: מה יודפס בקוד הבא? 
try:
    x = [1, 2, 3]
    print(x[5])
except ValueError:
    print("A")
except IndexError:
    print("B")
print("C")
תשובה: יודפס: 
B
C
הסבר: השגיאה היא IndexError (לא ValueError), לכן מתבצע except השני. אחר כך ממשיכים רגיל.
שאלה 2: מה הבעיה בקוד הזה? 
try:
    num = int(input("מספר: "))
except:
    print("שגיאה")
num = num * 2
print(num)
תשובה: אם תהיה שגיאה, המשתנה num לא יהיה מוגדר! כשננסה להשתמש בו נקבל NameError.

פתרון: להגדיר ערך ברירת מחדל או לשים את כל הקוד בתוך try.
שאלה 3: האם הקוד ב-finally ירוץ גם אם יש return ב-try?
תשובה: כן! הבלוק finally מתבצע תמיד - אפילו אם יש return, break או continue ב-try או ב-except.

🎯 תרגול מעשי

תרגיל 1: מחשבון בטוח

כתוב פונקציה calculator שמקבלת שני מספרים ופעולה (+, -, *, /) ומבצעת את החישוב.

הפונקציה צריכה לטפל בשגיאות:

  • חלוקה באפס
  • פעולה לא חוקית
נסה לכתוב את הפתרון: 
def calculator(a, b, op):
    try:
        if op == '+':
            return a + b
        elif op == '-':
            return a - b
        elif op == '*':
            return a * b
        elif op == '/':
            return a / b
        else:
            return "פעולה לא חוקית"
    except ZeroDivisionError:
        return "שגיאה: חלוקה באפס!"

# בדיקה:
print(calculator(10, 2, '+'))   # 12
print(calculator(10, 2, '/'))   # 5.0
print(calculator(10, 0, '/'))   # שגיאה: חלוקה באפס!

תרגיל 2: גישה למילון

כתוב פונקציה get_player_info שמקבלת מילון שחקנים ומזהה שחקן, ומחזירה את המידע עליו.

טפל בשגיאה אם השחקן לא קיים!


def get_player_info(players, player_id):
    try:
        return players[player_id]
    except KeyError:
        return f"שחקן {player_id} לא קיים!"

# בדיקה:
players = {
    "1": {"name": "דני", "score": 100},
    "2": {"name": "רונית", "score": 85}
}

print(get_player_info(players, "1"))
print(get_player_info(players, "99"))

🚀 אתגר מתקדם

⭐ אתגר: מערכת ניהול חשבון בנק

המשימה: צור מחלקה BankAccount עם הפעולות הבאות:

  • deposit(amount) - הפקדת כסף (חייב להיות מספר חיובי)
  • withdraw(amount) - משיכת כסף (בודק שיש מספיק יתרה)
  • get_balance() - קבלת יתרה

דרישות:

  • ✓ טפל בניסיון להפקיד/למשוך סכום שלילי
  • ✓ טפל בניסיון למשוך יותר מהיתרה
  • ✓ טפל בערכים שאינם מספרים
  • ✓ השתמש ב-try/except/finally לנעילת/שחרור נעילה (הדפס הודעות)
התחל לכתוב: 
class BankAccount:
    def __init__(self, initial_balance=0):
        self.balance = initial_balance
    
    def deposit(self, amount):
        try:
            print("🔒 נעילת חשבון...")
            if not isinstance(amount, (int, float)):
                raise TypeError("הסכום חייב להיות מספר!")
            if amount <= 0:
                raise ValueError("הסכום חייב להיות חיובי!")
            self.balance += amount
            print(f"✅ הופקדו {amount} ש\"ח")
        except (TypeError, ValueError) as e:
            print(f"❌ שגיאה: {e}")
        finally:
            print("🔓 שחרור נעילה\n")
    
    def withdraw(self, amount):
        try:
            print("🔒 נעילת חשבון...")
            if not isinstance(amount, (int, float)):
                raise TypeError("הסכום חייב להיות מספר!")
            if amount <= 0:
                raise ValueError("הסכום חייב להיות חיובי!")
            if amount > self.balance:
                raise ValueError("אין מספיק כסף!")
            self.balance -= amount
            print(f"✅ נמשכו {amount} ש\"ח")
        except (TypeError, ValueError) as e:
            print(f"❌ שגיאה: {e}")
        finally:
            print("🔓 שחרור נעילה\n")
שים לב איך finally משמש כאן לשחרור משאבים (נעילה) - זה דפוס נפוץ מאוד בתכנות!

💡 טיפים וכללי אצבע

תופס חריגות ספציפיות!
אל תשתמש ב-except כללי בלי צורך - זה עלול להסתיר באגים!
❌ לא טוב: 
try:
    # קוד מורכב...
except:
    pass  # מסתיר הכל!
✅ טוב: 
try:
    result = int(user_input)
except ValueError:
    print("קלט לא תקין")
except Exception as e:
    print(f"שגיאה לא צפויה: {e}")

כללי אצבע חשובים: 👍

  • תופס ספציפי: תפוס את החריגה הספציפית שאתה מצפה לה
  • אל תשתוק: לא להשתמש ב-pass ריק - לפחות הדפס הודעה
  • הודעות ברורות: הסבר למשתמש מה השגיאה ואיך לתקן
  • finally לניקוי: השתמש ב-finally לסגירת קבצים ושחרור משאבים
  • לא לכל דבר: לא כל תנאי צריך try/except - לפעמים if טוב יותר!

מתי להשתמש ב-try/except?

# ✅ טוב - קלט מהמשתמש
try:
    age = int(input("גיל: "))
except ValueError:
    print("לא מספר!")

# ❌ לא טוב - תנאי פשוט
try:
    if age < 0:
        # זה לא שגיאה, זה תנאי!
except:
    pass

🔍 חריגות ספציפיות

ValueError, KeyError, וכו'

המבנה הכללי:

try:
    # קוד שעלול לגרום לשגיאה
    
except SpecificError:
    # טיפול בשגיאה ספציפית
    
except AnotherError:
    # טיפול בשגיאה אחרת
    
except Exception as e:
    # טיפול בכל שגיאה אחרת
    
finally:
    # קוד שרץ תמיד

🎓 מה הלאה?

  • תרגל עם הקבצים שהעלת
  • נסה להוסיף טיפול בשגיאות לפרויקטים שלך
  • חקור חריגות מותאמות אישית (Custom Exceptions)
  • למד על context managers (with statement)
