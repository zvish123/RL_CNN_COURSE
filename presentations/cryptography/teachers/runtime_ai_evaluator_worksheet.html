בשעה 07:42 מתקבלת התרעה במערכת בית הספר: קובץ טקסט מוצפן הופיע בתיקיית שיתוף פנימית. הקובץ אינו מסוכן, אבל הוא חלק מתרגיל חקירה. ידוע לכם שההודעה הוצפנה בצופן נאיבי, ושהמילים המקוריות כתובות באנגלית.

צוות אחד מציע לנסות את כל המפתחות האפשריים. צוות שני אומר: “נשתמש ב־AI, הוא בטח יגיד לנו כמה זמן זה ייקח”. אבל ראש הצוות עוצר את כולם:

“AI יכול להישמע בטוח בעצמו גם כשהוא מניח הנחות שגויות. לפני שמאמינים לו — בונים הערכה, מבקשים ממנו לבדוק אותה, ואז מודדים בעצמנו.”

כעת אתם מקבלים משימה: לבדוק האם הערכת זמן הריצה של AI באמת עומדת במבחן המדידה.

2 הבנה תאורטית: מה מודדים?

חיפוש ממצה

חיפוש ממצה בודק מפתח אחרי מפתח. עבור כל מפתח מפענחים את ההודעה ובודקים אם התוצאה נראית נכונה.

אם יש k מפתחות וכל ניסיון עובר על n תווים, הערכה בסיסית היא O(k*n).

אבל זמן בשניות תלוי גם ב...

  • מהירות המחשב והדפדפן/פייתון.
  • מה בדיוק עושים בתוך הלולאה.
  • האם יש הדפסות למסך.
  • האם טוענים קבצים בזמן המדידה.
  • האם בדיקת “נראה כמו אנגלית” יקרה בעצמה.
פתחו תזכורת קצרה: O(k*n)

הסימון O(k*n) אינו אומר “כמה שניות”, אלא איך כמות העבודה גדלה כאשר מגדילים את מספר המפתחות או את אורך ההודעה.

אם מכפילים את k פי 10 והכול נשאר זהה, נצפה שהזמן יגדל בערך פי 10. אם מכפילים את n פי 10, גם אז נצפה לגידול דומה.

5 מדידה בפועל

אפשר למדוד בפייתון בעזרת time.perf_counter(), או להשתמש בסימולטור הדפדפן כאן כדי להבין את המגמה. המדידה בדפדפן אינה מחליפה ניסוי Python, אבל מסייעת לראות את הרעיון.

קוד מדידה לדוגמה בפייתון
import time def simulated_brute_force(k, n): counter = 0 for key in range(k): for i in range(n): counter += 1 return counter def measure_runtime(k, n): start = time.perf_counter() simulated_brute_force(k, n) end = time.perf_counter() return end - start
