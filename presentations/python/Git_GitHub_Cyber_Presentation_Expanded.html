🚀 Git ו‑GitHub

איך מפתחים ואנשי סייבר שומרים על קוד, עובדים בצוות, מתקנים טעויות ומוכיחים מי שינה מה?

המטרה של השיעור

בסוף המצגת תדעו להסביר מה ההבדל בין Git ל‑GitHub, למה הם חשובים בפרויקט סייבר, ואיך משתמשים בהם בעבודה אמיתית.

😱 תעלומת הקוד האבוד

דמיינו שעבדתם חודש על פרויקט: שרת, לקוח, מסד נתונים, התחברות משתמשים וכלי סייבר קטן.

יום לפני ההגשה...

מישהו מחק קובץ חשוב, שינוי חדש גרם לתוכנה לקרוס, והמחשב של אחד מחברי הצוות לא נדלק. בלי ניהול גרסאות קשה לדעת מה קרה ואיך חוזרים אחורה.

שאלה לכיתה

איך אפשר להגן על הפרויקט מפני טעויות, מחיקות, דריסת קוד ותקלות מחשב?

📂 הבעיה: גרסאות מבולגנות

כאשר אין כלי מסודר, תלמידים יוצרים עותקים ידניים של הקבצים.

project.py project_final.py project_final2.py project_final_REAL.py project_final_REAL_FIXED.py project_final_REAL_FIXED2.py

הבעיה: אף אחד לא יודע איזו גרסה באמת עובדת, מה השתנה בין הקבצים, ומי ביצע את השינוי.

💾 Git = מערכת לניהול גרסאות

Git הוא כלי שמותקן במחשב ושומר היסטוריה מסודרת של הפרויקט.

מה Git עושה?

Git עוקב אחרי שינויים בקבצים, שומר נקודות שמירה, מאפשר לחזור לגרסה קודמת, ומשווה בין מה שהיה לבין מה שקיים עכשיו.

🎮 Git כמו Save במשחק

במשחק מחשב שומרים התקדמות. אם נפלתם לבור או הפסדתם, אפשר לחזור לנקודת שמירה.

בעולם הקוד

Commit הוא נקודת שמירה של הפרויקט. אם שינוי חדש שבר את המערכת, אפשר לבדוק מה השתנה ואף לחזור אחורה.

דימוי פשוט

Git לא שומר רק את "הקובץ האחרון". הוא שומר את הסיפור המלא של הפרויקט לאורך זמן.

📸 Commit = צילום מצב

Commit הוא שמירה רשמית של מצב הפרויקט ברגע מסוים.

git add . git commit -m "Fix login validation bug"

מה חשוב בהודעת Commit?

ההודעה צריכה להסביר בקצרה מה השתנה: תיקון באג, הוספת מסך התחברות, שיפור בדיקת קלט, או שינוי בפרוטוקול.

למה זה עוזר?

כשמתגלה באג, אפשר למצוא באיזה Commit הוא נכנס ולבדוק מי עבד על החלק הזה.

🧠 מה Git זוכר?

Git שומר מידע חשוב

מי ביצע שינוי, מתי הוא בוצע, אילו שורות השתנו, ומה הייתה מטרת השינוי לפי הודעת ה‑Commit.

חשיבות בסייבר

בפרויקט סייבר חייבים עקיבות. אם נוספה חולשה, סיסמה נשמרה בטעות בקוד, או נמחקה בדיקת אבטחה — Git עוזר לאתר זאת.

☁️ GitHub = בית לפרויקטים בענן

GitHub הוא אתר שמאחסן פרויקטים שמנוהלים באמצעות Git.

ההבדל המרכזי

Git עובד אצלכם במחשב. GitHub מאפשר להעלות את הפרויקט לאינטרנט, לשתף אותו עם חברי צוות ולגשת אליו מכל מקום.

🆚 Git מול GitHub

Git

תוכנה לניהול גרסאות. היא יודעת לעקוב אחרי שינויים, ליצור Commitים, לפתוח ענפים ולחזור לגרסאות קודמות.

GitHub

שירות ענן שמאחסן מאגרי Git, מאפשר שיתוף פעולה, סקירת קוד, תיעוד פרויקט וניהול משימות.

משפט לזכור: Git הוא המנוע. GitHub הוא החניון, הלוח והמרכז השיתופי של הפרויקט.

📦 Repository = מאגר פרויקט

Repository, או בקיצור Repo, הוא תיקיית פרויקט שמנוהלת על ידי Git.

CyberProject/ ├── client.py ├── server.py ├── database.py ├── requirements.txt └── README.md

במאגר נמצאים הקוד, ההיסטוריה, התיעוד ולעיתים גם קבצי התקנה והוראות הרצה.

👨‍💻 למה זה חשוב בעבודה בצוות?

בלי GitHub

שולחים קבצי ZIP, מעתיקים בוואטסאפ, דורכים אחד על הקוד של השני, ולא יודעים איזו גרסה היא האחרונה.

עם GitHub

כל הצוות עובד על אותו מאגר. כל שינוי מתועד, אפשר לראות מי שינה מה, ולחבר עבודה של כמה תלמידים בצורה מסודרת.

⬆️ Push = העלאה ל‑GitHub

אחרי שעשינו Commit במחשב, נרצה לשלוח את השינוי למאגר שבענן.

git push
המחשב שלי │ ▼ GitHub

כך חברי הצוות והמורה יכולים לראות את השינוי.

⬇️ Pull = הורדה מ‑GitHub

כאשר חבר צוות העלה שינוי, אנחנו מורידים אותו אל המחשב שלנו.

git pull
GitHub │ ▼ המחשב שלי

כך כולם נשארים מסונכרנים ועובדים על הגרסה העדכנית.

🔁 זרימת עבודה בסיסית

כותבים קוד ↓ git add ↓ git commit ↓ git push ↓ הפרויקט מעודכן ב‑GitHub

זו השגרה היומיומית של מפתח: לשנות, לשמור, להסביר, ולהעלות.

🌿 Branch = ענף פיתוח

ענף מאפשר לעבוד על רעיון חדש בלי להרוס את הקוד הראשי.

דוגמה מפרויקט סייבר

הקוד הראשי עובד. תלמיד רוצה להוסיף זיהוי ניסיונות התחברות חשודים. הוא פותח ענף חדש, מפתח ובודק שם.

למה לא לעבוד ישר על main?

כי שינוי לא בדוק עלול לשבור את כל הפרויקט. Branch נותן אזור ניסויים בטוח.

🔀 Merge = חיבור ענפים

אחרי שהפיצ'ר נבדק ועובד, מחברים אותו אל הקוד הראשי.

main ────────────────●────── \ / \ feature ● / \________/

Merge הוא שלב חשוב: לא מחברים קוד לפני שבודקים שהוא תקין ולא שובר חלקים קיימים.

⚠️ Merge Conflict

לפעמים שני תלמידים משנים את אותה שורה בקובץ. Git לא מנחש מי צודק.

מה עושים?

בודקים את שתי הגרסאות, מחליטים מה צריך להישאר, מתקנים ידנית, ואז מבצעים Commit חדש. זה רגע מצוין ללמידה על תקשורת בצוות.

קונפליקט הוא לא תקלה מפחידה — הוא סימן שצריך לקבל החלטה מקצועית.

🛡️ Git בעולם הסייבר

שימושים אמיתיים

ניהול סקריפטים, כלי בדיקות, תיעוד חקירות, קבצי קונפיגורציה, דוחות אירוע ושיפור מתמשך של כלי הגנה.

למה זה קריטי?

בתחום הסייבר שינוי קטן יכול לפתוח חולשה או לסגור אותה. Git מאפשר להבין בדיוק מה השתנה ומתי.

🔐 זהירות: לא מעלים סודות

GitHub יכול להיות ציבורי. אסור להעלות סיסמאות, מפתחות API, טוקנים או קבצי credentials.

# לא לעשות: PASSWORD = "123456" API_KEY = "secret_token_here"

בפרויקטים מקצועיים משתמשים בקבצי סביבה וב‑.gitignore כדי למנוע העלאת מידע רגיש.

📄 README.md

README הוא דף ההסבר של הפרויקט. מי שנכנס למאגר צריך להבין מה הפרויקט עושה ואיך מריצים אותו.

מה לכלול?

שם הפרויקט, מטרות, הוראות התקנה, הוראות הרצה, דוגמאות שימוש, שמות חברי הצוות ותיאור קצר של מבנה הקבצים.

למה זה חשוב למגן?

README טוב מראה שהצוות מבין את הפרויקט ולא רק כתב קוד. הוא גם עוזר לבוחן להריץ ולהעריך את העבודה.

🏆 אתגר כיתתי

הכיתה מתחלקת לקבוצות. כל קבוצה מפתחת פיצ'ר קטן באותו פרויקט.

משימה לדוגמה

קבוצה אחת מוסיפה התחברות, קבוצה שנייה מוסיפה לוגים, קבוצה שלישית מוסיפה בדיקת סיסמה חזקה, וקבוצה רביעית כותבת README.

המטרה

להבין כיצד Commit, Push, Pull, Branch ו‑Merge משתלבים בעבודה קבוצתית אמיתית.
