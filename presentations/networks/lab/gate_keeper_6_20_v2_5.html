🎯 המשימה - מה עליכם לבנות

עליכם לכתוב תוכנית Python עם Scapy שמבצעת סריקת אבטחה פנימית ומייצרת דוח מקיף.

1️⃣ קלט מהמשתמש

התוכנית תקבל:

  • כתובת IP של המערכת לסריקה (ברירת מחדל: 127.0.0.1)
  • טווח פורטים לסריקה (ברירת מחדל: 1-1024)
  • אישור חוקי - התוכנית חייבת לדרוש מהמשתמש לאשר שיש לו הרשאה לסרוק

2️⃣ ביצוע סריקת SYN

התוכנית תסרוק כל פורט בטווח הנבחר באמצעות:

  • שליחת חבילת SYN לכל פורט
  • קבלת התשובה וזיהוי המצב:
    • SYN+ACK → פורט פתוח ✅
    • RST → פורט סגור ❌
    • אין תגובה → פורט מסונן (חומת אש) 🧱

התוכנית חייבת לכלול:

  • סגירת חיבור תקינה (שליחת RST) לפורטים פתוחים
  • טיפול בשגיאות (timeout, exceptions)
  • הצגת התקדמות במהלך הסריקה

3️⃣ ניתוח אבטחה חכם - בחירת גישה

לאחר שמצאתם אילו פורטים פתוחים, עליכם לנתח את רמת האבטחה שלהם. יש שתי גישות לביצוע המשימה:

🔄 בחרו גישה: מילונים סטטיים VS בינה מלאכותית 🤖

📚 גישה 1: מילונים סטטיים (קלאסי)

איך זה עובד?

תכינו 3 מילונים בקוד שלכם עם מידע מוכן מראש על פורטים:

  1. מילון שירותים - מיפוי בין מספר פורט לשם השירות
    SERVICES = {22: "SSH", 80: "HTTP", ...}
  2. מילון סיכונים קריטיים - פורטים מסוכנים + הסבר
    CRITICAL_PORTS = {21: "FTP שולח סיסמאות בטקסט גלוי...", ...}
  3. מילון אזהרות - פורטים רגישים + המלצות
    WARNING_PORTS = {22: "SSH - וודאו סיסמה חזקה...", ...}

מה צריך להכין?

  • לפחות 15 פורטים נפוצים עם מידע מלא
  • חקרו בעצמכם: מה כל פורט עושה, למה הוא מסוכן/בטוח
  • כתבו הסברים ברורים בעברית
✅ יתרונות:
  • ✅ מהיר - אין המתנה ל-API
  • ✅ עובד ללא אינטרנט
  • ✅ אמין 100% (אתם כותבים את המידע)
  • ✅ פשוט לתחזוקה
❌ חסרונות:
  • ❌ מוגבל לפורטים שהכנתם
  • ❌ עבודה ידנית רבה (חקירה)
  • ❌ צריך עדכון ידני

🤖 גישה 2: בינה מלאכותית - Gemini API (מתקדם)

איך זה עובד?

לכל פורט פתוח, תשלחו בקשה ל-Gemini API עם פרומפט מובנה שמבקש:

  • שם השירות
  • רמת סיכון (critical/warning/safe)
  • הסבר מפורט למה זה מסוכן/בטוח
  • התקפות נפוצות
  • המלצות אבטחה

מבנה הפרומפט:

"אתה מומחה אבטחת סייבר. נמצא פורט {PORT} פתוח.
החזר JSON עם:
{
  "port": {PORT},
  "service_name": "...",
  "risk_level": "critical/warning/safe",
  "explanation": "...",
  "common_attacks": [...],
  "recommendations": [...]
}
חשוב: רק JSON, ללא טקסט נוסף."

מה צריך לעשות?

  • קבלו API Key חינמי מ-Google AI Studio
  • כתבו פונקציה ששולחת בקשה ל-API
  • פרסמו את תשובת ה-JSON
  • שמרו cache (אם שאלתם על פורט פעם - שמרו את התשובה)
✅ יתרונות:
  • ✅ מידע עשיר ומפורט
  • ✅ עובד על כל פורט (גם נדירים)
  • ✅ מידע מעודכן (פגיעויות חדשות)
  • ✅ למידה על Prompt Engineering + APIs
❌ חסרונות:
  • ❌ צריך אינטרנט
  • ❌ איטי יותר (כל בקשה = 1-3 שניות)
  • ❌ מוגבל בכמות בקשות (API limits)
  • ❌ AI יכול לטעות

⭐ גישה 3: היברידית - המומלצת! (מצטיינים)

שלבו את שתי הגישות לתוצאה מושלמת:

  1. מילונים סטטיים ל-15-20 הפורטים הנפוצים ביותר (22, 80, 443, 3306, וכו')
  2. Gemini API לפורטים נדירים שלא במילון
  3. Cache - כל תשובה מ-API נשמרת, אז פעם הבאה היא במילון!

תהליך: מצאתי פורט → יש במילון? → כן: קח משם (מהיר!) → לא: שאל AI (חכם!) → שמור לפעם הבאה

🎖️ זו הגישה המקצועית ביותר!
מהירות + גמישות + אמינות = מנצח!

הערכת סיכונים (לכל הגישות):

לא משנה איזו גישה בחרתם, התוצאה הסופית חייבת לכלול:

  • 🚨 סיכון קריטי: פרוטוקולים לא מוצפנים (FTP, Telnet, HTTP Basic Auth)
  • ⚠️ אזהרה: שירותים רגישים שדורשים הגנה נוספת (SSH, RDP, MySQL)
  • ✅ תקין: שירותים סטנדרטיים עם אבטחה מתאימה (HTTPS, DNS)

לכל קטגוריה - הסבירו למה זה מסוכן/בטוח

4️⃣ דוח סריקה

התוכנית תייצר דוח הכולל:

  • כותרת עם פרטי הסריקה (IP, תאריך, שעה, גישה שנבחרה)
  • סיכום ויזואלי: כמה פורטים פתוחים/סגורים/מסוננים
  • טבלה מפורטת של כל הפורטים הפתוחים עם:
    • מספר פורט
    • שם השירות
    • רמת סיכון (צבע: אדום/כתום/ירוק)
    • הסבר קצר
  • המלצות אבטחה (אופציונלי אבל מומלץ)

💡 טיפים מהמורה

טיפים כלליים:

  1. התחילו מהפשוט: קודם כתבו פונקציה שסורקת פורט אחד, ואז הרחיבו לטווח
  2. בדקו על localhost קודם: 127.0.0.1 הוא המקום הבטוח ביותר לבדיקות
  3. הוסיפו timeout: אל תחכו לנצח לתשובה - timeout של 2 שניות מספיק
  4. שמרו על ההתקדמות: משתמשים אוהבים לראות שהתוכנית עובדת
  5. תעדו בקוד: כתבו הערות בעברית שמסבירות מה כל חלק עושה
  6. טפלו בשגיאות: השתמשו ב-try-except כדי לטפל במצבים לא צפויים

טיפים למילונים סטטיים:

  • 📚 חקרו פורטים אחד-אחד - קראו על כל פורט לפני שכותבים עליו
  • ✍️ כתבו הסברים בשפה שלכם - לא להעתיק מהאינטרנט
  • 🎯 התמקדו בפורטים הנפוצים: 21, 22, 23, 25, 80, 443, 3306, 3389

טיפים ל-Gemini API:

  • 🔑 קבלו API Key מוקדם - לוקח 5 דקות
  • 📝 בדקו את הפרומפט ב-Google AI Studio קודם (ידנית) לפני שמטמיעים בקוד
  • 💾 שמרו cache - אחרי בקשה ראשונה, שמרו את התוצאה בקובץ JSON
  • ⚠️ טפלו בשגיאות - מה אם אין אינטרנט? יש תוכנית B (fallback למילונים)
  • ⏱️ הגבילו מספר בקשות - אל תשלחו 1000 בקשות, רק לפורטים פתוחים

טיפים לגישה היברידית:

  • 🎯 קודם בנו את המילונים הסטטיים (15-20 פורטים)
  • 🤖 אחר כך הוסיפו את ה-API כ-fallback
  • 💾 שמרו cache חכם - כל תשובה מ-API הופכת למילון סטטי!
  • ⚖️ תנו למשתמש לבחור: --mode=static/api/hybrid
