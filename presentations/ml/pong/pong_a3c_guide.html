01

מה זה Reinforcement Learning?

בלמידת מכונה רגילה יש לנו תוויות — תמונה של חתול ואנחנו יודעים שזה חתול. ב-Reinforcement Learning אין תוויות. הסוכן חי בתוך סביבה, מנסה פעולות, ומקבל reward — מספר שאומר "עד כמה טוב מה שעשית".

🎮

סביבה (Environment)

המשחק עצמו — Pong. מחזיר מצב חדש ו-reward אחרי כל פעולה.

🤖

סוכן (Agent)

הרשת שלנו. מקבלת פיקסלים, מחזירה פעולה. לומדת מטעויות.

⚡

Reward

+1 כשהסוכן מנצח נקודה, -1 כשמפסיד. 0 כל שאר הזמן.

🎯

מדיניות (Policy)

הפונקציה שממפה מצב לפעולה — הלב של מה שהרשת לומדת.

מושג מפתח — Sparse Reward

ב-Pong, reward מגיע רק בסוף נקודה — אחרי עשרות צעדים. 99% מהצעדים הם reward=0. האתגר: הרשת צריכה לייחס את הניצחון/הפסד לפעולות שקרו קודם לכן.

הרשת לא מקבלת קואורדינטות X,Y של הכדור. היא מקבלת פיקסלים גולמיים, בדיוק כמו שאדם רואה את המסך.

נורמליזציה: חלוקה ב-256

פיקסל בטווח 0–255 הופך ל-0.0–1.0. מייצב את האימון. בקוד: fx = x.float() / 256

למה הרשת לא "יודעת" היכן הכדור?

אין קואורדינטות! הרשת לומדת בעצמה לזהות דפוסים בפיקסלים — מה שנראה כמו "כדור" הוא קבוצת פיקסלים בהירים שמשנה מיקום בין פריימים. הקונבולוציה מחלצת דפוסים אלה ללא הנחיה מפורשת.

03

ארכיטקטורת הרשת — model.py

הרשת היא Actor-Critic משותפת: גוף CNN משותף שמחלץ features, ואחריו שני ראשים נפרדים לשתי מטרות שונות.

# model.py — הרשת שאיתי מימש
class ModelA3C(nn.Module):
    def __init__(self, input_shape, n_actions):
        super(ModelA3C, self).__init__()

        # ── גוף משותף: 3 שכבות קונבולוציה ──
        self.conv = nn.Sequential(
            nn.Conv2d(input_shape[0], 32, kernel_size=8, stride=4),  # 4→32 filters
            nn.ReLU(),
            nn.Conv2d(32, 64, kernel_size=4, stride=2),            # 32→64 filters
            nn.ReLU(),
            nn.Conv2d(64, 64, kernel_size=3, stride=1),            # 64→64 filters
            nn.ReLU()
        )
        conv_out_size = self._get_conv_out(input_shape)  # = 3136

        # ── ראש Actor: מה לעשות? ──
        self.policy = nn.Sequential(
            nn.Linear(conv_out_size, 512),
            nn.ReLU(),
            nn.Linear(512, n_actions)   # 6 פעולות ב-Pong
        )
        # ── ראש Critic: כמה שווה המצב? ──
        self.value = nn.Sequential(
            nn.Linear(conv_out_size, 512),
            nn.ReLU(),
            nn.Linear(512, 1)             # מספר אחד: V(s)
        )

    def forward(self, x):
        fx = x.float() / 256                          # נורמליזציה
        conv_out = self.conv(fx).flatten(start_dim=1) # → וקטור 3136
        return self.policy(conv_out), self.value(conv_out)
        #       ^Actor: logits[6]   ^Critic: V(s)[1]
04

מאיפה מגיע המספר 1.8M פרמטרים?

כל חיבור בין נוירון לנוירון הוא פרמטר — מספר float32 שהרשת לומדת. בואו נחשב שכבה אחרי שכבה.

חישוב גודל הפלט של ה-CNN

נוסחת גודל פלט קונבולוציה: ⌊(W - F) / S⌋ + 1

⌊(84-8)/4⌋+1 = 20 → 20×20×32
⌊(20-4)/2⌋+1 = 9 → 9×9×64
⌊(9-3)/1⌋+1 = 7 → 7×7×64
7 × 7 × 64 = 3,136

פרוט כל שכבה — חישוב פרמטרים

נוסחה לשכבת קונבולוציה: params = (F×F×C_in + 1) × C_out (×1 לbias)

נוסחה לשכבה צפופה: params = (input_size + 1) × output_size

שכבה חישוב פרמטרים תפקיד
Conv1 (8×8×4 + 1) × 32 8,224 זיהוי קצוות ובסיסי
Conv2 (4×4×32 + 1) × 64 32,832 דפוסים מורכבים
Conv3 (3×3×64 + 1) × 64 36,928 features גבוהות רמה
סה"כ CNN 8,224 + 32,832 + 36,928 77,984 גוף משותף
Policy FC1 (3,136 + 1) × 512 1,606,144 הכי גדול!
Policy FC2 (512 + 1) × 6 3,078 6 פעולות Pong
Value FC1 (3,136 + 1) × 512 1,606,144 כמו Policy FC1
Value FC2 (512 + 1) × 1 513 V(s) — מספר אחד
סה"כ 77,984 + 1,609,222 + 1,606,657 ~3.29M שני heads מלאים
מאיפה 1.8M?

המספר 1.8M מתייחס לרוב רק לגוף + ראש אחד (בחלק מהמימושים Policy head בלבד), או לגרסה קצת שונה עם FC קטן יותר. המימוש של איתי עם שני heads מלאים מגיע ל-~3.3M. בכל מקרה — הרוב המוחלט של הפרמטרים נמצא בשכבות ה-FC (לא ב-CNN).

05

V(s) — הערכת ערך מצב

הצ'ריטיק לומד לענות על שאלה אחת: "אם אני נמצא במצב s, כמה reward אקבל בממוצע מכאן עד סוף המשחק?"

V(s) = E[ rₜ + γ·rₜ₊₁ + γ²·rₜ₊₂ + ... ] ציפייה על סכום ה-rewards המוזלים מהמצב הנוכחי

Temporal Difference — איך הרשת לומדת

הרשת לא יכולה לראות את העתיד. היא לומדת מהשוואה בין ניחוש לניחוש:

V_ref = r + γ · V(s_next)
שגיאה (TD-error) = V_ref − V(s)
עדכון: V(s) ← V(s) + lr · שגיאה כל עדכון שיפור קטן. אחרי אלפי עדכונים — V(s) מתכנס

γ = 0.99 — למה כל כך גבוה?

Pong הוא משחק ארוך. reward הגיע אחרי 50-100 צעדים. γ=0.99 מאפשר למידע להתפשט 100 צעדים אחורה עם ערך סביר: 0.99¹⁰⁰ ≈ 0.37.

STEPS_COUNT = 4 — n-step return

במקום ניחוש אחרי צעד אחד, צוברים 4 צעדים ואז מסתכלים קדימה. מאיץ את הלמידה ומייצב את ה-gradient.

מפת חום — V(s) לפי מיקום כדור

ירוק = V גבוה (מצב טוב). אדום = V נמוך (מצב גרוע). לחץ להפעיל למידה.

06

Advantage — לב A3C

Advantage שואל שאלה חכמה: "האם הפעולה שעשיתי הייתה טובה יחסית למה שציפיתי?"

A(s, a) = V_ref − V(s) V_ref = מה שקרה בפועל  |  V(s) = מה שציפינו

A > 0 — טוב מהצפוי

הפעולה הניבה יותר ממה שהרשת ניחשה. → Policy מתחזק לבחור פעולה זו.

A < 0 — גרוע מהצפוי

הפעולה הניבה פחות. → Policy מתחלש. הסוכן ילמד להימנע מפעולה זו.

A ≈ 0 — כציפייה

הפעולה לא מפתיעה. אין עדכון חזק. הרשת כבר "ידעה" שזה יקרה.

למה Advantage ולא reward ישיר?

בלי baseline (V(s)), הסוכן היה לומד: "כל פעולה שקדמה לניצחון היא טובה". זה רועש ואיטי. עם Advantage, הסוכן לומד: "הפעולה הזו הייתה טובה *ביחס לציפייה*". V(s) משמש כ-baseline שמפחית variance ומאיץ את הלמידה.

# חישוב Advantage ב-a3c.py
logits_v, value_v = net(states_v)          # forward pass

# Advantage = מה שקרה − מה שציפינו
adv_v = vals_ref_v - value_v.detach()      # .detach() חשוב!
# .detach() = לא מחשבים gradient דרך V(s) בחישוב Policy Loss

# Policy Loss: מחזק פעולות עם Advantage חיובי
log_prob_v = F.log_softmax(logits_v, dim=1)
log_prob_actions_v = adv_v * log_prob_v[range(BATCH_SIZE), actions_t]
loss_policy_v = -log_prob_actions_v.mean()  # מינוס = maximize
07

Loss — מה הרשת ממזערת?

ה-Loss הכולל מורכב משלושה חלקים, כל אחד עם תפקיד:

L_total = L_value + L_policy + L_entropy כל שלושה מחושבים בבת אחת ו-backprop עובר דרך כולם
Loss נוסחה תפקיד ב-Pong
L_value MSE(V(s), V_ref) מלמד את ה-Critic לנחש נכון הפרש בריבוע בין ניחוש לאמת
L_policy -log π(a|s) × A מחזק פעולות טובות אם עליתי וניצחתי → A>0 → חזק עלייה
L_entropy β × Σ π·log(π) מונע התכנסות מוקדמת מעודד חקירה. β=0.01 קטן אך חיוני 
# חישוב Loss מלא ב-a3c.py

# 1. Value Loss — Critic לומד לנחש V(s)
loss_value_v = F.mse_loss(value_v.squeeze(-1), vals_ref_v)

# 2. Policy Loss — Actor לומד מה לעשות
log_prob_v = F.log_softmax(logits_v, dim=1)
adv_v = vals_ref_v - value_v.detach()
log_prob_actions_v = adv_v * log_prob_v[range(BATCH_SIZE), actions_t]
loss_policy_v = -log_prob_actions_v.mean()

# 3. Entropy Bonus — מונע "קיפאון"
prob_v = F.softmax(logits_v, dim=1)
entropy_loss_v = ENTROPY_BETA * (prob_v * log_prob_v).sum(dim=1).mean()

# Loss כולל — backprop דרך כולם
loss_v = entropy_loss_v + loss_value_v + loss_policy_v
loss_v.backward()

# Clip Grad — מגן מ-exploding gradients
nn_utils.clip_grad_norm_(net.parameters(), CLIP_GRAD)  # = 0.1
optimizer.step()
למה CLIP_GRAD = 0.1?

עם A3C מקביל, נדיפות גבוהה של gradients היא בעיה נפוצה. gradient שגדול מ-0.1 נחתך. ערך קטן (0.1) מסוכן — אם החיתוך קורה תמיד, הלמידה איטית מאוד. ערך גדול מדי — הרשת עלולה להתפרץ. 0.1 הוא ערך מאוד קטן ושמרני.

08

A3C — מקביליות ואסינכרוניות

ה-A מ-A3C — Asynchronous. זה הרעיון הגדול: במקום Worker אחד שאוסף ניסיון לינארי, 4 Workers רצים בו-זמנית, כל אחד עם 15 סביבות Pong.

למה מקביליות עוזרת?

אם Worker אחד שחק 1000 פריימים — כולם מסצנה אחת, מצב אחד, מסלול אחד. ה-batch יהיה מאוד מתואם. עם 4 Workers × 15 סביבות = 60 משחקים במקביל, כל batch מגיע ממצבים שונים לגמרי → הגרדיאנט יציב יותר, הלמידה מהירה יותר.

# a3c.py — הרצת Workers
for _ in range(PROCESSES_COUNT):   # = 4
    data_proc = mp.Process(
        target=data_func,
        args=(net, device, train_queue)
    )
    data_proc.start()

# כל Worker מריץ:
def data_func(net, device, train_queue):
    envs = [make_env() for _ in range(ENV_COUNT)]  # 15 סביבות
    agent = Agent(lambda x: net(x)[0], ...)           # Policy head בלבד
    exp_source = ExperienceSourceFirstLast(envs, agent, ...)

    for exp in exp_source:
        train_queue.put(exp)    # שולח ניסיון לתהליך הראשי

קבלת החלטות — איך בוחרים פעולה?

בזמן אימון ובזמן בדיקה — הסוכן בוחר פעולה דרך Policy head בלבד. V(s) לא משתתף בקבלת ההחלטה.

# lib/Agent.py — קבלת החלטה
class Agent:
    def __call__(self, states):
        # forward pass — רק Policy head
        logits = self.model(states)          # → [6 מספרים]

        if self.apply_softmax:               # = True
            probs = F.softmax(logits, dim=1)  # הסתברויות [0,1]
        
        # דגימה הסתברותית — לא argmax!
        action = np.random.choice(probs, p=probs)
        return action
למה דגימה ולא argmax?

argmax תמיד בוחרת את הפעולה הטובה ביותר — אין חקירה. אם ניצחנו פעם אחת על ידי "עלה" לפני ה-שמאל, argmax יבחר "עלה" תמיד ולעולם לא יגלה שיש פתרונות טובים יותר. דגימה הסתברותית מאפשרת חקירה מתמשכת גם בבדיקה.

זרימת הקוד המלאה — test_model.py

אחרי אימון שמגיע ל-reward>18, נטענת המודל הנלמד ומשחקים:

# test_model.py — הפעלת הסוכן הלמוד
def main():
    device = torch.device('cuda' if torch.cuda.is_available() else 'cpu')
    
    # טעינת המודל הנלמד (כל המשקלות)
    net = torch.load('models/complete.pt')
    net = net.to(device)
    net.eval()  # מצב בדיקה — BatchNorm/Dropout מושבתים
    
    # סוכן משתמש רק ב-Policy head
    agent = Agent(lambda x: net(x)[0], device=device, apply_softmax=True)
    
    env = make_env()
    state = [env.reset()]

    while True:
        env.render()                        # הצג על המסך
        action = agent(state)               # 1. קבל פעולה מהסוכן
        next_state, reward, done, _ = env.step(action)  # 2. בצע
        state = [next_state]                # 3. עדכן מצב
        
        if done:
            state = [env.reset()]           # 4. משחק חדש

ציר הזמן המלא — מ-pixel לפעולה

1

פריים נכנס

Atari מחזיר numpy array 210×160×3. wrap_dqn ממיר ל-84×84 grayscale ומוסיף לתור של 4 פריימים.

2

tensor → CNN → flatten

4×84×84 → Conv1(32) → Conv2(64) → Conv3(64) → flatten → וקטור 3136.

3

Policy head → logits

FC(3136→512) → ReLU → FC(512→6). פלט: 6 מספרים לפני softmax.

4

softmax → הסתברויות

e^x_i / Σe^x_j. הופך לוגיטים ל-6 הסתברויות שמסתכמות ל-1.

5

דגימה → פעולה

np.random.choice עם ה-probs. מחזיר מספר 0–5 המייצג פעולה (NOOP/FIRE/UP/DOWN/...).

6

env.step(action)

Atari מבצע את הפעולה, מחזיר פריים חדש + reward. חוזר לשלב 1.

תוצאה — ביצועי על אנושיים

לאחר אימון מלא, הסוכן מנצח בהפרש של 15+ נקודות כמעט תמיד. בני אדם ממוצעים מגיעים ל-~9 נקודות. הסוכן למד לשחק טוב יותר מאדם ממוצע מבלי לקבל הוראה אחת — רק פיקסלים ו-reward.
