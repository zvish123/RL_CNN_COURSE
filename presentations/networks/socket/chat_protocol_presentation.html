📦 ProtocolMessage — dataclass
@dataclass class ProtocolMessage: action: str # חובה תמיד username: Optional[str] = None # register/login password: Optional[str] = None # register/login message: Optional[str] = None # שליחת הודעה status: Optional[str] = None # "ok" / "error" error: Optional[str] = None # טקסט שגיאה from_user: Optional[str] = None # broadcast def to_dict(self) -> dict: # כולל רק שדות שאינם None data = {"action": self.action} if self.username is not None: data["username"] = self.username # ... שאר השדות... return data @classmethod def from_dict(cls, data: dict) -> "ProtocolMessage": # בנייה מ-dict (בא מ-JSON) return cls( action=data.get("action", ""), username=data.get("username"), from_user=data.get("from"), # שים לב! # ... )
📋 שדות ותפקידיהם
שדה טיפוס נדרש מטרה
action str ✓ register/login/message/logout/broadcast
username str? — שם משתמש (register/login)
password str? — סיסמה (register/login)
message str? — תוכן הודעה / תשובה מהשרת
status str? — "ok" או "error"
error str? — טקסט שגיאה מהשרת
from_user str? — שולח ב-broadcast
💡 למה dataclass?
במקום dict רגיל — הכי קריא, יש השלמת קוד ב-IDE, ברור מה השדות האפשריים, ולא ניתן "לשכוח" שדה.
from_user ↔ "from" — ההמרה נעשית פעם אחת ב-from_dict בלבד.

⚙️ ChatProtocol — מתודות
class ChatProtocol: DELIMITER = "\n" # מפריד הודעות def __init__(self): self.buffer = "" # buffer פרטי לכל instance def encode(self, msg: ProtocolMessage) -> bytes: # ProtocolMessage → JSON bytes עם \n raw = json.dumps(msg.to_dict()) + self.DELIMITER return raw.encode("utf-8") def feed_data(self, raw_data: bytes) -> List[ProtocolMessage]: # bytes → רשימת ProtocolMessage self.buffer += raw_data.decode("utf-8") messages = [] while self.DELIMITER in self.buffer: line, self.buffer = self.buffer.split(self.DELIMITER, 1) if line.strip(): msg = ProtocolMessage.from_dict(json.loads(line)) messages.append(msg) return messages def validate(self, msg) -> Optional[str]: if not msg.action: return "Missing action" if msg.action in ("register", "login"): if not msg.username: return "Missing username" if not msg.password: return "Missing password" if msg.action == "message": if not msg.message: return "Missing message" return None # תקין
🏭 Factory Methods (static)
@staticmethod def make_ok(action, text): return ProtocolMessage( action=action, status="ok", message=text ) @staticmethod def make_error(action, error_text): return ProtocolMessage( action=action, status="error", error=error_text ) @staticmethod def make_broadcast(from_user, text): return ProtocolMessage( action="broadcast", from_user=from_user, message=text )
💡 Factory Methods
במקום לבנות dict ידנית בכל מקום — קוראים לפונקציה בעלת שם תיאורי. make_ok, make_error, make_broadcast זה Factory Pattern — דפוס עיצוב קלאסי.
💡 instance per connection
protocol = ChatProtocol() נוצר בתוך handle_client() — כל לקוח מקבל buffer משלו. אין race condition!

⚡ Delimiter — \n
כל הודעה מסתיימת ב-\n. זה מאפשר לשלוח כמה הודעות ב-recv אחד ולפצל אותן נכון ב-feed_data().
📥 תשובות מהשרת ללקוח
◀ OK RESPONSE
{"action": "login", "status": "ok", "message": "Login successful"}\n
◀ ERROR RESPONSE
{"action": "login", "status": "error", "error": "Invalid username or password"}\n
◀ BROADCAST
{"action": "broadcast", "from": "bob", "message": "היי!"}\n
💡 Buffer — הבעיה והפתרון
הבעיה: TCP לא מבטיח שכל recv() יחזיר הודעה שלמה.
הפתרון: feed_data() צובר bytes ב-buffer, ורק כשנמצא \n — מוציא הודעה שלמה.

def handle_client(client_socket, addr): # instance נפרד לכל לקוח protocol = ChatProtocol() while True: # שלב 4 — קבלת bytes raw_data = client_socket.recv(1024) # שלב 5 — feed_data → הודעות msgs = protocol.feed_data(raw_data) for req in msgs: # שלב 6 — ולידציה מרכזית err = protocol.validate(req) if err: client_socket.sendall(protocol.encode( ChatProtocol.make_error(req.action, err) )) continue # שלב 7 — טיפול לפי action if req.action == "message": broadcast( ChatProtocol.make_broadcast( sender, req.message ) )

SUMMARY 09 / 09
🧠 רעיונות מרכזיים
  • 📦
    dataclass — מגדיר מבנה נתונים עם שדות, ברירות מחדל, ותיעוד ברור. עדיף על dict רגיל כשהמבנה קבוע ומוכר מראש.
  • ⚙️
    Buffer Pattern — TCP הוא stream ללא גבולות הודעות. buffer + delimiter (כמו \n) פותרים את הבעיה.
  • 🎯
    Separation of Concerns — השרת לא יודע על JSON. הוא עובד עם ProtocolMessage. ChatProtocol הוא "מתורגמן" בין Python לרשת.
  • 🏭
    Factory Methods — במקום לבנות אובייקט ידנית, קוראים לפונקציה בעלת שם תיאורי. make_broadcast("alice","היי") ברור יותר מ-dict.
  • ♻️
    קוד משותף — protocol.py מיובא גם בשרת וגם בקליינט. שינוי אחד מתפשט לשניהם אוטומטית.
