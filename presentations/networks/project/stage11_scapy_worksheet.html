Scapy ככלי איסוף וניתוח

⏱ זמן משוער: 60-90 דק' ✏️ עבודה בצוות הפרויקט 📋 דורש: בחירת תבנית חריגה משלב 10

פונקציה: analyze_traffic(packets) → list[dict]

התוצר של דף העבודה הזה הוא פונקציית פייתון יחידה, שמקבלת תעבורה (מ-sniff או rdpcap) ומחזירה רשימת אירועים שזוהו — לפי התבנית שהצוות בחר בשלב 10.

⚠ תזכורת מחייבת: בתרגיל הזה משתמשים ב-Scapy רק לקליטה וניתוח (sniff, rdpcap, קריאת שדות). אין לבנות או לשלוח פקטות לצורך תקיפה, הצפה, או הטעיה (כמו ARP spoofing בפועל) — גם אם הנושא של הצוות הוא זיהוי התקפה כזו. כל בדיקה מול תעבורה "חיה" מתבצעת רק במעבדה הסגורה של הכיתה ובאישור מורה.
1

קריאת תעבורה ב-Scapyמתרגלים sniff / rdpcap ובוחנים שכבות בקוד

השתמשו בקובץ ה-pcap משלב 10 (או בהסנפה קצרה ומבוקרת, אם המורה אישר). הריצו בפועל את הקוד הבא ושנו אותו לפי הצורך:

from scapy.all import rdpcap
packets = rdpcap("capture.pcap")
print(len(packets))
packets[0].show()
משימה 1.1

כמה פקטות נקלטו, ואילו שכבות מופיעות בפקטה הראשונה?

הריצו את הקוד מהכרטיס שלמעלה ורשמו את הפלט בקצרה.

משימה 1.2

הדפיסו שדה ספציפי מתוך שכבה

לדוגמה: packets[0][IP].dst או packets[0][TCP].dport. בחרו שדה הרלוונטי לתבנית החריגה שלכם, והדביקו את השורת קוד ואת התוצאה שהתקבלה.

2

כתיבת lfilter לתבנית שלכםהופכים את הכלל מפעילות 2 בשלב 10 לקוד פייתון

תזכורת לתבנית שבחרתם בשלב 10 (DNS Watchdog / ARP Guardian / NetGuard / Chat Security / רעיון אחר). עכשיו כותבים פונקציית lfilter שמבודדת רק את הפקטות הרלוונטיות לתבנית הזו.

# דוגמה מהתרגול - לסנן רק שאילתות DNS מסוג A
def dns_filter(p):
    return DNS in p and p[DNS].opcode == 0 and p[DNSQR].qtype == 1

פונקציה שמקבלת פקטה אחת (p) ומחזירה True רק אם היא רלוונטית לתבנית החריגה שבחרתם.
