Registry — "מוח" Windows

ה-Registry הוא מסד נתונים היררכי שמאחסן כל הגדרה ב-Windows — עמוק בדיסק ובזיכרון. כל תוכנה, כל משתמש, כל התקן.

🕰️ מה לפני Registry?

Windows 3.1 השתמש בקבצי .ini (טקסט פשוט). Registry החליף אותם — מהיר יותר, מאובטח יותר, היררכי יותר. עדיין יש קבצי .ini ישנים (win.ini, system.ini).

🔐 למה חשוב לסייבר?

Registry = מפה של כל מה שקורה במחשב. Malware חייב לגעת ב-Registry לpersistence. Forensics מוצא ראיות שם שנים אחרי. EDR מנטרת Registry changes.

📁 קבצי Registry בדיסק

C:\Windows\System32\config\ — SYSTEM, SAM, SECURITY, SOFTWARE
C:\Users\<user>\NTUSER.DAT — הגדרות משתמש
C:\Users\<user>\AppData\Local\Microsoft\Windows\UsrClass.dat

🏗️ מושגים בסיסיים

Hive = עץ ראשי (HKLM...)
Key = "תיקייה" ב-Registry
Value = שם הערך (כמו שם קובץ)
Data = הנתון עצמו
Type = סוג הנתון

5 ה-Hives — שנה מה הן עושות

HKEY_LOCAL_MACHINE (HKLM) ├── HARDWARE\ ← donat עצ — נוצר בכל boot ├── SAM\ ← password hashes — נעול! ├── SECURITY\ ← policies — נעול! ├── SOFTWARE\ ← כל התוכנות המותקנות │ └── Microsoft\Windows\CurrentVersion\ │ └── Run\ ← AUTORUN — Persistence! └── SYSTEM\ ← services, drivers HKEY_CURRENT_USER (HKCU) └── SOFTWARE\Microsoft\Windows\CurrentVersion\ └── Run\ ← AUTORUN per-user HKEY_CLASSES_ROOT (HKCR) ← file associations HKEY_USERS (HKU) ← כל המשתמשים HKEY_CURRENT_CONFIG (HKCC) ← hardware config נוכחי

🔴 SAM — Security Account Manager

HKLM\SAM מכיל password hashes של כל המשתמשים המקומיים. נעול ב-runtime. Attackers משתמשים ב-Volume Shadow Copy לגניבה. פקודה: reg save HKLM\SAM דורש SYSTEM.

🔑 HKLM vs HKCU

HKLM = כל המשתמשים. דורש Admin.
HKCU = משתמש נוכחי בלבד. ללא Admin.
malware מעדיף HKCU — לא צריך הרשאות!

📝 Persistence Locations

HKLM\...\Run
HKCU\...\Run
HKLM\...\RunOnce
HKLM\System\CurrentControlSet\Services

💡 קיצורים

HKCU = HKEY_CURRENT_USER
HKLM = HKEY_LOCAL_MACHINE
HKCR = HKEY_CLASSES_ROOT
regedit מציג את כולם.

Malware ב-Registry — איפה?

תוקף רוצה persistence שתפעל ללא הרשאות Admin ותרוץ בכל התחברות של המשתמש. איזה מפתח הוא ישתמש?

א HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Run — HKLM כי הוא יציב יותר
ב HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\Run — לא דורש Admin, רץ עם logon
ג HKLM\SAM — SAM מכיל passwords
ד HKCR\.exe — שינוי file associations

סוגי Value Types

סוגתיאורדוגמה
REG_SZ מחרוזת טקסט Unicode "C:\program.exe"
REG_EXPAND_SZ מחרוזת עם משתני סביבה "%SystemRoot%\calc.exe"

REG_BINARY נתונים בינאריים גולמיים 01 00 00 00 ...
REG_MULTI_SZ מחרוזות מרובות (null separated) "val1\0val2\0\0"
REG_NONE ללא סוג מוגדר —

🔴 REG_BINARY ב-Malware

Malware מאחסן shellcode מוצפן ב-REG_BINARY. נראה כנתון לגיטימי, קשה לזיהוי. Base64 ב-REG_SZ — תמרור נוסף.

🐍 Python — קרא ערכי Run key

כתוב פונקציה שמחזירה את כל תוכניות ה-autorun של המשתמש הנוכחי:

import winreg

def get_autorun():
    RUN_KEY = r"Software\Microsoft\Windows\CurrentVersion\Run"
    results = []
    try:
        key = winreg.OpenKey(
            winreg.,
            RUN_KEY
        )
        i = 0
        while True:
            try:
                name, data, _ = winreg.(key, i)
                results.append((name, data))
                i += 1
            except OSError:
                break
        winreg.CloseKey(key)
    except FileNotFoundError:
        print("Key לא קיים")
    return results

for name, path in get_autorun():
    print(f"{name}: {path}")

כמו קבצים — כל Registry key יש לו ACL (Access Control List) שקובע מי יכול לקרוא, לכתוב ולמחוק.

KEY_ALL_ACCESS     = 0xF003F  # הכל
KEY_QUERY_VALUE    = 0x0001   # קריאת ערכים
KEY_SET_VALUE      = 0x0002   # כתיבת ערכים
KEY_CREATE_SUB_KEY = 0x0004   # יצירת subkeys
KEY_ENUMERATE_SUB_KEYS = 0x0008

🔍 Autoruns (Sysinternals)

הכלי הטוב ביותר לראות כל persistence mechanisms. מציג Run keys, Services, Drivers, Browser extensions, Scheduled Tasks — כולם יחד.

🕵️ Python — סריקת Run Keys

import winreg, json
from datetime import datetime

def scan_autoruns():
    HIVES = [
        (winreg.HKEY_LOCAL_MACHINE, "HKLM"),
        (winreg.HKEY_CURRENT_USER,  "HKCU"),
    ]
    RUN_PATHS = [
        r"SOFTWARE\Microsoft\Windows"
        r"\CurrentVersion\Run",
        r"SOFTWARE\Microsoft\Windows"
        r"\CurrentVersion\RunOnce",
    ]
    results = []
    for hive, hive_name in HIVES:
        for path in RUN_PATHS:
            try:
                k = winreg.OpenKey(hive, path)
                i = 0
                while True:
                    try:
                        n, d, _ = winreg.EnumValue(k, i)
                        results.append({
                          "hive": hive_name,
                          "name": n, "value": d,
                          "scanned": datetime.now().isoformat()
                        })
                        i += 1
                    except OSError: break
                winreg.CloseKey(k)
            except: pass
    return results

runs = scan_autoruns()
print(f"נמצאו {len(runs)} autorun entries")
for r in runs:
    print(f"  [{r['hive']}] {r['name']}: {r['value']}")

🐍 כתוב ערך ל-Registry

פונקציה שיוצרת key ב-HKCU וכותבת ערך REG_SZ:

import winreg

def write_reg_value(key_path: str, name: str, data: str):
    try:
        # פתח/צור key
        key = winreg.(
            winreg.HKEY_CURRENT_USER,
            key_path,
            0, winreg.KEY_WRITE
        )
        # כתוב ערך REG_SZ
        winreg.(
            key, name,
            0,
            winreg.,
            data
        )
        winreg.CloseKey(key)
        print(f"✓ נכתב: {name} = {data}")
    except PermissionError:
        print("✗ אין הרשאות כתיבה")

write_reg_value(r"Software\MyTool",
               "Version", "1.0")
