⏱ זמן משוער: 60-90 דק' ✏️ עבודה בצוות הפרויקט 📋 בחירה אחת מתוך חמש אפשרויות

מודול דיאגנוסטיקה / ניתוח שכבות נמוכות

בחרו את האפשרות המתאימה ביותר לסוג הפרויקט שלכם, ובנו אותה כתוספת לרכיב הניטור הקיים — לא חובה שתהיה מורכבת, אבל היא צריכה לרוץ בפועל ולתת תשובה אמיתית.

1

בחירת אפשרות הדיאגנוסטיקהבהתאם לסוג הפרויקט שלכם

שולחים ICMP Echo, מודדים זמן תגובה, מדווחים זמינות.

מסך שמציג את כל הלקוחות המחוברים, עם כתובת IP ו-MAC שלהם.

המשך ל-ARP Guardian משלב 10 - בניית טבלת IP↔MAC מקובץ הסנפה.

שמירת MAC נוכחי של ה-Gateway, והתרעה כשהוא משתנה.

רישום הודעות DHCP (DISCOVER/OFFER/REQUEST/ACK) ו/או ICMP שעברו בתעבורה, ליומן קריא.

משימה 1.1

למה האפשרות הזו מתאימה לפרויקט שלכם?

2

מדריך קצר לפי האפשרות שבחרתםקראו רק את הקטע הרלוונטי לבחירה שלכם

🟢 בדיקת ping
# בפייתון, באמצעות subprocess (לא דורש הרשאות מנהל):
import subprocess
result = subprocess.run(["ping", "-c", "1", target_ip], capture_output=True)
is_alive = result.returncode == 0
🟢 הצגת IP/MAC של לקוחות

בכל חיבור לקוח לשרת בפרויקט שלכם, שמרו את socket.getpeername() (IP) ואם נדרש MAC - שלפו מתוך פקטות ARP/Ethernet שנקלטו עם Scapy (כמו בשלב 10-11).

🟢 ניתוח ARP מתוך pcap
from scapy.all import rdpcap, ARP
packets = rdpcap("capture.pcap")
ip_to_mac = {}
for pkt in packets:
    if pkt.haslayer(ARP):
        ip_to_mac.setdefault(pkt[ARP].psrc, set()).add(pkt[ARP].hwsrc)
🟢 זיהוי שינוי ב-Gateway

בנו על הקוד מדף ההסבר: שמרו MAC ידוע של ה-Gateway, והשוו לכל פקטת ARP שמגיעה מ-IP של ה-Gateway.

🟢 תיעוד DHCP/ICMP

השתמשו ב-lfilter (משלב 11) שמסנן BOOTP/DHCP או ICMP, ורשמו לכל פקטה שורה קריאה ביומן (זמן, סוג הודעה, IP מקור/יעד).
