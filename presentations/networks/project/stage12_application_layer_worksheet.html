שכבת האפליקציה - ניתוח עומק וממשק UI

⏱ זמן משוער: 60-90 דק' ✏️ עבודה בצוות הפרויקט 📋 דורש: מודול ופלט אנומליה משלב 10-11

מסך UI לניתוח פרוטוקול שכבת האפליקציה

בדף הזה הצוות בוחר פרוטוקול אחד (HTTP / DNS / SMTP), מאתר בקובץ הסנפה את השדות הרלוונטיים, ומתכנן מסך שמציג אותם בצורה קריאה למשתמש — לא כטבלת לוג גולמית.

1

בחירת פרוטוקול לניתוח עומקבהתאם למודול שהצוות בונה

מתאים ל-Packet Explorer, NetGuard.

מתאים ל-DNS Watchdog, Phishing Analyzer.

מתאים ל-Phishing Analyzer.

משימה 1.1

למה בחרתם בפרוטוקול הזה?

קשרו את הבחירה למודול הספציפי שהצוות בונה (משלב 10) - מה הפרוטוקול הזה "מספק" שאף פרוטוקול אחר לא מספק לכם.

⚠ ייבדק ע"י המורה
2

איתור שדות בקובץ הסנפהמוצאים בפועל את הערכים הרלוונטיים

דוגמאות לשדות לפי פרוטוקול (להרחיב/לשנות לפי הצורך):

HTTP:  status code, method (GET/POST), Host header, User-Agent
DNS:   dns.qry.name, dns.qry.type, dns.flags.response, dns.flags.rcode
SMTP:  EHLO response, AUTH LOGIN, פקודה ותגובה (response code), \r\n בסוף שורה
משימה 2.1

אתרו ב-Wireshark שלוש דוגמאות שדה+ערך

פתחו את קובץ ההסנפה, פלטרו לפי הפרוטוקול שבחרתם (לדוגמה http / dns), ומצאו שלוש פקטות מעניינות. לכל אחת רשמו שדה וערך.

משימה 2.2

מה ערך "תקין" לעומת ערך "חשוד" בשדה הזה?

לדוגמה: status code 200 לעומת 403 חזרתי הרבה; RCODE 0 לעומת RCODE 3; EHLO תקין לעומת ניסיון AUTH חוזר.

3

תכנון מסך ה-UIמתרגמים שדות גולמיים לתצוגה קריאה

בנו "ויירפרים" טקסטואלי של המסך: אילו שדות יוצגו, מה התרגום האנושי שלהם, ואיך מסומנת חריגה.

משימה 3.1

בנו את שדות המסך (4-6 שדות)

לכל שדה: שם תצוגה (בעברית, ברור למשתמש) + הערך הגולמי שהוא מציג מהפרוטוקול.

איך מסומנת חריגה במסך (צבע, תג, אייקון)?

משימה 3.2 — הגשה

הריצו את שלב החילוץ (extraction) מקובץ הסנפה

כתבו פונקציית פייתון (יכולה להתבסס על analyze_traffic משלב 11) שמחלצת בפועל את השדות שבחרתם במשימה 3.1 מכל פקטה רלוונטית, ומחזירה אותם כ-dict מוכן להצגה במסך.

  • הפונקציה רצה בלי שגיאות מול קובץ ההסנפה
  • השדות שמוחזרים תואמים בדיוק לאלה שתוכננו במשימה 3.1
  • שם קובץ הקוד: protocol_view_שם_הצוות.py

רפלקציה קצרה לסיום

לפני שממשיכים לשלב הבא של הפרויקט — חשוב, כתוב, בדוק עם השותף, ואז שאלו את המורה אם נשארה אי-בהירות.


⚠ רפלקציה אישית — לא נבדקת "נכון/לא נכון"

בסיום מילוי הדף — לחצו על הכפתור כדי לשמור את כל התשובות שלכם כקובץ JSON, ולהגיש אותו למורה בנוסף לקובץ הקוד מפעילות 3.2.
