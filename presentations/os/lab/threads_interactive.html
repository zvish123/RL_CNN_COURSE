01
Join

המתנה לסיום thread

// מושג

מה זה join()?

כשה-thread הראשי קורא ל-t.join(), הוא עוצר ומחכה עד שה-thread t מסיים לרוץ.

import threading, time

def worker(name, delay):
    time.sleep(delay)
    print(f"[{name}] סיים ✓")

t1 = threading.Thread(target=worker, args=("A", 2))
t2 = threading.Thread(target=worker, args=("B", 1))

t1.start(); t2.start()

t1.join()  # ממתין ל-t1
t2.join()  # ממתין ל-t2

print("כולם סיימו!")  # רק כאן!

⚠️ join(timeout=N)

לא תמיד רוצים לחכות לנצח. join(timeout=5) ממתין עד 5 שניות ואז ממשיך בכל מקרה.

🏁

בלי join

הפלט מגיע בסדר לא צפוי. "כולם סיימו!" יכול להדפס לפני שה-threads בכלל התחילו!

Condition Variable

כמו Push Notification — thread אחד שולח הודעה ("הנתון מוכן"), thread אחר מחכה להודעה.

import threading, time

cond = threading.Condition()
ready = False

def producer():
    global ready
    with cond:
        time.sleep(2)
        ready = True
        cond.notify_all()  # מעיר את כולם

def consumer(name):
    with cond:
        while not ready:
            cond.wait()  # ישן עד notify
        print(f"{name}: קיבל!")

t1 = threading.Thread(target=producer)
t2 = threading.Thread(target=consumer, args=("one",))
t3 = threading.Thread(target=consumer, args=("two",))
t1.start()
t2.start()
t3.start()

wait()

משחרר את ה-Lock באופן אטומי וממתין. לא בוזבז CPU!

notify()

מעיר thread אחד שמחכה. notify_all() מעיר את כולם.

⚠️ Spurious Wakeups

תמיד כתוב while not condition ולא if not!

Barrier — נקודת מפגש

🎮 אנלוגיה: כולם ממתינים בחדר בריחה עד שכל הצוות מגיע — רק אז עוברים לשלב הבא.

import threading, time, random

barrier = threading.Barrier(3)  # 3 threads

def phase_worker(name):
    # שלב א׳ — כל thread עושה משהו אחר
    duration = random.uniform(0.5, 2.5)
    print(f"{name}: מתחיל שלב א׳ (ייקח {duration:.1f} שניות)")
    time.sleep(duration)
    barrier.wait()  # ⬅ כולם עוצרים כאן
                   # עוברים יחד כשהאחרון מגיע

    # שלב ב׳ — מתחיל רק כשכולם מוכנים
    print(f"{name}: מתחיל שלב ב׳ ✓")
    time.sleep(0.5)
    print(f"{name}: שלב ב׳ הסתיים ✓")

threads = [
    threading.Thread(target=phase_worker, args=(name,))
    for name in ["Alice", "Bob", "Charlie"]
]

print("=== מתחילים ===")
for t in threads:
    t.start()

for t in threads:
    t.join()

print("=== כולם סיימו ===")

⚠️ BrokenBarrierError

אם thread אחד קורס, ה-Barrier "נשבר" וכולם מקבלים שגיאה. תמיד השתמש ב-timeout!

Condition vs Barrier — איזה להשתמש?

לכל תרחיש, החלט איזה מנגנון מתאים:

תרחיש A

Thread A מוריד קובץ, Threads B ו-C מחכים לו ורק אחר כך מעבדים את הנתונים.

תרחיש B

3 threads מחשבים חלקים שונים של מטריצה, ואחרי שכולם מסיימים — ממשיכים לשלב הבא יחד.

א A → Condition Variable  |  B → Barrier
ב A → Barrier  |  B → Condition Variable
ג שניהם — Barrier לשניהם
ד Lock רגיל מספיק לשניהם

Queue — Producer/Consumer

📱 אנלוגיה: TikTok feed — הרשת "מכניסה" סרטונים לתור, ה-app "מוציא" ומציג. queue.Queue הוא thread-safe מהקופסה.

from queue import Queue
import threading, time

q = Queue(maxsize=5)  # מקסימום 5 בתור

def producer():
    for i in range(10):
        q.put(f"packet_{i}")  # חוסם אם תור מלא
        print(f"[Producer] 📦 שלח: packet_{i}  |  תור: {q.qsize()}/5")
        time.sleep(0.5)
    print("[Producer] ✅ סיים לשלוח הכל")

def process(item):
    time.sleep(1.2)                 # עיבוד לוקח זמן
    print(f"[Consumer] ⚙️  עיבד: {item}")

def consumer():
    while True:
        item = q.get()  # חוסם עד שיש פריט
        if item is None:            # סימן עצירה
            q.task_done()
            break
        process(item)
        q.task_done()   # חובה!
    print("[Consumer] 🏁 עצר")

if __name__ == "__main__":
    print("=== מתחיל Producer/Consumer ===\n")

    t_producer = threading.Thread(target=producer)
    t_consumer = threading.Thread(target=consumer, daemon=True)

    t_consumer.start()
    t_producer.start()

    t_producer.join()               # מחכה שה-producer יסיים

    q.join()                        # מחכה שכל ה-packets יעובדו

    q.put(None)                     # סימן עצירה ל-consumer
    t_consumer.join()

    print("\n=== כולם סיימו ===")

Deadlock

כשהמערכת נתקעת לנצח

// ארבעת תנאי קופמן

מתי נוצר Deadlock?

כל 4 התנאים חייבים להתקיים בו-זמנית:

01 // MUTUAL EXCLUSION

בלעדיות

רק thread אחד יכול להחזיק משאב בכל זמן נתון.

02 // HOLD AND WAIT

אחזוק והמתנה

Thread מחזיק משאב אחד בעודו ממתין למשאב אחר.

03 // NO PREEMPTION

אי-הפרעה

לא ניתן לקחת משאב בכוח מ-thread שמחזיק בו.

04 // CIRCULAR WAIT

המתנה מעגלית ⚠️

A מחכה ל-B, B מחכה ל-A — מעגל אינסופי.

💡 הפתרון: שבור תנאי אחד מארבעת התנאים — deadlock בלתי אפשרי.
// הדגמה חיה

Deadlock — ויזואליזציה

Thread A
→ מחזיק
Lock 1
מחכה⟵
⟶ מחכה
Lock 2
← מחזיק
Thread B
lock_a, lock_b = threading.Lock(), threading.Lock()

def thread_A():
    with lock_a:          # ✓ רכש lock_a
        time.sleep(0.1)
        with lock_b: ...  # ✗ מחכה ל-lock_b — נתקע!

def thread_B():
    with lock_b:          # ✓ רכש lock_b
        time.sleep(0.1)
        with lock_a: ...  # ✗ מחכה ל-lock_a — DEADLOCK

⚠️ הקוד הזה יתקע לנצח. אל תריץ אותו בלי Ctrl+C מוכן!

// שאלת הבנה #3

🔍 האם יש כאן Deadlock?

lock_x = threading.Lock()
lock_y = threading.Lock()

def thread_1():
    with lock_x:
        with lock_y:
            do_work()

def thread_2():
    with lock_x:    # שים לב — X לפני Y
        with lock_y:
            do_work()
א כן — שני threads רוצים את אותם locks
ב לא — שניהם רוכשים locks באותו סדר (X→Y)
ג אולי — תלוי בתזמון הריצה
ד כן — כי שני threads רצים בו-זמנית
// מניעת Deadlock

3 אסטרטגיות מניעה

📋

Lock Ordering

תמיד רכוש locks באותו סדר. אם כולם עושים A→B, אי-אפשר Circular Wait.

# תמיד lock_a לפני lock_b
with lock_a:
    with lock_b: ...
⏰

Timeout

נסה לרכוש lock עם timeout. אם לא הצלחת — שחרר הכל ונסה שוב.

if lock.acquire(timeout=1):
    # עבד
    lock.release()
else:
    # retry...
📦

השתמש ב-Queue

במקום shared state עם locks — העבר הודעות. Queue מובנה, ללא deadlock.

# לא צריך lock !
q.put(data)     # ↑ producer
q.get()        # ↓ consumer
// משימה אישית

💀 Deadlock Hunter

בדוק את שלושת קטעי הקוד — האם יש deadlock? מה הפתרון?

קוד A
with L1:
    with L2:
        work()
---
with L2:
    with L1:
        work()
✓ נכון! L1→L2 vs L2→L1 = Circular Wait
קוד B
with L1:
    with L2:
        work()
---
with L1:
    with L2:
        work()
✓ נכון! אותו סדר L1→L2 בשניהם — בטוח!
קוד C
with L1:
    q.put(data)
---
item = q.get()
with L2:
    work(item)
✓ נכון! Queue → אין שיתוף ישיר → אין deadlock
// שאלת אתגר

🏆 שאלת הבונוס

מה יקרה בקוד הזה?

cond = threading.Condition()
barrier = threading.Barrier(2)

def t1():
    with cond:
        cond.wait()   # ⬅ ממתין להודעה
    barrier.wait() # ⬅ ממתין ל-t2

def t2():
    barrier.wait() # ⬅ ממתין ל-t1
    with cond:
        cond.notify() # ⬅ מודיע ל-t1
א הקוד יעבוד תקין — t1 מחכה, t2 מודיע
ב Deadlock — t1 מחכה ל-cond, t2 מחכה ל-t1 ב-barrier
ג Race Condition — תלוי בתזמון
ד שגיאה ב-Python — לא ניתן לשלב Condition ו-Barrier
