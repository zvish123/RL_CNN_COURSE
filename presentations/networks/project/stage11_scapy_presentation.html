Scapy ככלי איסוף וניתוח

בשלב 10 בנינו את PacketAnalyzer מסביב לתאוריה של Wireshark. בשלב הזה אנחנו פותחים את הקופסה השחורה: לומדים איך Scapy עצמו עובד בפייתון — sniff, filter, lfilter — ומשתמשים בו כדי שהמודול יידע לקרוא תעבורה בעצמו, לא רק מקובץ Wireshark מוכן.

תזכורת: Packet ב-Scapy

Scapy מייצג כל פקטה כאוסף שכבות מקוננות — בדיוק כמו ב-Wireshark, רק שכאן זה אובייקט פייתון שאפשר לתשאל בקוד.

>>> packets[0].show()
###[ Ethernet ]###
  dst= ...
###[ IP ]###
     src= 10.0.0.3
     dst= 82.166.201.177
###[ TCP ]###
        sport= 49420
        dport= 80
###[ Raw ]###
           load= 'GET / HTTP/1.1...'

כל שכבה היא שדה בתוך השכבה הקודמת — בדיוק כמו ה-encapsulation שלמדנו ב-Wireshark.

sniff() — לכידת תעבורה חיה

הפונקציה המרכזית לקליטת פקטות בזמן אמת. כמה פרמטרים שכדאי להכיר:

from scapy.all import sniff

packets = sniff(
    count=20,        # כמה פקטות לקלוט ואז לעצור
    timeout=10,       # או לעצור אחרי X שניות, המוקדם מביניהם
    iface="eth0",     # איזה כרטיס רשת להאזין לו
    filter="tcp"      # פילטר BPF - כמו ב-Wireshark
)

הסנפה בלי count ובלי timeout רצה לנצח עד שעוצרים אותה ביד. בעבודה על הפרויקט (ובמיוחד בבדיקות אוטומטיות) תמיד להגביל אחד מהשניים — אחרת התרגיל "נתקע".

filter מול lfilter

שני מנגנוני סינון שונים בתכלית — לבחור נכון חוסך הרבה זמן עיבוד:

filter (BPF)lfilter (פייתון)
מתי מסנןברמת מערכת ההפעלה, לפני שהפקטה מגיעה ל-Scapyבתוך פייתון, אחרי שהפקטה כבר נקלטה
מהירותמהיר יותר (פחות עומס)איטי יותר אך הרבה יותר גמיש
תחבירמחרוזת BPF, כמו ב-Wireshark capture filterפונקציית פייתון שמקבלת פקטה ומחזירה True/False
מתאים לסינון פשוט (פרוטוקול, פורט, IP)תנאי מורכב (שדה ספציפי, ערך מחושב, כמה תנאים יחד)

דוגמת lfilter מהתרגול

פילטר תיכנותי שמחפש שאילתת DNS מסוג A (כתובת רגילה) — בדיוק כמו שראינו בתרגול ה-Scapy:

def dns_filter(p):
    return DNS in p and p[DNS].opcode == 0 and p[DNSQR].qtype == 1

packets = sniff(count=4, lfilter=dns_filter)

שימו לב: filter ו-lfilter אפשר לשלב יחד — filter="udp" מסנן ברמת המערכת, ו-lfilter בודק תנאי מדויק יותר על מה שעבר.

גישה לשכבות ול-Raw data

  • packet.haslayer(TCP) — בדיקה אם השכבה קיימת בפקטה
  • packet[TCP].dport — גישה לשדה ספציפי בתוך שכבה
  • packet[Raw].load — התוכן הגולמי של שכבת האפליקציה (bytes) — חייבים להמיר ל-str לפני חיפוש טקסט בו
if packet.haslayer(Raw):
    content = packet[Raw].load.decode(errors="ignore")
    if "GET /admin" in content:

⚠ עיקרון מחייב

לא בונים כלי תקיפה

Scapy מאפשר גם לבנות ולשלוח פקטות (לא רק לקלוט) — אבל בפרויקט הזה אנחנו משתמשים בו רק לקריאה, ניתוח וזיהוי.

  • מותר: sniff, rdpcap, ניתוח שדות, ספירה, התרעה
  • לא מותר בפרויקט: בניית פקטות לשליחה (send/sr) לצורך תקיפה, הצפה, או הטעיה (ARP spoofing בפועל, DoS, וכו')
  • גם אם הנושא שלכם הוא "זיהוי ARP spoofing" — אתם מזהים אותו, לא מדמים אותו על רשת אמיתית ללא אישור מורה מפורש ובסביבת מעבדה סגורה

⏸ תחנת עצירה

מה עושים השבוע?

פותחים את דף העבודה — Scapy לאיסוף וניתוח, וביחד עם הצוות:

  • קוראים תעבורה עם sniff או rdpcap, ובוחנים את השכבות בפועל
  • כותבים lfilter שמתאים לתבנית החריגה שבחרתם בשלב 10
  • ממשים את analyze_traffic() שמחזירה רשימת אירועים, ומגישים קובץ JSON עם התשובות

בדיקת הבנה

מתי הכי הגיוני להשתמש ב-lfilter ולא רק ב-filter?

כשרוצים לסנן רק לפי פרוטוקול, כמו "רק TCP"
כשהתנאי תלוי בערך שדה ספציפי או בחישוב, כמו "qtype == 1 וגם opcode == 0"
כשרוצים לכלוך הסנפה מהירה יותר ככל האפשר
