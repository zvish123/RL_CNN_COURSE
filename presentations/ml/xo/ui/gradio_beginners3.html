שלד ריק — gr.Blocks()
🧱 מה זה ממשק (UI)?

עד עכשיו הרצתם קוד Python בטרמינל — רואים טקסט, מזינים פרמטרים בתוך הקוד, ורואים תוצאות בשחור-לבן. ממשק משתמש (UI) הוא שכבה גרפית שמאפשרת לכל אחד להשתמש בקוד שלכם — בדפדפן, עם כפתורים, סליידרים וגרפים.

Gradio היא ספריית Python שבונה את הממשק בשבילכם. אתם כותבים Python — Gradio הופך אותו לעמוד web.

🏠
אנלוגיה: חשבו על ממשק כמו בית. gr.Blocks() הוא קירות הבית הריקים. עכשיו נתחיל לשים בהם ריהוט — כותרות, תיבות קלט, כפתורים.
⚙️ איך gr.Blocks() עובד?

כותבים with gr.Blocks() as demo: ובתוך ה-with מגדירים את כל הרכיבים. בסוף קוראים ל-demo.launch() שמפעיל שרת web ופותח את הממשק בדפדפן.

חשוב: כל רכיב שתוסיפו חייב להיות בתוך ה-with (עם indentation). זה כמו לשים ריהוט בתוך הבית ולא מחוצה לו.

PYTHON — שלד ריק 
import gradio as gr          # טוענים את ספריית Gradio

with gr.Blocks() as demo:    # פותחים את "הבית" הריק
    pass                        # בינתיים — ריק. נמלא אותו בהמשך.

demo.launch()                  # מפעילים את השרת ופותחים דפדפן

כותרת ותיאור — gr.Markdown()
📝 מה זה gr.Markdown?

gr.Markdown() מציג טקסט בממשק. הטקסט כתוב בפורמט Markdown — שפה פשוטה לעיצוב טקסט שמוכרת מ-README ב-GitHub.

זה לא שדה קלט — המשתמש לא מקליד שם. זו תצוגה בלבד, כמו כיתוב על שלט.

🪧
אנלוגיה: אם הממשק הוא בית, gr.Markdown() הוא השלטים והכיתובים על הקירות — "חדר שינה", "מטבח". הם לא עושים כלום, רק עוזרים להבין מה יש שם.
תחביר Markdownמה זה יוצרדוגמה
# טקסטכותרת ראשית (h1) — גדולהgr.Markdown("# שלום")
## טקסטכותרת משנה (h2) — בינוניתgr.Markdown("## קטע")
### טקסטכותרת קטנה (h3)gr.Markdown("### פרט")
**טקסט**טקסט מודגשgr.Markdown("**חשוב**")
*טקסט*טקסט נטויgr.Markdown("*הערה*")
`קוד`קוד inlinegr.Markdown("`gamma`")
📂 ממשיכים על אותו קובץ — xo_ui.py. מוחקים את ה-pass ומוסיפים את השורות החדשות.

תיבת טקסט — gr.Textbox()
📦 מה זה gr.Textbox?

gr.Textbox() הוא שדה טקסט — המקום שבו המשתמש מקליד, או שהממשק מציג תוצאה. אותו רכיב משמש לקלט ולפלט! ההבדל הוא רק איך אתם משתמשים בו — האם כ-input או כ-output של פונקציה.

📬
אנלוגיה: תיבת Textbox היא כמו תיבת דואר. אפשר להכניס אליה מכתב (קלט מהמשתמש) ואפשר לקבל ממנה מכתב (פלט מהפונקציה).
פרמטרסוגמה הוא עושה
label=strהכיתוב שמופיע מעל השדה — חובה לתאר מה זה
placeholder=strטקסט אפור בתוך השדה שנעלם כשמתחילים להקליד — רמז למשתמש
lines=intכמה שורות גובה יהיה לשדה. lines=1 = שדה רגיל, lines=5 = אזור טקסט גדול
value=strערך ברירת המחדל — מה כתוב בשדה כשהממשק נפתח
interactive=boolTrue = ניתן לעריכה (ברירת מחדל), False = קריאה בלבד (לפלט)
📂 ממשיכים על אותו קובץ — מוסיפים שני Textbox: אחד לשם הניסוי ואחד לפלט.

שלב 3 — חיבור קוד לממשק
כפתור ואירועים — gr.Button() + .click()
🔘 מה זה gr.Button ו-.click()?

gr.Button() הוא כפתור שהמשתמש לוחץ עליו. לבד, הכפתור לא עושה כלום. כדי שיקרה משהו, צריך לחבר אותו לפונקציה Python עם .click().

זה הלב של Gradio — חיבור בין פעולת משתמש (לחיצה) לפונקציה Python. הממשק לוקח ערכים מרכיבי קלט, מעביר לפונקציה, ומציג את ה-return ברכיבי פלט.

🔌
אנלוגיה: הכפתור הוא מתג חשמל. .click() הוא החיווט — מחבר את המתג לנורה (הפונקציה). בלי חיווט המתג חסר משמעות.
⚙️ פרמטרי .click()

btn.click(fn=..., inputs=..., outputs=...)

fn= — הפונקציה Python שתרוץ בלחיצה.

inputs= — רכיב (או רשימה) שהערך שלו יועבר לפרמטר הראשון של הפונקציה.

outputs= — רכיב (או רשימה) שיתעדכן עם ה-return של הפונקציה.

📂 ממשיכים על אותו קובץ — מוסיפים פונקציה, כפתור, וחיבור ביניהם.
PYTHON — שלב 3: כפתור + click 
import gradio as gr

# ── פונקציה שתרוץ בלחיצה על הכפתור ──
def start_experiment(name):
    if not name:
        return "⚠️ נא להזין שם לניסוי!"
    return f"✅ ניסוי '{name}' הופעל בהצלחה!"                    

with gr.Blocks() as demo:
    gr.Markdown("# 🧪 מערכת ניסויי XO")
    gr.Markdown("ממשק לניהול וניתוח ניסויי Reinforcement Learning")

    exp_name = gr.Textbox(label="שם הניסוי", placeholder="לדוגמה: baseline_01")
    output   = gr.Textbox(label="סטטוס", interactive=False)

    run_btn = gr.Button("🚀 הפעל ניסוי", variant="primary")       

    run_btn.click(
        fn=start_experiment,   # הפונקציה שתרוץ                  
        inputs=exp_name,         # מה להעביר לפונקציה             
        outputs=output           # לאן לשים את התוצאה             
    )                                                                   

demo.launch()
💡 variant="primary" — גורם לכפתור להיות בצבע ראשי בולט (כתום ב-Gradio). אפשר גם "secondary" לכפתור פחות בולט ו-"stop" לכפתור ביטול אדום.

שלב 4 — פריסה על המסך
שורות ועמודות — gr.Row() + gr.Column()
📐 מה זה Row ו-Column?

ברירת המחדל של Gradio: כל רכיב שמוסיפים מופיע מתחת לקודם. כמו שורות בדף.

gr.Row() — מיכל שמסדר רכיבים אופקית (זה-לצד-זה). gr.Column() — מיכל שמגדיר עמודה בתוך Row, עם שליטה על הרוחב דרך scale=.

🗞️
אנלוגיה: עיתון מחולק לעמודות. Row הוא "שורת עמודות" בעיתון, וכל Column הוא עמודה אחת. scale=2 אומר "עמודה זו כפולה ברוחב מעמודה עם scale=1".
↔️ gr.Row()

כל רכיב בתוכו יופיע לצד הקודם. ללא Row — כל רכיב בשורה חדשה.

↕️ gr.Column(scale=)

מגדיר עמודה בתוך Row. scale=2 = פי 2 רחב מ-scale=1 לצדו.

📂 ממשיכים על אותו קובץ — מסדרים את הקלט והפלט זה-לצד-זה.
PYTHON — שלב 4: Row + Column 
import gradio as gr

def start_experiment(name):
    if not name:
        return "⚠️ נא להזין שם לניסוי!"
    return f"✅ ניסוי '{name}' הופעל בהצלחה!"

with gr.Blocks() as demo:
    gr.Markdown("# 🧪 מערכת ניסויי XO")
    gr.Markdown("ממשק לניהול וניתוח ניסויי Reinforcement Learning")

    with gr.Row():                         # פותחים שורה אופקית
        with gr.Column(scale=1):            # עמודה שמאלית — קלט
            gr.Markdown("### ⚙️ הגדרות")
            exp_name = gr.Textbox(
                label="שם הניסוי",
                placeholder="baseline_01"
            )
            run_btn = gr.Button("🚀 הפעל", variant="primary")
        with gr.Column(scale=1):            # עמודה ימנית — פלט
            gr.Markdown("### 📊 תוצאות")
            output = gr.Textbox(
                label="סטטוס",
                interactive=False,
                lines=4
            )                                 

    run_btn.click(fn=start_experiment, inputs=exp_name, outputs=output)

demo.launch()

שלב 5 — פרמטרים מספריים
מחוון ומספר — gr.Slider() + gr.Number()
🎚️ מה זה gr.Slider?

gr.Slider() הוא מחוון שמאפשר לבחור מספר על ידי גרירה. מושלם לפרמטרים כמו gamma ו-exploration_rate שיש להם טווח ידוע (למשל 0.0 עד 1.0).

🔢 מה זה gr.Number?

gr.Number() הוא שדה מספרי מדויק — המשתמש מקליד מספר או לוחץ על חצים. מושלם לפרמטרים כמו תגמולים שצריך דיוק מספרי (win_reward=1.0).

פרמטרי gr.Slider()
minimum=ערך מינימום (ברירת מחדל: 0)
maximum=ערך מקסימום (ברירת מחדל: 100)
step=גודל צעד. 0.05 = קפיצות של 0.05
value=ערך ברירת מחדל כשהממשק נפתח
label=הכיתוב מעל המחוון
פרמטרי gr.Number()
value=ערך ברירת מחדל
label=הכיתוב מעל השדה
step=גודל קפיצה בחצים
minimum=מגביל ערך מינימלי
📂 ממשיכים על אותו קובץ — מוסיפים פרמטרים מספריים לעמודת הקלט.
PYTHON — שלב 5: Slider + Number 
import gradio as gr

def start_experiment(name, games, gamma, win_r):
    if not name:
        return "⚠️ נא להזין שם לניסוי!"
    return (
        f"✅ ניסוי '{name}' הופעל!\n"
        f"משחקים: {games}\n"
        f"Gamma: {gamma}\n"
        f"פרס ניצחון: {win_r}"
    )

with gr.Blocks() as demo:
    gr.Markdown("# 🧪 מערכת ניסויי XO")
    gr.Markdown("ממשק לניהול וניתוח ניסויי Reinforcement Learning")

    with gr.Row():
        with gr.Column(scale=1):
            gr.Markdown("### ⚙️ הגדרות")
            exp_name = gr.Textbox(label="שם הניסוי", placeholder="baseline_01")

            num_games = gr.Slider(         # מחוון למספר משחקים
                minimum=100,
                maximum=50000,
                step=100,
                value=5000,
                label="מספר משחקים"
            )
            gamma = gr.Slider(             # מחוון ל-gamma (0-1)
                minimum=0.0, maximum=1.0,
                step=0.05, value=0.9,
                label="Gamma"
            )
            win_r = gr.Number(             # שדה מספרי לפרס
                value=1.0,
                label="פרס ניצחון",
                step=0.1
            )                               

            run_btn = gr.Button("🚀 הפעל", variant="primary")

        with gr.Column(scale=1):
            gr.Markdown("### 📊 תוצאות")
            output = gr.Textbox(label="סטטוס", interactive=False, lines=4)

    run_btn.click(
        fn=start_experiment,
        inputs=[exp_name, num_games, gamma, win_r],  # ← עכשיו רשימה!
        outputs=output
    )

demo.launch()
💡 שים לב: כשיש יותר מקלט אחד, inputs= מקבל רשימה: inputs=[a, b, c]. הסדר ברשימה = סדר הפרמטרים בפונקציה.

שלב 6 — ארגון מתקדם
טאבים וקיבוץ — gr.Tabs() + gr.Group()
📑 מה זה gr.Tabs?

gr.Tabs() הוא מיכל שמחולק לטאבים — כמו תיקיות בארון. המשתמש לוחץ על טאב ורואה תוכן שונה. שימושי כשיש כמה "מצבים" שונים לממשק.

📦 מה זה gr.Group?

gr.Group() מוסיף מסגרת ויזואלית סביב קבוצת רכיבים שקשורים זה לזה. לא משנה את הפריסה — רק עוזר למשתמש להבין שהרכיבים שייכים יחד.

📂 הממשק הסופי — הכול ביחד: Tabs, Groups, Row, Column, Sliders, Numbers, Button, Markdown.
PYTHON — שלב 6: ממשק מלא

def start_experiment(name, games, strategy, g, eps, win, loss, tie, block):

שלב 7 — בחירה מתוך אפשרויות
תיבות סימון ובחירה — gr.Checkbox() + gr.Radio()
☑️ מה זה gr.Checkbox?

gr.Checkbox() הוא תיבת סימון — המשתמש מסמן או מבטל. הערך שמועבר לפונקציה הוא True אם מסומן, False אם לא. מושלם ל"כן/לא", הפעלה/כיבוי של פיצ'ר.

✅
אנלוגיה: Checkbox הוא כמו תיבת סימון בטופס נייר — "האם אתה מסכים לתנאים?" — מסמן V או משאיר ריק.
פרמטרסוגמה הוא עושה
label=strהכיתוב שמופיע לצד תיבת הסימון
value=boolTrue = מסומן כברירת מחדל, False = לא מסומן (ברירת מחדל)
info=strטקסט הסבר קטן שמופיע מתחת לתיבה
🔘 מה זה gr.Radio?

gr.Radio() מציג מספר אפשרויות ומאפשר לבחור אחת בלבד. הערך שמועבר לפונקציה הוא המחרוזת של האפשרות שנבחרה. כבר ראינו אותו — עכשיו נלמד אותו לעומק.

🔘
אנלוגיה: Radio buttons הם כמו בחירת מגדר בטופס — "זכר / נקבה / אחר" — ניתן לסמן רק אחד. ברגע שבוחרים אפשרות חדשה, הקודמת מתבטלת אוטומטית.
פרמטרסוגמה הוא עושה
choices=listרשימת האפשרויות לבחירה, לדוגמה: ["random", "heuristic"]
value=strהאפשרות שנבחרת כברירת מחדל כשהממשק נפתח
label=strכותרת מעל קבוצת הכפתורים
info=strהסבר קצר מתחת לכותרת
📋 מה ההבדל בין Checkbox ל-Radio?

Checkbox — שאלת כן/לא אחת. הערך הוא True/False.
Radio — בחירה אחת מתוך רשימה. הערך הוא מחרוזת.

אם צריך לאפשר מספר בחירות מתוך רשימה — יש gr.CheckboxGroup() (בונוס).

📂 ממשיכים על אותו קובץ — מוסיפים Checkbox ו-Radio לפרמטרי הניסוי.

שלב 8 — עיצוב הממשק
ערכות נושא ו-CSS — theme= + css=
🎨 מה זה CSS?

CSS (Cascading Style Sheets) היא שפת עיצוב לדפי web. היא קובעת צבעים, גדלי טקסט, מרווחים, גבולות — את כל מה שקשור למראה. בעוד שה-Python/Gradio קובע מה יש בממשק, CSS קובע איך זה נראה.

ב-Gradio יש שתי דרכים לעצב: Themes מוכנות מראש, ו-CSS מותאם אישית שכותבים בעצמנו.

👔
אנלוגיה: Gradio בונה את הרהיטים בבית (כפתורים, שדות). CSS הוא הצבע על הקירות, הבד על הספה, הגופן על השלטים. אותו בית, מראה שונה לגמרי.
🎨 Themes — ערכות נושא מובנות

Gradio מגיעה עם ערכות עיצוב מוכנות. מגדירים אותן ב-gr.Blocks(theme=...):

🔑 הפרמטר elem_classes

כדי שה-CSS שלנו ידע על איזה כפתור ספציפי לפעול, אנחנו מוסיפים לו "תג" בעזרת elem_classes="run-btn". אז ב-CSS כותבים .run-btn { ... } — הנקודה אומרת "class בשם run-btn".

בלי elem_classes — ה-CSS יפעל על כל הכפתורים. עם elem_classes — רק על הכפתור הספציפי שסימנו.

⚠️ !important — Gradio מגדיר עיצובים משלו. כדי "לנצח" אותם, מוסיפים !important אחרי הערך. בלעדיו, העיצוב שלנו עלול להתעלם.

שלב 9 — ממשק בעברית
RTL — הצמדת טקסט ואלמנטים לימין
↔️ מה זה RTL?

RTL = Right-To-Left — כיוון קריאה מימין לשמאל. עברית וערבית הן שפות RTL. ברירת המחדל של דפי web היא LTR (Left-To-Right) — לכן בלי הגדרה מיוחדת, טקסט עברי יוצמד לשמאל ויראה מוזר.

📖
אנלוגיה: כשפותחים ספר עברי, הדף הראשון הוא מה-"סוף" לפי לוגיקה אנגלית — הכל הפוך. RTL אומר לדפדפן: "הספר הזה נקרא מימין לשמאל".
🛠️ 4 דרכים לטפל ב-RTL ב-Gradio
דרךמה היא עושהמתי להשתמש
css direction: rtl מצמיד את כל הממשק לימין בבת אחת ✅ הדרך הטובה ביותר — ממשק שלם בעברית
rtl=True ב-Textbox מצמיד את הטקסט בתוך שדה ספציפי לימין כשרק שדה אחד הוא בעברית
HTML div עם dir='rtl' מצמיד טקסט Markdown ספציפי לימין כשחלק מהממשק אנגלית, חלק עברית
text-align: right ב-CSS מיישר כיתובים (labels) לימין בנוסף ל-direction — לכיתובים מעל שדות
PYTHON — שלב 9: RTL מלא לממשק בעברית

טבלת ניסויים — gr.Dataframe()
📊 מה זה gr.Dataframe?

gr.Dataframe() מציג טבלת נתונים אינטראקטיבית בממשק — המשתמש יכול לראות, לערוך ולמלא ערכים בתאים. מה שמגיע לפונקציה Python הוא Pandas DataFrame — אובייקט שמאפשר לעבד את הנתונים שורה-שורה.

זה הרכיב המושלם למשימה שלנו: טבלת פרמטרים לניסויים מרובים — כל שורה = ניסוי אחד עם הגדרות שונות.

📋
אנלוגיה: דמיינו גיליון Excel בתוך הממשק. כל שורה היא ניסוי, כל עמודה היא פרמטר. המשתמש ממלא את הטבלה — לוחץ כפתור — הקוד רץ על כל השורות.
פרמטרסוגמה הוא עושה
headers=list[str]שמות העמודות — ["שם", "gamma", "משחקים"]
datatype=list[str]סוג הנתונים לכל עמודה: "str", "number", "bool"
row_count=intמספר שורות ריקות כברירת מחדל כשהממשק נפתח
col_count=intמספר עמודות (חייב להתאים למספר ה-headers)
value=list / DataFrameנתונים ראשוניים שמופיעים כבר בטבלה בפתיחה
interactive=boolTrue = ניתן לעריכה, False = תצוגה בלבד לפלט
🧠 תובנה חשובה — ההבדל בין מתחילים למקצוענים

Gradio הוא event-driven על רכיבים שלמים — הוא לא כמו React או JavaScript שמאפשרים "כפתור בתוך תא". כשלוחצים כפתור, כל הטבלה עוברת לפונקציה כ-DataFrame שלם.

לכן: עובדים על כל הטבלה ב-loop — לא על שורה בודדת.

✅ שימוש בסיסי — עיבוד כל השורות

הפונקציה מקבלת את הטבלה כולה כ-DataFrame ועוברת על כל שורה עם iterrows():

PYTHON — שימוש בסיסי ב-Dataframe 
import gradio as gr

def process_table(df):
    results = []
    for _, row in df.iterrows():   # עוברים שורה-שורה
        results.append(
            f"שם: {row['שם']} | gamma: {row['gamma']}"
        )
    return "\n".join(results)

🟢 פתרון 1 — בחירת שורה ספציפית עם Number

כשרוצים לפעול על שורה אחת: המשתמש מזין את מספר השורה בשדה Number, והפונקציה שולפת אותה עם df.iloc[index]:

PYTHON — עיבוד שורה לפי אינדקס 
def process_row(df, selected_index):
    try:
        row = df.iloc[int(selected_index)]   # שורה לפי מספר
        return f"עיבוד שורה {selected_index}:\n{row.to_dict()}"
    except IndexError:
        return "⚠️ מספר שורה לא קיים!"

🟢 פתרון 2 — עמודת "הפעל" (bool) — הפתרון המקצועי ✅

הוסף עמודת bool בשם "הפעל" לטבלה. המשתמש מסמן V בשורות שרוצה להריץ. הפונקציה מסננת רק את השורות המסומנות ומריצה רק אותן.

זה הפתרון הפדגוגי הכי טוב — מחבר Dataframe, bool, ו-batch processing לתפיסה אחת מלוכדת.

PYTHON — עמודת "הפעל" לבחירת שורות 
import gradio as gr

def run_selected(df):
    results   = []
    skipped   = 0

    for _, row in df.iterrows():
        if row["הפעל"]:                        # רק שורות מסומנות
            name  = row["שם ניסוי"]
            gamma = row["gamma"]
            games = int(row["משחקים"])
            results.append(f"✅ '{name}' | gamma={gamma} | {games} משחקים")
        else:
            skipped += 1

    if not results:
        return "⚠️ לא סומנה אף שורה להפעלה!"

    summary = f"הופעלו {len(results)} ניסויים | דולגו {skipped}\n\n"
    return summary + "\n".join(results)

DEFAULT_DATA = [
    ["baseline",      0.9, 5000,  True],
    ["low_gamma",     0.3, 5000,  False],
    ["high_explore",  0.9, 10000, True],
    ["neg_reward",    0.9, 5000,  False],
]

        headers  =["שם ניסוי", "gamma", "משחקים", "הפעל"],
