🚀 ExperimentRunner: הכוח שבאוטומציה

1. מה זה בעצם עושה?

ה-ExperimentRunner הוא שכבת הניהול העליונה. היתרונות המרכזיים שלו הם:

אוטומציה מלאה: במקום להריץ ידנית כל פעם tournament.run(), אנחנו נותנים לו רשימה של ניסויים והוא מבצע אותם אחד אחרי השני.
ממשק חיצוני (CSV): זו היכולת החשובה ביותר. ניתן להגדיר את כל הפרמטרים (כמו gamma או exploration_rate) בתוך קובץ Excel, והקוד יקרא אותם משם אוטומטית.

2. טעינת ניסויים מקובץ CSV

הפונקציה load_experiments_from_csv מאפשרת לנו להפריד לחלוטין בין הקוד לבין הנתונים.

experiment_runner.py
def load_experiments_from_csv(self, filename: str = "experiments.csv"):
    experiments = []
    with open(filename, mode='r', encoding='utf-8') as f:
        reader = csv.DictReader(f)
        for row in reader:
            config = ExperimentConfig(
                name=row['name'],
                num_games=int(row['num_games']),
                gamma=float(row['gamma']),
                exploration_rate=float(row['exploration_rate']),
                win_reward=float(row['win_reward'])
            )
            experiments.append(config)
    return experiments

המטרה: לאפשר לאנשים שאינם מתכנתים (או לנו, כדי לחסוך זמן) להגדיר ניסויים חדשים בלי לגעת בשורת קוד אחת.

3. הרצה ומדידת ביצועים

בזמן הרצת הניסוי, ה-Runner מודד כמה זמן לקח לכל אימון ומנהל את התיקיות באופן מסודר.

experiment_runner.py
def run_single_experiment(self, config: ExperimentConfig):
    start_time = time.time()
    tournament = XOTournament(config)

    # הרצת האימון
    tournament.run()

    duration = time.time() - start_time
    stats = tournament.get_statistics()
    stats['duration_sec'] = duration # הוספת נתון זמן הריצה
    return stats

4. ריכוז תוצאות (Aggregation)

בסיום כל הניסויים, ה-Runner יוצר קובץ CSV אחד מרכזי שמסכם את כל אחוזי הניצחון של כל ה-Configs השונים.

מבנה קובץ התוצאות הסופי
Experiment_Name, Win_Rate_X, Avg_Moves, Duration_Sec
"High_Gamma", 0.85, 7.2, 12.4
"Low_Gamma", 0.62, 8.1, 11.8

בדיקת הבנה (שאלות לתלמיד) 🧠

שאלה 1: מה יקרה אם נוסיף שורה חדשה לקובץ experiments.csv ונריץ את ה-Runner?

הקוד יריץ ניסוי נוסף עם הפרמטרים החדשים באופן אוטומטי.
הקוד יקרוס כי הוא לא מכיר את הניסוי החדש.
הקוד יריץ רק את הניסוי הראשון שמופיע בקובץ.

שאלה 2: מדוע ה-Runner משתמש ב-Path(self.output_dir).mkdir(parents=True, exist_ok=True)?

כדי למחוק את כל הניסויים הישנים לפני שמתחילים.
כדי לוודא שקיימת תיקייה לשמירת התוצאות, ואם לא - ליצור אותה מבלי לגרום לשגיאה.
כדי להצפין את הקבצים בתוך התיקייה.
