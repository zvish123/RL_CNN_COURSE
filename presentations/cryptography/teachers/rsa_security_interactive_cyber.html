הביטחון של RSA

למה קל להכפיל שני ראשוניים, אבל קשה מאוד למצוא אותם בחזרה?

מטרת השיעור: להבין את הרעיון של ביטחון חישובי, פירוק לגורמים, Padding והצפנה היברידית.

1. מה בעצם צריך לפרוץ?

ב-RSA המפתח הציבורי מכיל מספר גדול:

n = p × q

כולם יודעים את n, אבל רק בעל המפתח הפרטי יודע את שני הראשוניים p ו-q.

p סודיq סודיn ציבורי

המשימה של תוקף

לקחת את n ולגלות:

n → p, q

זה נקרא פירוק לגורמים ראשוניים.

8. PKCS#1 מול OAEP

RSA “נאיבי”

לא מתאים לשימוש אמיתי.

PKCS#1 v1.5

שיטת Padding ותיקה.

נמצאת במערכות ישנות, אך דורשת זהירות.

OAEP

שיטה מודרנית יותר להצפנת RSA.

כלל זהב: לא מממשים RSA לבד במערכת אמיתית. משתמשים בספריות קריפטוגרפיות מוכרות.

9. למה משלבים RSA + AES?

RSA איטי ומתאים להצפנת מידע קטן — למשל מפתח.

AES מהיר ומתאים להצפנת קבצים והודעות גדולות.

10. הדגמת “מעטפה דיגיטלית”

מה נשלח ברשת?

כללי המשחק

  • המסר עצמו מוצפן ב-AES.
  • מפתח ה-AES מוצפן ב-RSA.
  • המפתח הפרטי לא נשלח לעולם.

11. האיום העתידי: מחשבים קוונטיים

אלגוריתם שור מסוגל, תאורטית, לפרק מספרים גדולים ביעילות על מחשב קוונטי גדול ויציב.

12. מיני-מעבדה: פריצה למספר קטן

נסו בעצמכם

הזינו מספר n קטן, והקוד ינסה למצוא גורמים.

הקוד הרעיוני

for i in range(2, sqrt(n)): if n % i == 0: return i, n // i

זה מדגים את הרעיון בלבד. RSA אמיתי גדול בהרבה.

14. משימות הרחבה לתלמידי סייבר

משימה 1: מדידת זמן

כתבו בפייתון פונקציה שמפרקת מספרים קטנים לגורמים. מדדו כמה זמן נדרש כאשר n גדל.

תוצר: טבלה וגרף זמן ריצה.

משימה 2: זיהוי חולשות

מה יקרה אם p ו-q קרובים מדי? מה יקרה אם משתמשים באותו p פעמיים?

תוצר: הסבר + הדגמה במספרים קטנים.

משימה 3: Padding

צרו סימולציה שבה אותה הודעה מוצפנת כמה פעמים: פעם בלי אקראיות ופעם עם אקראיות.

תוצר: מסקנה כתובה: למה Padding משנה את המשחק?

משימה 4: הצפנה היברידית

בנו תרשים זרימה ל-RSA + AES: שליחה, קבלה, פענוח ושגיאות אפשריות.

תוצר: תרשים + פסאודו-קוד.

15. סיכום מקצועי

  • פיצוח RSA שקול בעיקר לפירוק n לגורמים.
  • RSA בטוח בזכות קושי חישובי, לא בזכות קסם.
  • Padding הוא חלק חיוני מהשימוש הבטוח.
  • במערכות אמיתיות משתמשים לרוב ב-RSA כדי להצפין מפתח AES.
RSA לא בלתי ניתן לפריצה.
הוא לא מעשי לפריצה כיום.
